# IActiveScript::AddNamedItem
Добавляет имя элемента корень\- уровня в пространстве имен обработчика скриптов.  Элемент корень\- уровня объекта со свойствами и методами, источником события или всеми 3.  
  
## Синтаксис  
  
```  
HRESULT AddNamedItem(  
    LPCOLESTR pstrName,  // address of item name  
    DWORD dwFlags          // item flags  
);  
```  
  
#### Параметры  
 `pstrName`  
 \[in\] адрес буфера, содержащее имя элемента в виде доступна из скрипта.  Имя должно быть уникальным и постоянного.  
  
 `dwFlags`  
 \[in\] флаги, связанные с элементом.  Может оказаться сочетание этих значений:  
  
|Значение|Значение|  
|--------------|--------------|  
|SCRIPTITEM\_CODEONLY|Указывает, что именованный элемент представляющий объект кода \- только и что основное приложение не имеет `IUnknown`, который необходимо связать с данным объектом кода \- только для чтения.  Основное приложение содержится только имя для этого объекта.  В объект\- ориентированных языков, как C\+\+, этот создатьTm бы пометить класс.  Не все языки поддерживают этот пометить.|  
|SCRIPTITEM\_GLOBALMEMBERS|Указывает, что элемент коллекцию глобальных свойств и методов, связанных со скриптом.  Обычно обработчик скриптов игнорироватьTm мере имя объекта \(за исключением использования его в качестве файла cookie для метода [IActiveScriptSite::GetItemInfo](../../winscript/reference/iactivescriptsite-getiteminfo.md) или для разрешения области действия\), а точный предоставлятьTm мере ее члены как глобальные переменные и методы.  Это позволяет узлу расширение доступно библиотеки \(функциям среды выполнения и т д\) к скрипту.  Он выйден на обработчик скриптов для обработки конфликтов имени \(например, когда 2 элемента SCRIPTITEM\_GLOBALMEMBERS содержат методы с тем же именем\), несмотря на то, что ошибка не должна быть возвращенной ситуацией по этой причине.|  
|SCRIPTITEM\_ISPERSISTENT|Указывает, что элемент должен быть сохранен, если обработчик скриптов сохранить.  Аналогично, установка этого пометить указывает, что вернитесь к состоянию инициализированному должен сохранять имя элемента и сведения о типе \(обработчиком сценариев, однако должен освободить все указатели к интерфейсам в фактическом объекта\).|  
|SCRIPTITEM\_ISSOURCE|Указывает, что события источников элемента, скрипт может утонуть.  Дочерние объекты \(свойства объекта, в себя объекты\) могут также события источника к скрипту.  Это не рекурсивно, но он обеспечивает удобный механизм в общий случай, например, создать контейнер и все его элементы управления элемента.|  
|SCRIPTITEM\_ISVISIBLE|Указывает на то, что имя элемента содержатся в пространстве имен скрипта, обеспечивая доступ к свойствам, методам и событиям элемента.  По соглашению свойства элементов включают дочерние элементы элемента; поэтому все свойства дочерних объектов и методы \(и их потомки, рекурсивно\) будут доступны.|  
|SCRIPTITEM\_NOCODE|Указывает, что элемент просто имя добавляемого в пространстве имен сценария и обрабатываться как элемент, для которого должен быть связан код.  Например, без этого, пометить устанавливанным, VBScript создает отдельный модуль для именованного элемента и C\+\+ может создать отдельный класс\-оболочка для именованного элемента.|  
  
## Возвращаемое значение  
 Возвращать одно из следующих значений:  
  
|Возвращаемое значение|Значение|  
|---------------------------|--------------|  
|`S_OK`|Успех.|  
|`E_INVALIDARG`|Аргумент был недопустимым.|  
|`E_POINTER`|Недопустимый указатель был определен.|  
|`E_UNEXPECTED`|Вызов не ожидался \(например, обработчик скриптов еще не был загружен или не был инициализирован\).|
