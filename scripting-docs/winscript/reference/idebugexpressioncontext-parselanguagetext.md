# IDebugExpressionContext::ParseLanguageText
Создает выражение отладки для указанного текста.  
  
## Синтаксис  
  
```  
HRESULT ParseLanguageText(  
   LPCOLESTR           pstrCode,  
   UINT                nRadix,  
   LPCOLESTR           pstrDelimiter,  
   DWORD               dwFlags,  
   IDebugExpression**  ppe  
);  
```  
  
#### Параметры  
 `pstrCode`  
 \[in\] содержит текст выражения или выписок.  
  
 `nRadix`  
 \[in\] корневой элемент, который требуется использовать.  
  
 `pstrDelimiter`  
 \[in\] разделитель элемент \- скрипт\- блока.  При `pstrCode` анализироватьо из потока текста, основное приложение обычно использует разделитель, например 2 одинарных кавычки \("\), чтобы обнаружить конец блока скрипта.  Этот параметр задает разделитель, используемый основным приложением, что обработчик скриптов, чтобы обеспечить какую\-либо условную предобработку примитивов \(например, заменящ одинарная кавычка \('\) с 2 одинарными кавычками для использования в качестве разделителя\).  То, как \(и, если\), то обработчик скриптов использует этот зависит от информации обработчика скриптов.  Установите этот параметр в `NULL` если основное приложение не использует разделитель для обозначения конца блока скрипта.  
  
 `dwFlags`  
 \[in\] сочетание отладки флаги следующий текст:  
  
|Константа|Значение|Описание|  
|---------------|--------------|--------------|  
|DEBUG\_TEXT\_ISEXPRESSION|0x00000001|Указывает, что текст выражения в отличие от выписки.  Этот пометить может повлиять на способ текст анализироватьо некоторых языках.|  
|DEBUG\_TEXT\_RETURNVALUE|0x00000002|Если возвращаемое значение доступно, то оно будет использоваться вызывающим объектом.|  
|DEBUG\_TEXT\_NOSIDEEFFECTS|0x00000004|Не следует разрешать побочные эффекты.  Если этот пометить установлен, то вычисление выражения не должна изменить состояние среды выполнения.|  
|DEBUG\_TEXT\_ALLOWBREAKPOINTS|0x00000008|Разрешает точки останова во время вычисления текста.  Если этот пометить не установлен, то точки останова игнорироватьы во время вычисления текста.|  
|DEBUG\_TEXT\_ALLOWERRORREPORT|0x00000010|Разрешает отчеты об ошибках во время вычисления текста.  Если этот пометить не установлен, то ошибки не передаются к узлу во время оценки.|  
|DEBUG\_TEXT\_EVALUATETOCODECONTEXT|0x00000020|Указывает, что необходимо оценить выражение кода, а не к контексту выполнения само выражение|  
  
 `ppe`  
 \[out\] возвращает выражение отладки для указанного текста.  
  
## Возвращаемое значение  
 Метод возвращает `HRESULT`.  Допустимые значения включают, но не ограничиваются см. в следующей таблице.  
  
|Значение|Описание|  
|--------------|--------------|  
|`S_OK`|Метод успешно выполнен.|  
  
## Заметки  
 Этот метод создает выражение отладки для указанного текста.
