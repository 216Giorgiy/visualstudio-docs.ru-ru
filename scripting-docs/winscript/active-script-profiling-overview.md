# Обзор профилирования активных скриптов
[Интерфейсы профилировщика активных скриптов](../winscript/reference/active-script-profiler-interfaces.md) включает профилирование обработчика скриптов.  Профилирование активного сценария состоит из следующих частей:  
  
-   Обработчик языка  
  
-   Основное приложение  
  
-   Профилировщик  
  
## Обработчик языка  
 Обработчик языка выполняется скрипт.  Он предоставляет методы, позволяющие профилирование кода скрипта по мере его выполнения.  При профилировании включен, обработчик языка принимает идентификатор класса \(CLSID\) COM\-объект профилировщика в качестве аргумента.  Он создает экземпляр COM\-объект и затем вызовов профилировщика профилировщик, когда возникают различные события.  
  
 Обработчик языка реализует интерфейс [Интерфейс IActiveScriptProfilerControl](../winscript/reference/iactivescriptprofilercontrol-interface.md).  
  
> [!NOTE]
>  Выполнение языка [!INCLUDE[javascript](../javascript/includes/javascript-md.md)] проверяет переменную среды JS\_PROFILER при создании для определения, должен ли быть включено профилирование.  Если эта переменная принимает значение CLSID профилировщика, среда выполнения создает экземпляр профилировщика языка com\-объекта, значение переменной, чтобы определить, профилировщик, которую необходимо создать.  
  
## Основное приложение  
 Основное приложение создает обработчик языка и предоставляет обработчик скриптов языка с, которая должна выполняться.  Промежуточный узел также предоставляет контекст документа, который может использоваться отладчиком или профилировщиком для обеспечения оптимальных сведения при отладке или профилируете.  
  
## Профилировщик  
 Профилировщик получает вызовы из обработчика языка при различные события возникают.  Профилировщик должен зарегистрировать в качестве com\-объекта и должен реализовывать интерфейс [Интерфейс IActiveScriptProfilerCallback](../winscript/reference/iactivescriptprofilercallback-interface.md).
