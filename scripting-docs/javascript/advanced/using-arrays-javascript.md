# Использование массивов (JavaScript)
Массивы в [!INCLUDE[javascript](../../javascript/includes/javascript-md.md)] являются *разреженными*.  То есть если имеется массив с тремя элементами, пронумерованными 0, 1 и 2, можно создать элемент 50, не задумываясь об элементах от 3 до 49.  Если массив имеет автоматически обновляемую переменную длины \(информацию об автоматическом мониторинге длины массива см. в разделе [Встроенные объекты](../../javascript/intrinsic-objects-javascript.md)\), переменной длины присваивается значение 51, а не 4.  Можно создавать массивы, в которых нет промежутков в нумерации элементов, но это не обязательно.  
  
 В [!INCLUDE[javascript](../../javascript/includes/javascript-md.md)] объекты и массивы практически не отличаются.  Два основных различия состоят в том, что объекты, которые не являются массивами, не имеют автоматически обновляемого свойства длины, а массивы не содержат свойств и методов объекта.  
  
## Обращение к массивам  
 Для обращения к массивам следует использовать квадратные скобки \(\[\]\), как показано в следующем примере.  В скобки заключается или числовое значение, или выражение, результат которого является целым числом.  
  
```javascript  
var entryNum = 5;  
  
sample = new Array();  
  
sample[1] = "Maple Street";  
sample[entryNum] = 25;  
  
document.write (sample[1]);  
document.write (" ");  
document.write (sample[entryNum]);  
  
// Output: Maple Street 25  
  
```  
  
## Объекты как ассоциативные массивы  
 Для доступа к свойствам объекта обычно используется оператор ".".  Пример.  
  
```javascript  
myObject.aProperty  
```  
  
 В этом случае имя свойства является идентификатором.  Для доступа к свойствам объекта можно также использовать оператор индекса \(\[\]\).  В этом случае объект обрабатывается как *ассоциативный массив*, в котором значения данных связаны со строками.  Пример.  
  
```javascript  
myObject["aProperty"] // Same as above.  
```  
  
 Оператор индекса чаще связывают с доступом к элементам массива.  Если он используется с объектами, индекс является строковым литералом, представляющим имя свойства.  
  
 Обратите внимание на важное различие между двумя способами доступа к свойствам объекта.  
  
1.  Имя свойства, которое рассматривается как идентификатор \(синтаксис точки \(.\)\), нельзя обрабатывать, как данные.  
  
2.  Имя свойства, которое рассматривается как индекс \(синтаксис квадратных скобок \(\[\]\), можно обрабатывать, как данные.  
  
 Это различие становится полезным, если имена свойств неизвестны до времени выполнения \(например, при создании объектов на основе ввода пользователя\).  Чтобы извлекать свойства из ассоциативного массива, нужно использовать цикл `for…in`.
