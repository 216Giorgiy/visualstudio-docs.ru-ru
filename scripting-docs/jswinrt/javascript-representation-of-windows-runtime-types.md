# Представление типов среды выполнения Windows в JavaScript
Следующая таблица описывает способ представления типов среды выполнения Windows в JavaScript.  
  
> [!IMPORTANT]
>  Функции среды выполнения Windows недоступны для приложений, работающих в Internet Explorer.  
  
## Типы среды выполнения Windows в JavaScript  
  
|Тип среды выполнения Windows|Представление JavaScript|Описание|  
|----------------------------------|------------------------------|--------------|  
|`UInt8`|`Number`|`Uint8` среды выполнения Windows — это целое 8\-разрядное число без подписи, представленное в виде числа JavaScript.  Значение JavaScript сначала преобразуется в число JavaScript, а затем следует процесс `ToUint32`.  Если значение числа JavaScript находится вне диапазона Uint8, к результату будет применяться остаток от деления 2^8.|  
|`Int32`|`Number`|`Int32` среды выполнения Windows — это целое 32\-разрядное число с подписью, представленное в виде числа JavaScript.  Значение JavaScript сначала преобразуется в число JavaScript, а затем следует процесс `ToInt32`.  Если значение числа JavaScript находится вне диапазона `Int32`, к результату будет применяться остаток от деления 2^16.|  
|`Int64`|`Number`|`Int64` среды выполнения Windows — это целое 64\-разрядное число с подписью, представленное как стандартный номер, если он попадает в диапазон \[\- 2^53, 2^53\].  Если оно находится за пределами этого диапазона, представляется как значение числа, которое содержит пользовательское резервное хранилище, которое поддерживает все 64 бита целочисленных данных.  Все математические операции с этими пользовательскими значениями чисел Int64 вызывают необходимость преобразования значения в стандартное число диапазона \[\- 2^53, 2^53\].  Если значение вне этого диапазона, то возникает ошибка типа.  Исключения этого процесса преобразования — операции `==`, `===`, `SameValue` и  `RelationalComparison`, которые сравнивают аргументы двух представленных 64\-разрядных значений по всем 64 разрядам.  Если какой\-либо аргумент является стандартным числом JavaScript, в ходе этих операций аргумент перед сравнением преобразуется в число JavaScript.  Значение JavaScript присваивается непосредственно, если это значение Int64; в противном случае результат применения преобразования `ToInteger` к этому значению передается в среду выполнения Windows.  Если приведение типа завершается ошибкой, возвращается исключение.|  
|`Uint64`|`Number`|`Uint64` среды выполнения Windows — это целое 64\-разрядное число без подписи, представленное в виде стандартного числа, если оно попадает в диапазон \[0, 2^53\].  Если оно находится за пределами этого диапазона, представляется как число, которое содержит пользовательское резервное хранилище, которое поддерживает все 64 бита целочисленных данных без знака.  Все математические операции с этими пользовательскими значениями Uint64 вызывают необходимость преобразования значения в стандартное число диапазона \[0, 2^53\].  Если значение вне этого диапазона, то возникает ошибка типа.  Исключения этого процесса преобразования — операции `==`, `===`, `SameValue` и  `RelationalComparison`, которые сравнивают аргументы двух 64\-разрядных значений по всем 64 разрядам.  Если какой\-либо аргумент является стандартным числом JavaScript, в ходе этих операций аргумент перед сравнением преобразуется в число JavaScript.  Значение JavaScript присваивается непосредственно, если это значение Uint64; в противном случае результат применения преобразования `ToInteger` к этому значению с остатком деления 2^52 передается в среду выполнения Windows.  Если преобразование типа завершается ошибкой, возвращается исключение.|  
|`Single`|`Number`|`Single` среды выполнения Windows — это 32\-разрядное число с плавающей запятой, представленное в виде числа JavaScript путем выбора ближайшего значения двойной точности к значению одиночной точности.  Значение JavaScript преобразуется к числу JavaScript, а затем проверяется по диапазону, чтобы убедиться, что значение можно представить в виде 32\-разрядного числа с плавающей запятой \(одиночная точность\), соответствующего стандарту IEEE 754.  Если преобразование или проверка в диапазоне завершаются ошибкой, возвращается ошибка маршалинга.|  
|`Double`|`Number`|Тип `Double` среды выполнения Windows представляет собой 64\-битное 8\-разрядное число с плавающей запятой.  `ToNumber` используется для преобразования среды выполнения Windows `Double` в число JavaScript.  Если преобразование завершается ошибкой, возвращается ошибка маршалинга.|  
|`Boolean`|`Boolean`|`Boolean` среды выполнения Windows — это 8\-разрядное значение, где нуль — `false`, а любое значение, отличное от нуля — `true`, представленное как значение `Boolean` в JavaScript.  Значение JavaScript сначала преобразуется в значение `Boolean` JavaScript, используя метод `ToBoolean`.  Это означает, что функция среды выполнения Windows, объявленная как `void func(BOOL);`, может быть записана на языке JavaScript в качестве `obj.func("test");`.  Функция среды выполнения Windows называется с параметром `BOOL`, переданным в качестве `TRUE`.  Если преобразование завершается ошибкой, возвращается ошибка маршалинга.|  
|`Char16`|`String`|`Char16` среды выполнения Windows — это 16\-разрядный символ Юникода, представленный в виде строки JavaScript, которая содержит один символ.  Значение JavaScript преобразовано в строку JavaScript с использованием метода `ToString`, а затем проверено, чтобы убедиться, что строка содержит только один символ.  Один символ затем передается в виде значения `Char16`.  Если преобразование или проверка длины завершаются ошибкой, возвращается ошибка маршалинга.|  
|`String`|`String`|`String` среды выполнения Windows — это неизменяемая последовательность объектов `Char16`.  Строки представляются в виде объектов `String` JavaScript.  Строки в среде выполнения Windows не имеют разных значений для `null` и пустой строки \(""\).  `null` и пустые строковые значения преобразованы к пустой строке JavaScript.  Примечание. При передаче значения `null` JavaScript параметру среды выполнения Windows, который ожидает строку, создается строковое значение null, а не `null` или пустая строка \(""\).  Экземпляры строк, передаваемых среде выполнения Windows, всегда должны создаваться в пустую строку.|  
|`Enumeration`|`Object`|`Enumeration` среды выполнения Windows — это 32\-разрядное целое число со знаком или без знака, которое имеет соответствующий набор именованных констант.  На языке JavaScript перечисление представляется как объект, содержащий поле только для чтения для каждого именованного значения.  Значения перечисления преобразованы в числа JavaScript, как определено ранее для `Int32` и `UInt32`.  Если значение JavaScript преобразовано к перечислению среды выполнения Windows, оно преобразуется, усекается и проверяется на диапазон, как описано выше.  Однако результирующее значение не проверяется относительно определенных именованных значений, которые определены в перечислении.|  
|`Structure`|`Object`|`Structure` среды выполнения Windows — это разнородная коллекция именованных полей данных.  На языке JavaScript структура представлена в виде объекта JavaScript, который содержит именованное свойство для каждого поля в структуре.  Если преобразование какого\-либо значения структуры завершается ошибкой, возвращается ошибка маршалинга.  Поля в объекте JavaScript, которые не имеют эквивалентов в структуре среды выполнения Windows, игнорируются.  Примечание. Невозможно создать экземпляр структуры среды выполнения Windows на языке JavaScript с ключевым словом `new`.|  
|`Array`|`Array`|`Array` среды выполнения Windows преобразуется в массив JavaScript.  Однако изменить размер массива среды выполнения Windows невозможно, поэтому некоторые операции с массивами в JavaScript невозможны.  Внутреннее свойство \[\[Класс\]\] преобразованного массива не имеет значение Array, потому что это не разрешено в соответствии со спецификацией ECMAScript 5.  Это означает, что `Array.isArray(v)` возвращает `false`.  Значение JavaScript преобразовано в массив среды выполнения Windows следующим образом: если значение — `null` или `undefined`, оно преобразуется к собственному `null`.  Если внутреннее свойство \[\[Класс\]\] имеет значение Array, оно копируется в собственный массив и передается ссылка на этот массив.  Если массив преобразован, как описано выше, передается базовый собственный массив.  Примечание. Передача значения массива JavaScript методу среды выполнения Windows вызывает полное копирование массива.|  
|Делегат \(обратный вызов функции\)|`Function`|Делегат среды выполнения Windows — это ссылка на одиночный метод.  Делегат среды выполнения Windows представлен в JavaScript как `Function` JavaScript, которая гарантированно вызывается в правильном потоке.  Если вызывается `Function`, аргументы преобразуются в эквивалентные типы параметров среды выполнения Windows.  Если имеется меньше аргументов JavaScript, чем параметров `in` делегата, то вызов делегата завершается ошибкой.  Если имеется больше аргументов JavaScript, чем параметров `in` в делегате, дополнительные аргументы JavaScript игнорируются.  После вызова делегата параметры `out` преобразуются в типы JavaScript и возвращаются.  Если собственный объект функции JavaScript преобразуется в делегат среды выполнения Windows, создается программа\-оболочка в делегате среды выполнения Windows соответствующего типа.  Если вызывается делегат, параметры `in` преобразуются в типы JavaScript.  После вызова делегата возвращаемое значение преобразуется в параметры `out` делегата.|  
|Интерфейс|`Object`|Интерфейсы и группы интерфейсов непосредственно не представлены на языке JavaScript в виде объектов.  Однако интерфейсы представлены как типы параметров и возвращаемые типы методов среды выполнения Windows.  Экземпляр интерфейса среды выполнения Windows преобразуется следующим образом:<br /><br /> 1.  Если имеется допустимый класса среды выполнения, объект представлен как экземпляр этого класса.<br />2.  Если допустимый класс среды выполнения отсутствует, объект представляется как экземпляр класса среды выполнения без имени, реализующий точно тот интерфейс, который, как известно, реализуется интерфейсом, и все необходимые его интерфейсы.<br /><br /> Значение JavaScript проверяется, чтобы определить, является ли оно экземпляром класса среды выполнения или интерфейса.  Если да и исходное значение среды выполнения Windows реализует соответствующий интерфейс, то объект передается среде выполнения Windows.|  
|Классы среды выполнения|`Object`|Объекты среды выполнения Windows могут быть экземплярами классов среды выполнения, которые реализуют один или несколько интерфейсов.  Объекты среды выполнения Windows представлены на языке JavaScript в виде объектов.  Объединение методов, свойств и событий, определенных для всех реализованных интерфейсов класса, представляет именованные свойства в прототипе объекта JavaScript.  Объекты\-получатели JavaScript могут обращаться к любому члену класса напрямую.  Объекты среды выполнения Windows имеют заполнитель, который содержит все элементы из всех реализованных интерфейсов класса среды выполнения.|
