# Практическое руководство. Указание расположения, из которого будет производиться установка пользователями
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

При публикации приложения [!INCLUDE[ndptecclick](../deployment/includes/ndptecclick_md.md)] место, куда пользователи переходят для загрузки и установки приложения, не обязательно является местом, где первоначально было опубликовано приложение.  Например, в некоторых организациях разработчик может опубликовать приложение на промежуточный сервер, а затем администратор перемещает приложение на веб\-сервер.  
  
 В этом случае можно использовать свойство `Installation URL` для указания веб\-сервера, на который пользователи будут переходить для загрузки приложения.  Это необходимо, чтобы манифест приложения знал, где искать обновления.  
  
 Свойство `Installation URL` может устанавливаться на странице **Публикация** в **конструкторе проектов**.  
  
 **Примечание.** Свойство `Installation URL` может быть также установлено с помощью **мастера публикации**.  Дополнительные сведения см. в разделе [Практическое руководство. Публикация приложения ClickOnce с помощью мастера публикации](../Topic/How%20to:%20Publish%20a%20ClickOnce%20Application%20using%20the%20Publish%20Wizard.md).  
  
### Задание URL\-адреса установки  
  
1.  Выбрав проект в **обозревателе решений**, в меню **Проект** выберите пункт **Свойства**.  
  
2.  Выберите вкладку **Публикация**.  
  
3.  В поле "URL\-адрес установки" введите место установки, используя полный URL\-адрес в формате http:\/\/www.microsoft.com\/имя\_приложения, или UNC\-путь в формате \\\\Сервер\\имя\_приложения.
