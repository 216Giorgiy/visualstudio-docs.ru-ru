# <a name="publish-a-web-app-to-a-web-site-using-visual-studio"></a>Публикация веб-приложения на веб-сайт, с помощью Visual Studio

Можно использовать **публикации** средство для публикации приложения ASP.NET, ASP.NET Core, .NET Core и Python на веб-сайт с Visual Studio. Для Node.js действия поддерживаются, но имеют разные пользовательские интерфейсы.

[!INCLUDE [quickstart-prereqs](includes/quickstart-prereqs.md)]

## <a name="publish-to-a-web-site"></a>Публикация на веб-сайт

1. В обозревателе решений щелкните правой кнопкой мыши проект и выберите **публикации** (или используйте **построения** > **публикации** пункт меню).

    ![Команда публикации в контекстном меню проекта в обозревателе решений](../deployment/media/quickstart-publish.png "выберите публикации")

1. Если ранее вы настроили все профили публикации **публикации** откроется панель. Выберите **создать новый профиль**.

1. В **выберите целевой объект публикации** диалоговом окне выберите **IIS, FTP, д**.

    ![Выберите IIS, FTP и т. д.](../deployment/media/quickstart-publish-iis-ftp.png "выберите IIS, FTP и т. д.")

1. Нажмите **Публиковать**. Откроется диалоговое окно параметров публикации профиля.

    ![Выберите папку](../deployment/media/quickstart-publish-settings-web.png "выбрать папку")

1. В **метод публикации** выберите метод, такой как **веб-развертывания** или **FTP**. Параметры, которые вы увидите Далее соответствуют способа публикации. Веб-развертывания упрощает развертывание веб-приложений и веб-сайтов на серверах IIS и должен быть установлен как приложения на сервере. Используйте [установщика веб-платформы](https://www.microsoft.com/web/downloads/platform.aspx) установить его.

1. Настройка обязательных параметров для метода публикации и выберите **проверить подключение**. Если сервер или целевой объект доступен и настройки верны, сообщение, которое указывает соединение, которое проверяется, и вы готовы к публикации.

    ![Проверка подключения к](../deployment/media/quickstart-publish-web-deploy.png "Проверка подключения к")

1. Выберите **параметры** Чтобы настроить другие параметры развертывания, например следует ли развернуть конфигурацию отладки или выпуска, а затем выберите **Сохранить**. Если вы удаленной отладки, конфигурации отладки не требуется.

1. Для этого щелкните **публикации**. В окне вывода отображаются ход выполнения развертывания и результатов.
