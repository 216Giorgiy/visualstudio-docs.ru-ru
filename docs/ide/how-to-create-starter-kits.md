# Практическое руководство. Создание начальных наборов
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Начальный набор содержит код полного приложения и документацию о том, как изменить или расширить это приложение.  Процедура создания начального набора отличается от процедуры создания обычного шаблона проекта только тем, что начальный набор содержит файлы документации, которые отрываются при создании проекта на основе начального набора.  
  
## Проектирование и разработка начального набора  
 Во\-первых, необходимо определить тип начального набора, который требуется разработать, и определить целевую аудиторию.  Далее, нужно разработать проект и документацию, соответствующую существующим целям.  
  
 При создании примера приложения или подключаемого модуля:  
  
-   Создайте проект, для которого построение выполняется без ошибок.  
  
-   Добавьте код шаблона для реализации дополнительных задач \(необязательно\).  
  
-   Подготовьте документацию.  
  
 При создании средства обучения:  
  
-   Создайте проект, для которого построение выполняется без ошибок.  
  
-   Организуйте ресурсы, такие как фрагменты кода и шаблоны элементов.  
  
-   Подготовьте документацию.  
  
## Создание шаблона  
 После завершения создания проекта и документации можно приступить к созданию шаблона проекта начального набора.  Этот процесс не отличается от процесса создания шаблона.  
  
 Следующие разделы содержат сведения о создании шаблонов.

 Содержит описание использования мастера **Экспорт шаблона** для создания шаблона.
