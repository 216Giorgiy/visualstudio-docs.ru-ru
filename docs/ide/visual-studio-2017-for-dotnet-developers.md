## <a name="a-idguide-productivity-guide"></a><a id="guide"/> Руководство по повышению производительности
[Visual Studio 2017](https://www.visualstudio.com/downloads/) выводит производительность труда разработчиков на новый уровень. Мы повысили производительность и надежность при запуске и загрузке решений, обнаружении тестов, а также сократили задержки при вводе. Кроме того, мы улучшили существующие функции и добавили новые, чтобы помочь вам быстрее создавать более качественный код. Вот некоторые из этих возможностей: переход к декомпилированным сборкам, варианты имен переменных, предлагаемые в процессе ввода, иерархическое представление в обозревателе тестов, команда "Перейти ко всем" (**CTRL+T**) для перехода к объявлениям файлов, типов, членов и символов, интеллектуальный помощник по исправлению ошибок, настройка и применение стиля кода, а также рефакторинг и исправление множества ошибок в коде. 

Чтобы повысить производительность, следуйте этому руководству.

###  <a name="im-used-to-my-keyboard-shortcuts-from-a-different-extensioneditoride"></a>Мне привычнее использовать сочетания клавиш из другого расширения, редактора или интегрированной среды разработки.

Если вы переходите из другой интегрированной среды разработки или среды написания кода, возможно, вам поможет установка одного из следующих расширений:

- [Эмуляция Emacs](https://marketplace.visualstudio.com/items?itemName=JustinClareburtMSFT.EmacsEmulation)
- [Сочетания клавиш в Visual Studio (IntelliJ)](https://marketplace.visualstudio.com/items?itemName=JustinClareburtMSFT.HotKeys)
- [VSVim](https://marketplace.visualstudio.com/items?itemName=JaredParMSFT.VsVim)

Ниже приведены широко используемые сочетания клавиш в Visual Studio.

> Некоторые расширения переопределяют установленные по умолчанию сочетания клавиш в Visual Studio, и для использования приведенных ниже команд необходимо восстановить эти сочетания. Чтобы восстановить заданные по умолчанию сочетания клавиш в Visual Studio, выберите **Сервис > Импорт и экспорт параметров... > Сбросить все параметры** или **Сервис > Параметры > Клавиатура > Сброс**.

| **CTRL**+**D** | Дублировать строку | Дублирование строки кода, в которой находится курсор (доступно в **Visual Studio 2017 версии 15.6** и более поздних версиях) |
| **SHIFT**+**ALT**+**+**/**-** | Расширение или сужение выделенного фрагмента | Расширение или сужение выделенного в настоящее время фрагмента в редакторе (доступно в **Visual Studio 2017 версии 15.5** или более поздней) |

### <a name="i-need-a-way-to-quickly-navigate-to-files-or-types"></a>Мне необходимо быстро переходить к файлам или типам.
В Visual Studio 2017 есть функция _Перейти ко всем_ (**CTRL+T**). Она позволяет быстро перейти к объявлению любого файла, типа, члена или символа.
- Изменить расположение этой панели поиска или отключить динамический предварительный просмотр навигации можно с помощью значка **шестеренки**.
- Для фильтрации результатов используйте синтаксис запроса (например, "t mytype"). Область поиска можно также ограничить текущим документом.
- При сопоставлении поддерживается "верблюжий" стиль.

![Функция "Перейти ко всем" в Visual Studio](../ide/media/VS2017Guide-go-to-all.png "VS2017Guide-go-to-all")

### <a name="my-team-enforces-code-style-rules-on-our-codebase"></a>В моей команде к базе кода применяются правила стиля кода.
Для определения соглашений о написании кода можно использовать файл EDITORCONFIG. Мы рекомендуем установить [расширение языковых служб EditorConfig](https://aka.ms/editorconfig) для добавления и редактирования файла EDITORCONFIG. Ознакомиться со всеми вариантами соглашений о написании кода в .NET можно в [этой документации](https://aka.ms/editorconfigDocs).

Пример файла EDITORCONFIG см. в [этой публикации Gist](https://gist.github.com/kuhlenh/5471666a7a2c57fea427e81cf0a41da8).

![Применение стиля кода в Visual Studio](../ide/media/VS2017Guide-code-style.png "VS2017Guide-code-style")

### <a name="i-need-more-refactorings-and-code-fixes"></a>Мне нужны дополнительные возможности рефакторинга и исправления кода.
В Visual Studio 2017 включено множество возможностей рефакторинга, действий по созданию кода и исправлений кода, с которыми можно ознакомиться в [нашей документации](https://aka.ms/refactorings). Красной волнистой линией обозначаются ошибки, зеленой волнистой линией — предупреждения, а тремя серыми точками — предлагаемые варианты кода.

Чтобы получить доступ к исправлениям кода, щелкните значок лампочки или отвертки либо нажмите клавиши **CTRL+.** или **ALT+ВВОД**. Для каждого исправления отображается окно предварительного просмотра, в котором в реальном времени можно увидеть, как будет работать исправление.

Вот некоторые популярные быстрые исправления и варианты рефакторинга: "Переименовать", "Извлечь метод", "Изменить сигнатуру метода", "Создать конструктор", "Создать метод", "Переместить тип в файл", "Добавить проверку значения NULL", "Добавить параметр", "Удалить ненужные директивы using".

Рефакторинг и исправления кода можно легко создавать с помощью [анализаторов Roslyn](https://github.com/dotnet/roslyn/wiki/Getting-Started-Writing-a-Custom-Analyzer-&-Code-Fix). Участники сообщества написали *бесплатные расширения*, которые реализуют дополнительные проверки кода: Roslynator и SonarLint for Visual Studio. 

![Рефакторинг в Visual Studio](../ide/media/VS2017Guide-refactoring.png "VS2017Guide-refactoring")

### <a name="i-need-find-usages-go-to-implementation-navigate-to-decompiled-assemblies"></a>Мне нужны функции поиска использования, перехода к реализации и перехода к декомпилированным сборкам
В Visual Studio 2017 имеется множество функций для поиска и перехода по базе кода, включая функции "Найти все ссылки" (**SHIFT+F12**), "Перейти к реализации" (**CTRL+F12**) и "Перейти к определению" (**F12** или **CTRL+щелчок мышью**). Функция перехода к декомпилированным сборкам была добавлена в версии 15.6. Чтобы включить эту функцию, перейдите в раздел **Сервис > Параметры > Текстовый редактор > C# > Дополнительно > Включить навигацию по декомпилированным источникам**.

![Навигация по декомпилированным источникам в Visual Studio](../ide/media/VS2017Guide-navigate-to-source.png "VS2017Guide-navigate-to-source")

### <a name="i-want-to-run-and-see-my-unit-tests"></a>Мне необходимо выполнять и просматривать модульные тесты.
В Visual Studio 2017 имеется два решения для модульного тестирования: обозреватель тестов и _Live Unit Testing_. В обозревателе тестов версии 15.6 значительно возросла скорость обнаружения тестов. Кроме того, был переработан пользовательский интерфейс, чтобы обеспечить иерархическую сортировку.

В Visual Studio также есть функция модульного тестирования, которая называется [Live Unit Testing](/test/live-unit-testing). Live Unit Testing постоянно работает в фоновом режиме, выполняет тесты, на которые влияют изменения в коде, и обновляет значки в редакторе, сообщая о результатах тестирования.

![Иерархическое представление для обозревателя тестов в Visual Studio](../ide/media/VS2017Guide-hiearchy-test-explorer.png "VS2017Guide-hiearchy-test-explorer")

### <a name="what-other-features-do-i-need-to-know-about"></a>О каких еще функциях мне нужно знать?
Ниже перечислены функции редактора и возможности, повышающие производительность и эффективность написания кода. Некоторые из них может потребоваться включить, так как по умолчанию они отключены (например, потому, что они индексируют данные на компьютере, конфликтуют с другими функциями или в настоящее время являются экспериментальными).
- *Нахождение файла в обозревателе решений* — выделяет активный файл в обозревателе решений.
  - **Сервис > Параметры > Проекты и решения > Отслеживать активный элемент в обозревателе решений**
- *Добавление директив using для типов в базовых сборках и пакетах NuGet* — для типа, на который нет ссылок, выводится значок лампочки с предложением установить пакет NuGet.
  - **Сервис > Параметры > Текстовый редактор > C# > Дополнительно > Предлагать using для типов в эталонных сборках** и **Предлагать using для типов в пакетах NuGet**
- *Включение полного анализа решения* — просмотр всех ошибок в решении в списке ошибок.
  - **Сервис > Параметры > Текстовый редактор > C# > Дополнительно > Включить полный анализ решения**
- *Включение навигации к декомпилированным источникам* — позволяет использовать функцию "Перейти к определению" для типов и членов из внешних источников и применять декомпилятор ILSpy для отображения тел методов.
  - **Сервис > Параметры > Текстовый редактор > C# > Дополнительно > Включить навигацию к декомпилированным источникам**
- *Режим завершения или предложений* в IntelliSense позволяет изменять поведение завершения. Разработчики, использующие фоновые действия IntelliJ, как правило, изменяют этот режим на отличный от режима по умолчанию.
  - **Меню > Правка > IntelliSense > Переключить режим завершения**
- Мы предлагаем *фрагменты кода*, которые можно использовать в качестве стандартных заглушек (дважды нажмите клавишу TAB). См. [полный список](/ide/visual-csharp-code-snippets).

![Фрагменты кода в Visual Studio](../ide/media/VS2017Guide-code-snippet.png "VS2017Guide-code-snippet")

### <a name="missing-a-feature-that-makes-you-productive-or-experiencing-poor-performance"></a>Не хватает функции, которая могла бы сделать вашу работу эффективнее, или испытываете низкую производительность?
Есть несколько способов отправить нам отзыв.
- В нашем [репозитории GitHub](https://github.com/dotnet/roslyn/issues) можно подать запрос функции для .NET.
- Запросы функций, а также сообщения об ошибках и проблемах производительности в Visual Studio можно отправлять с помощью значка **Отправить отзыв** в верхней части окна Visual Studio.

### <a name="live-code-analysis-for-code-quality"></a>Динамический анализ кода для обеспечения качества кода

[Документация: рефакторинг и быстрые действия](refactoring-code-generation-quick-actions.md)

В Visual Studio предусмотрена динамическая диагностика кода, помогающая повысить качество кода за счет обнаружения ошибок и проблем в нем. Мы предоставляем быстрые действия (**CTRL**+**.**) для устранения обнаруженных проблем во всем документе, проекте или решении. Включите *анализ всего решения*, чтобы обнаружить проблемы в рамках всего решения, даже если соответствующие файлы не открыты в редакторе.

Кроме того, вы можете использовать предложения кода для ознакомления с рекомендациями, применять заглушки или генерацию кода, выполнять рефакторинг кода и внедрять новые возможности языка с помощью сочетания клавиш **CTRL**+**.** .

![Применение быстрых исправлений и рефакторинга с помощью меню лампочки](../ide/media/VSIDE_Productivity_CodeAnalysis.png "VSIDE_Productivity_CodeAnalysis")

Вы можете просматривать и проверять тесты в *обозревателе тестов* или мгновенно определять влияние изменений кода на модульные тесты в редакторе с помощью *Live Unit Testing* (только для корпоративного SKU).

### <a name="code-consistency-and-style"></a>Стиль и согласованность кода

- [Документация: параметры переносимого пользовательского редактора](create-portable-custom-editor-options.md)
- [Документация: параметры стиля кода EditorConfig для .NET](editorconfig-code-style-settings-reference.md)

Visual Studio позволяет настраивать конфигурацию соглашения о написании кода, обнаруживает нарушения стилей кода и предоставляет быстрые средства для устранения проблем со стилями через сочетание клавиш **CTRL**+**.** . Вы можете настроить и применить для своей команды соглашения о форматировании, именовании и стилях кода в репозиторий, что позволяет переопределить значения на уровне проектов и файлов с помощью *EditorConfig*.

### <a name="debugging"></a>Отладка

[Документация: отладка в Visual Studio](../debugger/index.md)

Visual Studio включает в себя первоклассный отладчик, позволяющий отлаживать приложения .NET, предназначенные для платформ .NET Framework, .NET Standard и .NET Core. Вы можете задавать условные точки останова и переключаться между ними (**F9**), выполнять вызовы методов по шагам, вычислять LINQ и лямбда-выражения, задавать контрольные значения для переменных, повторно подключаться к процессам, проверять стек вызовов или даже оперативно вносить изменения в код при отладке с помощью функции  *Изменить и продолжить*.

Если ваша служба запущена в Azure, вы можете использовать функцию *Отладка моментальных снимков* для оперативной диагностики проблем в развернутых облачных приложений в Visual Studio 2017 Enterprise.

![Отладка в Visual Studio](../ide/media/VSIDE_Productivity_Debugging.png "VSIDE_Productivity_Debugging")

### <a name="version-control"></a>Управление версиями

[Документация: управление версиями в Visual Studio](/vsts/index)

Вы можете использовать Git или TFVC для хранения и обновления кода в Visual Studio. В редакторе можно организовать локальные изменения с помощью Team Explorer, а также использовать строку состояния для отслеживания ожидающих фиксаций и изменений. Вы можете настроить непрерывную интеграцию и поставку в Visual Studio с помощью расширения [Инструменты непрерывной поставки для Visual Studio](https://marketplace.visualstudio.com/items?itemName=VSIDEDevOpsMSFT.ContinuousDeliveryToolsforVisualStudio), чтобы внедрить гибкий рабочий процесс разработки.

### <a name="extensibility"></a>Расширение среды

[Документация: расширение Visual Studio](../extensibility/index.md)

Visual Studio обладает обширной экосистемой расширений, которые можно устанавливать и создавать по мере необходимости. Установите расширения с *коллекции расширений* или *Visual Studio Marketplace*, создайте собственный подключаемый модуль редактора с помощью *VS SDK* или создайте собственный рефакторинг или анализатор динамического кода с помощью *пакета SDK для платформы компилятора .NET*. Вы можете найти дополнительные исправления кода и предложения, скачав расширение [Microsoft Code Analysis](https://marketplace.visualstudio.com/items?itemName=VisualStudioPlatformTeam.MicrosoftCodeAnalysis2017).
