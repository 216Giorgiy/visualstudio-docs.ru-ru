# <a name="visual-studio-2017-for-net-developers"></a>Visual Studio 2017 для разработчиков на платформе .NET

## <a name="smart-code-editor"></a>Быстрый редактор кода

[Документация: использование IntelliSense](using-intellisense.md)  
[Документация: возможности быстрого редактора](writing-code-in-the-code-and-text-editor.md)

Visual Studio глубоко анализирует код с помощью компилятора .NET ("Roslyn"), чтобы предоставить вам функции быстрого редактирования, такие как раскраска синтаксиса, завершение кода, проверка правописания неправильно введенных переменных, разрешение типов без импорта, структурирование, визуализаторы структуры, [CodeLens](find-code-changes-and-other-history-with-codelens.md), иерархия вызовов, краткие сведения при наведении курсора, справка по параметрам, а также средства для рефакторинга, применения быстрых действий и генерации кода.

![Быстрый редактор кода Visual Studio](../ide/media/VSIDE_Productivity_SmartCodeEditor.png "VSIDE_Productivity_SmartCodeEditor")

## <a name="navigate-and-search-your-codebase"></a>Навигация и поиск в базе кода

[Документация: перемещение по коду](navigating-code.md)

Вы можете быстро перемещаться по коду .NET, переходя прямо к нужному файлу, типу, члену или объявлению символа с помощью команды *Перейти ко всем* (**CTRL+T**). Вы можете найти все ссылки на символ или литерал в коде, включая ссылки на языках .NET (**SHIFT+F12**). Кроме того, команды целевой навигации помогут вам перейти непосредственно к определениям символов (**F12**) или реализациям (**CTRL+F12**).

!["Перейти ко всем" и "Найти все ссылки"](../ide/media/VSIDE_Productivity_Navigation.png "VSIDE_Productivity_Navigation")

## <a name="live-code-analysis-for-code-quality"></a>Динамический анализ кода для обеспечения качества кода

[Документация: рефакторинг и быстрые действия](refactoring-code-generation-quick-actions.md)

В Visual Studio предусмотрена динамическая диагностика кода, помогающая повысить качество кода за счет обнаружения ошибок и проблем в нем. Мы предоставляем быстрые действия (**CTRL+.**) для устранения обнаруженных проблем во всем документе, проекте или решении. Включите *анализ всего решения*, чтобы обнаружить проблемы в рамках всего решения, даже если соответствующие файлы не открыты в редакторе.

Кроме того, вы можете использовать предложения кода для ознакомления с рекомендациями, применять заглушки или генерацию кода, выполнять рефакторинг кода и внедрять новые возможности языка с помощью сочетания клавиш **CTRL+.** .

![Применение быстрых исправлений и рефакторинга с помощью меню лампочки](../ide/media/VSIDE_Productivity_CodeAnalysis.png "VSIDE_Productivity_CodeAnalysis")

## <a name="unit-testing"></a>Модульное тестирование

[Документация: модульное тестирование в Visual Studio](../test/improve-code-quality.md)

Вы можете запускать модульные тесты на основе платформ MSTest, NUnit или XUnit и выполнять их отладку для любого приложения, предназначенного для платформы .NET Framework, .NET Standard или .NET Core. Вы можете просматривать и проверять тесты в *обозревателе тестов* или мгновенно определять влияние изменений кода на модульные тесты в редакторе с помощью *Live Unit Testing* (только для корпоративного SKU). 

![Live Unit Testing в Visual Studio](../ide/media/VSIDE_Productivity_LiveUnitTesting.png "VSIDE_Productivity_LiveUnitTesting")

## <a name="code-consistency-and-style"></a>Стиль и согласованность кода

[Документация: параметры переносимого пользовательского редактора](create-portable-custom-editor-options.md)  
[Документация: параметры стиля кода EditorConfig для .NET](editorconfig-code-style-settings-reference.md)

Visual Studio позволяет настраивать конфигурацию соглашения о написании кода, обнаруживает нарушения стилей кода и предоставляет быстрые средства для устранения проблем со стилями через сочетание клавиш **CTRL+.** . Вы можете настроить и применить для своей команды соглашения о форматировании, именовании и стилях кода в репозиторий, что позволяет переопределить значения на уровне проектов и файлов с помощью *EditorConfig*.

![Настройка и применение соглашений о написании кода с помощью EditorConfig](../ide/media/VSIDE_Productivity_CodeStyle.png "VSIDE_Productivity_CodeStyle")

## <a name="debugging"></a>Отладка

[Документация: отладка в Visual Studio](../debugger/index.md)

Visual Studio включает в себя первоклассный отладчик, позволяющий отлаживать приложения .NET, предназначенные для платформ .NET Framework, .NET Standard и .NET Core. Вы можете задавать условные точки останова и переключаться между ними (**F9**), выполнять вызовы методов по шагам, вычислять LINQ и лямбда-выражения, задавать контрольные значения для переменных, повторно подключаться к процессам, проверять стек вызовов или даже оперативно вносить изменения в код при отладке с помощью функции  *Изменить и продолжить*.

Если ваша служба запущена в Azure, вы можете использовать функцию *Отладка моментальных снимков* для оперативной диагностики проблем в развернутых облачных приложений в Visual Studio 2017 Enterprise.

![Отладка в Visual Studio](../ide/media/VSIDE_Productivity_Debugging.png "VSIDE_Productivity_Debugging")

## <a name="version-control"></a>Управление версиями

[Документация: управление версиями в Visual Studio](/vsts/index)

Вы можете использовать Git или TFVC для хранения и обновления кода в Visual Studio. В редакторе можно организовать локальные изменения с помощью Team Explorer, а также использовать строку состояния для отслеживания ожидающих фиксаций и изменений. Вы можете настроить непрерывную интеграцию и поставку в Visual Studio с помощью расширения [Инструменты непрерывной поставки для Visual Studio](https://marketplace.visualstudio.com/items?itemName=VSIDEDevOpsMSFT.ContinuousDeliveryToolsforVisualStudio), чтобы внедрить гибкий рабочий процесс разработки.

## <a name="popular-extensions--shortcuts"></a>Популярные расширения и сочетания клавиш

Если вы переходите из другой интегрированной среды разработки или среды написания кода, возможно, вам поможет установка одного из следующих расширений:

- [Эмуляция Emacs](https://marketplace.visualstudio.com/items?itemName=JustinClareburtMSFT.EmacsEmulation )
- [Сочетания клавиш в Visual Studio (IntelliJ)](https://marketplace.visualstudio.com/items?itemName=JustinClareburtMSFT.HotKeys)
- [VSVim](https://marketplace.visualstudio.com/items?itemName=JaredParMSFT.VsVim)

Ниже приведены широко используемые сочетания клавиш в Visual Studio. Обратите внимание, что некоторые расширения переопределяют установленные по умолчанию сочетания клавиш в Visual Studio и для использования приведенных ниже команд необходимо восстановить эти сочетания. Чтобы восстановить заданные по умолчанию сочетания клавиш в Visual Studio, выберите **Сервис > Импорт и экспорт параметров... > Сбросить все параметры**.

| Сочетание клавиш (все профили) | Команда | Описание: |
|-|-|-|
| **CTRL+T** | Перейти ко всем | Переход к любому объявлению файла, типа, элемента или символа |
| **F12** (также **CTRL+щелчок**) | Перейти к определению | Переход к месту определения символа |
| **CTRL+F12** | Перейти к реализации | Переход от любого базового типа или элемента к его различным реализациям |
| **SHIFT+F12** | Найти все ссылки | Просмотр всех символьных или литеральных ссылок |
| **CTRL+.** (также **ALT+ВВОД** в профиле C#) | Быстрые действия и операции рефакторинга | Просмотр доступных исправлений кода, действия для создания кода, операций рефакторинга или других быстрых действий в выбранной позиции курсора или для выбранного кода |
| **CTRL**+**E**,**V** | Дублировать строку | Дублирование строки кода, в которой находится курсор (доступно в **Visual Studio 2017 версии 15.6 — предварительная версия 2** и более поздних версиях) |
| **CTRL**+**W** | Развернуть выделение | Развертывание текущего выделения на одну структурную единицу (доступно в **Visual Studio 2017 версии 15.5**) |
| **CTRL**+**SHIFT**+**W** | Свернуть выделение | Свертывание (уменьшение) текущего выделения на одну структурную единицу (доступно в **Visual Studio 2017 версии 15.5**) |
| **CTRL+Q** | Быстрый запуск | Поиск всех параметров Visual Studio |
| **F5** | Начало отладки | Начало отладки приложения |
| **CTRL+F5** | Запуск без отладки | Запуск приложения без отладки |
| **CTRL+K,D** (профиль по умолчанию) или **CTRL+E,D** (профиль C#) | Форматировать документ | Очистка нарушений форматирования в файле на основе заданных параметров перевода строки, интервалов и отступов |
| **CTRL+\\E** (профиль по умолчанию) или **CTRL+W,E** (профиль C#) | Просмотреть список ошибок | Просмотр всех ошибок в документе, проекте или решении |
