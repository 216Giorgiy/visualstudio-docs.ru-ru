# Локализация приложений
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

*Локализация* — это процесс настройки приложения на работу в данных региональных прараметрах или языковой среде.  Основной частью локализации является перевод пользовательского интерфейса на другой язык.  
  
## В этом подразделе  
 [Иерархическая организация ресурсов для локализации](../ide/hierarchical-organization-of-resources-for-localization.md)  
 Хранение локализованных ресурсов и обращение к ним из Visual Studio.  
  
 [Безопасность и локализованные вспомогательные сборки](../ide/security-and-localized-satellite-assemblies.md)  
 Подписывание вспомогательных сборок с помощью пар, состоящих из открытого и закрытого ключей.  
  
 [Номера версий основных и вспомогательных локализованных сборок](../ide/version-numbers-for-main-and-localized-satellite-assemblies.md)  
 Знакомство с классом <xref:System.Resources.SatelliteContractVersionAttribute>, определяющим, какие вспомогательные сборки будут работать с основной сборкой приложения.  
  
 [Языки нейтральных ресурсов для локализации](../ide/neutral-resources-languages-for-localization.md)  
 Знакомство с классом <xref:System.Resources.NeutralResourcesLanguageAttribute>, определяющим язык и региональные параметры ресурсов, включаемых в основную сборку приложения.
