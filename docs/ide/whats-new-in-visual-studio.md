# <a name="what39s-new-in-visual-studio-2017"></a>Новые возможности Visual Studio 2017
Добро пожаловать в версию-кандидат Visual Studio 2017, интегрированный набор средств обеспечения производительности разработчика, облачных служб и расширений, которые позволяют вам и вашей команде создавать уникальные приложения и игры для Интернета, Магазина Windows, настольных компьютеров, для Android и iOS.  

В этой версии-кандидате последней версии Visual Studio мы уделили основное внимание улучшениям производительности. С точки зрения производительности мы ускорили запуск, повысили оперативность реагирования Visual Studio, сократив при этом объем используемой памяти. А с точки зрения повышения эффективности мы добавили или обновили функции, которые помогут вам более продуктивно работать в Visual Studio.

> [!TIP]
> Новую версию-кандидат в действии можно увидеть в видеоролике [What's New in Visual Studio](https://channel9.msdn.com/events/connect/2016/159) (Новые возможности Visual Studio) на канале Channel 9.   

Далее приводится общий обзор внесенных изменений.

* **Повышенная производительность**. Усовершенствования для навигации по коду, IntelliSense, рефакторинга, исправлений кода и отладки экономят ваше время при выполнении повседневных задач независимо от того, какой язык или какую платформу вы используете. Для команд, осваивающих DevOps, Visual Studio 2017 упрощает внутренний цикл и ускоряет продвижение кода с помощью новых функций реального времени, таких как динамическое модульное тестирование и проверка архитектурных зависимостей в режиме реального времени.
* **Пересмотренные основы**.  Фокус сместился в сторону повышения эффективности основополагающих задач, с которыми разработчикам приходится сталкиваться постоянно. От новой упрощенной и модульной установки, которую разработчики могут настроить в соответствии со своими потребностями, ускорения всего цикла работы IDE и до нового способа просмотра, изменения и отладки любого кода без проектов и решений Visual Studio 2017 позволяет разработчикам держать в поле зрения всю структурную картину.
* **Оптимизированная разработка под Azure**. Встроенный набор инструментов Azure, позволяющий разработчикам без проблем создавать ориентированные на облако приложения на базе Microsoft Azure. Visual Studio упрощает настройку, сборку, отладку, упаковку и развертывание приложений и служб в Microsoft Azure прямо из IDE.
* **Широкие возможности для разработки мобильных приложений**. Благодаря улучшенным инструментам отладки и профилирования, а также средствам создания модульных тестов Visual Studio 2017 с Xamarin значительно ускоряет и упрощает сборку, подключение к облаку и настройку мобильных приложений для Android, iOS и Windows. Специалисты также могут разрабатывать мобильные приложения с помощью кроссплатформенных библиотек Visual C++ или Apache Cordova в Visual Studio.  

Далее приведены дополнительные сведения о наиболее существенных изменениях.

## <a name="performance-improvements"></a>Улучшения производительности

### <a name="a-new-setup-experience"></a>Новые возможности установки  
[Скачайте версию-кандидат Visual Studio Enterprise 2017](https://aka.ms/vs/15/preview/vs_enterprise) или [ознакомьтесь с требованиями к системе для Visual Studio](https://www.visualstudio.com/en-us/productinfo/vs2017-system-requirements-vs).

 Мы переработали возможности установки Visual Studio, чтобы еще больше упростить установку только необходимых компонентов. Мы также сократили минимальный объем памяти, поэтому установка Visual Studio выполняется с еще быстрее незначительным влиянием на функционирование системы. Удаление тоже происходит без ошибок.

 Самое важное изменение, которое вы заметите при установке Visual Studio, — это новый интерфейс установки. На вкладке **Рабочие нагрузки** находятся варианты установки, которые представляют распространенные платформы и языки — от разработки классических приложений для .NET до обработки и анализа данных на языках R, Python и F#.  

 ![Диалоговое окно установки версии-кандидата Visual Studio 2017](../ide/media/willow1.png "экран_установки_VS2017RC")

Выбирайте нужные вам рабочие нагрузки и при необходимости изменяйте их. Минимальная установка занимает всего несколько сотен мегабайт, но по-прежнему обеспечивает набор базовых инструментов для редактирования кода на более чем&20; языках, а также управление исходным кодом.

Мы также добавили несколько способов установки Visual Studio. Хотите выбрать собственные компоненты вместо рабочих нагрузок? Просто откройте вкладку **Отдельные компоненты** в установщике. Хотите установить языковые пакеты, не изменяя параметр "Язык Windows"? Откройте вкладку **Языковые пакеты** в установщике.  

Более подробные сведения о новых возможностях установки, включая пошаговые инструкции по их реализации, см. на странице [Install Visual Studio](../install/install-visual-studio.md) (Установка Visual Studio).


### <a name="start-visual-studio-faster"></a>Быстрый запуск Visual Studio
Если Visual Studio обнаруживает замедление запуска IDE, можно обратиться в новый центр производительности Visual Studio, чтобы ускорить процесс. В центре производительности указываются все расширения и окна инструментов, которые снижают скорость запуска IDE. Его можно использовать для повышения производительности запуска, определив запуск расширения или необходимость открытия окон инструментов во время запуска.

### <a name="decrease-solution-load-time"></a>Сокращение времени загрузки решения
Работа с решениями, которые содержат до 100 проектов, не означает, что вам нужно концентрироваться на всех файлах или проектах одновременно. Теперь вы можете выполнять задачи редактирования и отладки без ожидания загрузки каждого проекта в Visual Studio. Чтобы испробовать эти возможности в управляемых проектах, установите флажок **Загрузка упрощенного решения**, предварительно выбрав "Инструменты" -> "Параметры"-> "Проекты и решения".

  ![Диалоговое окно "Параметры" в версии-кандидате Visual Studio 2017](../ide/media/vs2017ide-LightweightSolutionLoad.PNG "диалоговое окно "Параметры" VS2017RC: "Загрузка упрощенного решения"")

### <a name="faster-on-demand-loading-of-extensions"></a>Ускоренная загрузка расширений по требованию
Перед нами стояла простая задача: загружать расширения тогда, когда они понадобятся, а не при запуске Visual Studio. Во-первых, мы перенесли расширения Python и Xamarin для загрузки по требованию. Далее мы работаем над переносом в эту модель всех расширений, входящих в состав Visual Studio, и расширений от сторонних поставщиков. Хотите узнать, какие расширения влияют на скорость запуска, загрузки решений и ввода данных? Эту информацию можно найти, выбрав "Справка"-> "Управление производительностью Visual Studio".

  ![Диалоговое окно "Параметры" в версии-кандидате Visual Studio 2017](../ide/media/vs2017ide-ManageVSperf.png "Диалоговое окно "Справка" VS2017RC: "Управление производительностью"")

## <a name="productivity-improvements"></a>Улучшения продуктивности работы

### <a name="sign-in-across-multiple-accounts"></a>Вход для нескольких учетных записей  
В версии-кандидате Visual Studio 2017 представлена новая служба удостоверений, которая позволяет совместно использовать учетные записи пользователей в инструментах разработчика Майкрософт, таких как Team Explorer, Microsoft Azure Tools, публикация для Магазина Windows и т. д.

Кроме того, увеличено время пребывания в системе — вам не требуется выполнять повторный вход каждые 12 часов. Дополнительные сведения см. в записи блога [Fewer Visual Studio Sign-in Prompts](https://blogs.msdn.microsoft.com/visualstudio/2016/08/15/fewer-visual-studio-sign-in-prompts/) (Сокращение количества запросов на вход в Visual Studio).

### <a name="manage-your-extensions-with-roaming-extensions-manager"></a>Управление расширениями с помощью диспетчера перемещаемых расширений
Теперь при входе в Visual Studio настройка каждой среды разработки с использованием избранных модулей стала еще проще. Новый диспетчер перемещаемых расширений отслеживает все ваши избранные модули, создав синхронизируемый список в облаке.  

Чтобы просмотреть список расширений в Visual Studio, выберите "Инструменты" > "Расширения и обновления", а затем — "Диспетчер перемещаемых расширений".

![Visual Studio 2017: диалоговое окно "Расширения и обновления"](../ide/media/vs2017ide-ExtensionsAndUpdates.png "Visual Studio 2017: "Инструменты > Расширения и обновления"")

Диспетчер перемещаемых расширений отслеживает все установленные расширения, но вы можете выбрать те, которые нужно добавить в список перемещаемых.

![Visual Studio 2017: диалоговое окно "Расширения и обновления"](../ide/media/vs2017ide-RoamingExtensionManager.png "Visual Studio 2017: диспетчер перемещаемых расширений")

При использовании диспетчера перемещаемых расширений вы заметите в списке 3 типа значков.
* ![Значок перемещаемого расширения](../ide/media/vs2017ide-roamedicon.png "Значок перемещаемого расширения")***Значок перемещаемого расширения***: расширение включено в список перемещаемых, но еще не установлено на этом компьютере.
  (Такие расширения можно установить с помощью кнопки **Скачать**.)
* ![Значок перемещаемого и установленного расширения](../ide/media/vs2017ide-roamedinstalledicon.png "Значок перемещаемого и установленного расширения")***Значок перемещаемого и установленного расширения***: все расширения включены в список перемещаемых и установлены в среде разработки.
  (Если вы решите, что перемещать расширение не нужно, его можно удалить с помощью кнопки **Остановить перемещение**.)
* ![Значок установленного расширения](../ide/media/vs2017ide-installedicon.png "Значок установленного расширения")***Значок установленного расширения***: все расширения установлены в среде, но не включены в список перемещаемых.
  (Расширения можно добавить в список перемещаемых с помощью кнопки **Начать перемещение**.)

Эти значки отражают текущее состояние вашего списка. Меняйте состояние любых расширений и настраивайте содержимое среды как вам нравится. А если захотите, мы сделаем это за вас. Любое расширение, скачанное после того, как вы выполнили вход в систему, будет добавлено в список перемещаемых как **перемещаемое и установленное**, а значит, будет доступно вам на любом компьютере.

### <a name="experience-live-architecture-dependency-validation-and-live-unit-testing"></a>Динамическая проверка зависимостей архитектуры и динамическое модульное тестирование

Если настроены схемы проверки зависимостей (также называемые "схемы слоев"), в версии-кандидате Visual Studio Enterprise 2017 теперь можно получать уведомления о нарушениях правил архитектурных зависимостей в режиме реального времени при вводе кода в редакторе кода.

Ошибки отображаются в списке ошибок, а точное расположение проблемы обозначается в текстовом редакторе волнистыми линиями. Это снижает вероятность привнесения нежелательных зависимостей.

![Динамическая проверка архитектуры](../ide/media/vs2017ide-LiveArchitectureDepedendencyValidation.png "Динамическая проверка зависимостей архитектуры")

#### <a name="live-unit-testing"></a>Динамическое модульное тестирование

Динамическое модульное тестирование — это новая возможность, которая присутствует только в выпуске Visual Studio Enterprise. Эта функция визуализирует результаты модульного теста и объем протестированного кода в редакторе при написании кода. Она работает в проектах C# и Visual Basic для .NET Framework и поддерживает три платформы тестирования — MSTest, NUnit и xUnit.

#### <a name="interact-with-git"></a>Взаимодействие с Git
Элементы управления в нижнем углу интегрированной среды разработки Visual Studio позволяют быстро фиксировать и публиковать проекты в Git и управлять репозиториями Git.

![Версия-кандидат Visual Studio 2017: диалоговое окно настройки](../ide/media/vsIDE-GitInteraction.png "Инструменты Git в интегрированной среде разработки версии-кандидата VS2017")

#### <a name="view-and-navigate-code-with-structure-visualizer"></a>Просмотр кода и навигация по коду с помощью визуализатора структуры
В редакторе кода Visual Studio доступна новая функция, называемая визуализатором структуры. Эта функция отображает вертикальные направляющие между вложенными областями кода, упрощая просмотр кода и навигацию по нему. Эта функция доступна для всех языков с поддержкой TextMate, а также для Visual C#, Visual Basic и XAML.

![Версия-кандидат Visual Studio 2017: диалоговое окно настройки](../ide/media/vsIDE-StructureVisualizer.png "Визуализатор структуры в версии-кандидате VS2017")

#### <a name="experience-an-improved-navigate-to"></a>Улучшенная функция "Перейти к"
Мы улучшили функцию "Перейти к". Мы упростили окно "Перейти к" и добавили поддержку дополнительных символов фильтра для сужения областей поиска в коде.

#### <a name="create-apps-in-even-more-programming-languages"></a>Создание приложений на дополнительных языках программирования
В Visual Studio увеличено количество языков программирования для создания приложений, поэтому решения и проекты больше не требуются. Код поддерживает цветовое выделение синтаксиса, завершение базовых операторов и, в некоторых случаях, функцию "Перейти к" и другие возможности. Если предпочитаемый вами язык не поддерживается, для него можно создать поддержку с помощью грамматик TextMate.

### <a name="visual-c"></a>Visual C++
В версии-кандидате Visual Studio 2017 реализовано множество изменений и исправлений для среды Visual C++. Мы устранили более 250 ошибок и заявленных проблем в компиляторе и других инструментах. Информация о многих из этих ошибок поступила к нам от клиентов через [Microsoft Connect](https://connect.microsoft.com/VisualStudio "Microsoft Connect").

Мы внесли несколько улучшений, а именно: включили распространение основных рекомендаций C++ в Visual Studio, обновили компилятор, добавив расширенную поддержку функций C ++&11; и C++, добавили и обновили функциональные возможности библиотек C++ и повысили производительность интегрированной среды разработки C++, рабочих нагрузок установки и т. д.

Дополнительные сведения см. на странице [What's New for Visual C++ in Visual Studio 2017 RC](/cpp/top/what-s-new-for-visual-cpp-in-visual-studio) (Новые возможности Visual C++ в версии-кандидате Visual Studio 2017).  


### <a name="debugging-and-diagnostics"></a>Отладка и диагностика
Теперь отладка выполняется быстрее и не приводит к задержкам при редактировании.

Например: в более ранней версии Visual Studio мы представили так называемый процесс размещения для WPF, Windows Forms и управляемых консольных проектов, чтобы ускорить процесс отладки, активировав его в фоновом режиме для использования в следующем сеансе отладки. Эта разработанная с благими намерениями функция приводила к ситуации, в которой Visual Studio временно перестает отвечать после остановки отладки или использования Visual Studio по завершении сеанса отладки.

В Visual Studio 2017 мы отключили процесс размещения и оптимизировали отладку, чтобы она так же быстро выполнялась и без процесса размещения и даже быстрее для проектов, в которых никогда не использовался процесс размещения (например, для проектов ASP.NET, универсальных приложений Windows и C++).

#### <a name="run-to-click"></a>Выполнение до щелкнутого:
теперь во время отладки вы можете щелкнуть значок рядом со строкой кода, чтобы выполнить эту строку. Вам больше не нужно задавать временные точки останова или совершать несколько действий, чтобы выполнить код и остановиться на требуемой строке.

![Отладка в версии-кандидате Visual Studio 2017 — значок "Выполнение до щелкнутого"](../ide/media/vs2017ide-RunToClick.png "Значок "Выполнение до щелкнутого" при отладке и диагностике в Visual Studio 2017")

#### <a name="the-new-exception-helper"></a>Новый помощник по исправлению ошибок:

используйте новый помощник по исправлению ошибок для беглого просмотра сведений об исключении в компактном немодальном диалоговом окне с мгновенным доступом к внутренним исключениям.

При диагностике исключения NullReferenceException вы сможете быстро увидеть, какой параметр имел значение NULL, прямо в помощнике по исправлению ошибок.

Можно также указать не прерывать работу на типах исключений, создаваемых определенными модулями. Для этого нужно установить флажок добавления условия во время остановки на созданном исключении.

![Диалоговое окно нового помощника по исправлению ошибок](../ide/media/vs2017ide-ExceptionHelper.png "Диалоговое окно нового помощника по исправлению ошибок")

Дополнительные сведения см. в публикации [Using the New Exception Helper in Visual Studio](https://blogs.msdn.microsoft.com/visualstudioalm/2016/03/31/using-the-new-exception-helper-in-visual-studio-15-preview/) (Использование нового помощника по исправлению ошибок в Visual Studio).

## <a name="talk-to-us"></a>Обращайтесь к нам  
 Зачем отправлять отзыв группе Visual Studio? Потому что мы серьезно относимся к отзывам клиентов. Они влияют на многие наши действия.  

Если вы хотите внести предложение по улучшении Visual Studio или сообщить о проблеме, см. страницу [Talk to Us](../ide/talk-to-us.md) (Обращайтесь к нам).  

### <a name="report-a-problem"></a>Сообщите о проблеме  
 Иногда для передачи всех последствий возникшей проблемы простого сообщения недостаточно. В случае зависания, сбоя или других проблем с производительностью вы можете предоставить нам шаги для воспроизведения и вспомогательные файлы (например, снимки экрана и файлы дампа трассировки и кучи) с помощью средства **Сообщить о проблеме**. Дополнительные сведения об использовании этого средства см. на странице [How to Report a Problem](how-to-report-a-problem-with-visual-studio-2017.md) (Как сообщить о проблеме).  

### <a name="track-your-issue-in-connect"></a>Отслеживание проблемы в Connect  
 Если требуется отследить состояние отзыва о Visual Studio, перейдите в модуль [Connect](http://connect.microsoft.com/) и сообщите там об ошибке. После отправки можно вернуться в Connect и отслеживать его состояние.
