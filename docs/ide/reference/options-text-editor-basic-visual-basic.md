# &quot;Параметры&quot;, &quot;Текстовый редактор&quot;, Basic (Visual Basic)
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

Страница свойств **Только для VB**, находящаяся в подпапке **Basic** папки **Текстовый редактор** диалогового окна **Параметры** \(меню **Сервис**\), содержит следующие свойства.  
  
 **Автоматическая вставка завершающих конструкций**  
 При вводе, к примеру, первой строки объявления процедуры `Sub Main—`и нажатия клавиши ENTER текстовый редактор добавляет соответствующую строку `End Sub`.  Аналогично, при добавлении цикла [For](/dotnet/visual-basic/language-reference/statements/for-next-statement) текстовый редактор добавляет соответствующий оператор `Next`.  Если выбран этот параметр, редактор кода автоматически добавляет конструкцию end.  
  
 **Автоматическое форматирование кода**  
 Текстовый редактор переформатирует код надлежащим образом.  Если выбран этот параметр, редактор кода выполняет следующие действия:  
  
-   выравнивает код по правильной позиции табуляции;  
  
-   исправляет регистр ключевых слов, переменных и объектов;  
  
-   добавляет отсутствующее слово `Then` в оператор `If...Then`;  
  
-   добавляет скобки к вызовам функций;  
  
-   добавляет недостающие закрывающие кавычки в строки;  
  
-   переформатирует экспоненциальное представление;  
  
-   переформатирует даты.  
  
 **Включать режим создания структуры**  
 При открытии файла в редакторе кода документ можно просматривать в режиме создания структуры.  Дополнительные сведения см. в разделе [Структура](../../ide/outlining.md).  Если выбран этот параметр, режим создания структуры будет включаться при открытии файла.  
  
 **Автоматическая вставка членов Interface и MustOverride**  
 При добавлении оператора `Implements` или `Inherits` для класса текстовый редактор вставляет прототипы членов, которые должны быть реализованы или переопределены, соответственно.  
  
 **Показывать разделительные линии процедур**  
 Текстовый редактор показывает визуальную область действия процедур.  В исходных файлах проекта \(VB\-файлы\) отображается линия в местах, перечисленных в следующей таблице.  
  
|Позиция в исходном VB\-файле|Пример расположения линии|  
|----------------------------------|-------------------------------|  
|После закрытия конструкции объявления блока|-   В конце класса, структуры, модуля, интерфейса или перечисления<br />-   После свойства, функции или вложенного объекта<br />-   Не между предложениями get и set в свойстве|  
|После набора конструкций из одной строки|-   После операторов импорта, до определения типа в файле класса<br />-   После объявленных в классе переменных, до любых процедур|  
|После объявлений из одной строки \(но не объявлений на уровне блока\)|-   После операторов Import, Inherits, объявлений переменных, событий, делегатов и операторов объявления библиотек DLL|  
  
 **Включать предложения по исправлению ошибок**  
 Текстовый редактор может предлагать решения стандартных ошибок, позволяя пользователю выбирать необходимое исправление, и применять их к коду.  
  
 **Включение выделения ссылок и ключевых слов**  
 Текстовый редактор может выделить все экземпляры символа или все ключевые слова в предложении, например `If..Then` `While...End While` или `Try...Catch...Finally`.  Можно перемещаться между отмеченными ссылками и ключевыми словами с помощью сочетаний клавиш CTRL\+SHIFT\+СТРЕЛКА ВНИЗ или CTRL\+SHIFT\+СТРЕЛКА ВВЕРХ.
