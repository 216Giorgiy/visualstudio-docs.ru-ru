# Страница &quot;Параметры&quot;, свойства узла отладки
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

В следующих таблицах описываются страницы \(или коллекции свойств\) диалогового окна **Параметры**, связанные с категорией **Отладка** — `DTE.Properties("Debugging", <Property Page>)`.  
  
## Общие  
 `DTE.Properties("Debugging", "General")`  
  
|Имя элемента свойства|Значение|Описание|  
|---------------------------|--------------|--------------|  
|PromptOnBreakpointDelete|Get\/Set \(Boolean\)|Определяет, должен ли отладчик запрашивать разрешение перед удалением всех точек останова в проекте.|  
|BreakAllProcesses|Get\/Set \(Boolean\)|Определяет, должен ли отладчик прерывать все процессы при остановке одного процесса.|  
|BreakAtBoundaries|Get\/Set \(Boolean\)|Определяет, следует ли отладчику прерывать выполнение, если исключение переходит границу между доменами AppDomain или границу между управляемым и машинным кодом.|  
|EnableAddressLevelDebugging|Get\/Set \(Boolean\)|Указывает, включены ли функции отладки на уровне адресов.|  
|ShowDisassemblyIfNoSource|Get\/Set \(Boolean\)|Определяет, следует ли отладчику отображать дизассемблированный код, если исходный код недоступен.|  
|EnableBreakpointFilters|Get\/Set \(Boolean\)|Указывает, включена ли фильтрация точек останова.|  
|EnableExceptionAssistant|Get\/Set \(Boolean\)|Определяет, используется ли для управления исключениями помощник по исключениям.|  
|UnwindCallstack|Get\/Set \(Boolean\)|Определяет, следует ли отладчику очищать стек вызовов для необработанных исключений.|  
|EnableJustMyCode|Get\/Set \(Boolean\)|Определяет, включен ли режим "Только мой код" для кода C\# и Visual Basic.|  
|ShowAllMembers|Get\/Set \(Boolean\)|Для объектов, не являющихся пользовательскими, определяет, следует ли отладчику отображать все члены объектов в окнах переменных.  Этот параметр применяется только при включении режима "Только мой код".|  
|WarnIfNoUserCode|Get\/Set \(Boolean\)|Определяет, должен ли отладчик выводить предупреждение, если пользователь пытается присоединить процесс, которой не содержит пользовательский код.  Этот параметр применяется только при включении режима "Только мой код".|  
|EnablePropertyEvaluation|Get\/Set \(Boolean\)|Определяет, следует ли отладчику автоматически вычислять свойства и неявные вызовы функции в управляемом коде.|  
|CallStringConversion|Get\/Set \(Boolean\)|Определяет, следует ли отладчику неявно вызывать функцию преобразования строк для объектов в окнах переменных.  Этот параметр применяется только для кода C\# и JScript.|  
|EnableSourceServer|Get\/Set \(Boolean\)|Определяет, может ли отладчик получать доступ кода с сервера системы управления версиями.|  
|PrintSourceServerDiagnostics|Get\/Set \(Boolean\)|Определяет, отображаются ли в окне вывода диагностические сообщения, связанные с сервером системы управления версиями.  Этот параметр применяется только при включении доступа к серверу системы управления версиями.|  
|HighlightEntireLine|Get\/Set \(Boolean\)|Определяет, следует ли отладчику выделять всю строку исходного кода для точек останова и текущего оператора.|  
|RequireSourceToMatch|Get\/Set \(Boolean\)|Определяет, следует ли отладчику запрашивать файлы исходного кода, чтобы выполнять точное сравнение с исходной версией при открытии файлов для отладки.|  
|RedirectOutputToImmediate|Get\/Set \(Boolean\)|Определяет, перенаправляются ли выходные данные из окна вывода в окно интерпретации.|  
|ShowRawVariableStructure|Get\/Set \(Boolean\)|Определяет, отображаются ли объекты в окнах переменных в необработанном виде.|  
|SuppressJitOptimization|Get\/Set \(Boolean\)|Для управляемого кода определяет, подавляется ли отладчиком JIT\-оптимизация.|  
|WarnIfNoSymbols|Get\/Set \(Boolean\)|Определяет, должен ли отладчик выводить предупреждение, если при запуске процесса недоступны отладочные символы.|  
|WarnIfScriptDisabled|Get\/Set \(Boolean\)|Определяет, должен ли отладчик выводить предупреждение, если при запуске процесса не включена отладка скриптов.|  
|ShowMarkersForAllThreads|Get\/Set \(Boolean\)|Определяет, следует ли отладчику отображать маркеры потоков.|  
|StepOverPropertiesAndOperators|Get\/Set \(Boolean\)|Указывает, следует ли обходить свойства и операторы \(только в управляемом коде\).|  
  
## Режим "Изменить и продолжить"  
 `DTE.Properties("Debugging", "EditAndContinue")`  
  
|Имя элемента свойства|Значение|Описание|  
|---------------------------|--------------|--------------|  
|EnableEditAndContinue|Get\/Set \(Boolean\)|Указывает, включен ли режим "Изменить и продолжить".  Этот параметр применяется ко всем языкам, поддерживающим режим "Изменить и продолжить".|  
|InvokedByCommands|Get\/Set \(Boolean\)|Определяет, применяются ли автоматически в режиме "Изменить и продолжить" изменения кода, когда пользователь выбирает такие команды отладки, как **Шаг** или **Продолжить**.  Этот параметр применяется только к машинному коду.|  
|InvokedByCommandsAskFirst|Get\/Set \(Boolean\)|Определяет, следует ли выводить в режиме "Изменить и продолжить" запрос на применение изменений кода, когда пользователь выбирает такие команды отладки, как **Шаг** или **Продолжить**.  Этот параметр применяется только к машинному коду.|  
|WarnAboutStaleCode|Get\/Set \(Boolean\)|Определяет, следует ли отладчику выводить предупреждение, если включение режима "Изменить и продолжить" приведет к выполнению необновленного или устаревшего кода.  Этот параметр применяется только к машинному коду.|  
|RelinkChangesOnStop|Get\/Set \(Short\)|Определяет, следует ли выполнять в среде Visual Studio повторную компоновку изменений кода, примененных в режиме "Изменить и продолжить", если выполнение приложения останавливается.  Этот параметр применяется только к машинному коду.|  
|AllowPrecompiling|Get\/Set \(Short\)|Определяет, разрешается ли в режиме "Изменить и продолжить" загружать предварительно скомпилированные заголовки в фоновом режиме.  Этот параметр применяется только к машинному коду.|  
  
## JIT  
 `DTE.Properties("Debugging", "JustInTime")`  
  
|Имя элемента свойства|Значение|Описание|  
|---------------------------|--------------|--------------|  
|JitManaged|Get\/Set \(Boolean\)|Определяет, включена ли JIT\-отладка для управляемого кода.|  
|JitNative|Get\/Set \(Boolean\)|Определяет, включена ли JIT\-отладка для машинного кода.|  
|JitScript|Get\/Set \(Boolean\)|Определяет, включена ли JIT\-отладка для кода скриптов.|  
  
## Машинный код  
 `DTE.Properties("Debugging", "Native")`  
  
|Имя элемента свойства|Значение|Описание|  
|---------------------------|--------------|--------------|  
|LoadDllExports|Get\/Set \(Boolean\)|Определяет, следует ли отладчику загружать таблицы экспорта библиотеки DLL.|  
|EnableRPC|Get\/Set \(Boolean\)|Определяет, может ли отладчик выполнить пошаговые вызовы удаленных процедур COM.|
