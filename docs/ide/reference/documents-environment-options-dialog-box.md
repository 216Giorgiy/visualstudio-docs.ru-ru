# Страница &quot;Документы&quot;, папка &quot;Среда&quot;, диалоговое окно &quot;Параметры&quot;
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

Данная страница диалогового окна **Параметры** служит для управления отображением документов в среде разработки и внешними изменениями, касающимися различных документов и файлов.  Чтобы открыть это диалоговое окно, в меню **Сервис** выберите **Параметры**, затем откройте папку **Среда** и перейдите на страницу **Документы**.  Если страница **Документы** отсутствует в списке, установите флажок **Показать все параметры** в диалоговом окне **Параметры**.  
  
> [!NOTE]
>  Доступные в диалоговых окнах параметры, а также названия и расположение команд меню могут отличаться от описанных в справке в зависимости от текущих параметров или версии.  Чтобы изменить параметры, в меню **Сервис** выберите команду **Импорт и экспорт параметров**.  Дополнительные сведения см. в разделе [Customizing Development Settings in Visual Studio](http://msdn.microsoft.com/ru-ru/22c4debb-4e31-47a8-8f19-16f328d7dcd3).  
  
 **Повторно использовать окно текущего документа, если он сохранен**  
 Загрузка нового документа в окно закрываемого текущего документа, если последний сохранен.  Если текущий документ не сохранен, он остается открытым, а новый открывается в отдельном окне.  Если данный параметр снят, новый документ всегда открывается в отдельном окне.  
  
 Если при работе с несколькими документами вырезание и вставка данных применяются нечасто, и при этом нужно снизить число открытых документов и окон в рабочей области, можно включить данную функцию.  
  
 **Обнаруживать факт изменения файла вне среды**  
 Если этот параметр выбран, то при внесении изменений в открытый файл с помощью редактора, не входящего в состав интегрированной среды разработки, будет немедленно отображено соответствующее сообщение.  Получив данное сообщение, можно повторно загрузить файл из хранилища.  
  
 **Автоматически загружать изменения, сделанные вне среды**  
 Если установлен флажок **Обнаруживать факт изменения файла вне среды**, а в файл, открытый в интегрированной среде разработки, были внесены изменения не в IDE, то по умолчанию создается предупреждающее сообщение.  Если этот флажок установлен, предупреждающее сообщение не отображается, а документ повторно загружается в интегрированную среду разработки с учетом внесенных изменений.  
  
 **Изменение файлов только для чтения с запросом при сохранении**  
 Разрешение на открытие и изменение файлов, предназначенных только для чтения.  Когда вы закончите, необходимо использовать  **Сохранить как**команду, чтобы сохранить файл под новым именем, если вы хотите сохранить запись изменений.  
  
 **Открывать файлы в каталоге текущего активного документа**  
 При выбранном параметре включается отображение папки текущего активного документа в диалоговом окне **Открыть файл**.  Если данный параметр снят, в диалоговом окне **Открыть файл** отображается папка, в которой располагается последний из открытых файлов.  
  
 **Проверять при загрузке окончания строк на однотипность**  
 Проверка однородности окончаний строк и отображение сообщения в случае несоответствия.  
  
 **Отображать предупреждение, если глобальная отмена влияет на измененные файлы**  
 Выберите этот параметр, чтобы отображать сообщение, если при выполнении команды **Глобальная отмена** будут отменены изменения оптимизации в файлах, которые были изменены после операции рефакторинга.  При возвращении файлов в состояние, которое было до оптимизации, могут быть утрачены последующие изменения файлов.  
  
 **Показывать прочие файлы в обозревателе решений**  
 Если установить данный флажок, в **обозревателе решений** будет отображаться узел **Прочие файлы**.  В узле "Прочие файлы" представлены файлы, не связанные с проектом или решением, но отображаемые в **обозревателе решений** для удобства пользователя.  
  
> [!NOTE]
>  Если выбрать данный элемент управления, для веб\-документов, не включенных в активное веб\-приложение, можно будет использовать расположенную в меню **Файл** команду **Просмотр в браузере**.  
  
 **\<** *n* **\> элементов сохранено в проекте "Прочие файлы"**  
 Указывает число файлов, которое будет отображаться в **обозревателе решений** в узле **Прочие файлы**.  Эти файлы будут отображаться, даже если они не открыты в редакторе.  Можно задать любое целое число от 0 до 256.  По умолчанию включается 0 файла.  
  
 Например, если установить этому параметру значение 5 и открыть 10 файлов из узла "Прочие файлы", то при закрытии всех 10 файлов первые 5 файлов будут по\-прежнему отображаться в папке **Прочие файлы**.  
  
 **Использовать Юникод, если данные невозможно сохранить с кодовой страницей**  
 Выберите этот параметр, чтобы в файлах, содержащих данные, которые несовместимы с выбранной кодовой страницей, по умолчанию при сохранении использовать Юникод.
