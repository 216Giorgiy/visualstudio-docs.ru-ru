# Диалоговое окно &quot;Дополнительные параметры служб&quot;
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

Службы клиентских приложений упрощают доступ к службам входа, ролей и профилей [!INCLUDE[ajax_current_short](../../ide/reference/includes/ajax_current_short_md.md)] из приложений Windows Forms и WPF \(Windows Presentation Foundation\).  Для настройки служб клиентских приложений используется страница **Службы** в **Конструкторе проектов**.  Дополнительные сведения о странице **Службы** см. в разделе [Страница "Службы" в конструкторе проектов](../../ide/reference/services-page-project-designer.md).  
  
 Для настройки дополнительных параметров клиентских служб приложений используйте диалоговое окно **Дополнительные параметры служб** страницы **Службы** в **Конструкторе проектов**.  С помощью этих параметров можно переопределить поведение служб приложений по умолчанию, чтобы реализовать менее распространенные сценарии.  Дополнительные сведения см. в разделе [Службы клиентских приложений](../Topic/Client%20Application%20Services.md).  
  
 Чтобы открыть окно **Дополнительные параметры служб**, выберите узел проекта в **Обозревателе решений** и затем в меню **Проект** выберите команду **Свойства**.  В окне **Конструктор проектов** откройте вкладку **Службы** и нажмите кнопку **Дополнительно**.  Если службы клиентских приложений отключены, эта кнопка недоступна.

## Список элементов пользовательского интерфейса  
 **Сохранить хэш пароля в локальной системе для входа в автономном режиме**  
 Указывает, что в локальной системе будет в зашифрованном виде кэшироваться пароль пользователя, чтобы пользователь мог осуществлять вход, когда приложение находится в автономном режиме.  Дополнительные сведения см. в разделе [How to: Work Offline with Client Application Services](http://msdn.microsoft.com/ru-ru/f792cb16-8520-4a0f-9dc9-07bfbc454e38).  Этот параметр устанавливается по умолчанию.  
  
 **Требовать, чтобы пользователи повторяли вход, если у файла cookie сервера истек срок действия**  
 Указывает, будут ли прошедшие проверку подлинности пользователи автоматически получать доступ, когда приложение обращается к ролям или службе профиля, а срок действия файла Cookie проверки подлинности на сервере истек.  Выберите этот вариант, чтобы запретить доступ к службам приложений и требовать явной повторной проверки подлинности после истечения срока действия файла Cookie.  Этот параметр полезен при развертывании приложений на общедоступных системах, чтобы пользователи, которые прекратили пользоваться приложением, не входили в систему автоматически.  По умолчанию этот параметр не установлен.  
  
 **Срок действия кэша службы ролей**  
 Указывает срок, в течение которого поставщик клиентских ролей будет использовать значения ролей из кэша, вместо того чтобы обращаться к службе ролей.  Установите небольшой срок действия для часто обновляемых ролей и более длительный срок для редко обновляемых ролей.  Значение по умолчанию — один день.  
  
 Поставщик ролей получает кэшированные значения ролей или службы ролей при вызове метода <xref:System.Web.Security.RolePrincipal.IsInRole%2A>.  Для программной очистки кэша и принудительного доступа к удаленной службе требуется вызвать метод <xref:System.Web.ClientServices.Providers.ClientRoleProvider.ResetCache%2A>.  
  
 **Использовать пользовательскую строку подключения**  
 Указывает, что поставщики клиентских служб будут использовать в качестве локального кэша пользовательское хранилище данных.  По умолчанию поставщики служб используют в качестве кэша локальную файловую систему.  Если выбран этот параметр, в текстовое поле будет автоматически подставляться строка подключения по умолчанию.  Строка подключения по умолчанию позволяет создать и использовать базу данных SQL Server Compact Edition. Можно также указать строку подключения к существующей базе данных SQL Server.  Дополнительные сведения см. в разделе [Практическое руководство. Настройка служб клиентских приложений](../Topic/How%20to:%20Configure%20Client%20Application%20Services.md).  По умолчанию этот параметр не установлен.
