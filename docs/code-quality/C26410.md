# <a name="c26410--noreftoconstuniqueptr"></a>C26410 NO_REF_TO_CONST_UNIQUE_PTR
Как правило ссылки на константный уникальный указатель не имеют смысла. Их можно безопасно заменить необработанные ссылкой или указателем.

## <a name="remarks"></a>Примечания
- Уникальный указатель проверок имеют довольно широкий условия для определения интеллектуальные указатели. Правило R.31: *при наличии не std интеллектуальные указатели, выполните базовый шаблон из std описывает уникальный указатель и общие понятия указатель*. Эвристика прост, но может привести к сюрпризов: интеллектуальный указатель типа является любой тип, который определяет один оператор "->" или оператор\*; может копировать тип (общий указатель) должен иметь открытый конструктор копий или оператор присваивания перегруженных которой обрабатывает R значение ссылочного параметра.
- Код шаблона может создавать много шума. Имейте в виду, что можно создавать экземпляры шаблонов с различными параметрами типа с разными уровнями косвенного обращения, включая ссылки. Некоторые предупреждения могут быть неявными и исправления может потребовать повторной шаблонов (например, явного удаления косвенного обращения ссылки). Если намеренно универсального шаблона код, предупреждение можно подавить.

## <a name="example-1-unnecessary-reference"></a>Пример 1: Ненужные ссылки
```cpp
std::vector<std::unique_ptr<Tree>> roots = GetRoots();
std::for_each(
    roots.begin(), 
    roots.end(), 
    [](const auto &root) { Rebalance(root.get()); });   // C26410
```
