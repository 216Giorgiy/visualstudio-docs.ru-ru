# Практическое руководство. Настройка анализа кода для веб-приложения ASP.NET
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

[!INCLUDE[vsPreShort](../code-quality/includes/vspreshort_md.md)] и [!INCLUDE[vsUltShort](../code-quality/includes/vsultshort_md.md)] позволяют выбрать в списке анализа кода *наборы правил* для применения к проекту веб\-приложения [!INCLUDE[vstecasp](../code-quality/includes/vstecasp_md.md)].  Набор правил по умолчанию — минимальные правила и рекомендации корпорации Майкрософт.  При желании к веб\-сайту можно применить другой набор правил.  
  
### Настройка набора правил для проекта платформы веб\-страниц ASP.NET  
  
1.  Выберите веб\-сайт в окне **Обозреватель решений**.  
  
2.  В меню **Анализ** выберите команду **Настройка анализа кода для веб\-сайта**.  
  
3.  Если выбрано решение, и в нем содержится более одного проекта, выберите конфигурацию построения и целевую операционную систему в списках **Конфигурация** и **Платформа**.  
  
4.  Для каждого проекта в решении щелкните столбец **Набор правил**, затем щелкните имя запускаемого набора правил.  
  
5.  По умолчанию анализ кода запускается для всех проектов решения.  Чтобы отключить или включить анализ кода для конкретного проекта, выполните следующие действия.  
  
    1.  Щелкните правой кнопкой имя проекта и выберите "Свойства".  
  
    2.  Установите или снимите флажок **Включить анализ кода**.  Также можно выполнить анализ кода в ручном режиме, открыв меню **Анализ** и выбрав пункт **Выполнить анализ кода на веб\-узле**.  
  
6.  В раскрывающемся списке **Выполнить этот набор правил** выполните следующие действия.  
  
    -   Выберите набор правил, который следует использовать.  
  
    -   Выберите **\<Обзор...\>**, чтобы указать существующий настраиваемый набор правил, если его нет в списке.  
  
    -   Определите настраиваемый набор правил.  Для получения дополнительной информации см. [Создание настраиваемых наборов правил](../code-quality/creating-custom-code-analysis-rule-sets.md).
