# <a name="c6278"></a>C6278
Предупреждение C6278: \<переменная > выделена при помощи оператора new [] массив, а удаляется скалярным оператором delete. Не будут вызваны деструкторы

Это предупреждение возникает только в коде C++ и указывает, что вызывающая функция выделила память с массивом **new []** оператор, но освободила ее скалярным **удалить** оператор. Это неопределенное поведение в соответствии со стандартом C++ и реализации Microsoft Visual C++. Существует по крайней мере трех причин, это может привести к проблемам.

- Вызываются конструкторы для отдельных объектов в массиве, но не вызываются деструкторы.

- Если глобальный или класса, специфичные для **оператор new** и **оператор delete** не совместимы с **оператор new []** и **оператор delete []**, могут возникнуть непредвиденные результаты.

- Всегда рискованно полагаться на неопределенное поведение.

Точное последствия этой неполадки предсказать их трудно. Это может привести к утечки сведений для классов с деструкторами, выполнить отмену выделения памяти; несогласованное поведение для классов с деструкторами, выполняющими семантически различные операции; или возможно повреждение памяти и сбои при переопределении операторов. В других случаях несоответствие может не иметь существенного значения в зависимости от реализации компилятора и его библиотек. Средство анализа не всегда различает эти ситуации.

Если память выделяется с массивом **new []**, она должна быть обычно освобождаться с массивом **delete []**.

## <a name="example"></a>Пример

Это предупреждение возникает в следующем примере кода:

```cpp
class A
{
  // members
};

void f( )
{
  A *pA = new A[5];
  // code ...
  delete pA;
}
```

Чтобы устранить это предупреждение, используйте следующий образец кода:

```cpp
void f( )
{
  A *pA = new A[5];
  // code ...
  delete[] pA;
}
```

Если нижележащий объект в массиве является типом-примитивом например `int`, `float`, `enum`, или указатель, нет деструктор для вызова. В таких случаях предупреждение [C6283](../code-quality/c6283.md) вместо этого передается.

Использование нового и delete имеют множество ловушек с точки зрения утечки памяти и исключения. Чтобы избежать такого рода утечки и исключение проблемы, используйте механизмы, предоставляемые по C++ стандартной библиотеки шаблонов (STL). К ним относятся [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class), и [вектор](/cpp/standard-library/vector). Дополнительные сведения см. в разделе [интеллектуальные указатели](/cpp/cpp/smart-pointers-modern-cpp) и [стандартной библиотеки C++](/cpp/standard-library/cpp-standard-library-reference).
