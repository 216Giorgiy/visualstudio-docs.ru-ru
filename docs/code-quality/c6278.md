# <a name="c6278"></a>C6278
Предупреждение C6278: \<переменной > выделена с массивом new [], а удаляется скалярным оператором delete. Деструкторы не будут вызваны.

 Это предупреждение возникает только в коде C++ и указывает, что вызывающая функция выделила памяти с массивом **new []** оператор, но освобождается его скалярным **удалить** оператор. Это неопределенное поведение в соответствии со стандартом C++ и реализации Microsoft Visual C++. Существуют по крайней мере три причины, это может привести к проблемам.

-   Конструкторы для отдельных объектов в массиве вызываются, но не вызываются деструкторы.

-   Если глобальный или данного класса **оператор new** и **оператор delete** не совместимы с **оператор new []** и **оператор delete []**, могут возникнуть непредвиденные результаты.

-   Всегда рискованно полагаться на неопределенное поведение.

 Точные последствия этой неполадки предсказать их трудно. Он может привести к утечке классов с деструкторами, выполняющие отмены выделения памяти; неоднородное поведение классов с деструкторами, выполняющими семантически различные операции; или возможно повреждение памяти и сбои при переопределении операторов. В других случаях может быть несовпадение имеют значения, в зависимости от реализации компилятора и его библиотек. Средство анализа не всегда различает эти ситуации.

 Если память выделяется с массивом **new []**, он должен быть высвобождать с помощью массива **delete []**.

## <a name="example"></a>Пример
 Это предупреждение возникает в следующем примере кода:

```
class A
{
  // members
};

void f( )
{
  A *pA = new A[5];
  // code ...
  delete pA;
}
```

 Чтобы устранить это предупреждение, используйте в следующем примере кода:

```
void f( )
{
  A *pA = new A[5];
  // code ...
  delete[] pA;
}
```

 Если нижележащий объект в массиве является типом-примитивом например `int`, `float`, `enum`, или указатель, нет деструктор для вызова. В таких случаях предупреждение [C6283](../code-quality/c6283.md) вместо сообщается.

 Использование нового и delete имеют множество ловушек с точки зрения утечек памяти и исключения. Чтобы избежать такого рода утечки и исключение проблемы, используйте механизмов, которые содержатся в C++ стандартной библиотеки шаблонов (STL). К ним относятся [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class), и [вектор](/cpp/standard-library/vector). Дополнительные сведения см. в разделе [интеллектуальные указатели](/cpp/cpp/smart-pointers-modern-cpp) и [стандартной библиотеки C++](/cpp/standard-library/cpp-standard-library-reference).
