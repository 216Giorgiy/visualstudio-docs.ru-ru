# <a name="ca1020-avoid-namespaces-with-few-types"></a>CA1020: не используйте пространства имен с несколькими типами
|||  
|-|-|  
|TypeName|AvoidNamespacesWithFewTypes|  
|CheckId|CA1020|  
|Категория|Microsoft.Design|  
|Критическое изменение|Критическое|  
  
## <a name="cause"></a>Причина  
 Пространство имен, отличном от глобального пространства имен содержит меньше пяти типов.  
  
## <a name="rule-description"></a>Описание правила  
 Убедитесь, что каждое из используемых пространств имен имеет логическую организацию, и что существует веская причина для размещения типов в разреженное пространство имен. Пространства имен должны содержат типы, которые используются совместно в большинстве сценариев. Если их приложения являются взаимоисключающими, типы должен находиться в отдельном пространстве имен. Например <xref:System.Web.UI> пространство имен содержит типы, используемые в веб-приложениях и <xref:System.Windows.Forms> пространство имен содержит типы, используемые в [!INCLUDE[TLA#tla_mswin](../code-quality/includes/tlasharptla_mswin_md.md)]-приложений на основе. Несмотря на то, что оба пространства имен содержат типы, которые управляют некоторыми аспектами пользовательского интерфейса, эти типы не предназначены для использования в одном приложении. Таким образом они находятся в отдельных пространствах имен. Организовывать пространства имен также может быть полезно, так как это увеличит обнаружения функций. Изучив иерархию пространства имен, пользователи библиотек должны иметь возможность найти типы, реализующие функциональность.  
  
> [!NOTE]
>  Типы во время разработки и разрешения не должны сливаться с другими пространствами имен в соответствии с данным правилом. Эти типы принадлежат собственные пространства имен ниже основного пространства имен и пространств имен должны оканчиваться на `.Design` и `.Permissions`соответственно.  
  
## <a name="how-to-fix-violations"></a>Устранение нарушений  
 Чтобы устранить нарушение данного правила, попробуйте объединить пространства, содержащие несколько типов, в одно пространство имен.  
  
## <a name="when-to-suppress-warnings"></a>Отключение предупреждений  
 Это безопасно подавить предупреждение из этого правила, если пространство имен содержит типы, используемые с типами в других пространств имен.
