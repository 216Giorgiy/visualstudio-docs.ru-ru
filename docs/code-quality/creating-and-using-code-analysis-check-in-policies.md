# Создание и использование политик возврата с анализом кода
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

При использовании системы управления версиями Team Foundation можно создать политику возврата с анализом кода для проектов кода .NET Framework и проектов в машинном коде \(C\/C\+\+\) в командном проекте.  Политика возврата с анализом кода позволяет управлять кодом, возвращаемым в базу кода, и повышать его качество.  
  
 Политика выполняется, если локальное построение актуально, анализ кода применяется к самым последним исходным файлам.  Правила анализ кода, применяемые в проекте кода должны, состоять как минимум из тех же правил, которые определены в политике возврата в командном проекте.  Правила, заданные в виде ошибок в параметрах командного проекта, также должны быть заданы в виде ошибок в проекте кода.  
  
> [!IMPORTANT]
>  Политики возврата с анализом кода нельзя применять к проектам веб\-сайтов.  Их можно применять к проектам веб\-приложений.  
  
 Для Создания политик возврата с анализом кода используются параметры командного проекта [!INCLUDE[esprscc](../code-quality/includes/esprscc_md.md)].  Политики возврата задаются и применяются для командного проекта, однако анализ кода настраивается и выполняется для отдельных проектов кода на локальных компьютерах разработчиков.  В этом разделе описана процедура определения политик возврата с анализом кода для командного проекта, а также процедура внедрения настраиваемой политики анализа управляемого кода.
