# <a name="ca1700-do-not-name-enum-values-39reserved39"></a>CA1700: Называть значения перечислений именем &#39;зарезервировано&#39;

|||
|-|-|
|TypeName|DoNotNameEnumValuesReserved|
|CheckId|CA1700|
|Категория|Microsoft.Naming|
|Критическое изменение|Критическое|

## <a name="cause"></a>Причина

Имя члена перечисления содержит слово «reserved».

## <a name="rule-description"></a>Описание правила

В данном правиле предполагается, что член перечисления, имя которого содержит слово "reserved", не используется в настоящее время, а является местозаполнителем, который будет в дальнейшем переименован или удален. Переименование или удаление элемента — это критическое изменение. Не следует ожидать пользователям пропускать члена, просто потому, что его имя содержит «зарезервированный», а также можно зависит от пользователей на чтение и следовать документации. Кроме того поскольку зарезервированные элементы отображаются в браузерах объектов и интеллектуальные интегрированные среды разработки, они могут привести к путанице, о том, какие члены фактически используются.

Вместо использования зарезервированного элемента следует добавьте новый элемент в перечисление в будущей версии. В большинстве случаев Добавление нового члена не является критическим изменением, до тех пор, пока добавление не приводит значения исходных элементов.

В ограниченном числе вариантов Добавление члена является критическим изменением, даже в том случае, если исходные элементы сохраняют свои исходные значения. В первую очередь, новый элемент не может возвращаться из существующих путей кода без изменения вызывающих объектов, использующих `switch` (`Select` в [!INCLUDE[vbprvb](../code-quality/includes/vbprvb_md.md)]) оператором в возвращаемое значение, которое охватывает весь элемент списка, и, создает исключение вариант по умолчанию. Вторичный проблемой является, что клиентский код может не обрабатывать, изменение в поведении методов отражения например <xref:System.Enum.IsDefined%2A?displayProperty=fullName>. Соответственно, если новый член должен быть возвращены из существующих методов или известная несовместимость приложения возникает из-за неверного использования отражения, единственным неразрывный решением является:

1. Добавление нового перечисления, содержащий исходные и новые элементы.

2. Пометка исходного перечисления с <xref:System.ObsoleteAttribute?displayProperty=fullName> атрибута.

   Выполните ту же процедуру для видимого типы и члены, которые предоставляют исходного перечисления.

## <a name="how-to-fix-violations"></a>Устранение нарушений

Чтобы устранить нарушение этого правила, удалите или переименуйте член.

## <a name="when-to-suppress-warnings"></a>Отключение предупреждений

Его можно безопасно подавить предупреждение из этого правила для члена, который используется в настоящее время или библиотек, которые были предоставлены ранее.
