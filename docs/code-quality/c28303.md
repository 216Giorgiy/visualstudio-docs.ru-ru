# C28303
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Предупреждение C28303: Для ссылочного параметра \<имя\_параметра\> языка C\+\+ в \<аннотация\> обнаружен неоднозначный оператор `_Deref_`.  
  
 Это предупреждение аналогично предупреждению C28302 и сообщает об использовании в параметре дополнительного уровня `_Deref_` .  
  
 SAL2 не требует использования дополнительного уровня `_Deref_` при работе со ссылочными параметрами.  Данное примечание неоднозначно для уровня разыменования, предназначенного для создания заметки.  Может потребоваться использовать `_At_` для ссылки на конкретный объект для создания заметки.  
  
## Пример  
 Это предупреждение возникает в следующем коде, поскольку использование `__deref_out_ecount(n)` является неоднозначным:  
  
```  
void ref(__deref_out_ecount(n) int **&buff, int &n)  
```  
  
 Вышеуказанная заметка может быть интерпретирована так же как:  
  
-   ссылка на массив \(из n значений\) указателей на integer \(интерпретация SAL1\)  
  
-   ссылка на указатель на массив целых чисел \(из n значений\) \(интерпретации SAL2\)  
  
 Последующее может исправить это предупреждение:  
  
```  
void ref(_Out_writes_(n) int **&buff, int &n)  
// or  
_At_(*buff), _Out_writes(n)) void ref(int **&buff, int &n)  
  
```
