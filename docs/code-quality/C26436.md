# <a name="c26436-needvirtualdtor"></a>C26436 NEED_VIRTUAL_DTOR
«Тип с виртуальной функцией должен либо открытый виртуальный или защищенный невиртуальный деструктор».

**C++ Core Guidelines**: C.35: деструктор базового класса должен быть либо открытый и виртуальный, или защищенные и невиртуальные

Если класс определяет виртуальную функцию становится полиморфных, что подразумевает, что производные классы могли изменять его поведение, в том числе логику управления и уничтожение ресурсов. Так как клиентский код может вызывать полиморфных типов с помощью указателей в базовые классы, нет способа, можно явно выбрать клиент без приведение подходит какое поведение. Чтобы убедиться в том, что постоянно управления ресурсами, и происходит удаление в соответствии с правилами фактический тип рекомендуется определить открытый виртуальный деструктор. Если в иерархии типов позволяет запретить клиентский код для уничтожения объектов непосредственно, деструкторы должны быть определены как защищенный невиртуальный.

## <a name="remarks"></a>Примечания
- Это предупреждение отображается на первой виртуальной функции определение типа (он может быть виртуальный деструктор, если он не является общим), один раз для каждого типа.
  -  Так как определение можно поместить отдельно от объявления, он не всегда может получить любые виртуальные спецификаторы. Но предупреждение по-прежнему действителен – он проверяет фактические «virtuality» функции.
