# <a name="c6279"></a>C6279
Предупреждение C6279: \<переменной > выделяется с помощью скалярное новое, удаленные с помощью массивов delete]  
  
 Это предупреждение возникает только в коде C++ и указывает, что вызывающая функция выделила память скалярным **новый** оператор, но освобождается его с массивом **delete []** оператор. Если память выделяется скалярной **новый**, ее следует высвобождать скалярной **удалить**.  
  
 Существуют по крайней мере три причины, это может привести к проблемам.  
  
-   Конструкторы для отдельных объектов в массиве не вызываются, тогда как деструкторы вызываются.  
  
-   Если глобальный (или относящийся к определенному классу) **оператор new** и **оператор delete** не совместимы с **оператор new []** и **оператор delete []**, могут возникнуть непредвиденные результаты.  
  
 Точные последствия этой неполадки предсказать их трудно. Это может привести к непредвиденному поведению или сбои из-за использования неинициализированной памяти, поскольку конструкторы не вызываются. Или может вызвать выделение памяти и сбои в ситуациях, где переопределении операторов. В редких случаях несоответствие может оказаться незначительной. Средство анализа не различаются между таких ситуациях.  
  
## <a name="example"></a>Пример  
 Это предупреждение возникает в следующем коде:  
  
```cpp  
class A  
{  
  // members  
};  
  
void f ( )  
{  
  A *pA = new A;  
  //code ...  
  delete[] pA;  
}  
```  
  
 Чтобы устранить это предупреждение, используйте следующий код:  
  
```cpp  
void f( )  
{  
  A *pA = new A;  
  //code ...  
  delete pA;  
}  
```  
  
 Чтобы избежать подобные проблемы выделения, используйте механизмы, предоставляемые библиотекой стандартных шаблонов (STL) C++. К ним относятся [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class), и [вектор](/cpp/standard-library/vector). Дополнительные сведения см. в разделе [интеллектуальные указатели](/cpp/cpp/smart-pointers-modern-cpp) и [стандартной библиотеки C++](/cpp/standard-library/cpp-standard-library-reference).
