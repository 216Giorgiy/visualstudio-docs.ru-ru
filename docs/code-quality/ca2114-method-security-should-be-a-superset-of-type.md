# <a name="ca2114-method-security-should-be-a-superset-of-type"></a>CA2114: безопасность метода должна быть надмножеством типа
|||  
|-|-|  
|TypeName|MethodSecurityShouldBeASupersetOfType|  
|CheckId|CA2114|  
|Категория|Microsoft.Security|  
|Критическое изменение|Критическое|  
  
## <a name="cause"></a>Причина  
 Тип имеет декларативной безопасности и один из его методов имеет декларативную безопасность для одного действия безопасности, и действие по обеспечению безопасности не [требования связывания](/dotnet/framework/misc/link-demands) или [требования к наследованию](http://msdn.microsoft.com/en-us/28b9adbb-8f08-4f10-b856-dbf59eb932d9)и разрешения проверяемые типом не являются подмножеством разрешений, проверяемых методом.  
  
## <a name="rule-description"></a>Описание правила  
 Метод не должен обладать как на уровне метода и на уровне типа декларативной безопасности для одного действия. Эти две проверки не объединяются; применяется только запросу уровне метода. Например, если тип запрашивает разрешение `X`, и один из его методов разрешение `Y`, код не имеет разрешений `X` для выполнения метода.  
  
## <a name="how-to-fix-violations"></a>Устранение нарушений  
 Просмотрите код, чтобы убедиться в том, что оба действия необходимы. Если оба действия необходимы, убедитесь, что действие уровня метод включает безопасность, определенную на уровне типа. Например, если тип запрашивает разрешение `X`, и его метод также должен запросить разрешение `Y`, метод должен явным образом запрашивать `X` и `Y`.  
  
## <a name="when-to-suppress-warnings"></a>Отключение предупреждений  
 Можно безопасно подавить предупреждение из этого правила, если метод не требуется безопасность, указанная типом. Однако это не является обычным сценарием и может указывать на необходимость проверки тщательного планирования.
