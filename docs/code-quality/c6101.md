# C6101
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Предупреждение C6101: Возврат неинициализированной памяти  
  
 Успешный путь через функцию не устанавливает именованный параметр `_Out_`.  Это сообщение создано на основе аннотации SAL, которая указывает, что анализируемая функция всегда завершается успешно.  Функция, которая не возвращает сигнала об успехе или ошибке, должна установить значения во все свои параметры `_Out_`, так как анализатор предполагает, что параметр `_Out_` является неинициализированными данными до вызова функции, и что функция инициализирует параметры, чтобы они больше не были неинициализированными.  Если функция указывает на успешное завершение или сбой, то в случае сбоя параметр `_Out_` не следует задавать, а также можно выявить неинициализированные расположения и избежать их.  В любом случае цель \- избежать чтения из неинициализированных расположений.  Если функция иногда не касается параметра `_Out_`, который затем используется, то этот параметр должен быть инициализирован перед вызовом функции и помечен с аннотацией `_Inout_` или, при необходимости, более точной аннотацией `_Pre_null_` или `_Pre_satisfies_()`. «Частичный успех» можно обработать заметкой `_When_`.  Для получения дополнительной информации см. [Использование аннотаций SAL для сокращения количества дефектов в коде C\/C\+\+](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md).
