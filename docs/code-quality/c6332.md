# C6332
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Предупреждение C6332: Недопустимый параметр: передача нуля в качестве параметра dwFreeType в \<функция\> не допускается.  В результате возникает сбой вызова.  
  
 Данное предупреждение указывает, что в функцию VirtualFree или VirtualFreeEx передан недопустимый параметр.  Функции VirtualFree и VirtualFreeEx не разрешают передавать ноль в качестве параметра dwFreeType.  Параметр dwFreeType может принимать значение MEM\_DECOMMIT или MEM\_RELEASE.  Однако значения MEM\_DECOMMIT и MEM\_RELEASE нельзя одновременно использовать в одном вызове.  Кроме того, следует убедиться, что возвращаемое значение функции VirtualFree не пропущено.  
  
## Пример  
 В следующем коде создается данное предупреждение из\-за передачи в функцию VirtualFree недопустимого параметра:  
  
```  
#include <windows.h>  
#define PAGELIMIT 80              
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID f( VOID )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  BOOL bSuccess;             
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo( &sSysInfo );    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc(  
                         NULL,                // system selects address  
                         PAGELIMIT*dwPageSize, // size of allocation  
                         MEM_RESERVE,          
                         PAGE_NOACCESS );       
  if (lpvBase)  
  {  
    // code to access memory   
  }  
  else  
  {  
    return;  
  }  
  
  bSuccess = VirtualFree( lpvBase, 0, 0 );   
  // code ...  
}  
```  
  
 Для устранения этого предупреждения измените вызов функции VirtualFree, как указано в следующем коде:  
  
```  
#include <windows.h>  
#define PAGELIMIT 80              
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID f( VOID )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  BOOL bSuccess;             
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo( &sSysInfo );    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc(  
                         NULL,                // system selects address  
                         PAGELIMIT*dwPageSize, // size of allocation  
                         MEM_RESERVE,          
                         PAGE_NOACCESS );       
  if (lpvBase)  
  {  
    // code to access memory   
  }  
  else  
  {  
    return;  
  }  
  
  bSuccess = VirtualFree( lpvBase, 0, MEM_RELEASE );  
  // code ...  
}   
```  
  
 Использование VirtualAlloc и VirtualFree имеет много опасностей в терминах утечки памяти и исключений.  Чтобы избежать все такие проблемы утечек и исключений, используйте механизмы, предоставляемые библиотекой стандартных шаблонов \(STL\) C\+\+.  Они включают [shared\_ptr](/visual-cpp/standard-library/shared-ptr-class), [unique\_ptr](/visual-cpp/standard-library/unique-ptr-class) и [vector](../Topic/%3Cvector%3E.md).  Дополнительные сведения см. в разделах [Интеллектуальные указатели](/visual-cpp/cpp/smart-pointers-modern-cpp) и [Стандартная библиотека C\+\+](/visual-cpp/standard-library/cpp-standard-library-reference).
