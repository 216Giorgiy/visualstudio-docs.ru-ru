# Встроенные функции
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Выражение в SAL может быть выражением C\/C\+\+ при условии, что выражение не имеет побочных эффектов, например, \+\+, \-\- и все вызовы функций имеют побочные эффекты в этом контексте.  Однако SAL предоставляет некоторые объекты, подобные функциям, и некоторые зарезервированные символы, которые могут быть использованы в выражениях SAL.  Они называются *встроенными функциями*.  
  
## Общее назначение  
 Следующие заметки встроенных функций предоставляют общие служебные функции для SAL.  
  
|Заметка|Описание|  
|-------------|--------------|  
|`_Curr_`|Синоним для объекта, который в данный момент аннотируется.  Если заметка `_At_` в данный момент используется, то `_Curr_` является такой же, как первый параметр в `_At_`.  В противном случае она является параметром или всей функцией\/возвращаемым значением, с которым лексически связана заметка.|  
|`_Inexpressible_(expr)`|Выражает ситуацию, когда размер буфера слишком сложен для представления с использованием выражения заметки, например, когда оно вычисляется с помощью просмотра входного набора данных и последующего подсчета выбранных членов.|  
|`_Nullterm_length_(param)`|`param` является количеством элементов в буфере, без учета знака завершения null.  Оно может быть применено к любому буферу типа, отличного от статистической обработки и отличного от void.|  
|`_Old_(expr)`|Когда оно вычисляется в предварительном условии, `_Old_` возвращает входное значение `expr`.  Когда оно вычисляется в постусловии, оно возвращает значение `expr`, как будто оно вычислялось в предусловии.|  
|`_Param_(n)`|Параметр `n` функции, считающий от 1 до `n`, где `n` является символьной константой.  Если параметр имеет имя, то эта заметка схожа с доступом к параметру по имени. **Note:**  `n` может ссылаться на позиционные параметры указанные многоточием, или может использоваться в прототипах функций, в которых не используются имена.|  
|`return`|Зарезервированное ключевое слово C\/C\+\+ `return` можно использовать в выражении SAL для указания возвращаемого значения функции.  Значение доступно только в состоянии отправки; при использовании в предварительном состоянии вызывается синтаксическая ошибка.|  
  
## Специфика строк  
 Следующие заметки встроенной функции позволяют управлять строками.  Все четыре функций служат для осуществления одной цели: возвращать количество элементов типа, найденного перед заключительным знаком null.  Различными являются те типы данных в элементах, на которые ссылаются.  Обратите внимание, что если требуется указать длину буфера, завершающегося символом null, который не состоит из символов, используйте заметку `_Nullterm_length_(param)` из предыдущего раздела.  
  
|Заметка|Описание|  
|-------------|--------------|  
|`_String_length_(param)`|`param` является количество элементов в строке, без учета заключающего символа null.  Эта заметка зарезервирован для строково\-символьных типов.|  
|`strlen(param)`|`param` является количество элементов в строке, без учета заключающего символа null.  Эта заметка зарезервированы для использования с массивами символов и похожа на функцию среды выполнения языка C [strlen\(\)](/visual-cpp/c-runtime-library/reference/strlen-wcslen-mbslen-mbslen-l-mbstrlen-mbstrlen-l).|  
|`wcslen(param)`|`param` является количество элементов в строке до \(без учета\) заключающего символа null.  Эта заметка зарезервированы для использования с массивами расширенных символов и похожа на функцию среды выполнения языка C [wcslen\(\)](/visual-cpp/c-runtime-library/reference/strlen-wcslen-mbslen-mbslen-l-mbstrlen-mbstrlen-l).|
