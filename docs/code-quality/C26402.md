# <a name="c26401-dontdeletenonowner"></a>C26401 DONT_DELETE_NON_OWNER
Эта проверка определяет места там, где переход на `owner<T>` может оказаться удачным вариантом для первого этапа рефакторинга. Как C26400 он применяет правила I.11 и R.3, но посвящена «выпуск» часть времени существования указатель. Оно предупреждает о любой вызов оператора `delete` Если цели не является ни `owner<T>` , ни неявно принятой владельца. Дополнительные сведения см. в разделе [C26400](c26400.md) относительно объявления автоматически. Это включает выражения, которые ссылаются на глобальные переменные, формальных параметров и т. д.


Предупреждения C26400 и C26401 всегда выполняться с [C26409](c26409.md), но они больше подходят для сценариев, где немедленно выполнить миграцию на смарт-указатели не представляется возможным. В таких случаях `owner<T>` концепции, может быть использована сначала и C26409 могут блокироваться временно.
