# Практическое руководство. Фильтрация отчетов из командной строки
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

С помощью параметров команды **VSPerfReport**  можно фильтровать отчеты за определенный период времени файла данных профилирования или ограничивать данные одним или несколькими процессами или потоками.  Дополнительные сведения об этой команде см. в разделе [VSPerfReport](../profiling/vsperfreport.md).  
  
|Параметры|Описание|  
|---------------|--------------|  
|**StartTime:**\[*Value*\]|Отображаются только данные, собранные после значения "value" \(в миллисекундах\).|  
|**EndTime:**\[*Value*\]|Отображаются только данные, собранные до значения "value" \(в миллисекундах\).|  
|**FilterFile:** `VSPFFile`|Задает расположение файла фильтра, созданного в окне **Отчет производительности Visual Studio**.|  
|**MsFilter:**\[*StartTime,Duration*\]|Отображаются только данные, собранные в период c `StartTime` до конца интервала `Duration` \(в миллисекундах\).|  
|**Process:**\[*Pid*\]|Отображаются только данные из указанного процесса.|  
|**Thread:**\[*ThreadID*\]|Отображаются только данные из указанного потока.|  
|**Thread:**\[*ThreadID,ProcessID*\]|Отображаются только данные из указанного потока, связанного с указанным процессом.|
