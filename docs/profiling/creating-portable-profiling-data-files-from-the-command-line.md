# Создание переносимых файлов данных профилирования в командной строке
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Внедряя символы для запуска профилирования в VSP\-файл с помощью программы командной строки [VSPerfReport](../profiling/vsperfreport.md), можно упростить совместный доступ к данным профилирования.  
  
 Кроме того, можно создать файл предварительно проанализированных данных профилирования \(VSPS\-файл\), который меньше по размеру и быстрее загружается в интерфейс IDE.  
  
> [!NOTE]
>  Убедитесь, что для **VSPerfReport** доступны файлы символов \(PDB\-файлы\).  Для получения дополнительной информации см. [Практическое руководство. Определение расположения файлов символов с помощью командной строки](../profiling/how-to-specify-symbol-file-locations-from-the-command-line.md).  
>   
>  Сведения о пути к **VSReport** см. в разделе [Указание пути к средствам командной строки](../profiling/specifying-the-path-to-profiling-tools-command-line-tools.md).  
>   
>  Данные профилирования в VSPS\-файле фильтровать нельзя.  
  
### Процедура внедрения символов для запуска профилирования в файл данных профилирования \(VSP\-файл\)  
  
-   В окне командной строки введите следующую команду:  
  
     \<Путь\>**VSPerfReport \<**VSP\-файл\> **\/PackSymbols**  
  
     По умолчанию имя VSPS\-файла совпадает с основным именем VSP\-файла.  С помощью параметра **Output** можно задать альтернативное имя.  
  
### Процедура создания сводного файла данных профилирования  
  
-   В окне командной строки введите следующую команду:  
  
     \<Путь\>**VSPerfReport \<**VSP\-файл\> **\/SummaryFile** \[**\/Output:**\<Имя файла\>\]  
  
     По умолчанию имя VSPS\-файла совпадает с основным именем VSP\-файла.  С помощью параметра **Output** можно задать альтернативное имя.
