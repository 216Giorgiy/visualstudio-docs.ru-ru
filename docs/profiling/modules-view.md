# Представление &quot;Модули&quot;
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

В представлении "Модули" перечисляются модули данных профилирования.  Каждый модуль представляет собой корневой узел иерархического дерева.  Функции модуля, профилирование которых было выполнено в ходе сеанса, указываются в узле модуля.  Если данные профилирования собирались с помощью метода выборки, сведения о строке указываются в узле функции, а данные указателя инструкций — в узле строки.  
  
 Чтобы отобразить или скрыть данные о производительности модуля, следует развернуть или свернуть имя модуля.  
  
 Чтобы добавить или удалить столбцы, щелкните окно отчета правой кнопкой мыши и выберите пункт **Добавить или удалить столбцы**.  Щелкнув имя столбца, можно сортировать данные.  Для получения дополнительной информации см. [Практическое руководство. Настройка столбцов представлений отчета](../profiling/how-to-customize-report-view-columns.md).  
  
 Набор столбцов, доступных в преставлении "Модули", зависит от метода профилирования \(выборка или инструментирование\), с помощью которого собирались данные, а также от того, собирались ли в ходе сеанса профилирования данные о выделении памяти .NET.
