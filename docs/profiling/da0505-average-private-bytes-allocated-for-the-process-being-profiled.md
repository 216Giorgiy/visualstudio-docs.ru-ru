# DA0505. Среднее число байт исключительного пользования, распределенное для профилируемого процесса
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

|||  
|-|-|  
|Идентификатор правила|DA0505|  
|Категория|Управление ресурсами|  
|Метод профилирования|Все|  
|Сообщение|Эти сведения были собраны исключительно в информационных целях.  Счетчик "Байты исключительного пользования процесса" измеряет виртуальную память, выделенную профилируемым процессом.  Содержащееся в отчете значение является средним, вычисленным по всем интервалам измерения.|  
|Тип правила|Сведения|  
  
 При профилировании методом выборки, памяти .NET или конфликтов ресурсов необходимо собрать не меньше 10 экземпляров, чтобы вызвать срабатывание правила.  
  
## Описание правила  
 В этом сообщении указывается средний объем виртуальной памяти, выделенной в данный момент процессом, в байтах \(байтах исключительного пользования\).  Байты исключительного пользования соответствуют участкам виртуальной памяти, которые были выделены процессом, и обращаться к которым могут только потоки, выполняющиеся внутри данного процесса.  
  
 Для 32\-разрядных процессов, выполняющихся на 32\-разрядных компьютерах, верхний предел участка исключительного пользования адресного пространства процесса – 2 Гб.  С помощью переключателя [\/3GB](http://go.microsoft.com/fwlink/?LinkId=177831) Boot.ini, 32\-разрядные процессы могут получить до 3 ГБ виртуальной памяти.  32\-разрядный процесс, выполняющийся на 64\-разрядном компьютере, может получить до 4 Гб виртуальной памяти исключительного пользования.  
  
 64\-разрядный процесс, выполняющийся на 64\-разрядном компьютере, может получить до 8 Тб виртуальной памяти исключительного пользования.  
  
 Указываемое значение является средним по всем интервалам измерения, на которых был активен профилируемый процесс.  
  
 Для получения дополнительных сведений об адресных пространствах процессов см. раздел [Virtual Address Space](http://go.microsoft.com/fwlink/?LinkId=177832) в документации по управлению памятью Windows.  
  
## Использование данных правил  
 Указываемые значения используются для сравнения производительности разных версий или построений программы или анализа производительности приложения в различных сценариях профилирования.
