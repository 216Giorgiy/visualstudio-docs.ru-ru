# DA0506. Максимальное число байт исключительного пользования, распределенное для профилируемого процесса
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

|||  
|-|-|  
|Идентификатор правила|DA0506|  
|Категория|Отслеживание ресурсов|  
|Метод профилирования|Все|  
|Сообщение|Эти сведения были собраны исключительно в информационных целях.  Счетчик "Байты исключительного пользования процесса" измеряет виртуальную память, выделенную профилируемым процессом.  Содержащееся в отчете значение является максимальным по всем интервалам измерения.|  
|Тип правила|Сведения|  
  
 При профилировании методом выборки, памяти .NET или конфликтов ресурсов необходимо собрать не меньше 10 экземпляров, чтобы вызвать срабатывание правила.  
  
## Описание правила  
 В этом сообщении указывается максимальный объем виртуальной памяти, выделенной в данный момент процессом, в байтах \(байтах исключительного пользования\).  Байты исключительного пользования соответствуют участкам виртуальной памяти, которые были выделены процессом, и обращаться к которым могут только потоки, выполняющиеся внутри данного процесса.  
  
 Для 32\-разрядных процессов, выполняющихся на 32\-разрядных компьютерах, верхний предел участка исключительного пользования адресного пространства процесса – 2 Гб.  С помощью переключателя [\/3 GB](http://go.microsoft.com/fwlink/?LinkId=177831) Boot.ini, 32\-разрядные процессы могут получить до 3 ГБ виртуальной памяти.  32\-разрядный процесс, выполняющийся на 64\-разрядном компьютере, может получить до 4 Гб виртуальной памяти исключительного пользования.  
  
 64\-разрядный процесс, выполняющийся на 64\-разрядном компьютере, может получить до 8 Тб виртуальной памяти исключительного пользования.  
  
 Содержащееся в отчете значение является максимальной величиной, полученной на всех интервалах измерения, в которых профилируемый процесс был активным.  
  
 Для получения дополнительных сведений об адресных пространствах процессов см. раздел [Virtual Address Space](http://go.microsoft.com/fwlink/?LinkId=177832) в документации по управлению памятью Windows.  
  
## Использование данных правил  
 Указываемые значения используются для сравнения производительности разных версий или построений программы или анализа производительности приложения в различных сценариях профилирования.  
  
 Максимальное значение для байтов исключительного пользования, близкое к предельному размеру адресного пространства процесса для данной архитектуры, приводит к исключениям, связанным с нехваткой памяти.  Дополнительные сведения см. в разделе [Изучение проблем памяти](http://go.microsoft.com/fwlink/?LinkID=177833) в журнале MSDN.
