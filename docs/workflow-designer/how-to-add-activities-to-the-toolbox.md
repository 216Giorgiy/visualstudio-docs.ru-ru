# Как добавить действия в область элементов
Действия можно добавить в **Область элементов** в решении несколькими различными способами.Это можно сделать из текущего проекта, ссылкой из другого проекта либо ссылкой из другой сборки.  
  
### Добавление действия из текущего проекта  
  
1.  Добавить новое пользовательское действие к текущему проекту рабочего процесса.[!INCLUDE[crabout](../test/includes/crabout_md.md)] добавлении нового пользовательского действия к проекту см. в разделе [Практическое руководство. Добавление нового элемента в проект рабочего процесса](../Topic/How%20to:%20Add%20a%20New%20Item%20to%20a%20Workflow%20Project.md).  
  
2.  Добавьте к действию пользовательскую логику.  
  
3.  Выполните сборку проекта.Если построение выполнено успешно, то в **Области элементов** появляется новая категория с именем \<*project name*\>, в которую включено пользовательское действие.  
  
    > [!NOTE]
    >  Если область инструментов сбрасывается, пользовательские действия удаляются, даже если выполняется повторное построение решения.Чтобы вновь заполнить область инструментов пользовательскими действиями после ее сброса, перезапустите [!INCLUDE[vs2010](../modeling/includes/vs2010_md.md)].  
  
    > [!NOTE]
    >  В области инструментов может быть отображено только одно действие с определенным именем.Если два различных действия из разных сборок имеют то же имя класса, будет отображено только одно.  
  
    > [!NOTE]
    >  Домен приложения совместно используется экземплярами редактора. Если используются статические переменные, то они также будут совместно использоваться экземплярами редактора.Если такое поведение нежелательно, следует использовать службу для отслеживания экземпляров переменных.Дополнительные сведения об использовании служб в конструкторе см. в разделе [Использование контекста редактирования ModelItem](../Topic/Using%20the%20ModelItem%20Editing%20Context.md).  
  
### Добавление действия из другого проекта  
  
1.  Откройте решение, содержащее хотя бы один проект рабочего процесса и либо пользовательский проект библиотеки действий, либо другой проект рабочего процесса, который определяет пользовательское действие.  
  
2.  Выполните построение обоих проектов.Если построения выполнены успешно, то в **Области элементов** появляется новая категория с именем \<*project name*\>, в которую включено пользовательское действие.  
  
### Добавление действия в область элементов из сборки  
  
1.  Откройте решение рабочего процесса.  
  
2.  В меню **Сервис** выберите пункт **Выбрать элементы панели элементов...**  
  
3.  В диалоговом окне **Выбор элементов панели элементов** откройте вкладку **System.Activities Components**, нажмите кнопку **Обзор…** и перейдите к сборке, содержащей добавляемое пользовательское действие.  
  
4.  Выберите сборку и нажмите кнопку **ОК**.Компонент пользовательского действия добавляется к списку компонентов и выбирается автоматически.  
  
    1.  Нажмите кнопку **ОК**, чтобы закрыть диалоговое окно.  
  
5.  Чтобы отобразить панель элементов, выберите пункт **Область элементов** в меню **Вид**.  
  
6.  Пользовательское действие появляется в той категории **Области элементов**, которая была в фокусе в момент добавления элемента.Например, если категория **Общие** была выбрана в **Области элементов** в момент добавления элемента, то действие появится в категории **Общие**.
