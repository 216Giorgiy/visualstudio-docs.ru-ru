# <a name="how-to-view-web-page-response-time-in-a-load-test-using-the-load-test-analyzer"></a>Практическое руководство. Просмотр времени ответа веб-страницы в нагрузочном тесте с помощью анализатора тестовой нагрузки

Время, которое требуется для загрузки каждой веб-страницы, называется *временем ответа*. При создании веб-теста производительности можно установить целевое значение времени отклика для каждого запроса веб-страницы в веб-тесте производительности.

Если веб-тест производительности выполняется под нагрузкой в нагрузочном тесте, то для каждой страницы можно проанализировать следующие сведения.

-   Среднее время ответа для страницы.

-   Процент итераций теста, удовлетворяющих целевому значению времени ответа для страницы.

-   С помощью представления "Таблицы" или "Диаграммы" анализатора тестовой нагрузки можно анализировать значения времени ответа веб-страниц:

-   Анализ значений времени ответа веб-страниц в представлении таблиц

-   Анализ значений времени ответа веб-страниц в представлении диаграмм

## <a name="view-response-time-data-in-a-table"></a>Просмотр данных времени ответа в виде таблицы

### <a name="to-view-response-time-data-in-a-table"></a>Чтобы просмотреть данные времени ответа в виде таблицы, выполните следующие действия.

1.  В анализаторе тестовой нагрузки нажмите кнопку **Таблицы** на панели инструментов и убедитесь, что сетка таблицы отображается.

2.  В раскрывающемся списке **Таблица** выберите **Страницы**.

3.  Данные для каждой страницы отображаются в таблице. В этой таблице, как правило, содержатся следующие столбцы.

    |||
    |-|-|
    |**Страница**|Имя веб-страницы.|
    |**Сценарий**|Имя сценария. Этот столбец важен в том случае, если в веб-тесте производительности имеется несколько сценариев.|
    |**Тест**|Имя веб-теста производительности. Этот столбец важен в том случае, если в нагрузочном тесте имеется несколько веб-тестов производительности.|
    |**Network**|Тип сети.<br /><br /> По умолчанию эти данные не собираются. Для включения сбора этих данных в **редакторе тестовой нагрузки** разверните узел **Параметры запуска** и выберите узел параметра запуска, который необходимо изменить. В окне **Свойства** выберите для свойства **Хранилище сведений о синхронизации** значение **Все индивидуальные сведения**.|
    |**Итого**|Общее количество запросов веб-страницы. В этом значении суммируются все итерации нагрузочного теста.|
    |**Среднее**|Среднее время ответа страницы.<br /><br /> По умолчанию эти данные не собираются. Для включения сбора этих данных в **редакторе тестовой нагрузки** разверните узел **Параметры запуска** и выберите узел параметра запуска, который необходимо изменить. В окне **Свойства** выберите для свойства **Хранилище сведений о синхронизации** значение **Все индивидуальные сведения**.|
    |**Min**|Минимальное время ответа страницы.<br /><br /> По умолчанию эти данные не собираются. Для включения сбора этих данных в **редакторе тестовой нагрузки** разверните узел **Параметры запуска** и выберите узел параметра запуска, который необходимо изменить. В окне **Свойства** выберите для свойства **Хранилище сведений о синхронизации** значение **Все индивидуальные сведения**.|
    |**Медиана**|Медиана значений времени ответа страницы.<br /><br /> По умолчанию эти данные не собираются. Для включения сбора этих данных в **редакторе тестовой нагрузки** разверните узел **Параметры запуска** и выберите узел параметра запуска, который необходимо изменить. В окне **Свойства** выберите для свойства **Хранилище сведений о синхронизации** значение **Все индивидуальные сведения**.|
    |**90 %**|90-й процентиль времени ответа. Он означает, что 90% страниц ответили быстрее, чем это число, а 10% — медленнее.<br /><br /> По умолчанию эти данные не собираются. Для включения сбора этих данных в **редакторе тестовой нагрузки** разверните узел **Параметры запуска** и выберите узел параметра запуска, который необходимо изменить. В окне **Свойства** выберите для свойства **Хранилище сведений о синхронизации** значение **Все индивидуальные сведения**.|
    |**95 %**|Процентиль значения 95 для времени ответа. Он означает, что 95 % страниц ответили быстрее, чем это число, а 5 % — медленнее.|
    |**99 %**|99-й процентиль времени ответа. Он означает, что 99 % страниц ответили быстрее, чем это число, а 1 % — медленнее.<br /><br /> По умолчанию эти данные не собираются. Для включения сбора этих данных в **редакторе тестовой нагрузки** разверните узел **Параметры запуска** и выберите узел параметра запуска, который необходимо изменить. В окне **Свойства** выберите для свойства **Хранилище сведений о синхронизации** значение **Все индивидуальные сведения**.|
    |**Max**|Максимальное время ответа страницы.<br /><br /> По умолчанию эти данные не собираются. Для включения сбора этих данных в **редакторе тестовой нагрузки** разверните узел **Параметры запуска** и выберите узел параметра запуска, который необходимо изменить. В окне **Свойства** выберите для свойства **Хранилище сведений о синхронизации** значение **Все индивидуальные сведения**.|
    |**Стандартное отклонение**|По умолчанию данные о стандартном отклонении не собираются. Для включения сбора этих данных в **редакторе тестовой нагрузки** разверните узел **Параметры запуска** и выберите узел параметра запуска, который необходимо изменить. В окне **Свойства** выберите для свойства **Хранилище сведений о синхронизации** значение **Все индивидуальные сведения**.|
    |**Время создания страницы**|Среднее время ответа для всех запросов веб-страницы.|
    |**Цель**|Целевое значение времени ответа страницы. Это значение является постоянным для страницы. **Примечание.** Целевое время ответа страницы отображается только в том случае, если оно определено для запроса в веб-тесте производительности.|
    |**% достижения цели**|Процент запросов страницы, удовлетворяющих целевому значению времени ответа страницы.|

## <a name="view-response-time-data-in-a-graph"></a>Просмотр данных времени ответа в виде диаграммы

Данные времени ответа можно просмотреть в виде графа, чтобы проанализировать изменение этого времени в ходе выполнения нагрузочного теста. Это особенно полезно в том случае, если нагрузка возрастает по мере выполнения теста (например при использовании шаблона пошаговой нагрузки). Дополнительные сведения см. в разделе [Изменение шаблонов нагрузки для моделирования действий виртуальных пользователей](../test/edit-load-patterns-to-model-virtual-user-activities.md).

### <a name="to-view-response-time-data-in-a-graph"></a>Чтобы просмотреть данные времени ответа в виде графа, выполните следующие действия.

1.  Выберите пункт **Диаграммы** на панели инструментов анализатора тестовой нагрузки и убедитесь, что диаграмма отображается.

2.  В окне **Счетчики** разверните узел сценария, который необходимо настроить (например, `Scenario1`).

3.  Разверните узел веб-теста производительности, который необходимо настроить.

4.  Разверните узел **Страницы**.

5.  Разверните узел страницы, которую необходимо настроить.

6.  Щелкните правой кнопкой мыши **Доля страниц, соответствующих цели** и выберите команду **Показать счетчик на диаграмме**.

     Данные добавляются в граф.

7.  (Необязательно) Повторите предыдущий шаг для среднего времени ответа страницы, целевого значения времени ответа страницы и общего количества страниц.

    > [!NOTE]
    > Целевое значение времени ответа страницы является постоянным.

 Дополнительные сведения см. в статье [Анализ результатов нагрузочного тестирования в представлении диаграмм](../test/analyze-load-test-results-in-the-graphs-view.md).
