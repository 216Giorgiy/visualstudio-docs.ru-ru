# <a name="choosing-a-domain-specific-language-solution-template"></a>Выбор шаблона решения на доменном языке
Чтобы создать решение предметно ориентированного языка, выберите один из шаблонов решений, которые доступны в мастере конструктора доменного языка. Выбрав шаблон, который больше всего похож на язык, который вы хотите создать, можно свести к минимуму изменения, которые необходимо внести в начальный решение.

 Следующие шаблоны решения доступны в мастере конструктора доменного языка.

|Шаблон|Компоненты|Описание:|
|-|-|-|
|Диаграммы классов|-Фигуры секций<br />-Наследование класс<br />-Связь наследования<br />-Наследование фигур<br />-Свойства связи|Этот шаблон решения используется в том случае, если ваш предметно ориентированного языка включает сущностей и связей, которые имеют свойства. Этот шаблон создает доменный язык, который похож на UML-схемах классов. К основным элементам относятся классы и интерфейсы, вместе с связи ассоциации, обобщения и реализации. Класс или интерфейс отображается как поле, которое содержит список атрибутов.|
|Схемы компонентов|-Порты|Этот шаблон решения используется в том случае, если ваш предметно ориентированного языка включает в себя компоненты, то есть части системы программного обеспечения. Этот шаблон создает доменный язык, который похож на UML-схемы компонентов. К основным элементам относятся компоненты и порты, которые отображаются в виде маленьких фигур с внешней стороны компонентов.|
|Задача технологических схем|— Изображений и геометрических фигур<br />-   *Дорожки*|Этот шаблон решения используется в том случае, если ваш предметно ориентированного языка включает рабочие процессы, состояний или последовательностей. Этот шаблон создает доменный язык, который похож на UML-схемы деятельности. Действие является основной сущности и основного связь является переход между действиями. Шаблон включает нескольких элементов, таких как начальное, конечное состояние и полосу синхронизации.|
|Минимальный язык|— Один класс и фигуру<br />— Один отношения и соединителя|Используйте этот шаблон решения, если ваш предметно ориентированного языка не похоже на другие шаблоны. Этот шаблон создает доменный язык, который содержит два класса и одной связи, которые представлены в **элементов** как **поле** и **строки**. Класс и связи имеют свойство примере строка.|
|Минимальный конструктор WinForm|— Модель с небольшим.<br />-Форму Windows, которая отображает модель.|Этот шаблон используется в том случае, если вы хотите создать приложение, в котором DSL привязан к формы Windows, а не графический конструктор.<br /><br /> Форма, который выступает в качестве пользовательского интерфейса для языка находится в папке Dsl\UI.<br /><br /> Проект соберется перед открытием конструктора формы.<br /><br /> Дополнительные сведения см. в разделе [Создание доменного языка Windows Forms-Based](../modeling/creating-a-windows-forms-based-domain-specific-language.md).|
|Конструктор минимальной WPF|— Модель с небольшим<br />— Windows Presentation Foundation пользовательский интерфейс, который отображает модель|Этот шаблон используется в том случае, если вы хотите создать приложение, в котором DSL привязан к интерфейса пользователя WPF, а не графический конструктор.<br /><br /> Конструктор для пользовательского интерфейса находится в папке Dsl\UI.<br /><br /> Проект соберется перед открытием конструктора пользовательского интерфейса.<br /><br /> Дополнительные сведения см. в разделе [Создание доменного языка WPF-Based](../modeling/creating-a-wpf-based-domain-specific-language.md).|
|Библиотека DSL|Минимальная библиотека|Этот шаблон используется в том случае, если вы хотите создать частичное определение DSL, который можно импортировать в другие определения DSL.|
