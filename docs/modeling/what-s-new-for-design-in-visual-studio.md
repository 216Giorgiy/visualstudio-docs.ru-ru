# <a name="what39s-new-for-design-in-visual-studio"></a>Новые возможности для разработки в Visual Studio

## <a name="live-dependency-validation"></a>Проверка зависимостей в режиме реального времени

Удаление нежелательных зависимостей является важной частью управления вашей технического долга.
Динамическая проверка зависимостей теперь включены, предоставляя точные сведения о проблемах и полностью пользоваться преимуществами новых возможностей список ошибок и редактор.

![Проверка динамической зависимостей в действии](~/docs/modeling/media/dep-validation-whatsnew-01.png)

Возможности разработки изменилось осуществлять проверку зависимостей более доступными и более доступным, изменения термины «Схема слоев» «Диаграмма зависимостей».

**Архитектура** меню теперь содержит команду, чтобы напрямую создать диаграмму зависимостей:

![Динамическая зависимостей "в меню" Архитектура](~/docs/modeling/media/dep-validation-whatsnew-02.png)

… и были изменены для повышения информативности имен свойств слоя в схеме зависимостей и их описания:

![Динамическая зависимостей обновлены имена свойств](~/docs/modeling/media/dep-validation-whatsnew-03.png)

Теперь отображается влияние изменения непосредственно в результаты анализа для текущего кода в решении при каждом сохранении схемы. Ожидания завершения команды «Проверить зависимости» больше не нужно.

Дополнительные сведения см. в разделе [в этой записи блога](https://blogs.msdn.microsoft.com/visualstudioalm/2016/10/07/live-architecture-dependency-validation-in-visual-studio-15-preview-5/). 
 
## <a name="uml-designers-have-been-removed"></a>Были удалены конструкторов UML

UML-конструкторов были удалены из этой версии Visual Studio Enterprise.

* UML-схемы теперь представлены в виде XML-файлов
* В обозревателе моделей UML больше не существует.
* Ссылки, больше не используются для проверки зависимостей проекта моделирования
* Узел «Ссылки слоя» в обозревателе решений не отображается
* Действие построения «Проверка» на схеме зависимостей (слоев) больше не используется — задача построения был удален 
* Структура проекта сохраняется кругового пути между версиями
* По-прежнему можно открыть, создать, изменить и сохранить диаграмму зависимостей (уровня) в формате XML
* TFS рабочих элементов, связанных в диаграмму зависимостей (уровня) не доступны в рабочей области конструирования
* Обратной ссылки на DSL или слой больше не поддерживается 
* Расширения UML в пакет SDK моделирования больше не поддерживается

Однако поддержка визуализации архитектуры .NET и C++ код доступен через [карты кода](map-dependencies-across-your-solutions.md)и значительные улучшения для проверки зависимостей, описанных выше.

Если вы являетесь пользователем значительные конструкторов UML, можно продолжать использовать Visual Studio 2015 или более ранних версий, при выборе средство для потребностей UML.

Дополнительные сведения см. в разделе [в этой записи блога](https://blogs.msdn.microsoft.com/visualstudioalm/2016/10/14/uml-designers-have-been-removed-layer-designer-now-supports-live-architectural-analysis/).

##  <a name="version-support-for-architecture-and-modeling-tools"></a>Поддержка версий для инструментов моделирования и архитектуры  

Система Visual Studio доступна в нескольких версиях. Не все они поддерживают инструменты моделирования и архитектуры. Следующая таблица содержит сведения о доступности каждого средства.  
  
|**Функция**|**Предприятия**|**Professional**|**Community**|**Express**|  
|-----------------|--------------------|----------------------|-------------------|-----------------|  
|**Карты кода**|Да|См. Примечание (1)|-|-|  
|**Диаграммы зависимостей**|Да|Примечание (2)|Примечание (2)|-|  
|**Направленные графы** (схемы DGML)|Да|Да|Да|-|  
|**Клон кода**|Да|-|-|-|  
  
Примечание (1). Поддерживает только чтение карт кода, фильтрацию карт кода, добавление новых универсальных узлов и создание направленных графов из выделенной области.

Примечание (2): Поддерживает только чтение схемы зависимостей.
