# <a name="c6279"></a>C6279
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Последнюю версию этого раздела можно найти в [C6279](https://docs.microsoft.com/visualstudio/code-quality/c6279).  
  
Предупреждение C6279: \<переменная > выделена помощи скалярного оператора new, удаляется оператором для массивов delete]  
  
 Это предупреждение возникает только в коде C++ и указывает, что вызывающая функция выделила памяти скалярным **новый** оператор, но освободила ее с массивом **delete []** оператор. Если память выделяется скалярной **новый**, его следует освобождать при помощи скалярного оператора **удалить**.  
  
 Существует по крайней мере трех причин, это может привести к проблемам.  
  
-   Конструкторы для отдельных объектов в массиве не вызываются, несмотря на то, что деструкторы — это.  
  
-   Если глобальный (или определенного класса) **оператор new** и **оператор delete** не совместимы с **оператор new []** и **оператор delete []**, могут возникнуть непредвиденные результаты.  
  
 Точное последствия этой неполадки предсказать их трудно. Это может привести к непредвиденному поведению или происходит аварийный сбой из-за использования неинициализированной памяти, поскольку конструкторы не вызываются. Или может вызвать выделение памяти и сбои в ситуациях, где были переопределены операторы. В редких случаях несоответствие может оказаться незначительной. Средство анализа не различают в настоящее время эти ситуации.  
  
## <a name="example"></a>Пример  
 В следующем коде создается данное предупреждение:  
  
```cpp  
class A  
{  
  // members  
};  
  
void f ( )  
{  
  A *pA = new A;  
  //code ...  
  delete[] pA;  
}  
```  
  
 Чтобы устранить это предупреждение, используйте следующий код:  
  
```cpp  
void f( )  
{  
  A *pA = new A;  
  //code ...  
  delete pA;  
}  
```  
  
 Чтобы избежать подобные проблемы выделения, используйте механизмы, предоставляемые библиотекой стандартных шаблонов (STL) C++. К ним относятся [shared_ptr](http://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](http://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a), и [вектор](http://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Дополнительные сведения см. в разделе [интеллектуальные указатели](http://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) и [стандартной библиотеки C++](http://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).
