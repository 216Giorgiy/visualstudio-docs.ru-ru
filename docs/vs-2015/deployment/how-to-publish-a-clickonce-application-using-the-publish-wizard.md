# <a name="how-to-publish-a-clickonce-application-using-the-publish-wizard"></a>Практическое руководство. Публикация приложения ClickOnce с помощью мастера публикации
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Последнюю версию этого раздела можно найти в [как: публикация приложения ClickOnce с помощью мастера публикации](https://docs.microsoft.com/visualstudio/deployment/how-to-publish-a-clickonce-application-using-the-publish-wizard).  
  
Чтобы сделать приложение ClickOnce доступным для пользователей, необходимо опубликовать его в общей папке, на FTP-сервере или на съемном носителе. Можно опубликовать приложение с помощью мастера публикации; Дополнительные свойства, связанных с публикацией, доступны на **публикации** странице **конструктор проектов**. Дополнительные сведения см. в разделе [публикация приложений ClickOnce](../deployment/publishing-clickonce-applications.md).  
  
 Перед запуском Мастера публикации необходимо правильно настроить свойства публикации. Например, если вы хотите назначить ключ для подписи приложения ClickOnce, можно использовать т.д **подписывание** странице **конструктор проектов**. Дополнительные сведения см. в разделе [Защита приложений ClickOnce](../deployment/securing-clickonce-applications.md).  
  
> [!NOTE]
>  Если вы устанавливаете больше одной версии приложения с использованием технологии ClickOnce, то более ранние версии приложения перемещаются в папку "Archive", созданную в указанном вами расположении публикации. Архивация более ранних версий предотвращает появление папок с ранними версиями в каталоге установки.  
  
> [!NOTE]
>  Отображаемые диалоговые окна и команды меню могут отличаться от описанных в справке в зависимости от действующих параметров или выпуска среды. Чтобы изменить параметры, выберите в меню **Сервис** пункт **Импорт и экспорт параметров** . Дополнительные сведения см. в статье [Настройка параметров разработки в Visual Studio](http://msdn.microsoft.com/en-us/22c4debb-4e31-47a8-8f19-16f328d7dcd3).  
  
### <a name="to-publish-to-a-file-share-or-path"></a>Публикация в общую папку  
  
1.  В **обозревателе решений**, выберите проект приложения.  
  
2.  На **построения** меню, щелкните **публикации**`Projectname`.  
  
     Откроется Мастер публикации.  
  
3.  В **место публикации приложения?** странице, введите допустимый адрес FTP-сервера или допустимый путь к файлу с помощью одного из указанных форматов и нажмите кнопку **Далее**.  
  
4.  В **как пользователи будут устанавливать приложение?** выберите расположение, где пользователи будут переходить для установки приложения:  
  
    -   Если пользователи будут устанавливать с веб-сайта, нажмите кнопку **из веб-сайта** и введите URL-адрес, соответствующий пути к файлу, введенные на предыдущем шаге. Нажмите кнопку **Далее**. (Данный параметр обычно используется, когда в качестве расположения публикации указывается FTP-адрес. Загрузка из FTP напрямую не поддерживается, поэтому URL-адрес необходимо ввести здесь вручную.)  
  
    -   Если пользователи будут устанавливать приложение непосредственно из общей папки, нажмите кнопку **из UNC-путь или общая папка**, а затем нажмите кнопку **Далее**. (Это необходимо для расположений публикации вида c:\deploy\myapp формы или \\\server\myapp.)  
  
    -   Если пользователи будут устанавливать со съемного носителя, выберите **с компакт-диска или DVD-диска**, а затем нажмите кнопку **Далее**.  
  
5.  На **будет ли приложение доступно в автономном режиме?** странице, выберите соответствующий параметр:  
  
    -   Если вы хотите включить приложение для запуска когда пользователь отключен от сети, нажмите кнопку **Да, это приложение будет доступно, сети или автономно**. Ярлык на **запустить** меню будет создан для приложения.  
  
    -   Если вы хотите запустить приложение непосредственно из места публикации, нажмите кнопку **нет, это приложение доступно только через Интернет**. Ярлык на **запустить** меню не создаются.  
  
     Чтобы продолжить, нажмите кнопку **Далее** .  
  
6.  Нажмите кнопку **Готово** для публикации приложения.  
  
     Статус публикации отображается в области уведомлений.  
  
### <a name="to-publish-to-a-cd-rom-or-dvd-rom"></a>Публикация на компакт-диск или DVD-диск  
  
1.  В **обозревателе решений**, щелкните правой кнопкой мыши проект приложения и нажмите кнопку **свойства**.  
  
     Открывается **конструктор проектов**.  
  
2.  Нажмите кнопку **публикации** вкладку, чтобы открыть **публикации** странице в **конструктор проектов**и нажмите кнопку **мастера публикации** кнопки.  
  
     Откроется Мастер публикации.  
  
3.  В **место публикации приложения?** введите путь к файлу или FTP расположение, где будет опубликовано приложение, например d:\deploy. Нажмите кнопку **Далее** для продолжения.  
  
4.  На **как пользователи будут устанавливать приложение?** щелкните из **компакт-диска или DVD-диска**, а затем нажмите кнопку **Далее**.  
  
    > [!NOTE]
    >  Если запрос на установку на автоматический запуск при вставке компакт-диска в дисковод, откройте **публикации** странице в **конструктор проектов** и нажмите кнопку **параметры** Кнопка, а затем в **параметры публикации** мастера выберите **установок для компакт-диска автоматически запускать Setup, когда вставлен ДИСК**.  
  
5.  Если приложение распространяется на компакт-дисках, вы можете предоставлять для него обновления на веб-сайте. В **где приложение будет искать обновления?** выберите вариант обновления:  
  
    -   Если приложение будет проверять наличие обновлений, нажмите кнопку **приложение будет проверять наличие обновлений из следующего расположения** и введите расположение, где обновления будут публиковаться. Это может быть расположение файла, веб-сайт или FTP-сервер.  
  
    -   Если приложение не будет проверять наличие обновлений, нажмите кнопку **приложение не будет проверять наличие обновлений**.  
  
     Чтобы продолжить, нажмите кнопку **Далее** .  
  
6.  Нажмите кнопку **Готово** для публикации приложения.  
  
     Статус публикации отображается в области уведомлений.  
  
    > [!NOTE]
    >  После завершения публикации необходимо воспользоваться устройством для записки компакт-дисков или DVD-дисков, чтобы скопировать файлы из расположения, указанного в шаге 3, на компакт-диск или DVD-диск.
