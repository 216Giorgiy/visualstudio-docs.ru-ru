# <a name="set-up-named-authentication-credentials"></a>Настройка учетных данных для проверки подлинности

Чтобы опубликовать приложение в Azure или отслеживать существующую облачную службу, Visual Studio требуются учетные данные для проверки подлинности запросов к Azure, а именно идентификатор подписки Azure и допустимый сертификат X.509 v3 с помощью ключа по крайней мере 2048 бит. Укажите эти учетные данные одним из следующих методов:

- В Visual Studio выберите **Просмотр > обозреватель серверов**, щелкните правой кнопкой мыши **Azure** выберите **подключиться к подписке Microsoft Azure**и войдите в систему.
- Создайте файл подписки (`.publishsettings`), который содержит открытый ключ для сертификата. Файл подписки может содержать учетные данные для нескольких подписок, как описано в этой статье.

Примечание: эти учетные данные отличаются от учетных данных, используемых для проверки подлинности запросов к службам хранилища Azure.

## <a name="create-a-subscription-file"></a>Создание файла подписки

В обозревателе сервера щелкните правой кнопкой мыши **Azure** узел и выберите **управление подписками и их фильтрация**. Затем выберите **сертификаты** вкладку, а затем выполните одно из следующих действий:

- Выберите **импорта** открыть **Импорт подписок Microsoft Azure** диалоговое окно. Выберите **загрузки файла подписки** связать и в браузере сохраните скачанный файл во временное расположение. В диалоговом окне перейдите к папке скачивания и затем импортируйте его для использования при проверке подлинности.
- Выберите активную подписку и выберите **изменить**, которая открывает диалоговое окно, в котором можно изменить существующую подписку для использования при проверке подлинности.
- Выберите **New** открыть **новую подписку** диалогового окна поле и укажите необходимые сведения. Чтобы отправить сертификат для своего облака службу, указанную в диалоговом окне, выполните вход на портал Azure, перейдите к облачной службе, выберите **параметры > сертификаты управления**выберите **отправить**, затем Укажите путь к `.cer` файл.

Если вы хотите создать сертификат самостоятельно, обратитесь к инструкциям [Создание и передача сертификата управления для Azure](https://msdn.microsoft.com/library/windowsazure/gg551722.aspx) и вручную передайте сертификат, который [портала Azure](https://portal.azure.com/).
