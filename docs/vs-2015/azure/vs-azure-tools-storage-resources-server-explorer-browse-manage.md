# <a name="browse-and-manage-storage-resources-by-using-server-explorer"></a>Просмотр и администрирование ресурсов хранилища с помощью обозревателя сервера

[!INCLUDE [storage-try-azure-tools](./includes/storage-try-azure-tools.md)]

## <a name="overview"></a>Обзор

Если вы установили инструменты Azure для Microsoft Visual Studio, можно просмотреть BLOB-объектов, очереди и таблицы данных из учетных записей хранения для Azure. Azure **хранения** узла в обозревателе сервера отображает данные, в вашей учетной записи эмулятора локального хранилища и других учетных записях хранения Azure.

Чтобы просмотреть обозреватель сервера в Visual Studio в строке меню, выберите **представление** > **обозревателя серверов**. **Хранения** узла отображаются все учетные записи хранения, которые существуют в каждой подписке Azure или сертификат, который вы подключены к. Если вашей учетной записи хранения не отображается, его можно добавить в соответствии с инструкциями [далее в этой статье](#add-storage-accounts-by-using-server-explorer).

Начиная с версии Azure SDK 2.7, можно также использовать Cloud Explorer для просмотра и управления ресурсами Azure. Дополнительные сведения см. в разделе [управление ресурсами Azure с помощью Cloud Explorer](vs-azure-tools-resources-managing-with-cloud-explorer.md).

## <a name="view-and-manage-storage-resources-in-visual-studio"></a>Просмотр и управление ресурсами хранилища в Visual Studio

Обозреватель серверов автоматически отображает список больших двоичных объектов, очередей и таблиц в учетной записи эмулятора хранилища. Учетная запись эмулятора хранения отображается в обозревателе сервера в **хранения** узла в виде **разработки** узла.

Чтобы просмотреть ресурсы учетной записи эмулятора хранилища, разверните **разработки** узла. Если эмулятор хранения не был запущен при развертывании **разработки** узла, он автоматически запускает. Этот процесс может занять несколько секунд. Можно продолжать работу в других частях Visual Studio запускает эмулятор хранения.

Для просмотра ресурсов в учетной записи хранения, разверните узел учетной записи хранения в обозревателе серверов, где встречается **большие двоичные объекты**, **очереди**, и **таблиц** узлов.

## <a name="work-with-blob-resources"></a>Работа с ресурсами BLOB-объектов

**Большие двоичные объекты** узел отображает список контейнеров для выбранной учетной записи хранения. Контейнеры больших двоичных объектов содержат файлы BLOB-объектов и эти большие двоичные объекты можно упорядочить в папки и вложенные папки. Дополнительные сведения см. в разделе [использование хранилища BLOB-объектов из .NET](/azure/storage/blobs/storage-dotnet-how-to-use-blobs).

### <a name="to-create-a-blob-container"></a>Чтобы создать контейнер BLOB-объектов

1. Откройте контекстное меню для **большие двоичные объекты** узел, а затем выберите **создать контейнер больших двоичных объектов**.
1. В **создать контейнер больших двоичных объектов** диалогового окна введите имя нового контейнера.  
1. Нажмите клавишу ВВОД на клавиатуре или вы можете щелкните или нажмите за пределами поля имени, чтобы сохранить контейнер больших двоичных объектов.

   > [!NOTE]
   > Имя контейнера BLOB-объектов должны начинаться с цифры (0 – 9) или строчные буквы (a-z).

### <a name="to-delete-a-blob-container"></a>Чтобы удалить контейнер BLOB-объектов

Откройте контекстное меню для контейнера больших двоичных объектов, который требуется удалить, а затем выберите **удалить**.

### <a name="to-display-a-list-of-the-items-in-a-blob-container"></a>Для отображения списка элементов в контейнер больших двоичных объектов

Откройте контекстное меню для имени контейнера BLOB-объектов в списке, а затем выберите **откройте**.

При просмотре содержимого контейнера больших двоичных объектов, он отображается на вкладке представления контейнера больших двоичных объектов.

![Представление контейнера BLOB-объектов](./media/vs-azure-tools-storage-resources-server-explorer-browse-manage/IC749016.png)

Нужно выполнить следующие операции с большими двоичными объектами с помощью кнопок в правом верхнем углу представления контейнера больших двоичных объектов.

* Ввести значение фильтра и применить его.
* Обновите список больших двоичных объектов в контейнере.
* Отправьте файл.
* Удалите большой двоичный объект. (При удалении файла из контейнера больших двоичных объектов не базовый файл удаляется. Он удаляется только из контейнера больших двоичных объектов.)
* Откройте большой двоичный объект.
* Сохраните большой двоичный объект на локальном компьютере.

### <a name="to-create-a-folder-or-subfolder-in-a-blob-container"></a>Чтобы создать папку или подпапку в контейнере больших двоичных объектов

1. Выберите контейнер больших двоичных объектов в Cloud Explorer. В окне контейнера выберите **отправки большого двоичного объекта** кнопки.

1. В **отправить новый файл** выберите **Обзор** кнопку, чтобы указать файл, который требуется отправить, а затем введите имя папки в **папка (необязательно)** поле.

   ![Отправка файла в папке больших двоичных объектов](./media/vs-azure-tools-storage-resources-server-explorer-browse-manage/IC766037.png)

   Вы можете добавлять вложенные папки в папки контейнера таким же способом. Если не указать имя папки, файл передается на верхний уровень контейнера BLOB-объектов. Файл отображается в указанной папке в контейнере.

   ![Папка добавлена в контейнер больших двоичных объектов](./media/vs-azure-tools-storage-resources-server-explorer-browse-manage/IC766038.png)

1. Дважды щелкните папку или нажмите клавишу ВВОД, чтобы просмотреть содержимое папки. Когда вы находитесь в папке контейнера, можно вернуться в корень контейнера, выбрав **открыть родительский каталог** (стрелку).

### <a name="to-delete-a-container-folder"></a>Удаление папки контейнера

Удалите все файлы в папке.

Так как папки в контейнерах больших двоичных объектов виртуальные, невозможно создать пустую папку. Вы также не может удалить папку, чтобы удалить содержимое файлов, но вместо этого необходимо удалить все содержимое папки, чтобы удалить саму папку.

### <a name="to-filter-blobs-in-a-container"></a>Фильтрация больших двоичных объектов в контейнере

Вы можете отфильтровать большие двоичные объекты, которые отображаются по общему префиксу.

Например, если ввести префикс **hello** в текстовое поле фильтра, а затем выберите **Execute** (**!**) кнопка, отображаться только большие двоичные объекты, которые начинаются со слова «hello».

![Текстовое поле фильтра](./media/vs-azure-tools-storage-resources-server-explorer-browse-manage/IC519076.png)

В текстовом поле фильтра учитывается регистр и не поддерживается фильтрация с подстановочными знаками. Большие двоичные объекты можно фильтровать только по префиксу. Префикс может содержать разделитель, если вы используете разделитель для упорядочивания больших двоичных объектов в виртуальной иерархии. Например, фильтрация по префиксу «HelloFabric /» возвращает все большие двоичные объекты, которые начинаются с этой строки.

### <a name="to-download-blob-data"></a>Чтобы загрузить данные больших двоичных объектов

В Cloud Explorer можно используйте любой из следующих методов:

* Откройте контекстное меню для одного или нескольких больших двоичных объектов, а затем выберите **откройте**.
* Выберите имя большого двоичного объекта, а затем выберите **откройте** кнопки.
* Дважды щелкните имя большого двоичного объекта.

Отображается ход выполнения загрузки большого двоичного объекта в **журнал действий Azure** окна.

Большой двоичный объект откроется в редакторе по умолчанию для этого типа файлов. Если операционная система распознает этот тип файла, файл откроется в локально установленном приложении. В противном случае будет предложено выбрать приложение, которое подходит для типа файла большого двоичного объекта. Локальный файл, который создается при загрузке большого двоичного объекта, помечен как доступный только для чтения.

Данные большого двоичного объекта кэшируются локально и проверяются на соответствие время последнего изменения большого двоичного объекта в хранилище BLOB-объектов Azure. Если большой двоичный объект был обновлен с момента последней загрузки, он загружается повторно. В противном случае большой двоичный объект загружается с локального диска.

По умолчанию большой двоичный объект загружается во временный каталог. Чтобы скачать большие двоичные объекты в определенный каталог, откройте контекстное меню для имена выбранных больших двоичных объектов и выберите **Сохранить как**. При таком способе сохранения большого двоичного объекта файл не открывается, и локальный файл создается с атрибутами чтения и записи.

### <a name="to-upload-blobs"></a>Отправка BLOB-объектов

Чтобы отправить большие двоичные объекты, выберите **отправки большого двоичного объекта** кнопки, когда контейнер открыт при просмотре представления контейнера больших двоичных объектов.

Можно выбрать один или несколько файлов для отправки и отправлять файлы любого типа. **Журнал действий Azure** окне отображается ход отправки. Дополнительные сведения о том, как работать с данными больших двоичных объектов см. в разделе [использование хранилища BLOB-объектов Azure в .NET](http://go.microsoft.com/fwlink/p/?LinkId=267911).

### <a name="to-view-logs-transferred-to-blobs"></a>Чтобы просмотреть журналы, которые передаются в большие двоичные объекты

Если при использовании системы диагностики Azure данные журнала из приложения Azure и журналы перенесены в вашу учетную запись хранения, вы увидите контейнеры, созданные Azure для этих журналов. Просмотр этих журналов в обозревателе сервера — это простой способ обнаружения проблем с приложением, особенно в том случае, если оно развернуто в Azure.

Дополнительные сведения о системе диагностики Azure см. в разделе [сбор данных журналов с помощью системы диагностики Azure](https://msdn.microsoft.com/library/azure/gg433048.aspx).

### <a name="to-get-the-url-for-a-blob"></a>Чтобы получить URL-адрес для большого двоичного объекта

Откройте контекстное меню для большого двоичного объекта, а затем выберите **Копировать URL-адрес**.

### <a name="to-edit-a-blob"></a>Чтобы изменить большого двоичного объекта

Выберите большой двоичный объект, а затем выберите **открыть BLOB-объект** кнопки.

Файл загружается во временную папку и открывается на локальном компьютере. После внесения изменений повторите отправку большого двоичного объекта.

## <a name="work-with-queue-resources"></a>Работа с ресурсами очередей

Очереди служб хранилища размещаются в учетной записи хранения Azure. Их можно использовать, чтобы разрешать облачных ролей службы для взаимодействия друг с другом и с другими службами с помощью механизма передачи сообщений. Очереди можно обращаться программно через облачную службу и через веб-службу для внешних клиентов. Можно также получить доступ к очереди напрямую с помощью обозревателя серверов в Visual Studio.

При разработке облачной службы, которая использует очереди, может потребоваться использовать Visual Studio для создания очереди и работать с ними в интерактивном режиме во время разработки и тестирования кода.

В обозревателе сервера можно просматривать очереди в учетной записи хранения, создавать и удалять очереди, открывать очередь для просмотра сообщений и добавлять сообщения в очередь. При открытии очереди для просмотра, можно просматривать отдельные сообщения, и можно выполнять следующие действия в очереди с помощью кнопок в левом верхнем углу:

* Обновите представление очереди.
* Добавьте сообщение в очередь.
* Удаление верхние сообщения из очереди.
* Очищайте всю очередь.

Ниже показана очередь из двух сообщений:

![Просмотр очереди](./media/vs-azure-tools-storage-resources-server-explorer-browse-manage/IC651470.png)

Дополнительные сведения о хранилище служб очередей, см. в разделе [приступить к работе с хранилищем очередей Azure с помощью .NET](http://go.microsoft.com/fwlink/?LinkID=264702). Сведения о веб-службе для очередей служб хранилища, см. в разделе [основные понятия службы очередей](http://go.microsoft.com/fwlink/?LinkId=264788). Сведения о том, как отправлять сообщения в очередь службы хранилища с помощью Visual Studio, см. в разделе [отправке сообщений в очередь служб хранилища](https://docs.microsoft.com/azure/visual-studio/vs-storage-cloud-services-getting-started-queues).

> [!NOTE]
> Очереди служб хранилища отличаются от очередей служебной шины Azure. Дополнительные сведения об очередях служебной шины см. в разделе [очереди служебной шины, разделы и подписки](https://docs.microsoft.com/azure/service-bus-messaging/service-bus-queues-topics-subscriptions).

## <a name="work-with-table-resources"></a>Работа с ресурсами таблиц

Хранилище таблиц Azure хранятся большие объемы структурированных данных. Эта служба — хранилище данных NoSQL, которая принимает вызовы с проверкой подлинности внутри и снаружи облака Azure. Таблицы Azure идеально подходят для хранения структурированных нереляционных данных.

### <a name="to-create-a-table"></a>Создание таблицы

1. В Cloud Explorer выберите **таблиц** узел учетной записи хранения, а затем выберите **Create Table**.
1. В **Create Table** диалоговом окне введите имя для таблицы.

### <a name="to-view-table-data"></a>Просмотр данных таблицы

1. В Cloud Explorer откройте **Azure** узел, а затем откройте **хранения** узла.
1. Откройте узел учетной записи хранения, которая вас интересует и откройте **таблиц** узел, чтобы увидеть список таблиц для учетной записи хранения.
1. Откройте контекстное меню для таблицы, а затем выберите **Просмотр таблицы**.

    ![Таблица Azure в обозревателе решений](./media/vs-azure-tools-storage-resources-server-explorer-browse-manage/IC744165.png)

Таблица упорядочена по сущностям (отображаются в строках) и свойствам (отображаются в столбцах). Например на следующем рисунке показаны сущности, содержащиеся в конструкторе таблиц.

### <a name="to-edit-table-data"></a>Изменение данных таблицы

В конструкторе таблиц откройте контекстное меню для сущности (одна строка) или свойства (одна ячейка), а затем выберите **изменить**.

    ![Add or edit a table entity](./media/vs-azure-tools-storage-resources-server-explorer-browse-manage/IC656238.png)

Сущности в одной таблице не обязательно должны иметь тот же набор свойств (столбцов). Учитывайте следующие ограничения при просмотре и изменении данных таблицы.

* Нельзя просматривать или изменять двоичные данные (`type byte[]`), но их можно сохранять в таблице.
* Нельзя изменить **PartitionKey** или **RowKey** значения, поскольку хранилище таблиц в Azure не поддерживает эту операцию.
* Не удается создать свойство с именем **Timestamp**. Свойство службы хранилища Azure с таким именем.
* Если ввести **DateTime** значение, следует придерживаться формата, соответствующий язык и региональные стандарты компьютера (например, мм/дд/гггг чч: мм: [AM | PM] для английского языка США).

### <a name="to-add-entities"></a>Добавление сущностей

1. В конструкторе таблиц выберите **Добавление сущности** кнопки.

    ![Сущности кнопка "Добавить"](./media/vs-azure-tools-storage-resources-server-explorer-browse-manage/IC655336.png)

1. В **Добавление сущности** диалоговом окне введите значения свойств **PartitionKey** и **RowKey** свойства.

    ![Добавить диалоговое окно «сущности»](./media/vs-azure-tools-storage-resources-server-explorer-browse-manage/IC655335.png)

    Значения следует вводить внимательно. Их нельзя изменить после закрыть диалоговое окно, если только удалив сущность и добавив ее снова.

### <a name="to-filter-entities"></a>Фильтрация сущностей

Вы можете настроить набор сущностей, отображаемые в таблице, при использовании построителя запросов.

1. Чтобы открыть построитель запросов, откройте таблицу для просмотра.
1. Выберите **построитель запросов** на панели инструментов в представлении таблицы.

    **Построитель запросов** откроется диалоговое окно. Ниже показан запрос, созданный в построителе запросов.

    ![Построитель запросов](./media/vs-azure-tools-storage-resources-server-explorer-browse-manage/IC652231.png)
1. После завершения построения запроса закройте диалоговое окно. Полученная текстовая форма запроса отображается в текстовом поле в качестве фильтра служб данных WCF.
1. Чтобы выполнить запрос, выберите значок зеленого треугольника.

Можно также фильтровать данные сущностей, отображаемые в конструкторе таблиц Если ввести строку фильтра служб данных WCF непосредственно в текстовое поле фильтра. Такая строка аналогична SQL-предложению WHERE, но отправляется на сервер в качестве HTTP-запроса. ## <a name="refresh-storage-data"></a>Обновление данных хранилища

Когда обозреватель серверов подключается к или получение данных из учетной записи хранения, операция может занять до минуты для завершения. Если обозреватель серверов не удается подключиться, операции может истечь время. Во время получения данных можно продолжать работать в других частях Visual Studio. Чтобы отменить операцию, которая выполняется слишком долго, выберите **остановить обновление** кнопку на панели инструментов обозревателя сервера.

### <a name="to-refresh-blob-container-data"></a>Обновление данных контейнера BLOB-объектов

* Выберите **большие двоичные объекты** под узлом **хранилища**, а затем выберите **обновить** кнопку на панели инструментов обозревателя сервера.
* Чтобы обновить список больших двоичных объектов, который отображается, выберите **Execute** кнопки.

### <a name="to-refresh-table-data"></a>Обновление данных таблицы

* Выберите **таблиц** под узлом **хранения**, а затем выберите **обновить** кнопку на панели инструментов обозревателя сервера.
* Чтобы обновить список сущностей, который отображается в конструкторе таблиц, выделите **Execute** кнопку в конструкторе таблиц.

### <a name="to-refresh-queue-data"></a>Обновление данных очереди

Выберите **очереди** под узлом **хранения**, а затем выберите **обновить** кнопку на панели инструментов обозревателя сервера.

### <a name="to-refresh-all-items-in-a-storage-account"></a>Обновление всех элементов в учетной записи хранения

Выберите имя учетной записи, а затем выберите **обновить** кнопку на панели инструментов обозревателя сервера.

## <a name="add-storage-accounts-by-using-server-explorer"></a>Добавление учетных записей хранения с помощью обозревателя сервера

Существует два способа добавления учетных записей хранения с помощью обозревателя сервера. Можно создать учетную запись хранения в подписке Azure, или можно присоединить существующую учетную запись хранения.

### <a name="to-create-a-storage-account-by-using-server-explorer"></a>Чтобы создать учетную запись хранения с помощью обозревателя сервера

1. В обозревателе сервера откройте контекстное меню для **хранения** узел, а затем выберите **создать учетную запись хранения**.

1. В **создать учетную запись хранения** диалогового окна выберите или введите следующие сведения:

   * Подписка Azure, к которому вы хотите добавить учетную запись хранения.
   * Имя, которое вы хотите использовать для новой учетной записи хранения.
   * Регион или территориальная группа (например, Западная часть США или Восточная Азия).
   * Тип репликации, которые вы хотите использовать для учетной записи хранения, такие как локально избыточное хранилище.

   ![Создание учетной записи хранения Azure](./media/vs-azure-tools-storage-resources-server-explorer-browse-manage/IC744166.png)

1. Выберите **Создать**.

Учетная запись хранения появится в **хранения** списка в обозревателе решений.

### <a name="to-attach-an-existing-storage-account-by-using-server-explorer"></a>Чтобы присоединить существующую учетную запись хранения с помощью обозревателя сервера

1. В обозревателе сервера откройте контекстное меню для Azure **хранения** узел, а затем выберите **присоединить внешнее хранилище**.

    ![Добавление существующей учетной записи хранения](./media/vs-azure-tools-storage-resources-server-explorer-browse-manage/IC766039.png)
1. В **создать учетную запись хранения** диалогового окна выберите или введите следующие сведения:

   * Имя существующей учетной записи хранения, которую требуется присоединить.
   * Ключ для выбранной учетной записи хранения. Это значение обычно предоставляются автоматически при выборе учетной записи хранения. Если требуется, чтобы Visual Studio запомнила ключ учетной записи хранения, выберите **запомнить ключ учетной записи** "флажок".
   * Протокол, используемый для подключения к учетной записи хранения, например HTTP, HTTPS или пользовательской конечной точки. Дополнительные сведения о настраиваемых конечных точках см. в разделе [как Настройка строк подключения](https://msdn.microsoft.com/library/azure/ee758697.aspx).

### <a name="to-view-the-secondary-endpoints"></a>Просмотр вторичных конечных точек

Если вы создали учетную запись хранения с помощью **Read-Access Geo Redundant** варианта репликации можно просмотреть его вторичные конечные точки, открыв контекстное меню для имени учетной записи и затем выберите **свойства**.

![Вторичные конечные точки хранилища](./media/vs-azure-tools-storage-resources-server-explorer-browse-manage/IC766040.png)

### <a name="to-remove-a-storage-account-from-server-explorer"></a>Чтобы удалить учетную запись хранения из обозревателя серверов

В обозревателе сервера откройте контекстное меню для имени учетной записи и выберите **удалить**. 

При удалении учетной записи хранения, сохраненные данные ключа для этой учетной записи также удаляется.

При удалении учетной записи хранения из обозревателя сервера, он не влияет на вашу учетную запись хранения или любые данные, которые он содержит. Он удаляется только ссылка из обозревателя серверов. Чтобы окончательно удалить учетную запись хранения, используйте [портала Azure](https://portal.azure.com/).
