# <a name="constructing-filter-strings-for-the-table-designer"></a>Создание строк фильтра для конструктора таблиц
## <a name="overview"></a>Обзор
Для фильтрации данных в таблицу Azure, который отображается в Visual Studio **конструктор таблиц**, создать строку фильтра и ввести ее в поле фильтра. Синтаксис строки фильтра определяется службами данных WCF и похожа на предложение SQL WHERE, но отправляется в службу таблиц через HTTP-запроса. **Конструктор таблиц** обрабатывает соответствующую кодировку, поэтому для фильтрации по значению свойства, необходимо вводить только имя свойства, оператор сравнения, значение критерия и при необходимости, логический оператор в поле фильтра. Не нужно включать параметр запроса $filter, как это делается при построении запроса к таблице через URL-адрес [Справочник по API REST служб хранилища](http://go.microsoft.com/fwlink/p/?LinkId=400447).

Службы данных WCF основаны на [Open Data Protocol](http://go.microsoft.com/fwlink/p/?LinkId=214805) (OData). Дополнительные сведения о параметре системного запроса фильтрации (**$filter**), см. в разделе [характеристики соглашений по OData URI](http://go.microsoft.com/fwlink/p/?LinkId=214806).

## <a name="comparison-operators"></a>Операторы сравнения
Для всех типов свойств поддерживаются следующие логические операторы:

| Логический оператор | Описание | Пример строки фильтра |
| --- | --- | --- |
| eq |Равно |City eq «Redmond» |
| gt |Больше |Цена gt 20 |
| GE |Больше или равно |Цена ge 10 |
| lt |Меньше |Цена lt 20 |
| LE |Меньше или равно |Цена le 100 |
| NE |Не равно |Ne города «Лондон» |
| и |И |Цена le 200 и Цена gt 3.5 |
| или |Или |Цена le 3.5 или цена gt 200 |
| not |не |не isAvailable |

При построении строки фильтра, важны следующие правила:

* Используйте логические операторы для сравнения свойства со значением. Обратите внимание, что нельзя сравнивать свойство и динамическое значение; одной из частей выражения должно быть константой.
* Все части строки фильтра учитывается регистр.
* Значение константы должны иметь тот же тип данных, как свойство в порядке для фильтра для получения допустимых результатов. Дополнительные сведения о поддерживаемых типах свойств см. в разделе [основные сведения о модели данных службы таблиц](http://go.microsoft.com/fwlink/p/?LinkId=400448).

## <a name="filtering-on-string-properties"></a>Фильтрация по строковым свойствам
При фильтрации по строковым свойствам заключайте строковую константу в одинарные кавычки.

В следующем примере выполняется фильтрация на **PartitionKey** и **RowKey** свойства; дополнительные неключевые свойства также могут быть добавлены в строку фильтра:

    PartitionKey eq 'Partition1' and RowKey eq '00001'

Каждое выражение фильтра можно заключить в круглые скобки, несмотря на то, что это не обязательно:

    (PartitionKey eq 'Partition1') and (RowKey eq '00001')

Обратите внимание на то, что служба таблиц поддерживает запросы с подстановочными знаками, и они не поддерживаются в конструкторе таблиц либо. Тем не менее можно выполнить с помощью операторов сравнения и нужный префикс сопоставления префиксов. Следующий пример возвращает сущности, свойства LastName, начинающимся с буквы «A».

    LastName ge 'A' and LastName lt 'B'

## <a name="filtering-on-numeric-properties"></a>Фильтрация по числовым свойствам
Чтобы отфильтровать целое число или число с плавающей запятой, указывайте число без кавычек.

Этот пример возвращает все сущности свойства Age, значение которого больше 30.

    Age gt 30

Следующий пример возвращает все сущности свойства AmountDue, значение которого меньше или равно 100.25:

    AmountDue le 100.25

## <a name="filtering-on-boolean-properties"></a>Фильтрация по логическим свойствам
Для фильтрации по логическим значениям указывайте **true** или **false** без кавычек.

В следующем примере возвращаются все сущности, в которых свойство IsActive имеет значение **true**:

    IsActive eq true

Можно также написать этот критерий фильтра без логического оператора. В следующем примере служба таблиц также вернет все сущности которых свойство IsActive имеет **true**:

    IsActive

Чтобы вернуть все сущности, в которых свойство IsActive имеет значение false, можно использовать not оператор:

    not IsActive

## <a name="filtering-on-datetime-properties"></a>Фильтрация по свойствам DateTime
Для фильтрации по значению DateTime укажите **datetime** ключевое слово, за которым введите константу даты и времени в одинарные кавычки. Константа даты и времени должна быть указана в комбинированном формате UTC, как описано в разделе [форматирование значений свойств DateTime](http://go.microsoft.com/fwlink/p/?LinkId=400449).

Следующий пример возвращает сущности, в которых свойство CustomerSince имеет значение 10 июля 2008:

    CustomerSince eq datetime'2008-07-10T00:00:00Z'
