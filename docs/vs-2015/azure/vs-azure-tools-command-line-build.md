# <a name="building-azure-projects-from-the-command-line"></a>Создание проектов Azure из командной строки
С помощью Microsoft Build Engine (MSBuild), можно создавать продукты в лабораторных средах, где не установлена среда Visual Studio. MSBuild использует формат XML для файлов проекта расширяемый и полностью поддерживается корпорацией Майкрософт. С помощью формата файлов MSBuild, можно описать элементы, которые должны быть построения для одного или нескольких платформ и конфигураций.

MSBuild можно также запускать из командной строки, а в этом разделе описывается именно такой метод. Задавая свойства в командной строке, можно собирать те или иные конфигурации проекта. Аналогичным образом можно также определить целевые объекты, которые создает MSBuild. Дополнительные сведения о параметрах командной строки и MSBuild см. в разделе [справочнике по командной строке MSBuild](https://msdn.microsoft.com/library/ms164311.aspx).

## <a name="msbuild-parameters"></a>Параметры MSBuild
Самый простой способ создать пакет — запустить MSBuild с `/t:Publish` параметр. По умолчанию эта команда создает каталог относительно корневой папки проекта, такие как `<ProjectDirectory>\bin\Configuration\app.publish\`. При построении проекта Azure создается два файла: собственно файл пакета и сопутствующий файл конфигурации:

* Файл пакета (`project.cspkg`)
* Файл конфигурации (`ServiceConfiguration.TargetProfile.cscfg`)

По умолчанию каждый проект Azure включает в себя один файл конфигурации службы для локальных сборок (Отладка), а другой для облачных (промежуточное хранение или производство). Тем не менее можно добавить или удалить файлы конфигурации службы, при необходимости. При построении пакета в Visual Studio будет предложено выбрать файл службы конфигурации для включения в пакет. При построении пакета с помощью MSBuild локальный файл конфигурации службы включается по умолчанию. Чтобы включить другой файл конфигурации службы, присвойте `TargetProfile` свойство команды MSBuild (`MSBuild /t:Publish /p:TargetProfile=ProfileName`).

Если вы хотите использовать другой каталог для хранения пакета и файлы конфигурации, задайте путь с помощью `/p:PublishDir=Directory\` параметр, включая конечные разделителя обратной косой черты.
