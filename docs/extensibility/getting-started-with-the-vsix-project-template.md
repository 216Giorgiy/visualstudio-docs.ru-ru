# Приступая к работе с использованием шаблона проекта VSIX
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Шаблон проекта VSIX можно использовать для создания расширения или существующее расширение для развертывания пакета. Шаблон проекта VSIX имеет версии Visual Basic и Visual C\# и устанавливаются как часть пакета SDK для Visual Studio.  
  
 Шаблон проекта VSIX просто состоит из source.extension.vsixmanifest файл, который содержит сведения о расширении и ресурсы, которые он поставляется.  
  
 Чтобы найти шаблон проекта VSIX, необходимо установить пакет SDK для Visual Studio. Для получения дополнительной информации см. [SDK для Visual Studio](../extensibility/visual-studio-sdk.md).  
  
## Развертывание пользовательского шаблона проекта с помощью шаблона проекта VSIX  
 Следующие шаги показывают, как использовать проект VSIX для упаковки шаблон проекта, можно использовать совместно с другими разработчиками или отправить в коллекции Visual Studio.  
  
1.  Создание шаблона проекта.  
  
    1.  Откройте проект, из которого создается шаблон. Этот проект может быть любого типа проекта.  
  
    2.  В меню **Файл** выберите команду **Экспорт шаблона**. Выполните шаги мастера.  
  
         ZIP\-файл создается в %USERPROFILE%\\My документы\\Visual Studio *\< версия \>*\\My Exported Templates\\.  
  
2.  Создайте пустой проект VSIX.  
  
     В меню **Файл** последовательно выберите пункты **Создать** и **Проект**. Выберите либо **Visual Basic** или **Visual C\#**. Выбранный узел и выберите **расширения**, а затем выберите **проект VSIX**.  
  
3.  Добавьте в проект файл с расширением ZIP. Задайте его **Копировать в выходной каталог** свойства `Copy Always`.  
  
4.  В **обозревателе решений**, дважды щелкните `source.extension.vsixmanifest` файл, чтобы открыть его в **Конструктор манифеста VSIX**, и внесите следующие изменения:  
  
    -   Задайте **Название продукта** на **Мой проект шаблона**.  
  
    -   Задайте **идентификатор продукта** на **MyProjectTemplate \- 1**.  
  
    -   Задайте **Автор** на **Fabrikam**.  
  
    -   Задайте **Описание** на **шаблон проекта**.  
  
    -   В **активы** добавьте **Microsoft.VisualStudio.ProjectTemplate** и значения его путь имя ZIP\-файла.  
  
5.  Сохраните и закройте файл source.extension.vsixmanifest.  
  
6.  Выполните сборку проекта.  
  
7.  В выходном каталоге дважды щелкните VSIX\-файл.  
  
8.  Объект **установщик VSIX** появится окно сообщения. Следуйте инструкциям для установки расширения.  
  
9. Закройте Visual Studio и откройте его снова.  
  
10. Выберите **расширения и обновления** \(на **средства** меню\) и выберите **шаблоны** категории. Один из доступных расширений должен быть **Мой проект шаблона**.  
  
11. Новый шаблон проекта отображается в **Новый проект** диалоговое окно в том же месте как исходный шаблон проекта. Например, если вы создали шаблон с именем **VB консоли** из консольного приложения Visual Basic, **VB консоли** появляется в той же области, как Visual Basic **Консольное приложение** шаблона.  
  
#### Чтобы указать расположение шаблона в диалоговом окне Новый проект  
  
1.  Шаблон папки, содержащиеся в *путь установки Visual Studio*\\Common7\\IDE\\ProjectTemplates и *путь установки Visual Studio*\\Common7\\IDE\\ItemTemplates каталоги. Имена разделов верхнего уровня в диалоговом окне Новый проект неполное соответствие имен папок шаблона. Там, где они различаются, используйте имя папки шаблонов.  
  
     Измените расширение файла .vsix на ZIP и откройте файл.  
  
2.  Создайте новую папку с тем же именем, как шаблон должен отобразиться в части диалогового окна нового проекта.  
  
3.  Если шаблон будет отображаться в подразделе, создайте вложенную папку с тем же именем.  
  
4.  Переместите ZIP\-файл шаблона в новую папку.  
  
5.  Измените расширение .zip на VSIX.  
  
6.  Откройте манифест VSIX.  
  
7.  В манифест VSIX, обновите **активов** путь к шаблону, чтобы она указывала на корневой каталог дерева, содержащий файл шаблона. Например если шаблон находится в \\CSharp\\Windows, ссылки должны указывать \\CSharp.
