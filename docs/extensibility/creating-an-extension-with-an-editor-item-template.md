# Создание расширения с помощью редактора шаблона элемента
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Можно использовать шаблоны элементов, включенных в Visual Studio SDK, чтобы создать простой редактор расширениями классификаторов, крайних элементов и полей в редактор. Шаблоны элементов редактора доступны для проектов Visual C\# или Visual Basic VSIX.  
  
## Предварительные требования  
 Начиная с Visual Studio 2015, не установить пакет SDK для Visual Studio из центра загрузки. Она будет включена в качестве дополнительного компонента в установку Visual Studio. VS SDK также можно установить позже. Для получения дополнительной информации см. [Установка Visual Studio SDK](../extensibility/installing-the-visual-studio-sdk.md).  
  
## Создание расширения классификатора  
 Шаблон элемента классификатора редактор создает редактор классификатор, цвета текста, соответствующего \(в этом случае все\) в любой текстовый файл.  
  
1.  В **Новый проект** диалогового окна разверните **Visual C\#** или **Visual Basic** и нажмите кнопку **расширения**. В **шаблоны** выберите **проект VSIX**. В **имя** введите `TestClassifier`. Нажмите кнопку **ОК**.  
  
2.  В **обозревателе решений**, щелкните правой кнопкой мыши узел проекта и выберите **Добавить или создать элемент**. Перейдите в Visual C\# **расширения** и выберите команду **классификатора редактор**. Оставьте имя файла по умолчанию \(EditorClassifier1.cs\).  
  
3.  Существует три файла кода следующим образом:  
  
    -   Содержит EditorClassifier1.cs `EditorClassifier1` класса.  
  
    -   Содержит EditorClassifier1ClassificationDefinition.cs `OEditorClassifier1ClassificationDefinition` класса.  
  
    -   Содержит EditorClassifier1Format.cs `EditorClassifier1Format`  класса.  
  
    -   Содержит EditorClassifier1Provider.cs `EditorClassifier1Provider` класса.  
  
4.  Выполните сборку решения и запустите отладку. Откроется экспериментальный экземпляр Visual Studio.  
  
     При открытии текстового файла, весь текст подчеркнут фиолетовые фоне.  
  
## Создание оформления текста относительно расширения  
 Шаблон оформления текста редактор создает обрамления относительно текста, которое помечает все экземпляры текстового символа «», использовав поле с красным контуром и синим фоном. Это относительно текста так как поле всегда наложения «» символов, даже если они перемещаются или переформатирован.  
  
1.  В **Новый проект** диалогового окна разверните **Visual C\#** или **Visual Basic** и нажмите кнопку **расширения**. В **шаблоны** выберите **проект VSIX**. В **имя** введите `TestAdornment`. Нажмите кнопку **ОК**.  
  
2.  В **обозревателе решений**, щелкните правой кнопкой мыши узел проекта и выберите **Добавить или создать элемент**. Перейдите в Visual C\# **расширения** и выберите команду **оформления текста редактора**. Оставьте имя файла по умолчанию \(TextAdornment1.cs\/vb\).  
  
3.  Существует два файла кода следующим образом:  
  
    -   Содержит TextAdornment1.cs `TextAdornment1` класса.  
  
    -   содержит extAdornment1TextViewCreationListener.cs `TextAdornment1TextViewCreationListener` класса.  
  
4.  Выполните сборку решения и запустите отладку. Откроется экспериментальный экземпляр. При открытии текстового файла, «» символов в текст были выделены красным контуром с синим фоном.  
  
## Создание расширения обрамления относительно окна просмотра  
 Шаблон оформления просмотра редактор создает обрамления относительно окна просмотра, который добавляет фиолетовые поле с красным контуром в правом верхнем углу окна просмотра.  
  
> [!NOTE]
>  *Просмотра* — это область представления текста, отображаемого на текущий момент.  
  
#### Для создания расширения обрамления просмотра с помощью шаблона оформления просмотра редактора  
  
1.  В **Новый проект** диалогового окна разверните **Visual C\#** или **Visual Basic** и нажмите кнопку **расширения**. В **шаблоны** выберите **проект VSIX**. В **имя** введите `ViewportAdornment`. Нажмите кнопку **ОК**.  
  
2.  В **обозревателе решений**, щелкните правой кнопкой мыши узел проекта и выберите **Добавить или создать элемент**. Перейдите в Visual C\# **расширения** и выберите команду **обрамления просмотра редактор**. Оставьте имя файла по умолчанию \(ViewportAdornment1.cs\/vb\).  
  
3.  Существует два файла кода следующим образом:  
  
    -   Содержит ViewportAdornment1.cs `ViewportAdornment1` класса.  
  
    -   Содержит ViewportAdornment1TextViewCreationListener.cs `ViewportAdornment1TextViewCreationListener` класса  
  
4.  Выполните сборку решения и запустите отладку. Откроется экспериментальный экземпляр. Если создается новый текстовый файл, фиолетовые поле с красным контуром отображается в правом верхнем углу окна просмотра.  
  
## Создание расширения полей  
 Шаблон поля редактор создает зеленую границу, которая отображается вместе со слова «Hello world\!» под горизонтальной полосы прокрутки.  
  
#### Для создания расширения полей с помощью шаблона края редактора  
  
1.  В **Новый проект** диалогового окна разверните **Visual C\#** или **Visual Basic** и нажмите кнопку **расширения**. В **шаблоны** выберите **проект VSIX**. В **имя** введите `MarginExtension`. Нажмите кнопку **ОК**.  
  
2.  В **обозревателе решений**, щелкните правой кнопкой мыши узел проекта и выберите **Добавить или создать элемент**. Перейдите в Visual C\# **расширения** и выберите команду **обрамления просмотра редактор**. Оставьте имя файла по умолчанию \(EditorMargin1.cs\/vb\).  
  
3.  Существует два файла кода следующим образом:  
  
    -   Содержит EditorMargin1.cs `EditorMargin1` класса.  
  
    -   Содержит EditorMargin1Factory.cs `EditorMargin1Factory` класса.  
  
4.  Создать этот проект и запустить отладку. Откроется экспериментальный экземпляр. При открытии текстового файла, зеленую границу, которая содержит слова «Hello EditorMargin1» отображается под горизонтальной полосы прокрутки.
