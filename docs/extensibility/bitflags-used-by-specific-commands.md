# Битовые флаги, используемые команды
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Поведение ряда функций в API подключаемого модуля управления источника можно изменить путем задания одного или нескольких битов в одно значение. Эти значения называются битовые флаги. Различные битовые флаги, используемые API подключаемого модуля управления источника подробно описываются, сгруппированных по функция, которая использует их.  
  
## Извлечен флаг  
 Этот флаг может быть задан для любого [SccAdd](../extensibility/sccadd-function.md) или [SccCheckin](../extensibility/scccheckin-function.md).  
  
|Flag|Значение|Описание|  
|----------|--------------|--------------|  
|`SCC_KEEP_CHECKEDOUT`|0x1000|Оставьте файл извлеченным.|  
  
## Добавление флагов  
 Эти флаги используются [SccAdd](../extensibility/sccadd-function.md).  
  
|Flag|Значение|Описание|  
|----------|--------------|--------------|  
|`SCC_FILETYPE_AUTO`|0x00|Подключаемый модуль системы управления версиями должен автоматически определить, является ли файл текстовым или двоичным.|  
|`SCC_FILETYPE_TEXT`|0x01|Тип файла — текст.|  
|`SCC_FILETYPE_BINARY`|0x04|Файл имеет двоичный. **Note:**  `SCC_FILETYPE_TEXT` и `SCC_FILETYPE_BINARY` флаги являются взаимоисключающими. Установите один или оба.|  
|`SCC_ADD_STORELATEST`|0x02|Хранить только последнюю версию \(не разности\).|  
  
## Флаги поиска различий  
 [SccDiff](../extensibility/sccdiff-function.md) Использует эти флаги для определения области действия операции поиска различий.`SCC_DIFF_QD_xxx` Флаги являются взаимоисключающими. Если указан один из них, не визуально — должен быть предоставлен. В «быстрый diff» \(QD\), подключаемый модуль не определить, как файл отличается, только в том случае, если он отличается. Если ни один из этих флагов указан, выполняется «visual diff»; Подробный файл различия вычисляются и отображаются. Если запрошенный QD не поддерживается, подключаемый модуль перемещается к следующему лучший. Для экземпляра если подключаемый модуль не поддерживает это IDE запрашивает контрольную сумму, подключаемый модуль не полное содержимое Проверьте \(по\-прежнему гораздо быстрее, чем визуального отображения\).  
  
|Flag|Значение|Описание|  
|----------|--------------|--------------|  
|`SCC_DIFF_IGNORECASE`|0x0002|Игнорируйте различия в регистре.|  
|`SCC_DIFF_IGNORESPACE`|0x0004|Игнорируйте различия пробела. **Note:**  `SCC_DIFF_IGNORECASE` И `SCC_DIFF_IGNORESPACE` флаги — необязательный битовые флаги.|  
|`SCC_DIFF_QD_CONTENTS`|0x0010|QD, сравнивая все содержимое файла.|  
|`SCC_DIFF_QD_CHECKSUM`|0x0020|QD по контрольной суммы.|  
|`SCC_DIFF_QD_TIME`|0x0040|QD штампом даты и времени файла.|  
|`SCC_DIFF_QUICK_DIFF`|0x0070|Это маска используется для проверки QD битовые флаги. Не следует передавать в функцию; три битовые флаги QD являются взаимоисключающими. QD всегда означает без отображения пользовательского интерфейса.|  
  
## Флаг PopulateList  
 Этот флаг используется [SccPopulateList](../extensibility/sccpopulatelist-function.md) в `fOptions` параметр.  
  
|Flag|Значение|Описание|  
|----------|--------------|--------------|  
|`SCC_PL_DIR`|0x00000001L|Каталоги, файлы не передает интегрированной среды разработки.|  
  
## Флаги PopulateDirList  
 Эти флаги используются [SccPopulateDirList](../extensibility/sccpopulatedirlist-function.md) в `fOptions` параметр.  
  
|Значение параметра|Значение|Описание|  
|------------------------|--------------|--------------|  
|SCC\_PDL\_ONELEVEL|0x0000|Проверьте только один уровень папок для каталогов \(это значение по умолчанию\).|  
|SCC\_PDL\_RECURSIVE|0x0001|Рекурсивно проверить все каталоги каждого из данного каталога.|  
|SCC\_PDL\_INCLUDEFILES|0x0002|Включить имена файлов в процесс проверки.|  
  
## Флаги OpenProject  
 Эти флаги используются [SccOpenProject](../extensibility/sccopenproject-function.md) в `dwFlags` параметр.  
  
|Значение параметра|Значение|Описание|  
|------------------------|--------------|--------------|  
|SCC\_OP\_CREATEIFNEW|0x00000001L|Если проект не существует в системе управления версиями, создайте его. Если этот флаг не установлен, запрашивать пользователя проекта для создания \(если `SCC_OP_SILENTOPEN` установлен флаг\).|  
|SCC\_OP\_SILENTOPEN|0x00000002L|Не запрашивать пользователя для создания проекта; только возвращаемым `SCC_E_UNKNOWNPROJECT`.|  
  
## Получить флаги  
 Эти флаги используются [SccGet](../extensibility/sccget-function.md) и [SccCheckout](../extensibility/scccheckout-function.md).  
  
|Flag|Значение|Описание|  
|----------|--------------|--------------|  
|`SCC_GET_ALL`|0x00000001L|IDE передается каталоги, файлы не: все файлы в этих каталогах.|  
|`SCC_GET_RECURSIVE`|0x00000002L|Передает каталоги интегрированной среды разработки: эти каталоги и всех подкаталогах.|  
  
## nOption значения  
 Эти флаги используются [SccSetOption](../extensibility/sccsetoption-function.md) в `nOption` параметр.  
  
|Flag|Значение|Описание|  
|----------|--------------|--------------|  
|`SCC_OPT_EVENTQUEUE`|0x00000001L|Установите состояние очереди событий.|  
|`SCC_OPT_USERDATA`|0x00000002L|Укажите данные пользователя для `SCC_OPT_NAMECHANGEPFN`.|  
|`SCC_OPT_HASCANCELMODE`|0x00000003L|Интегрированная среда разработки может обрабатывать "Отмена"|  
|`SCC_OPT_NAMECHANGEPFN`|0x00000004L|Задайте обратный вызов для изменения имени.|  
|`SCC_OPT_SCCCHECKOUTONLY`|0x00000005L|Отключение источника управления подключаемый модуль пользовательского интерфейса извлечения и не задан рабочий каталог.|  
|`SCC_OPT_SHARESUBPROJ`|0x00000006L|Добавьте из системы управления версиями, чтобы указать рабочий каталог. Попробуйте для совместного использования в связанный проект, если он является прямым потомком.|  
  
## битовые флаги dwVal  
 Эти флаги используются [SccSetOption](../extensibility/sccsetoption-function.md) в `dwVal` параметр.  
  
|Flag|Значение|Описание|Используемые `nOption` значение|  
|----------|--------------|--------------|-------------------------------------|  
|`SCC_OPT_EQ_DISABLE`|0x00L|Приостанавливает действие очереди событий.|`SCC_OPT_EVENTQUEUE`|  
|`SCC_OPT_EQ_ENABLE`|0x01L|Ведение журнала очереди событий.|`SCC_OPT_EVENTQUEUE`|  
|`SCC_OPT_HCM_NO`|0L|\(По умолчанию\) Режим без отмены; Подключаемый модуль необходимо указать при необходимости.|`SCC_OPT_HASCANCELMODE`|  
|`SCC_OPT_HCM_YES`|1L|Интегрированная среда разработки обрабатывает "Отмена".|`SCC_OPT_HASCANCELMODE`|  
|`SCC_OPT_SCO_NO`|0L|\(По умолчанию\) ОК, чтобы извлечь из пользовательского интерфейса подключаемого модуля; рабочий каталог установлен.|`SCC_OPT_SCCCHECKOUTONLY`|  
|`SCC_OPT_SCO_YES`|1L|Нет подключаемый модуль пользовательского интерфейса извлечения, не рабочий каталог.|`SCC_OPT_SCCCHECKOUTONLY`|
