# <a name="launching-a-program"></a>Запуск программы
Пользователи, желающие отладки программы, можно нажать клавишу F5 для запуска отладчика в интегрированной среде разработки. Это начинается последовательность событий, которые в конечном счете приводят к IDE соединиться модуля отладки (DE), которая в свою очередь подключен, или не присоединен, в программу следующим образом:  
  
1.  IDE сначала вызывает пакета проекта для получения параметров отладки решения активного проекта. Параметры включают начальный каталог, переменные среды, порт, в котором будет запускаться программа и DE, используемый для создания программы, если указано. Эти параметры передаются в отладочный пакет.  
  
2.  Если указано DE DE вызывает операционной системы, чтобы запустить программу. В результате запуска программы, загружается среды выполнения программы. Например если программа написанная на языке MSIL, общеязыковая среда выполнения будет вызываться для запуска программы.  
  
     - или -  
  
     Если DE не указан, порт вызывает операционной системы, чтобы запустить программу, вследствие чего среды выполнения программы для загрузки.  
  
    > [!NOTE]
    >  Если DE используется для запуска программы, вполне вероятно, что же DE будет присоединена к программе.  
  
3.  В зависимости от того, было ли DE или порт запустила программу DE или среда выполнения создает описание программы или узла и уведомляет порт, на котором выполняется программа.  
  
    > [!NOTE]
    >  Рекомендуется, среде выполнения создать узел программы, так как программа узел представляет собой упрощенный программы, можно отлаживать. Нет необходимости загрузить копию всей DE только для создания и регистрации узла программы. Если DE предназначен для запуска в процессе интегрированной среды разработки, но не интегрированной среды разработки на них работают, должен быть компонентом, который можно добавить узел программы в порт.  
  
 Вновь созданный программы, а также все другие программы, связанные с несвязанных, запущен или подключенные к одной интегрированной среде разработки, создания сеанса отладки.  
  
 Программным образом, при первом нажатии **F5**, [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]на отладочный пакет вызывает пакет проекта (она связана с типом запуска программы) через <xref:Microsoft.VisualStudio.Shell.Interop.IVsDebuggableProjectCfg.DebugLaunch%2A> метод, который в свою очередь заполнения <xref:Microsoft.VisualStudio.Shell.Interop.VsDebugTargetInfo2> структуру с параметры отладки для активного проекта решения. Эта структура передается обратно отладки пакета путем вызова <xref:Microsoft.VisualStudio.Shell.Interop.IVsDebugger2.LaunchDebugTargets2%2A> метод. Отладочный пакет создает экземпляр диспетчера отладочной сеанса (SDM), которая запускает программу, отладки и любые связанные отладчики.  
  
 Один из аргументов, переданных SDM — это GUID DE, который будет использоваться для запуска программы.  
  
 Если не является DE GUID `GUID_NULL`, SDM совместно создает DE, а затем вызывает его [LaunchSuspended](../../extensibility/debugger/reference/idebugenginelaunch2-launchsuspended.md) метод, чтобы запустить программу. Например, если программа записывается в машинном коде, затем `IDebugEngineLaunch2::LaunchSuspended` , скорее всего, вызовет `CreateProcess` и `ResumeThread` (функции Win32), чтобы запустить программу.  
  
 В результате запуска программы, загружается среды выполнения программы. Затем создается DE или среды выполнения [IDebugProgramNode2](../../extensibility/debugger/reference/idebugprogramnode2.md) интерфейса для описания программы и передает этот интерфейс для [AddProgramNode](../../extensibility/debugger/reference/idebugportnotify2-addprogramnode.md) известить порт, который программа запущена.  
  
 Если `GUID_NULL` передается, то программа открывает порт. После запуска программы, среда выполнения создает `IDebugProgramNode2` интерфейса для описания программы и передает их в `IDebugPortNotify2::AddProgramNode`. Это заставляет порт, на котором выполняется программа. Затем SDM Присоединяет отладчик к выполняющейся программе.
