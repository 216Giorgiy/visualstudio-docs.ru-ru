# <a name="visual-studio-debugger-glossary"></a>Глоссарий отладчика Visual Studio
Ниже приведены термины, используемые в [!INCLUDE[vsprvs](../../../code-quality/includes/vsprvs_md.md)] пакет SDK для отладки.  
  
## <a name="terms"></a>Термины  
 связанная точка останова  
 Абстракция для останова в коде. Существует однозначное соответствие между связанная точка останова и инструкцию точки останова в поток кода. Когда код выгрузки, связанных точек останова может отменить привязку.  
  
 причинно-следственных связей  
 Предоставляет возможность отслеживать логический поток выполнения на нескольких физических потоков, процессов и компьютеров и реконструкции стека вызовов, логический поток в любой момент в течение времени существования этого потока.  
  
 контекст кода  
 Предоставляет краткое описание позиции в коде в отладчик. Для большинства архитектур во время выполнения контекст кода представляет собой адрес в поток инструкций программы. Для нетрадиционных языков, в которых код не представлены инструкции, контекст кода может быть представлен другим способом.  
  
 Код пути  
 Представляет точку выполнения в коде, где будет выбрана ветвь или вызов функции. Трассировка стека представляет собой список путей кода вызов функции.  
  
 Подсистема отладки (DE)  
 Компонент, который позволяет выполнить отладку архитектуры во время выполнения. Отладчик работает совместно с интерпретатор или операционной системы и предоставляет службы отладки, такие как выполнение управления, точки останова и оценки выражений.  
  
 контекст документа  
 Предоставляет краткое описание позиции в файл исходного документа в отладчик. В большинстве языков контекст документа — позиция в исходном файле. Для нетрадиционных языков, для которых исходный файл не может быть текст, контекст документа могут быть представлены либо иным образом. См. также *позиция документа*.  
  
 Позиция в документе  
 Предоставляет краткое описание позиции в файле исходного кода в Интегрированной среде разработки. В большинстве языков позиции документа — положение в файле исходного кода. Для нетрадиционных языков позицию в документе может быть представлен по-другому. См. также *контексту документа*.  
  
 Ошибка точки останова  
 Абстракция для описания ошибки в ожидающая точка останова. Ошибка точки останова могут описывать ошибку в расположении, ожидающая точка останова, выражение, связанное с ожидающая точка останова или другие сведения, которые предотвращает привязки ожидающая точка останова расположение кода.  
  
 контекст оценки  
 Предоставляет краткое описание контекст программирования для оценки выражения. Как правило контекст вычисления является областью. При выполнении вычисления выражения в контексте выражения, выражения контекст предоставляет правила области, соответствующие точки создания. Например контексте выражения, созданные в кадре стека предоставляет контекст для оценки локальные переменные, параметры метода, члены класса (если применимо) и глобальные переменные.  
  
 перехваченные исключения  
 Исключение перехватывается обработчиком отладки, даже если нет механизма обработки исключений на месте в текущем кадре стека.  
  
 JustMyCode  
 Концепция отладки кода, который принадлежит пользователю и пропуск всех промежуточного кода, таких как системного кода, даже если исходный код доступен для этого кода системы.  
  
 ожидающая точка останова  
 Реализует абстракцию для точки останова до, во время и после кода загружается и способ виртуализировать точки останова. В ожидании точки останова:  
  
-   Содержит все сведения, необходимые для привязки точки останова в код в одну или несколько программ.  
  
-   Выполнить привязку в нескольких местах кода, в одну или несколько программ.  
  
-   Никогда не привязывает самого кода.  
  
 Загружает каждого кода времени, всех ожидающих точек останова в программе, проверяются на предмет того, можно привязать. Ожидающая точка останова называется содержат всех связанных точек останова, он выполняет привязку.  
  
 процесс  
 Физический процесс Win32. Процесс может содержать несколько программ. См. также *программы*.  
  
 программа  
 Одно пространство имен, выполняемых в рамках конкретной архитектуры во время выполнения. См. также *процесс*.  
  
 Диспетчер сеансов отладки (SDM)  
 Управляет любое количество обработчиков отладки, отладки любое количество программ в нескольких процессах на любом количестве компьютеров. На базовом уровне SDM является мультиплексор отладчики. Кроме того SDM предоставляет единое представление сеанса отладки в интегрированную среду разработки.  
  
 кадр стека  
 Представляет состояние процесса вычисления на определенный кадр и определенного уровня вложенных вызовов функций.  
  
 thread  
 Обобщенный понятие выполнения инструкций на основе стека в хотя бы одну программу.  
  
 Предупреждение точки останова  
 Абстракция для описания предупреждение в ожидающая точка останова. Предупреждение точки останова описывает причины, почему ожидающая точка останова не еще связана расположение кода. Возможно, что код не загружен еще для расположения, описываемого ожидающая точка останова или по другой причине.
