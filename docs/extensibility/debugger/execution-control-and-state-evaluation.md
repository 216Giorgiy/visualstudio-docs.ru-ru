# Управление выполнением и оценки состояния
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

Отладка приложения необходима реализация такие функции управления выполнения, как пошаговое выполнение функций, остановка на точках останова и продолжения выполнения. Visual Studio, отладка баз его выполнения элемента управления, на события обмениваются компоненты отладчика.  
  
## <a name="in-this-section"></a>Содержание  
 [Управление программой](../../extensibility/debugger/program-control.md)  
 Перечисляет следующие подпрограммы, которые выполняются на уровне программы: Установка следующего оператора, выполнения, пошаговое выполнение, продолжение, приостановка и возобновление.  
  
 [Методы, связанные с точки останова](../../extensibility/debugger/breakpoint-related-methods.md)  
 Определяет границу и ожидающих типы точек останова, Visual Studio поддерживает.  
  
 [Оценка стека вызова](../../extensibility/debugger/call-stack-evaluation.md)  
 Обсуждается реализация методы, позволяющие просмотра кадров стека, стека вызовов во время режима приостановки.  
  
 [Вычисление выражений](../../extensibility/debugger/expression-evaluation-visual-studio-debugging-sdk.md)  
 Объясняет, как ядро отладки (DE) диспетчера выражений оценки (EE) и сеанс отладки вовлечены синтаксического разбора и вычисление выражения, введенные в одно из окон интегрированной среды разработки.  
  
 [События элементов управления](../../extensibility/debugger/control-events.md)  
 Описывает интерфейс, используемый для отправки событий во время выполнения управляемой программы.
