# <a name="choosing-a-debug-engine-implementation-strategy"></a>Выбор стратегии реализации ядра отладки
Архитектура во время выполнения используется для определения стратегии реализации отладки ядра (DE). Модуль отладки могут быть созданы в процессе в программу, чтобы провести отладку, в работе диспетчера сеанса отладки для Visual Studio (SDM) или out of process для обеих функций. Следующие рекомендации помогут вам выбрать один из этих трех стратегий.  
  
## <a name="guidelines"></a>Рекомендации  
 Хотя возможна DE быть out of process как SDM и программу для отладки, нет обычно причин для этого. Вызовы через границы процессов выполняются относительно медленно.  
  
 Отладка модулей уже предоставлены для среды выполнения собственного Win32, а также для общеязыковой среды выполнения. Если необходимо заменить DE какую-либо из этих сред, необходимо создать процесс DE с SDM.  
  
 В противном случае можно выбрать между созданием DE в процессах в SDM или в процесс для отлаживаемой программы. Важно учитывать потребности в вычислителе выражений DE частый доступ к программе хранилища символов и хранилище символов могут загружаться в память для быстрого доступа. Также учтите следующее:  
  
-   Если не много вызовов между средство оценки выражений и хранилище символов или хранилище символов можно считать в области памяти SDM, создайте DE-процесс для SDM. После присоединения к программе необходимо вернуться к SDM CLSID ядро отладки. SDM использует этот идентификатор CLSID для создания экземпляра в процессе DE.  
  
-   Если DE необходимо вызвать программу для доступа к хранилищу символов, создайте DE в работе с программой. В этом случае программа создает экземпляр DE.
