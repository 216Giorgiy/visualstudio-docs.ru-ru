# <a name="session-debug-manager"></a>Диспетчер сеансов отладки
Диспетчер сеансов отладки (SDM) управляет любое количество обработчиков отладки (DE) отладка любое количество программ в нескольких процессах на любом числе компьютеров. Помимо мультиплексор модуля отладки, SDM обеспечивает единое представление сеанса отладки в Интегрированной среде разработки.  
  
## <a name="session-debug-manager-operation"></a>Операция диспетчера сеанса отладки  
 Диспетчер сеансов отладки (SDM) управляет DE. Может существовать более одного модуля отладки, выполняющийся на компьютере, в то же время. Для мультиплексировать DEs, SDM включает ряд интерфейсы из DEs и представляет их в интегрированную среду разработки, как единый интерфейс.  
  
 Чтобы повысить производительность, не мультиплексируются некоторые интерфейсы. Вместо этого они используются непосредственно из DE и вызовы этих интерфейсов не проходят через SDM. Например интерфейсы, используемые с памятью, кода и контексты документа не мультиплексируются, поскольку они ссылаются на определенной инструкции, памяти или документа в указанную программу отладки программы с определенной DE. Нет других DE должны заняться на данном уровне связи.  
  
 Это не относится всех контекстов. Вызовы интерфейса контекст оценки выражения проходят через SDM. Во время вычисления выражения упаковывает SDM [IDebugExpression2](../../extensibility/debugger/reference/idebugexpression2.md) интерфейс, который предоставляет интегрированную среду разработки, поскольку при вычислении этого выражения, оно может включать несколько DEs, отладке программ, в том же процессе, которые могут быть выполняется на том же потоке.  
  
 Обычно SDM выступает в качестве механизм делегирования, но он может действовать как механизме оповещения. Например при вычислении выражения SDM выступает в качестве широковещательных механизм для уведомления все DEs, их выполнение кода в указанный поток. Аналогично когда SDM получает событие остановки, он осуществляет вещание все программы, что им прекратить выполнение. При вызове шаг SDM осуществляет широковещательную передачу все программы, можно по-прежнему работает. Каждый DE также рассылаются точки останова.  
  
 SDM не отслеживает текущей программы, поток или кадр стека. Процесс, программа и потока данные передаются в SDM в сочетании с конкретными событиями отладки.
