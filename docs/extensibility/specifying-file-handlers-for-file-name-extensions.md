# Указание обработчиков файлов для расширений имен файлов
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Существует ряд способов для определения приложения, который обрабатывает файл, имеющий определенное расширение. Команды OpenWithList и подключу — два способа задания обработчиков файлов в разделе реестра для расширения файла.  
  
## Команда OpenWithList  
 Когда правой кнопкой мыши файл в проводнике Windows, появляется **Откройте** команды. Если более одного продукта связан с расширением, вы видите **Открыть с помощью** подменю.  
  
 Можно зарегистрировать различных приложений открывать расширение, установив ключ OpenWithList для расширения файла в разделе HKEY\_CLASSES\_ROOT. Приложения, перечисленные в этом разделе для расширения имени файла отображаются под **рекомендуемых программ** заголовок в **Открыть с помощью** диалоговое окно. В следующем примере показано приложений, зарегистрированных для открытия расширением VCPROJ\-файл.  
  
```  
HKEY_CLASSES_ROOT\  
   .vcproj\  
      (default)="VisualStudio.vcproj.14.0"  
      OpenWithList\  
         devenv.exe  
```  
  
> [!NOTE]
>  Ключи, указание приложений являются HKEY\_CLASSES\_ROOT\\Applications в списке.  
  
 Добавив ключ OpenWithList, объявить, что ваше приложение поддерживает расширение файла, даже если другое приложение получает право на владение расширения. Это может быть следующей версии приложения или другого приложения.  
  
## Подключу  
 Программные идентификаторы \(ProgID\), понятные версии идентификатор ClassID, определить версию приложения или COM\-объект. Каждый совместно создаваемый объект должен быть собственный идентификатор ProgID. Например, VisualStudio.DTE.7.1 запускает Visual Studio .NET 2003 при запуске VisualStudio.DTE.10.0 [!INCLUDE[vsprvs](../code-quality/includes/vsprvs_md.md)]. Как владелец проекта типа или типа элемента проекта необходимо создать зависящие от версии идентификатор ProgID для расширения файла. Эти идентификаторы ProgID может быть избыточным, в том, что более чем один программный идентификатор ProgID может запустить то же приложение. Дополнительные сведения см. в разделе [Регистрация команды для расширений имен файлов](../extensibility/registering-verbs-for-file-name-extensions.md).  
  
 Используется следующее соглашение об именовании для версий файла идентификаторы ProgID во избежание дублирования с регистрацией других поставщиков:  
  
|Расширение файла|Версии идентификатор ProgID|  
|----------------------|---------------------------------|  
|.Extension|ProductName. extension.versionMajor.versionMinor|  
  
 Можно зарегистрировать различных приложений, которые могут открывать с определенным расширением, добавив версии идентификаторы ProgID в качестве значения для этого*\< расширение \>*\\OpenWithProgids ключа. Этот раздел реестра со списком альтернативных идентификаторы ProgID, связанный с расширением файла. Приложения, связанные с перечисленных идентификаторы ProgID появляются в **Открыть с помощью***Марка* подменю. Если указано и то же приложение как `OpenWithList` и `OpenWithProgids` ключей, операционная система объединяет дубликаты.  
  
> [!NOTE]
>  `OpenWithProgids` Ключ поддерживается только в Windows XP. Из\-за других операционных системах игнорировать этот ключ, не используйте его как только регистрация для обработчиков файлов. Этот ключ можно используйте для улучшения качества работы пользователя в Windows XP.  
  
 Добавьте нужные идентификаторы ProgID в качестве значения типа REG\_NONE. Ниже приведен пример регистрации идентификаторы ProgID для расширения имени файла \(.*ext*\).  
  
```  
HKEY_CLASSES_ROOT\  
   .ext\  
      (default)="MyProduct.ext.14.0"  
      OpenWithProgids  
         progid        REG_NONE (zero-length binary value)  
         otherprogid   REG_NONE (zero-length binary value)  
```  
  
 Идентификатор ProgID, заданный как значение по умолчанию для расширения файла — обработчика файлов по умолчанию. Если изменить идентификатор ProgID для расширения имени файла с предыдущей версии [!INCLUDE[vsprvs](../code-quality/includes/vsprvs_md.md)] или может выполняться на другие приложения, то необходимо зарегистрировать `OpenWithProgids` ключа для расширения файла и укажите новый идентификатор ProgID в списке вместе с вам поддерживать старые идентификаторы ProgID. Например:  
  
```  
HKEY_CLASSES_ROOT\  
   .vcproj\  
      (default)="VisualStudio.vcproj.14.0"  
      OpenWithProgids  
         vcprojfile              //old progid  
         VisualStudio.vcproj.12.0 //old progid  
         VisualStudio.vcproj.14.0 //new progid  
```  
  
 Если старый идентификатор ProgID содержит команды, связанные с ним, а затем эти команды также отображаются в разделе **Открыть с помощью** *Марка* в контекстном меню.
