# <a name="command-flag-element"></a>Элемент Command флаг
Изменяет его родительского элемента.  
  
## <a name="syntax"></a>Синтаксис  
  
```  
<CommandFlag>DynamicVisibility</CommandFlag>  
```  
  
## <a name="attributes-and-elements"></a>Атрибуты и элементы  
 В следующем разделе описаны значения допустимый элемент.  
  
### <a name="attributes"></a>Атрибуты  
 Отсутствует.  
  
### <a name="child-elements"></a>Дочерние элементы  
  
|Значение|Описание|  
|-----------|-----------------|  
|AllowParams|Указывает, что пользователи могут вводить параметры команды в **команда** окна при вводе каноническое имя команды.<br /><br /> Допустимо для: `Button`|  
|AlwaysCreate|Меню создается, даже если он не имеет групп и кнопки.<br /><br /> Допустимо для: `Menu`|  
|CaseSensitive|Записи пользователя учитывается регистр.<br /><br /> Допустимо для: `Combo`|  
|CommandWellOnly|Примените этот флаг, если команда не отображается в меню верхнего уровня, и вы хотите сделать его доступным для настройки дополнительных оболочки, например, для привязки к сочетания клавиш. После установки пакета VSPackage можно настроить эти команды, открыв **параметры** диалоговое окно и последующего изменения размещения команд в разделе **клавиатуры среды** категории. Этот флаг не влияет на расположение на контекстных меню, панелей инструментов, контроллеры меню или подменю.<br /><br /> Допустимые для: `Button`, `Combo`|  
|DefaultDisabled|По умолчанию команда будет отключен, если пакет VSPackage, который его реализует не загружен или `QueryStatus` не был вызван метод.<br /><br /> Допустимые для: `Button`, `Combo`|  
|DefaultDocked|Закрепить по умолчанию. Этот параметр больше не применяется к панели инструментов, так как они всегда закреплены.|  
|DefaultInvisible|По умолчанию команда выполняется без внешних проявлений, если пакет VSPackage, который его реализует не загружен или `QueryStatus` не был вызван метод.<br /><br /> Рекомендуется сочетать с `DynamicVisibility` флаг.<br /><br /> Допустимые для: `Button`, `Combo`, `Menu`|  
|DontCache|Среда разработки не кэширует `QueryStatus` результаты метода для этой команды.<br /><br /> Для меню это значение определяет контроллер меню не следует кэшировать текста пунктов меню. Этот флаг используется, если меню содержит динамические элементы или элементы, имеющие динамический текст.<br /><br /> Допустимые для: `Button`, `Menu`|  
|DynamicItemStart|Обозначает начало динамического списка. Это позволяет среде для построения списка путем последовательного вызова `QueryStatus` метод для элементов списка, пока не будет возвращено OLECMDERR_E_UNSUPPORTED флаг. Это хорошо подходит для элементов, таких как недавно использовавшихся (файлов MRU) в списках и окна.<br /><br /> Допустимо для: `Button`|  
|DynamicVisibility|Видимость команды можно изменить с помощью `QueryStatus` метода или с помощью контекста GUID, включенный в `VisibilityConstraints` раздела.<br /><br /> Применяется к командам, которые отображаются в меню и на панели инструментов окна инструментов, но не на панели инструментов верхнего уровня, которые отображаются в главном окне. Элементы верхнего уровня панели инструментов может быть отключена, но не скрыт, если флаг OLECMDF_INVISIBLE возвращается из `QueryStatus` метод. Команды панели инструментов, которые отображаются на панели инструментов окна инструментов могут быть скрыты.<br /><br /> В меню этот флаг также указывает, что должен быть автоматически скрыт Если скрыты все его члены. Этот флаг обычно назначается подменю, так как это поведение уже в меню верхнего уровня.<br /><br /> Этот флаг следует использовать в сочетании с `DefaultInvisible` флаг.<br /><br /> Допустимые для: `Button`, `Combo`, `Menu`|  
|Фильтрация ввода|См. в разделе фильтрация ключи в разделе [элемент поля со списком](../extensibility/combo-element.md).<br /><br /> Допустимо для: `Combo`|  
|FixMenuController|Если эта команда находится на контроллер меню, команда используемую по умолчанию. то есть при каждом нажатии кнопки меню контроллера сам выборе команды. Если контроллер меню `TextIsAnchorCommand` набор флагов, то контроллер меню также принимает его текст из команды с `FixMenuController` флаг.<br /><br /> Должен иметь только одну команду на контроллере меню `FixMenuController` флаг. Если более одной команды помечен таким образом, последней команды в меню становится команды по умолчанию.<br /><br /> Допустимо для: `Button`|  
|IconAndText|Показать значок и текст в меню и панели инструментов.<br /><br /> Допустимые для: `Button`, `Combo`, `Menu`|  
|NoAutoComplete|Функция автозаполнения отключена.<br /><br /> Допустимо для: `Combo`|  
|NoButtonCustomize|Не позволяет пользователю настраивать эту кнопку.<br /><br /> Допустимые для: `Button`, `Combo`|  
|NoKeyCustomize|Не включайте Настройка клавиатуры.<br /><br /> Допустимые для: `Button`, `Combo`|  
|NoShowOnMenuController|Если эта команда находится на контроллер меню, команда не отображается в раскрывающемся списке.<br /><br /> Допустимо для: `Button`|  
|NotInTBList|Не отображается в списке доступных панелей инструментов. Это допустимо только для типов меню панели инструментов.<br /><br /> Допустимо для: `Menu`|  
|NoToolbarClose|Пользователь не может закрыть панель инструментов. Это допустимо только для типов меню панели инструментов.<br /><br /> Допустимо для: `Menu`|  
|PICT|Показать только значок на панели инструментов, но только для текста меню. Если значок не указан, показывает активную пустую область на панели инструментов.<br /><br /> Допустимо для: `Button`|  
|PostExec|Выполняет команду без блокировки. В среде разработки Откладывает выполнение до завершения всех предварительной обработки запросов.<br /><br /> Допустимо для: `Button`|  
|RouteToDocs|Команда направляется в активный документ.<br /><br /> Допустимо для: `Button`|  
|StretchHorizontally|Если этот флаг установлен, ширина становится минимальную ширину поле со списком, а если есть место на панели инструментов, поле со списком растягивается с целью заполнения доступного пространства. Это происходит только в том случае, если панель инструментов закреплена горизонтально и только одно поле со списком на панели инструментов можно использовать флаг (этот флаг игнорируется для всех, кроме перечисленных в первом поле со списком).<br /><br /> Допустимо для: `Combo`|  
|TextMenuUseButton|Используйте `ButtonText` для меню. Это поле по умолчанию является `MenuText` , если он указан.<br /><br /> Допустимо для: `Button`|  
|TextChanges|Текст команды или меню можно изменить во время выполнения, обычно через `QueryStatus` метод.<br /><br /> Допустимые для: `Button`, `Menu`|  
|TextChangesButton|Допустимо для: `Button`|  
|TextIsAnchorCommand|Для контроллера меню текста меню берется из команды по умолчанию (привязки). Команда привязки является последней выбранной или заблокирована. Если этот флаг не установлен, меню контроллер использует свой собственный `MenuText` поля. Тем не менее щелкнув меню контроллера по-прежнему позволяет последней выбранной команды в текущем контроллере.<br /><br /> Мы рекомендуем сочетании этот флаг с `TextChanges` флаг.<br /><br /> Этот флаг применяется только к меню типа MenuController или MenuControllerLatched.<br /><br /> Допустимо для: `Menu`|  
|TextMenuCtrlUseMenu|Используйте `MenuText` на контроллеры меню. Это поле по умолчанию является `ButtonText`.<br /><br /> Допустимо для: `Button`|  
|TextMenuUseButton|Используйте `ButtonText` для меню. Это поле по умолчанию является `MenuText` , если он указан.<br /><br /> Допустимо для: `Button`|  
|TextOnly|Показать только текст на панели инструментов или меню, но нет значка, даже если указан этот значок.<br /><br /> Допустимо для: `Button`|
