# Глоссарий Visual Studio SDK
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

В этом глоссарии содержатся определения терминов, используемых в [!INCLUDE[vsipsdk](../extensibility/includes/vsipsdk_md.md)] документации.  
  
## Термины  
 надстройки  
 Служебное приложение, драйвера или другого программного обеспечения, добавленных основного приложения. В среде разработки Visual Studio \(IDE\), надстройка — это приложение на основе автоматизации, который расширяет возможности интегрированной среды разработки.  
  
 модель автоматизации  
 Модель автоматизации, известная в предыдущих версиях Visual Studio как модель расширения является программный интерфейс, который предоставляет доступ к базовым процедурам этого диска IDE. Надстройки, мастера и макросы использовать объекты в модели автоматизации для управления или расширить функциональные возможности интегрированной среды разработки.  
  
 контекст пользовательского интерфейса командной строки  
 Ассоциация GUID с видимостью команды пользовательского интерфейса или элемента, например панель инструментов. Контекст пользовательского интерфейса командной строки — в отличие от выбора контекста в том, что он не присоединен к окну.  
  
 Контекст пользовательского интерфейса командной строки можно использовать для:  
  
-   Назначьте идентификатор GUID панель инструментов, которая появляется при активации определенного окна.  
  
-   Назначьте GUID доступности команды без необходимости загрузки или запуска VSPackage.  
  
-   Назначьте идентификатор GUID, влияют на активной привязки ключей.  
  
-   Назначьте идентификатор GUID, чтобы включить запись макроса.  
  
-   Назначьте идентификатор GUID для активации режима отладки или для переключения между режимами конструктора и выполнения в редакторе.  
  
 component  
 Часть программного обеспечения, могут быть сделаны части функциональных возможностей приложения без этого приложения, наличие любые существующие сведения о реализации программы. Связь между компонентом и приложения является исключительно через интерфейсы OLE стиля.  
  
 Диспетчер компонентов  
 Службы `SOleComponentManager`, который предоставляет координации служб без пользовательского интерфейса верхнего уровня компонентов.`SOleComponentManager` Службы реализует `IOleComponentManager` интерфейса.  
  
 Диспетчер компонентов пользовательского интерфейса  
 Службы `SOleComponentUIManager`, обеспечивающий координацию службы пользовательского интерфейса.`SOleComponentUIManager` Службы реализует `IOleComponentUIManager` и `IOleInPlaceComponentUIManager` интерфейсов.  
  
 контекст  
 `IVsUserContext` Объекта \(COM\), присоединенные к компоненту среды. Этот объект содержит ключевые слова поиска, ключевые слова справки F1 и атрибуты, относящиеся к компоненту. Кроме того контекста сумок пункты все сумок подконтекст, связанных с ними.  
  
 контекст поставщика  
 Компонент в IDE, который содержит контейнер контекст, связанный с ним. Такие компоненты включают окно инструмента, редактор или иерархии проекта.  
  
 конструктор  
 Программный интерфейс, который позволяет пользователям управлять элементами пользовательского интерфейса \(формы, кнопки и другие элементы управления\).  
  
 DocData  
 COM\-объект инкапсуляции базовых данных документа в мире, где разделение документов и представлений \(например, в случае редактор текста, это будет текстовый буфер базового все представления текстового редактора\). Если EditorFactory этот объект не указан, IDE изготовление на его имени. Ответственность за этот объект является управление сохранение данных и представления семантики общего доступа для нескольких за этот же `DocData`. Если `DocData` поддерживает `IOleCommandTarget` интерфейс, он будет включен в маршрутизации команд UIShell.  
  
 DocObject  
 Технология, используемая для размещения пользовательского интерфейса в рамке, предоставленный средой размещения. В частности, этот термин относится внедрения, поддерживающий `IOleDocument` и соответствующие интерфейсы. Эта технология имеет много потенциальных приложений, например реализации COM документов, окон инструментов в Visual Basic 5.0, конструкторы ActiveX в Visual Basic 6.0 и т. д.  
  
 документ  
 Используется для обозначения ко всему документу, как `DocData` и `DocView`. Например, содержит DocumentFrame `DocView`, но он также сохраняет ссылку на `DocData` для обработки сохраняемости.  
  
 DocView  
 DocObject и внедрение или области окна с которым взаимодействует пользователь для просмотра и управления базовый `DocData`. Обратите внимание, что пользователи не могут использовать разделения документов и представлений, который является частью `DocObject` интерфейса конструктора. Пользователям всего DocObject использоваться в качестве представления вместо более абстрактными \(и менее Формализованное\) представление о базовых данных, известный как `DocData`.`DocView` объекты всегда внедряются с объектами фрейма документа \(дочерних MDI\-окон\) интегрированной среды разработки.  
  
 DTE  
 `DTE` \(Расширения средств разработки\) объекта является точкой доступа верхнего уровня в модели автоматизации Visual Studio, которое позволяет программным путем автоматизации и расширения интегрированной среды разработки.  
  
 Окно динамической справки  
 Окно инструментов, которое реализуется в интегрированной среде разработки и отображает список ключевых слов для поиска или разделы справки F1.  
  
 редактор  
 Код \(код CLSID класса\), который реализует `DocView`. Он также реализует `DocData` Если поддерживается разделение представления и данных.  
  
 расширение  
 Функция, которая изменяет, настраивает или добавляет интегрированной среды разработки. Создание расширений с помощью модели автоматизации или пакеты VSPackage.  
  
 внешний редактор  
 Редактор, не относящиеся к Интегрированной среде разработки, таких как Microsoft Word. Он был зарегистрирован через собственные механизмы и может быть использована вне интегрированной среды разработки. Если этот редактор можно внедрять, он отображается внутри окна в Интегрированной среде разработки. Если он не может быть внедрен, создается отдельное окно верхнего уровня.  
  
 иерархия  
 Дерево узлов, каждый узел, связанный с набором свойств.  
  
 независимый компонент верхнего уровня  
 Компонент, который использует немодальное окно верхнего уровня может эффективно работать как окно автономного приложения, но реализуется как объект в процессе. Таким образом это независимый компонент верхнего уровня необходимо координировать модальность и служб цикла сообщений с помощью интегрированной среды разработки. В процессе объекты не имеют свои собственные цикл обработки сообщений.  
  
 сведения поставщика  
 Сведения поставщика — это модуль, можно найти ключевые слова и возвращают список разделов, в виде `IVsUserContextItem` объектов. Чтобы обеспечить элементов ключевое слово F1 или поставщика данных, зарегистрируйте скомпилированного файла справки \(. HxS\) с системой. Разделы справки в эти файлы используются для предоставления списка разделов отображаются в окне «Динамическая справка» и отображаются ли пользователь нажимает клавишу F1.  
  
 компонент на месте  
 Объект VSPackage, реализующий интерфейс `IOleInPlaceComponent` интерфейс для управления окна, визуально содержащихся в окне документа в Интегрированной среде разработки. Компоненты на месте не участвуют в стандартных OLE\-слияние меню; Вместо этого они интегрируют свои элементы пользовательского интерфейса в Интегрированной среде разработки.  
  
 Существует два типа компонентов на месте: жесткую привязку с заменой компонентов и элементов управления компонента.  
  
 Компоненты на месте жесткую привязку имеют меню, панелей инструментов и команд, которые тесно интегрированы в пользовательский интерфейс IDE, отображаются в виде, если они были созданы непосредственно в Интегрированной среде разработки.  
  
 Элементы управления компонента не имеют свои собственные элементы пользовательского интерфейса, интегрированы в Интегрированной среде разработки; Вместо этого они используют меню, команд и панелей инструментов интегрированной среды разработки. Например полужирный команда может использоваться полужирным выделенного слова в элементе управления форматированного текста, встроенные в форме. Элементы управления компонента можно запросить отображение элементов пользовательского интерфейса динамически установленного компонента.  
  
 Служба языка  
 Набор объектов, которая позволяет разработчикам реализовать функции редакторов кода языка компьютера, такие как текст, отмечая и выделения цветом VSPackage.  
  
 Проект прочих файлов  
 Проект, используемый для дома открытых файлов, которые не находятся в любой проект. Список элементов в этом проекте не сохраняется.  
  
 проект  
 Проекты состоят из объектов иерархии или COM\-объекты, реализующие `IVsHierarchy` интерфейса.  
  
 конструктор, определенный для проекта или редактора  
 Конструктор, который не может использоваться независимо от типа проекта. Все конструкторы конкретного проекта должен ввести данные фабрики редактора реестра. IDE затем можно создать экземпляр конструктора при открытии файлов определенного типа в конкретном проекте.  
  
 окно проекта  
 Окно, которое постоянно отслеживает текущий активный проект иерархии и элемент из контекста глобального выделения. Используйте тип проекта windows `SVsTrackSelectionEx` предупреждения IDE изменения и отображения отзывы пользователю службы. Обозреватель решений — пример окна типа проекта.  
  
 Окно \\"Свойства\\"  
 Обозреватель свойств ранее.  
  
 ссылка\-проектов  
 Проект, не требуются файлы проекта должны находиться в том же каталоге. Вместо этого ссылки на файлы из других каталогов несвязанных хранятся и обслуживается самого проекта.  
  
 таблицы запущенных документа  
 Внутренняя структура, в котором IDE поддерживает список всех открытых документов. Этот список включает все открытые документы в памяти независимо от ли документы момент редактируется. Документ является любой элемент, который сохраняется, включая хранимые процедуры, открывается в редакторе, файлы в проекте или основной файл проекта \(например, файл \*.vcproj\).  
  
 выделенный фрагмент  
 Данные, является частью подробности каждого окна в Интегрированной среде разработки и используется для отслеживания active выбранных элементов. Выделенный фрагмент состоит из:  
  
-   Указатель на `IVsHierarchy` интерфейс иерархии проекта  
  
-   Идентификатор элемента элемента проекта.  
  
-   Указатель на `ISelectionContainer` интерфейс предоставляет доступ к свойствам для активных объектов.  
  
-   Массив значений элементов.  
  
 service  
 Контракт для набора COM\-интерфейсов, которые находятся в COM\-объект. При создании службы, который определяется идентификатором GUID, можно определить набор COM\-интерфейсов, который выполняет службы. COM\-объекты использовать службы для связи друг с другом.  
  
 решение  
 Группа связанных проектов, с которыми работает пользователь.  
  
 стандартный конструктор  
 Конструктор, который может использоваться независимо от типа проекта. Все стандартные конструкторы необходимо ввести данные фабрики редактора реестра. IDE затем можно создать экземпляр конструктора при открытии файла с определенным расширением. Данные необходимо сохранить в файле.  
  
 стандартный редактор  
 Редактор, который может использоваться независимо от любого конкретного типа проекта. Такие редакторы имеют EditorFactories зарегистрирован в реестре. Это позволяет интегрированной среды разработки найти и запустить редактор.  
  
 стандартный редактор ОС  
 Внедрение, не является Visual Studio конкретного. Он зарегистрирован с помощью хорошо известных ключей Win32 \(например, Win32 обозреватель знает, как для вызова\). Если можно внедрить такие редактор, редактор будет по\-прежнему отображаться в его место в Интегрированной среде разработки. В противном случае — для таких редакторы создается отдельное окно верхнего уровня.  
  
 Переход на подконтекст контейнера  
 `IVsUserContext` Контейнер контекст связан объект. Этот объект содержит ключевые слова поиска, ключевые слова справки F1 и атрибуты для выбора компонента интегрированной среды разработки. Переход на подконтекст примеры команды в окне инструментов или ключевое слово в редакторе.  
  
 Список задач  
 Окно инструментов, которое реализуется в интегрированной среде разработки и отображает список активных задач.  
  
 текстовый буфер  
 Общее имя для объекта `VSTextBuffer`.  
  
 Текстовое представление  
 Общее имя для объекта `VSTextView`.  
  
 Средство компонент верхнего уровня  
 Компонент, который работает в качестве немодального всплывающее окно, тесно согласование с помощью пользовательского интерфейса интегрированной среды разработки. Как независимые компоненты верхнего уровня средство верхнего уровня компоненты также должны координировать модальность и служб цикла сообщений с помощью интегрированной среды разработки.  
  
 компонент верхнего уровня  
 Объект VSPackage, который управляет немодальное окно верхнего уровня вместо клиентской области окна интегрированной среды разработки. Реализация верхнего уровня компонентов `IOleComponent` интерфейс, чтобы воспользоваться преимуществами служб цикл сообщений, таких как доступ к времени простоя.  
  
 Активные пользовательского интерфейса  
 Объект VSPackage, который доступен и в данный момент имеет фокус.  
  
 Иерархии пользовательского интерфейса  
 COM\-объект, реализующий `IVsUIHierarchy` интерфейс для отображения иерархии. Окно иерархии пользовательского интерфейса реализует `ISelectionContainer` интерфейс для обновления окна свойств; в других типов проектов windows можно использовать эту реализацию, при необходимости.  
  
 VSCT  
 Таблица команда Visual Studio. Файл .vsct содержит сведения о размещении и поведения меню, панелей инструментов и команд в формате XML.  
  
 VSPackage  
 Устанавливаемые часть программного обеспечения, расширяет Visual Studio IDE, определяющих один или несколько из следующих действий: пользовательский интерфейс, службы, типы проектов или редактора или конструктора. VSPackage состоит из COM\-объект, реализующий `IVsPackage` интерфейс и один или несколько других COM\-объектов, реализующих другие интерфейсы для поддержки выбора и других функций. Кроме того VSPackage имеет требования конкретной регистрации.
