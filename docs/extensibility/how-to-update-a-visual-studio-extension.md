# Практическое руководство: обновление расширения Visual Studio
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Расширение Visual Studio в системе можно обновить с помощью **расширения и обновления** чтобы установить обновленную версию. При создании обновленная версия расширения можно обозначить ее как обновленную, увеличив номер версии в манифесте VSIX.  
  
 Обновления устанавливаются при манифест VSIX поступившего расширения имеет те же `ID` установленной версией и выше `Version` номер. Если `Version` номер равна или ниже, пакет не может быть установлен. Если `ID` не совпадают, пакет, который еще не установлен, распознается как отдельное расширение.  
  
 Для предотвращения конфликтов во время разработки, рекомендуется вы удаления предыдущих версий расширений и также удалить или отключить потенциально конфликтующие расширения.  
  
### Обновление расширения в системе  
  
1.  В меню **Сервис** выберите пункт **Расширения и обновления**.  
  
2.  В левой области щелкните **обновления**.  
  
3.  В средней области выберите обновление, которое вы хотите установить.  
  
     Номер версии обновленного расширения отображается в правой области вместе с другими данными.  
  
4.  В нижней части правой панели, щелкните **обновление**.  
  
### Публикация обновления расширения  
  
1.  В Visual Studio откройте решение для расширения, которое требуется обновить. Внесите необходимые изменения.  
  
    > [!IMPORTANT]
    >  Значение без знака, все пользовательские расширения не обновляется автоматически. Необходимо всегда подписывать модули.  
  
2.  В **обозревателе решений**, откройте: source.extension.manifest.  
  
3.  В конструкторе манифеста увеличьте значение числа в **версии** поле.  
  
4.  Сохраните решение и постройте его.  
  
5.  Отправьте файл .vsix \(в папке \\bin\\Debug\\ проекта\) [коллекции Visual Studio](http://go.microsoft.com/fwlink/?LinkID=123847) веб\-сайта.  
  
     При открытии пользователем, имеется более ранняя версия расширения **расширения и обновления**, новая версия отображается в **обновления** список, условии, что средство автоматически искать обновления.  
  
     Можно включить или отключить автоматическую проверку наличия обновлений в нижней части **обновления** области \(**Включить или отключить автоматическое обнаружение обновлений, доступных**\), какие изменения **проверять наличие обновлений** в **Сервис \/ Параметры \/ среды и расширения и обновления**.  
  
    > [!NOTE]
    >  Начиная с Visual Studio 2015 г. обновление 2, можно указать \(в **Сервис \/ Параметры \/ среда или расширения и обновления**\) требуется ли автоматическое обновление для расширения на уровне пользователя, все пользовательские расширения или оба \(значение по умолчанию\).
