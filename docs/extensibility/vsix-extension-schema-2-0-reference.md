# <a name="vsix-extension-schema-20-reference"></a>Справочник по схеме 2.0 расширения VSIX
Файл манифеста развертывания VSIX описывает содержимое пакета VSIX. Формат файла регулируется схемы. Эта схема версии 2.0 поддерживает добавление пользовательских типов и атрибутов.  Схема манифеста является расширяемой. XML-элементы и атрибуты, которые он не понимает, игнорирует манифеста загрузчика.  
  
> [!IMPORTANT]
>  Visual Studio 2015 можно загрузить VSIX-файлы в форматах Visual Studio 2010, Visual Studio 2012 или Visual Studio 2013.  
  
## <a name="package-manifest-schema"></a>Схема манифеста пакета  
 Корневой элемент файла манифеста XML — `<PackageManifest>`, с одним атрибутом `Version`, т.е. версии формат манифеста. При внесении значительных изменений в формат, формат версии будет изменен. В этом разделе описываются формат манифеста версия 2.0, которая задается в манифесте `Version` атрибуту значение версии = «2.0».  
  
### <a name="packagemanifest-element"></a>Элемент PackageManifest  
 В пределах `<PackageManifest>` корневого элемента, можно использовать следующие элементы:  
  
-   `<Metadata>`-Метаданных и рекламных сведений о самом пакете. Только один `Metadata` элемент допускается в манифесте.  
  
-   `<Installation>`-В этом разделе определяется тем, что этот пакет расширений могут быть установлены, включая номера SKU приложения, его можно установить в. Только один `Installation` элемент допускается в манифесте. Манифест должен иметь `Installation` элемента или этот пакет невозможно установить в любой SKU.  
  
-   `<Dependencies>`-Здесь определяются необязательный список зависимостей для этого пакета.  
  
-   `<Assets>`-В этом разделе перечислены все средств, входящих в состав этого пакета. Без этого раздела этого пакета не выявляются все содержимое.  
  
-   `<AnyElement>*`-Схеме манифеста является достаточно гибким, чтобы разрешить любые другие элементы. Дочерние элементы не распознаются манифеста загрузчика предоставляются в API диспетчера расширений, как дополнительных объектов XmlElement. С помощью этих дочерних элементов, расширения VSIX можно определить дополнительные данные в файле манифеста, код, выполняемый в Visual Studio можно получить доступ во время выполнения. В разделе <xref:Microsoft.VisualStudio.ExtensionManager.IExtension.AdditionalElements%2A>и <xref:Microsoft.VisualStudio.ExtensionManager.IExtension.LocalizedAdditionalElements%2A>.</xref:Microsoft.VisualStudio.ExtensionManager.IExtension.LocalizedAdditionalElements%2A> </xref:Microsoft.VisualStudio.ExtensionManager.IExtension.AdditionalElements%2A>  
  
### <a name="metadata-element"></a>Элемент метаданных  
 Этот раздел содержит метаданные о пакете, его идентификатор и рекламу сведения. `<Metadata>`содержит следующие элементы:  
  
-   `<Identity>`-Это определяет идентификационные данные для этого пакета и включает следующие атрибуты:  
  
    -   `Id`— Этот атрибут должен иметь уникальный идентификатор для пакета, выбранного по ее автора. Так же, как типы CLR, пространства имен должны быть квалифицированы именем: Company.Product.Feature.Name. `Id` Атрибут ограничено до 100 символов.  
  
    -   `Version`— Это определяет версию этого пакета и его содержимым. Этот атрибут имеет формат, управление версиями сборок CLR: основная.дополнительная.сборка.редакция (1.2.40308.00). Пакет с более высоким номером версии считается обновления пакета и может устанавливаться поверх существующего установленной версии.  
  
    -   `Language`— Атрибут это язык по умолчанию для пакета и соответствует текстовые данные в этом манифесте. Этот атрибут соответствует соглашению код языкового стандарта CLR сборок ресурсов, например: en-us, en, fr-fr. Можно указать `neutral` для объявления модуль от языка, который будет выполняться в любой версии Visual Studio. Значение по умолчанию — `neutral`.  
  
    -   `Publisher`— Этот атрибут идентифицирует издатель компании или отдельное имя этого пакета. `Publisher` Атрибут ограничено до 100 символов.  
  
-   `<DisplayName>`-Этот элемент задает имя понятное пакета, который отображается в пользовательском Интерфейсе диспетчера расширений. `DisplayName` Содержимое ограничено до 100 символов.  
  
-   `<Description>`— Это необязательный элемент — это краткое описание пакета и его содержимое, отображаемое в пользовательский Интерфейс диспетчера расширений. `Description` Содержимое может содержать любой текст, который вы хотите, но имеет более 1000 символов.  
  
-   `<MoreInfo>`— Это необязательный элемент является URL-адрес страницы через Интернет, содержит полное описание данного пакета. Протокол должен быть указан как http.  
  
-   `<License>`— Это необязательный элемент является относительным для файла лицензии (.txt, .rtf) содержатся в пакете.  
  
-   `<ReleaseNotes>`— Это необязательный элемент является либо относительный путь файла заметки о выпуске, содержащегося в пакет (.txt, .rtf) либо URL-адрес веб-сайта, отображается в заметках о выпуске.  
  
-   `<Icon>`— Это необязательный элемент является относительным путем к файлу изображения (png, bmp, jpeg, ico) содержатся в пакете. Изображение значка должно быть 32 x 32 пиксела (или будет сжат до размера) и отображается в listview пользовательского интерфейса. Если не `Icon` указан элемент, пользовательский Интерфейс использует значение по умолчанию.  
  
-   `<PreviewImage>`— Это необязательный элемент является относительным путем к файлу изображения (png, bmp, jpeg) содержатся в пакете. Изображение предварительного просмотра должен быть 200 x 200 пикселей и отображаются в области сведений пользовательского интерфейса. Если не `PreviewImage` указан элемент, пользовательский Интерфейс использует значение по умолчанию.  
  
-   `<Tags>`— Это необязательный элемент список тегов дополнительный текст, разделенный точкой с запятой, которые используются для указания поиска. `Tags` Элемент ограничено до 100 символов.  
  
-   `<GettingStartedGuide>`— Это необязательный элемент является либо относительный путь к файлу HTML либо URL-адрес веб-сайта, содержащий сведения об использовании расширения или содержимое этого пакета. В этом руководстве будет запущен как часть установки.  
  
-   `<AnyElement>*`-Схеме манифеста является достаточно гибким, чтобы разрешить любые другие элементы. Любые дочерние элементы, которые не распознаются манифеста загрузчиком представляются как список объектов XmlElement. С помощью этих дочерних элементов, расширения VSIX можно определить дополнительные данные в файле манифеста и перечислить их во время выполнения.  
  
### <a name="installation-element"></a>Элемент установки  
 В этом разделе определяется тем, что этот пакет может быть установлен и номера SKU приложения, его можно установить в. Этот раздел содержит следующие атрибуты:  
  
-   `Experimental`— Установите значение true, если у вас есть расширение, установленные для всех пользователей, но разрабатывается обновленную версию на том же компьютере. Например если вы установили MyExtension 1.0 для всех пользователей, но требуется отладка MyExtension 2.0 на том же компьютере, задайте экспериментально = «true». Этот атрибут является доступны в Visual Studio 2015 с обновлением 1 и более поздней версии.  
  
-   `Scope`— Этот атрибут может принимать значение «Глобальное» или «ProductExtension»:  
  
    -   «Глобальные» указывает, что установка не ограничены определенной Конфигурацией. Например это значение используется, если установлен пакет SDK расширения.  
  
    -   «ProductExtension» указывает, что традиционные расширения VSIX (версия 1.0) в отдельные SKU Visual Studio установлена. Это значение по умолчанию.  
  
-   `AllUsers`— Это необязательный атрибут указывает, является ли этот пакет будет установлен для всех пользователей. По умолчанию этот атрибут имеет значение false, указывает, что пакет находится на пользователя. (Если это значение равно true, пользователя, выполняющего установку необходимо повысить уровень прав администратора для установки получившийся проект VSIX.  
  
-   `InstalledByMsi`— Это необязательный атрибут указывает, установлен ли этот пакет с помощью установщика Windows. Пакеты, установленные с помощью установщика Windows устанавливаются и управлять по MSI (программы и компоненты), а не, диспетчер расширений Visual Studio.  По умолчанию этот атрибут имеет значение false, указывает, что пакет не установлен с помощью установщика Windows.  
  
-   `SystemComponent`— Это необязательный атрибут указывает, следует ли учитывать этот пакет компонентов системы. Компоненты системы отсутствуют в пользовательском Интерфейсе диспетчера расширений, не могут быть обновлены. По умолчанию этот атрибут имеет значение false, указывает, что пакет не системного компонента.  
  
-   `AnyAttribute*`— `Installation` Элемент принимает открытый набор атрибутов, предоставляемых во время выполнения как словарь пары имя значение.  
  
-   `<InstallationTarget>`— Этот элемент определяет расположение, в котором установщик VSIX устанавливает пакет. Если значение `Scope` атрибут — «ProductExtension» пакет должен быть предназначен SKU, который установлен как часть его содержимое, необходимо объявить его доступность для расширения файла манифеста. `<InstallationTarget>` Элемент имеет следующие атрибуты при `Scope` имеет явный атрибут или значение по умолчанию «ProductExtension»:  
  
    -   `Id`— Этот атрибут идентифицирует пакет.  Атрибут соответствует соглашению пространство имен: Company.Product.Feature.Name. `Id` Атрибута может содержать только буквы, цифры и не может превышать 100 знаков. Ожидаемые значения:  
  
        -   Microsoft.VisualStudio.IntegratedShell  
  
        -   Microsoft.VisualStudio.Pro  
  
        -   Microsoft.VisualStudio.Premium  
  
        -   Microsoft.VisualStudio.Ultimate  
  
        -   Microsoft.VisualStudio.VWDExpress  
  
        -   Microsoft.VisualStudio.VPDExpress  
  
        -   Microsoft.VisualStudio.VSWinExpress  
  
        -   Microsoft.VisualStudio.VSLS  
  
        -   My.Shell.App  
  
    -   `Version`— Этот атрибут задает диапазон версий с минимальной и максимальной версии поддерживаемых этого SKU. Пакет можно подробные сведения о версиях номера SKU, которые она поддерживает. Нотация версии диапазона является [10.0, 11.0], где  
  
        -   [— Минимальная версия включительно.  
  
        -   ] — максимальный номер версии включительно.  
  
        -   (-монопольного минимальной версии.  
  
        -   ) — максимальный номер версии exclusive.  
  
        -   Отдельная версия # — только указанную версию.  
  
        > [!IMPORTANT]
        >  Схема VSIX версии 2.0 появился в Visual Studio 2012. Чтобы использовать эту схему требуется Visual Studio 2012 или более поздней версии установлен на компьютере; используйте VSIXInstaller.exe, входит в состав этого продукта. Можно более ранних версий Visual Studio с помощью Visual Studio 2012 или более поздней версии VSIXInstaller, но только с помощью более поздней версии программы установки.  
  
    -   `AnyAttribute*`— `<InstallationTarget>` Элемент позволяет открытый набор атрибутов, которые будут предоставляться во время выполнения как словарь пары имя значение.  
  
### <a name="dependencies-element"></a>Dependencies-элемент  
 Этот элемент содержит список зависимостей, которые объявляются с помощью этого пакета. Если указаны все зависимости, эти пакеты (идентифицируются их `Id`) должны быть установлены перед.  
  
-   `<Dependency>`элемент — это дочерний элемент имеет следующие атрибуты:  
  
    -   `Id`— Этот атрибут должен иметь уникальный идентификатор зависимого пакета. Это значение идентификатора должно соответствовать `<Metadata><Identity>Id` атрибут пакета, зависящей от данного пакета. `Id` Атрибут соответствует соглашению пространство имен: Company.Product.Feature.Name. Атрибут может содержать только буквы, цифры и не может превышать 100 знаков.  
  
    -   `Version`— Этот атрибут задает диапазон версий с минимальной и максимальной версии поддерживаемых этого SKU. Пакет можно подробные сведения о версиях номера SKU, которые она поддерживает. Нотация версии диапазона является [12.0, 13.0], где:  
  
        -   [— Минимальная версия включительно.  
  
        -   ] — максимальный номер версии включительно.  
  
        -   (-монопольного минимальной версии.  
  
        -   ) — максимальный номер версии exclusive.  
  
        -   Отдельная версия # — только указанную версию.  
  
    -   `DisplayName`-Этот атрибут является отображаемое имя зависимый пакет, который используется в элементы пользовательского интерфейса, таких как диалоговые окна и сообщения об ошибках. Атрибут является необязательным, если зависимый пакет устанавливается по MSI.  
  
    -   `Location`— Это необязательный атрибут указывает относительный путь внутри этой VSIX вложенный пакет VSIX или URL-адрес в расположение загрузки для зависимости. Этот атрибут позволяет пользователю найти готовности к установке пакета.  
  
    -   `AnyAttribute*`— `Dependency` Элемент принимает открытый набор атрибутов, предоставляемых во время выполнения как словарь пары имя значение.  
  
### <a name="assets-element"></a>Элемент активы  
 Этот элемент содержит список `<Asset>` доступных тегов для каждого расширения или содержимое элемента, этот пакет.  
  
-   `<Asset>`-Этот элемент содержит следующие атрибуты и элементы.  
  
    -   `Type`— Это тип расширения или содержимое, представленное данным элементом. Каждый `<Asset>` элемент должен иметь один `Type`, но несколько `<Asset>` элементов может иметь такую же `Type`. Этот атрибут должен быть представлен как полное доменное имя, согласно соглашениям пространства имен. Ниже приведены известные типы.  
  
        1.  Microsoft.VisualStudio.VsPackage  
  
        2.  Microsoft.VisualStudio.MefComponent  
  
        3.  Microsoft.VisualStudio.ToolboxControl  
  
        4.  Microsoft.VisualStudio.Samples  
  
        5.  Microsoft.VisualStudio.ProjectTemplate  
  
        6.  Microsoft.VisualStudio.ItemTemplate  
  
        7.  Microsoft.VisualStudio.Assembly  
  
         Можно создать собственные типы и присвойте им уникальные имена. Во время выполнения в Visual Studio код можно перечислить и использовать эти пользовательские типы через API диспетчера расширений.  
  
    -   Путь — относительный путь к файлу или папке внутри пакета, который содержит ресурс.  
  
    -   `AnyAttribute*`— Открытый набор атрибутов, которые будут предоставляться во время выполнения как словарь пары имя значение.  
  
         `<AnyElement>*`— Все структурированного содержимого должны разделяться `<Asset>` начальный и конечный тег. Все элементы передаются в виде списка объектов XmlElement. Расширения VSIX можно определить структурированных метаданных конкретного типа в файле манифеста и перечислить их во время выполнения.  
  
### <a name="sample-manifest"></a>Пример манифеста  
  
```  
<?xml version="1.0" encoding="utf-8"?>  
<PackageManifest Version="2.0.0" xmlns="http://schemas.microsoft.com/developer/vsx-schema/2011" xmlns:d="http://schemas.microsoft.com/developer/vsx-schema-design/2011">  
  <Metadata>  
    <Identity Id="0000000-0000-0000-0000-000000000000" Version="1.0" Language="en-US" Publisher="Company" />  
    <DisplayName>Test Package</DisplayName>  
    <Description>Information about my package</Description>  
    <MoreInfo>http://www.fabrikam.com/Extension1/</MoreInfo>  
    <License>eula.rtf</License>  
    <ReleaseNotes>notes.txt</ReleaseNotes>  
    <Icon>Images\icon.png</Icon>  
    <PreviewImage>Images\preview.png</PreviewImage>  
  </Metadata>  
  <Installation InstalledByMsi="false" AllUsers="false" SystemComponent="false" Scope="ProductExtension">  
    <InstallationTarget Id="Microsoft.VisualStudio.Pro" Version="[11.0, 12.0]" />  
  </Installation>  
  <Dependencies>  
    <Dependency Id="Microsoft.Framework.NDP" DisplayName="Microsoft .NET Framework" d:Source="Manual" Version="[4.5,)" />  
    <Dependency Id="Microsoft.VisualStudio.MPF.12.0" DisplayName="Visual Studio MPF 12.0" d:Source="Installed" Version="[12.0]" />  
  </Dependencies>  
  <Assets>  
    <Asset Type="Microsoft.VisualStudio.VsPackage" d:Source="Project" d:ProjectName="%CurrentProject%" Path="|%CurrentProject%;PkgdefProjectOutputGroup|" />  
  </Assets>  
</PackageManifest>  
```
