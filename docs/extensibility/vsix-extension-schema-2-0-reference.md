# <a name="vsix-extension-schema-20-reference"></a>Справочник по схеме 2.0 расширения VSIX
Файл манифеста развертывания VSIX описывает содержимое пакета VSIX. Формат файла регулируется схемы. Эта схема версии 2.0 поддерживает добавление пользовательских типов и атрибутов.  Схема манифеста является расширяемым. Загрузчик манифеста пропускает XML-элементы и атрибуты, которые он не распознает.  
  
> [!IMPORTANT]
>  Visual Studio 2015 можно загружать VSIX-файлы в форматах Visual Studio 2010, Visual Studio 2012 или Visual Studio 2013.  
  
## <a name="package-manifest-schema"></a>Схема манифеста пакета  
 Корневой элемент XML-файл манифеста — `<PackageManifest>`, с одним атрибутом `Version`, представляющую собой версию функции формат манифеста. Если значительных изменений в формат, формат версии будет изменен. В этом разделе описывается формат манифеста версии 2.0, указанной в манифесте, задав `Version` равным значению версии = «2.0».  
  
### <a name="packagemanifest-element"></a>Элемент PackageManifest  
 В пределах `<PackageManifest>` корневого элемента, можно использовать следующие элементы:  
  
-   `<Metadata>`-Метаданные и рекламы сведения о самом пакете. Только один `Metadata` элемент разрешен в манифесте.  
  
-   `<Installation>`-Это раздел определяет способ этот пакет расширений могут быть установлены, включая номера SKU приложения, его можно установить в. Только один `Installation` элемент разрешен в манифесте. Манифест должен иметь `Installation` элемент, или этот пакет не устанавливается в SKU.  
  
-   `<Dependencies>`-Здесь определяются необязательный список зависимостей для этого пакета.  
  
-   `<Assets>`-Этот раздел содержит все необходимые активы, содержащихся в этом пакете. Без этого раздела этот пакет не будет контактной какое-либо содержимое.  
  
-   `<AnyElement>*`-Схемы манифеста отличается достаточной гибкостью, чтобы разрешить любые другие элементы. Все дочерние элементы, не распознаются манифеста загрузчика представлены в диспетчер расширений API как дополнительных объектов XmlElement. Используя эти дочерние элементы, расширения VSIX можно определить дополнительные данные в файле манифеста, код, выполняемый в Visual Studio можно получить доступ во время выполнения. См. разделы <xref:Microsoft.VisualStudio.ExtensionManager.IExtension.AdditionalElements%2A> и <xref:Microsoft.VisualStudio.ExtensionManager.IExtension.LocalizedAdditionalElements%2A>.  
  
### <a name="metadata-element"></a>Элемент метаданных  
 Этот раздел содержит метаданные о пакете, его идентификатор и рекламные сведения. `<Metadata>`содержит следующие элементы:  
  
-   `<Identity>`-Это определяет идентификационные сведения для этого пакета и включает следующие атрибуты:  
  
    -   `Id`-Этот атрибут должен быть уникальный идентификатор для выбранного автором пакета. Имя должно уточняться так же, как типы CLR, с пространством имен: Company.Product.Feature.Name. `Id` Атрибут ограничено до 100 символов.  
  
    -   `Version`-Это определяет версию этого пакета и его содержимым. Этот атрибут имеет формат, управление версиями сборок CLR: основная.дополнительная.сборка.редакция (1.2.40308.00). Пакет с более высокий номер версии считается обновления пакета и может быть установлен существующий установленную версию.  
  
    -   `Language`-Этот атрибут является языком по умолчанию для пакета и соответствует текстовые данные в этот манифест. Этот атрибут соответствует соглашению код языкового стандарта среды CLR для сборки ресурсов, например: en-us, en, fr-fr. Можно указать `neutral` для объявления расширения не зависящий от языка, который будет выполняться в любой версии Visual Studio. Значение по умолчанию — `neutral`.  
  
    -   `Publisher`-Этот атрибут идентифицирует издателя компании или отдельное имя этого пакета. `Publisher` Атрибут ограничено до 100 символов.  
  
-   `<DisplayName>`-Этот элемент задает имя понятное пакета, который отображается в пользовательском Интерфейсе диспетчера расширений. `DisplayName` Содержимое более 50 символов.  
  
-   `<Description>`— Это необязательный элемент представляет краткое описание пакета и его содержимое, который отображается в пользовательском Интерфейсе диспетчера расширений. `Description` Содержимое может содержать любой текст, который требуется, но она содержит более 1000 символов.  
  
-   `<MoreInfo>`— Это необязательный элемент является URL-адрес страницы документации, содержащий полное описание этого пакета. Протокол должен быть указан как http.  
  
-   `<License>`— Это необязательный элемент является относительным лицензии файл (.txt, .rtf), содержащиеся в пакете.  
  
-   `<ReleaseNotes>`— Это необязательный элемент является либо относительный путь в файл заметки о выпуске содержатся в пакете (.txt, .rtf), либо URL-адрес веб-сайта, отображенные в заметках о выпуске.  
  
-   `<Icon>`— Это необязательный элемент является относительным для файла изображения (png, bmp, jpeg, ico) содержащиеся в пакете. Изображение значка 32 x 32 пикселя (или будет сжат до размера) и отображается в listview пользовательского интерфейса. Если не `Icon` указан элемент, пользовательский Интерфейс использует значение по умолчанию.  
  
-   `<PreviewImage>`— Это необязательный элемент является относительным для файла изображения (png, bmp, jpeg) содержащиеся в пакете. В изображении предварительного просмотра должен составлять 200 x 200 пикселей и отображаются в области сведений пользовательского интерфейса. Если не `PreviewImage` указан элемент, пользовательский Интерфейс использует значение по умолчанию.  
  
-   `<Tags>`— Это необязательный элемент перечислены теги дополнительный текст, разделенный точками с запятой, которые используются для указания поиска. `Tags` Элемент ограничено до 100 символов.  
  
-   `<GettingStartedGuide>`— Это необязательный элемент — либо относительный путь к файлу HTML или URL-адрес веб-сайта со сведениями об использовании расширения или содержимое этого пакета. В этом руководстве запускается как часть установки.  
  
-   `<AnyElement>*`-Схемы манифеста отличается достаточной гибкостью, чтобы разрешить любые другие элементы. Любые дочерние элементы, которые не распознаются загрузчиком манифеста предоставляется как список объектов XmlElement. С помощью этих дочерних элементов, расширения VSIX можно определить дополнительные данные в файле манифеста и перечислить их во время выполнения.  
  
### <a name="installation-element"></a>Элемент установки  
 В этом разделе определяет способ этот пакет можно установить и номера SKU приложения, его можно установить в. Этот раздел содержит следующие атрибуты:  
  
-   `Experimental`-Установите этому атрибуту значение true, если расширение, которое в настоящее время установлено для всех пользователей, но вы разрабатываете обновленную версию на том же компьютере. Например если вы установили MyExtension 1.0 для всех пользователей, но требуется отладка MyExtension 2.0 на том же компьютере, задайте экспериментальный = «true». Этот атрибут является доступны в Visual Studio 2015 с обновлением 1 и более поздней версии.  
  
-   `Scope`-Этот атрибут может принимать значение «Global» или «ProductExtension»:  
  
    -   «Глобальные» указывает, что установки не ограничена определенной Конфигурацией. Например это значение используется, если установлен пакет SDK расширения.  
  
    -   «ProductExtension» указывает на установку традиционных расширения VSIX (версия 1.0) ограничивается только отдельные SKU Visual Studio. Это значение по умолчанию.  
  
-   `AllUsers`— Это необязательный атрибут указывает, является ли этот пакет будет установлен для всех пользователей. По умолчанию этот атрибут имеет значение false, указывает, что пакет находится на пользователя. (Если это значение равно true, пользователя, выполняющего установку необходимо повысить уровень прав администратора выполнить установку VSIX результирующее.  
  
-   `InstalledByMsi`— Это необязательный атрибут указывает, является ли установить этот пакет MSI-ФАЙЛ. Пакеты, установленные с MSI устанавливаются и управляемые с MSI (программы и компоненты) и не по Диспетчер расширений Visual Studio.  По умолчанию этот атрибут имеет значение false, указывает, что пакет не устанавливается по MSI-ФАЙЛ.  
  
-   `SystemComponent`— Это необязательный атрибут указывает, следует ли учитывать системный компонент этого пакета. Компоненты системы больше не показывать в пользовательском Интерфейсе диспетчера расширений и не может быть обновлен. По умолчанию этот атрибут имеет значение false, указывает, что пакет не является компонентом системы.  
  
-   `AnyAttribute*`- `Installation` Элемент принимает открытый набор атрибутов, которые будут предоставлены во время выполнения как словарь пары "имя значение".  
  
-   `<InstallationTarget>`-Этот элемент определяет расположение, где установщик VSIX устанавливает пакет. Если значение `Scope` атрибут — «ProductExtension» пакета должны предназначаться для SKU, в которой установлен как часть его содержимое для объявления его доступности для расширения файла манифеста. `<InstallationTarget>` Элемент имеет следующие атрибуты при `Scope` атрибутов содержат свойство или значение по умолчанию «ProductExtension»:  
  
    -   `Id`-Этот атрибут идентифицирует пакет.  Атрибут соответствует соглашению, пространство имен: Company.Product.Feature.Name. `Id` Атрибута может содержать только буквы, цифры и не должна превышать 100 символов. Ожидаемые значения:  
  
        -   Microsoft.VisualStudio.IntegratedShell  
  
        -   Microsoft.VisualStudio.Pro  
  
        -   Microsoft.VisualStudio.Premium  
  
        -   Microsoft.VisualStudio.Ultimate  
  
        -   Microsoft.VisualStudio.VWDExpress  
  
        -   Microsoft.VisualStudio.VPDExpress  
  
        -   Microsoft.VisualStudio.VSWinExpress  
  
        -   Microsoft.VisualStudio.VSLS  
  
        -   My.Shell.App  
  
    -   `Version`-Этот атрибут задает диапазон версий с минимальной и максимальной версии поддерживаемых этого SKU. Пакет можно подробные сведения о версиях номера SKU, он поддерживает. Версия диапазона представляет собой [10.0 11.0], где  
  
        -   [-Минимальная версия включительно.  
  
        -   ]-максимальный номер версии включительно.  
  
        -   (-Минимальная версия монопольного.  
  
        -   ) — максимальный номер версии монопольная.  
  
        -   Одна версия # — только указанную версию.  
  
        > [!IMPORTANT]
        >  В Visual Studio 2012 появилась в версии 2.0 схемой VSIX. Чтобы использовать эту схему требуется приобрести Visual Studio 2012 или более поздней версии установлена на этом компьютере и использовать VSIXInstaller.exe, входит в состав этого продукта. Можно ориентироваться на более ранних версиях Visual Studio с Visual Studio 2012 или более поздней версии VSIXInstaller, но только с помощью более поздней версии программы установки.  
  
    -   `AnyAttribute*`- `<InstallationTarget>` Элемент позволяет открытый набор атрибутов, которые будут предоставляться во время выполнения как словарь пары "имя значение".  
  
### <a name="dependencies-element"></a>Dependencies-элемент  
 Этот элемент содержит список зависимостей, которые объявляются с помощью этого пакета. Если указаны все зависимости, эти пакеты (определяется их `Id`) должны быть установлены перед.  
  
-   `<Dependency>`элемент - это дочерний элемент имеет следующие атрибуты:  
  
    -   `Id`-Этот атрибут должен быть уникальный идентификатор для зависимого пакета. Это значение идентификатора должно соответствовать `<Metadata><Identity>Id` атрибут, этот пакет является зависимым от пакета. `Id` Атрибут соответствует соглашению, пространство имен: Company.Product.Feature.Name. Атрибут может содержать только буквы, цифры и не должна превышать 100 символов.  
  
    -   `Version`-Этот атрибут задает диапазон версий с минимальной и максимальной версии поддерживаемых этого SKU. Пакет можно подробные сведения о версиях номера SKU, он поддерживает. Прототип диапазон версий имеет [12.0, 13.0], где:  
  
        -   [-Минимальная версия включительно.  
  
        -   ]-максимальный номер версии включительно.  
  
        -   (-Минимальная версия монопольного.  
  
        -   ) — максимальный номер версии монопольная.  
  
        -   Одна версия # — только указанную версию.  
  
    -   `DisplayName`-Этот атрибут является отображаемое имя зависимый пакет, который используется в элементы пользовательского интерфейса, таких как диалоговые окна и сообщения об ошибках. Атрибут является необязательным, если зависимый пакет не установлен, MSI-ФАЙЛ.  
  
    -   `Location`— Это необязательный атрибут указывает относительный путь внутри вложенной пакет VSIX этот VSIX-ФАЙЛ или URL-адрес в расположение загрузки для зависимости. Этот атрибут позволяет пользователю найти готовности к установке пакета.  
  
    -   `AnyAttribute*`- `Dependency` Элемент принимает открытый набор атрибутов, которые будут предоставлены во время выполнения как словарь пары "имя значение".  
  
### <a name="assets-element"></a>Элемент активы  
 Этот элемент содержит список `<Asset>` отображаются теги для каждого элемента расширения или содержимое этого пакета.  
  
-   `<Asset>`-Этот элемент содержит следующие атрибуты и элементы.  
  
    -   `Type`-Это тип расширения или содержимого, представленного данным элементом. Каждый `<Asset>` элемент должен иметь один `Type`, но несколько `<Asset>` элементов может иметь такую же `Type`. Этот атрибут должны представляться как полное доменное имя в соответствии с соглашениями пространства имен. Ниже приведены известные типы.  
  
        1.  Microsoft.VisualStudio.VsPackage  
  
        2.  Microsoft.VisualStudio.MefComponent  
  
        3.  Microsoft.VisualStudio.ToolboxControl  
  
        4.  Microsoft.VisualStudio.Samples  
  
        5.  Microsoft.VisualStudio.ProjectTemplate  
  
        6.  Microsoft.VisualStudio.ItemTemplate  
  
        7.  Microsoft.VisualStudio.Assembly  
  
         Можно создать собственные типы и дать им уникальные имена. Во время выполнения в среде Visual Studio код можно перечислить и использовать эти пользовательские типы через API диспетчера расширений.  
  
    -   Путь — относительный путь к файлу или папке в пределах пакета, который содержит средства.  
  
    -   `AnyAttribute*`— Открытый набор атрибутов, которые будут предоставляться во время выполнения как словарь пары имя значение.  
  
         `<AnyElement>*`— Любая Структурированное содержимое, разрешенные между `<Asset>` начинаются и заканчиваются тег. Все элементы передаются в виде списка объектов XmlElement. Расширения VSIX можно определить структурированных метаданных определенного типа в файле манифеста и перечислить их во время выполнения.  
  
### <a name="sample-manifest"></a>Пример манифеста  
  
```  
<?xml version="1.0" encoding="utf-8"?>  
<PackageManifest Version="2.0.0" xmlns="http://schemas.microsoft.com/developer/vsx-schema/2011" xmlns:d="http://schemas.microsoft.com/developer/vsx-schema-design/2011">  
  <Metadata>  
    <Identity Id="0000000-0000-0000-0000-000000000000" Version="1.0" Language="en-US" Publisher="Company" />  
    <DisplayName>Test Package</DisplayName>  
    <Description>Information about my package</Description>  
    <MoreInfo>http://www.fabrikam.com/Extension1/</MoreInfo>  
    <License>eula.rtf</License>  
    <ReleaseNotes>notes.txt</ReleaseNotes>  
    <Icon>Images\icon.png</Icon>  
    <PreviewImage>Images\preview.png</PreviewImage>  
  </Metadata>  
  <Installation InstalledByMsi="false" AllUsers="false" SystemComponent="false" Scope="ProductExtension">  
    <InstallationTarget Id="Microsoft.VisualStudio.Pro" Version="[11.0, 12.0]" />  
  </Installation>  
  <Dependencies>  
    <Dependency Id="Microsoft.Framework.NDP" DisplayName="Microsoft .NET Framework" d:Source="Manual" Version="[4.5,)" />  
    <Dependency Id="Microsoft.VisualStudio.MPF.12.0" DisplayName="Visual Studio MPF 12.0" d:Source="Installed" Version="[12.0]" />  
  </Dependencies>  
  <Assets>  
    <Asset Type="Microsoft.VisualStudio.VsPackage" d:Source="Project" d:ProjectName="%CurrentProject%" Path="|%CurrentProject%;PkgdefProjectOutputGroup|" />  
  </Assets>  
</PackageManifest>  
```
