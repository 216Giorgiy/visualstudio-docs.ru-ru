# <a name="test-area-3-check-outundo-checkout"></a>Область проверки 3: Извлечение / Отмена извлечения
Эта область тестирования подключаемый модуль системы управления версиями охватывает редактирования и отката элементов из хранилища версий через **извлечь** и **отменить извлечение** команд.  
  
 **Извлечение**: метки элемента в хранилище версий как извлеченный, изменяет локальную копию для чтения и записи.  
  
 **Отмена извлечения**: отмечает элемент в хранилище версий, при возврате, возвращается локальную копию для состояния перед извлечением (в зависимости от параметров).  
  
## <a name="command-menu-access"></a>Доступ к меню команд  
 Следующие [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] путей меню среды разработки, которые используются в тестовых случаях.  
  
##### <a name="check-out"></a>Извлечь:  
  
-   **Файл**, **система управления версиями**, **извлечь**.  
  
-   **Файл**, **извлечь**.  
  
-   Контекстное меню **извлечь**.  
  
-   Отмена извлечения: **файл**, **система управления версиями**, **отмены извлечения**.  
  
## <a name="common-expected-behavior"></a>Общее ожидаемое поведение.  
  
-   После извлечения операции удаляемые файлы или папки, помечаются как извлеченные в хранилище версий.  
  
-   Хранилище версий атрибутов извлечение соответствующему пользователю.  
  
-   Время и Дата извлечения являются правильными (согласно параметрам пользователя).

### <a name="case-3b-disconnected-check-out"></a>Регистр 3b: отключен извлечение  
 В отключенном режиме пользователи определенный уровень поддержки системы управления версиями постоянной, если не присоединен непосредственно в хранилище версий. Это делается путем кэширования вся информация о прикрепленной решение и проекты локально.  
  
 Монопольное извлечение операции возможно только при наличии подключения к системе управления версиями. Проверка общего операций происходит в любой момент ли подключением и отключением. Таким образом при отключении от хранилища версий, только **проверки для совместного использования** (COS) команда включена. При отсутствии подключения, **отменить извлечение** отключено, поскольку невозможно получить старую версию, чтобы заменить изменения, внесенные пользователем.  
  
 При повторном подключении пользователя к версии хранения состояния извлечения всех прикрепленная решений и проектов синхронизируются. При этом произойдет необходимые обновления в хранилище для извлечения, выполненные пользователем. После синхронизации был выполнен, пользователь может продолжать работу в обычном режиме (подключения).  
  
#### <a name="expected-behavior"></a>Ожидаемое поведение  
  
-   Нельзя использовать **проверьте Out исключительно** команды при отсутствии подключения из хранилища версий.  
  
-   Нельзя использовать **отменить извлечение** команды при отсутствии подключения из хранилища версий.  
  
-   **Общие извлечь** работу команды.

### <a name="case-3c-query-editquery-save-qeqs"></a>Вариант 3c: изменение запроса сохранить (QEQS)

При редактировании управляемой элемент, который «установлен в» QEQS перехватывает попытки изменить и запрашивает у пользователя, ему нужно извлечь файл, чтобы изменить его. В зависимости от **средства**, **параметры** параметры пользователя — либо принудительно проверьте выходной файл для редактирования или разрешено изменение копии в памяти, ознакомьтесь с более поздней версии. ### <a name="case-3e-undo-check-out"></a>Регистр 3e: Отмена извлечения  
 **Отменить извлечение** используется для отмены файл извлечен состояние и избежать проверки изменений, внесенных в файл.  
  
#### <a name="expected-behavior"></a>Ожидаемое поведение  
  
-   Значение по умолчанию создается на основе пользователя **извлечение локальной версии** параметр. Если пользователь выбрал извлечение локальной версии, а затем по умолчанию операция отмены извлечения, всегда вернуться к версии, извлечен.  
  
-   После принятия процесса отката, значки в **обозревателе решений** обновлены для затронутых файлов и элемент удаляется из **Отложенные возвраты** окна.
