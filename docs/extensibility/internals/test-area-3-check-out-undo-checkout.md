# Область тестирования 3: Извлечение и отменить извлечение
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

Эта область тестирования подключаемого модуля системы управления версиями охватывает редактирования и возврат элементов из хранилища версий через **извлечь** и **Отменить извлечение** команды.  
  
 **Извлечь**: метки элемента в хранилище версий извлечен, изменяет локальную копию для чтения и записи.  
  
 **Отменить извлечение**: помечает элемент как отмеченный хранилище версий, отменяет локальную копию состояния перед извлечением \(в зависимости от параметров\).  
  
## Доступ к меню команд  
 Следующие [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] в тестовых случаях используются пути меню среды разработки.  
  
##### Ознакомьтесь с:  
  
-   **Файл**, **систему управления версиями**, **извлечь**.  
  
-   **Файл**, **извлечь**.  
  
-   Контекстное меню **извлечь**.  
  
-   Отмена извлечения: **файл**, **систему управления версиями**, **Отменить извлечение**.  
  
## Общее ожидаемое поведение  
  
-   После операции извлечения удаляемые файлы и папки будут помечены как извлечен в хранилище версий.  
  
-   Хранилище версий атрибуты извлечение правильные имена.  
  
-   Время и Дата извлечения верны \(согласно параметрам пользователя\).  
  
## Тестовые случаи  
 Ниже приведены конкретных тестовых случаев для области проверки извлечения или Отмена извлечения.

### Case 3b: извлечение отключен  
 В отключенном режиме пользователи определенный уровень поддержки системы управления версиями постоянно, если не присоединен непосредственно в хранилище версий. Это делается путем локального кэширования вся информация о прикрепленной решения и проектов.  
  
 Монопольное извлечение операций может произойти только при подключении к системе управления версиями. Общие извлечения операций может произойти в любое время ли подключения. Таким образом при отключении от хранилища версий, только **проверки для совместного использования** \(COS\) включена команда. Во время отключения, **Отменить извлечение** отключено, поскольку невозможно получить старую версию, чтобы заменить изменения, внесенные пользователем.  
  
 При повторном подключении к версии хранения и извлечения состояния всех прикрепленных решений и проектов синхронизируются. Это выполняет необходимые обновления в хранилище для извлечения, выполненных пользователем. После синхронизации произошло, пользователь может продолжать работу в обычном режиме \(подключения\).  
  
#### Ожидаемое поведение  
  
-   Нельзя использовать **Проверьте Out исключительно** команды при отсутствии подключения хранилища версий.  
  
-   Нельзя использовать **Отменить извлечение** команды при отсутствии подключения хранилища версий.  
  
-   **Общего извлечь** работу команды.  
  
|Действие|Шаги теста|Ожидаемые результаты для проверки|  
|--------------|----------------|---------------------------------------|  
|Во время отключения, извлечь файл, а затем подключить для синхронизации|1.  Отключить управляемой проект с помощью диалогового окна Изменение системы управления версиями \(**файл**, **системы управления версиями**, **Система управления версиями изменений**l\).<br />2.  Извлечь файл.<br />3.  Нажмите кнопку извлечь \(нет подключения\) в диалоговом окне предупреждения.<br />4.  Измените файл.<br />5.  Подключение с помощью диалогового окна Изменение системы управления версиями.<br />6.  Получите последнюю версию файла.|Общее ожидаемое поведение|  
  
### Case 3c: сохранение запросов редактирования\/запроса \(QEQS\)  
 Элементы в системе управления версиями, отслеживаются для редактирования, изменения, и сохраняет, чтобы помочь пользователям легко управлять своих файлов. При редактировании элемент управляемой «установлен в» QEQS перехватывает попытки изменить и запрашивает у пользователя, ему нужно извлечь файл для редактирования. В зависимости от **средства**, **Параметры** параметры пользователя — либо принудительно проверьте выходной файл для изменения или разрешено изменение копии в памяти, ознакомьтесь с более поздней версии. Если пользователь **средства**, **Параметры** задан параметр для отображения диалоговое окно извлечения и просто извлечь его, а затем пользователь делает его изменить, файл автоматически проверяет, по возможности.  
  
#### Ожидаемое поведение  
  
-   После операции извлечения удаляемые файлы и папки будут помечены как извлечен в хранилище версий.  
  
-   Хранилище версий атрибуты извлечения пользователя.  
  
-   Время и Дата извлечения верны \(согласно параметрам пользователя\).  
  
-   Локальную копию целевой файл или папка доступен для чтения.

Откройте файл проекта в редакторе, попробуйте ввести новый текст в файле. Если этот шаг завершается успешно, перейдите к шагу Далее.<br />10. Щелкните **Изменить** в **Извлечение для изменения** диалоговое окно. Если этот шаг завершается успешно, перейдите к шагу Далее.<br />11. ### Case 3d: Автоматического извлечения  
 Вложенная область охватывает извлеченную сценарии где **извлечь** диалоговое окно не отображается на пользователя **средства**, **Параметры**, **Параметры системы управления версиями**.  
  
#### Ожидаемое поведение  
  
-   После операции извлечения удаляемые файлы и папки будут помечены как извлечен в хранилище версий.  
  
-   Хранилище версий атрибуты извлечения пользователя.  
  
-   Время и Дата извлечения правильный \(согласно параметрам пользователя\).  
  
-   Локальную копию целевой файл или папка доступен для чтения.

### Case 3Д: отменить извлечение  
 **Отменить извлечение** используется для отмены файл проверяется состояние извлечения и избежать возврата изменений, внесенных в файл.  
  
#### Ожидаемое поведение  
  
-   По умолчанию создается на основе пользователя **извлечение локальной версии** параметр. Если пользователь выбрал извлечение локальной версии, значение по умолчанию для отмены извлечения — всегда вернуться к версии, извлечен.  
  
-   Принять отмены значки в **обозревателе решений** обновлены для затронутых файлов и элемент удаляется из **Отложенные возвраты** окна.

|--------------|----------------|---------------------------------------|

Изменение файла.<br />5.  Отмена извлечения \(**файл**, **систему управления версиями**, **Отменить извлечение**\).|Общее ожидаемое поведение.|

Изменение файла.<br />5.  Отмена извлечения \(**файл**, **систему управления версиями**, **Отменить извлечение**\).|Общее ожидаемое поведение.|
