# <a name="providing-automation-for-vspackages"></a>Предоставляет сведения об автоматизации VSPackages
Существует два основных способа для автоматизации для вашего VSPackages: путем реализации VSPackage объекты и применяя автоматизации стандартных объектов. Как правило они используются совместно для расширения модели автоматизации среды.  
  
## <a name="vspackage-specific-objects"></a>Объекты, зависящие от VSPackage  
 Определенных местах внутри модели автоматизации требуется ввести объекты автоматизации, которые являются уникальными для VSPackage. Например новые проекты требуют различных объектов, которые предоставляет только VSPackage. Имена этих объектов вводятся в реестре и получить через вызовы к среде `DTE` объекта.  
  
 Также можно получить объекты VSPackage при автоматизации Потребитель использует объект, предоставляемый через свойство объекта стандартный объект. Например, стандартная `Window` объект имеет `Object` свойство, обычно называемый `Windows.Object` свойство. При вызове потребители `Window.Object` в окне, реализованные в VSPackage, передается обратно определенные автоматизации объект собственный.  
  
#### <a name="projects"></a>Проекты  
 Пакеты VSPackage можно расширить модель автоматизации для новых типов проектов через свои собственные объекты VSPackage. Основная цель предоставления новых объектов автоматизации для для различения уникальный проект VSPackage объектов из <xref:Microsoft.VisualStudio.VCProjectEngine.VCProject>или <xref:VSLangProj80.VSProject2>объект.</xref:VSLangProj80.VSProject2> </xref:Microsoft.VisualStudio.VCProjectEngine.VCProject> Это отличие он удобен, если требуется предоставить возможность выделить или итерации типа проекта отдельно от других типов проектов, они отображаются рядом друг с другом в решении. Дополнительные сведения см. в разделе [предоставление объектов проекта](../../extensibility/internals/exposing-project-objects.md).  
  
#### <a name="events"></a>События  
 Архитектура событий среды предлагает другой место для добавления собственных VSPackage конкретных объектов. Например путем создания собственных объектов событий с уникальным, можно расширить модель событий среды для проектов. Может потребоваться предоставить свои собственные события, при добавлении нового элемента типа проекта. Дополнительные сведения см. в разделе [предоставление событий](../../extensibility/internals/exposing-events-in-the-visual-studio-sdk.md).  
  
#### <a name="window-objects"></a>Объекты окон  
 Windows можно вернуть объект автоматизации конкретных VSPackage обратно в среду, при вызове. Реализация объекта, который является производным от <xref:Microsoft.VisualStudio.Shell.Interop.IVsExtensibleObject>, <xref:EnvDTE.IExtensibleObject>или `IDispatch` , передает обратно свойства, расширяя объект окна, в котором он размещен.</xref:EnvDTE.IExtensibleObject> </xref:Microsoft.VisualStudio.Shell.Interop.IVsExtensibleObject> Например этот подход можно использовать для автоматизации для элемента управления, находящегося в рамке окна. Этот объект и другие объекты, которые она может расширить семантика вашим для разработки. Дополнительные сведения см. в разделе [как: укажите автоматизации для Windows](../../extensibility/internals/how-to-provide-automation-for-windows.md).  
  
#### <a name="options-pages-on-the-tools-menu"></a>В меню Сервис Параметры-страницы  
 Можно создавать страницы для расширения средств модели автоматизации параметры посредством реализации страницы и добавления информации в реестр, чтобы создать свои собственные параметры. На страницах вызываются через объектную модель среды, как и другие параметры страницы. Если для разработки функции, добавляемый к среде через VSPackages требуются параметры страницы, следует добавить функции автоматизации. Дополнительные сведения см. в разделе [поддержки автоматизации для страницы параметров](../../extensibility/internals/automation-support-for-options-pages.md).  
  
## <a name="standard-automation-objects"></a>Объекты автоматизации стандартных  
 Расширения автоматизации для проектов, также реализовать автоматизации стандартных объектов (производный от `IDispatch`), выделить рядом других объектов проекта и реализовать стандартные методы и свойства. Стандартные объекты примеры объектов проекта, которые вставляются в иерархии решения, такие как `Projects`, `Project`, `ProjectItem`, и `ProjectItems`. Каждый новый тип проекта должен реализовывать эти объекты (и, возможно, другие строки в зависимости от семантики проекта).  
  
 В некотором смысле эти объекты предоставляют противоположной преимуществами объекты проекта VSPackage. Объекты автоматизации стандартных позволяют проекта, чтобы использовать обобщенный так, как любой другой проект, поддерживающий те же объекты. Таким образом, надстройки, написанные для Общие `Project` и `ProjectItem` объектов может работать с проектами любого типа. Дополнительные сведения см. в разделе [проект моделирования](../../extensibility/internals/project-modeling.md).
