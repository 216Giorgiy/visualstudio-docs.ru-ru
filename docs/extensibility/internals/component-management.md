# <a name="component-management"></a>Компонент управления
Единицы задач в установщик Windows, называются компоненты установщика Windows (также называемые WICs или просто компонентов). Идентификатор GUID идентифицирует каждый WIC, который представляет собой базовую единицу установки и подсчетом ссылок для настройки, которые используют установщик Windows.  
  
 Несмотря на то, что можно использовать несколько продуктов для создания установщика VSPackage, здесь предполагается использование файлов установщика Windows (MSI). При создании установщика, необходимо правильно управлять развертывание файлов, чтобы правильный подсчет ссылок на все время. Таким образом разные версии продукта будет не помешать или приостановка друг с другом в сочетание установки и удаления сценариев.  
  
 В установщике Windows подсчет ссылок происходит на уровне компонента. Необходимо тщательно организации ресурсов — файлы, параметры реестра и так далее — на компоненты. Существуют другие уровни организации, например модули, функции и продуктов —, может помочь в различных сценариях. Дополнительные сведения см. в разделе [основы установщика Windows](../../extensibility/internals/windows-installer-basics.md).  
  
## <a name="guidelines-of-authoring-setup-for-side-by-side-installation"></a>Правила разработки Установка Side-by-side  
  
-   Автор файлов и разделов реестра, которые являются общими для версии в собственных компонентах.  
  
     Это позволяет легко использовать их в следующей версии. Например библиотеки типов, зарегистрированных глобально, файла расширения, другие элементы, зарегистрированные в раздел HKEY_CLASSES_ROOT и т. д.  
  
-   Модули слияния отдельных сгруппировать общих компонентов.  
  
     Это позволяет автору правильно для side-by-side продвижение вперед.  
  
-   Установите общие файлы и разделы реестра с помощью разных версий одной компоненты установщика Windows.  
  
     Если вы используете другой компонент, файлы и параметры реестра удаляются при удаляется один с версиями пакета VSPackage, но по-прежнему установлен другой пакет VSPackage.  
  
-   Не следует смешивать с версиями и общие элементы в одном компоненте.  
  
     Это делает невозможным установки для общих элементов глобальном местоположении и элементов с версиями с изолированной расположения.  
  
-   Нет общего реестра, которые указывают на файлы с версиями.  
  
     В противном случае общие ключи будут перезаписаны при установке другого с версиями пакета VSPackage. После удаления вторая версия файла, на который указывает ключ удаляется.
