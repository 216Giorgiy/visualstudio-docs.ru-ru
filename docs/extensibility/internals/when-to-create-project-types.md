# <a name="when-to-create-project-types"></a>Когда следует создавать типы проектов
Создание нового проекта типа основу для настройки [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] для пользователей. Однако создание новый тип проекта не является обязательным для всех [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] настроек. Следующие рекомендации помогут вам определить, требуется ли новый тип проекта для вашего сценария.  
  
## <a name="create-a-new-project-type"></a>Создать новый тип проекта  
 Если вы хотите настроить, необходимо создать тип проекта [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] действовать в одной или нескольких из следующих способов:  
  
-   Участвовать в сборки, развертывания, управления версиями и конфигураций.  
  
-   Обеспечивают поддержку отладки.  
  
-   Отображать элементы проекта в **обозревателе решений**.  
  
-   Используйте **Открытие проекта** или **новый проект** диалоговое окно.  
  
-   Поддерживает вложение проекта.  
  
## <a name="extend-an-existing-project-type"></a>Для расширения существующего типа проекта  
 Может потребоваться создать новый тип проекта, можно использовать [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] одним из следующих способов изменить или расширить поведение существующего типа проекта, например изменение процесс построения [!INCLUDE[vcprvc](../../code-quality/includes/vcprvc_md.md)] проектов:  
  
-   Работать с несколькими файлами как единое целое.  
  
-   Отображение одного файла как иерархию вложенных элементов.  
  
-   Отображение контекста команды вокруг редакторов.  
  
-   Отображение контекст службы для редакторов.  
  
## <a name="use-an-existing-project-type"></a>Использовать существующий тип проекта  
 Создание нового проекта иногда не является обязательным. В следующей таблице показаны задачи, которые необходимо создать тип проекта для.  
  
|Задача|Описание|  
|----------|-----------------|  
|Обработка команд|Любой пакет VSPackage может обрабатывать команды.|  
|Сборка редактора|Пользовательские редакторы могут быть зарегистрированы. Дополнительные сведения см. в разделе [окон документов и редакторы](http://msdn.microsoft.com/en-us/603625e1-62b6-413a-bc44-089346e166bc).|  
|Windows-владелец|Окна инструментов и документов можно создать без добавления новый тип проекта.|  
|Предоставление доступа к свойствам в окне «Свойства»|Все объекты можно предоставлять свойства.|  
  
## <a name="create-a-project-subtype"></a>Создание подтипом проекта  
 Подтипы проекта можно использовать для расширения типа управляемого проекта без необходимости создавать новый тип проекта. Подтипы проекта используется агрегатное выражение COM для расширения в проекты управляемого кода на языке Microsoft [!INCLUDE[vbprvb](../../code-quality/includes/vbprvb_md.md)] или [!INCLUDE[csprcs](../../data-tools/includes/csprcs_md.md)]. С помощью COM статистическая обработка можно повторно использовать большую часть реализации системы управляемого проекта и настраивать для того или иного сценария через агрегирование и использование поддержки интерфейсов. Дополнительные сведения о подтипы проекта см. в разделе [подтипы проекта](../../extensibility/internals/project-subtypes.md).
