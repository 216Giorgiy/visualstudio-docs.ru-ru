# Шаблоны веб-сайта поддержки
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

[!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] Проект и шаблоны элементов веб\-сайты с возможностью повторного использования и предоставляют настраиваемые заглушки проекта и элемента веб\-сайты, ускоряют ход процесса разработки, удалив нужно создать новые проекты и элементы веб\-сайта с нуля.  Дополнительные сведения [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] шаблоны см. в разделе  [Создание пользовательских шаблонов проектов и элементов](../../ide/creating-project-and-item-templates.md).  
  
## Папка шаблонов проектов  
 Шаблоны шаблона веб\-проекта \[обычно устанавливаются в значение on*путь установки Visual Studio*\] \\ Common7 \\ ide \\ Шаблоны проекта \\ Интернет \\, каждое в подпапке с именем после языка программирования Интернета.  
  
## Файл проекта  
 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] интегрированная среда разработки \(ide\) требует расширений файлов проекта как способ сопоставления шаблона к нужному типу проекта.  Поскольку веб\-проект не имеют файл проекта, .webproj фиктивное расширение файла проекта зарегистрировано для поддержки это.  
  
 При необходимости строка названия языка можно добавить к шаблону, чтобы система является веб\-проектом, чтобы задать значение по умолчанию языка **Добавление нового элемента** диалоговое окно для элементов на основе шаблона.  Строка должна быть первой линии и должна соответствовать имени файла и зарегистрированному в AddItemLanguageName в регистрации обработчика Intellisense и имени зарегистрированному в разделе подтипом проекта \(VsTemplate\).  Дополнительные сведения см. в разделе [Атрибуты веб\-сайта поддержки](../../extensibility/internals/web-site-support-attributes.md).  
  
 Если строка отсутствует, то система веб\-проекта пытается определить язык по умолчанию на основе атрибуте языка и расширений файлов страниц, добавленных в веб\-проекте шаблоном шаблона проекта.  
  
## Шаблоны проектов  
 Шаблоны проектов веб\-сайтов используются для построения новых веб\-сайтов в ответ на **Новый веб\-узел** команда на  **Файл** меню.  3 Типа проекта веб\-сайта в настоящее время поддерживаются:  
  
-   Пустые проекты веб\-сайтов  
  
-   Проекты веб\-сайтов  
  
-   Проекты веб\-службы  
  
### Пустые проекты веб\-сайтов  
 Эти файлы создаются новое пустое в ответ на веб\-сайте **Пустой веб\-сайт** команды доступны после которой он указывал на  **Новый веб\-узел** на  **Файл** меню:  
  
-   EmptyWeb.vstemplate  
  
     Файл шаблона, который направляет создание нового пустого веб\-сайта.  
  
-   EmptyWeb.webproj  
  
     Этот файл артефакт системы шаблона проекта.  Он удовлетворяет ссылка на файл проекта в файле EmptyWeb.vstemplate.  
  
### Проекты веб\-сайтов  
 Эти файлы создаются нового веб\-сайта в ответ на **Веб\-сайт ASP.NET** команды доступны после которой он указывал на  **Новый веб\-узел** на  **Файл** меню:  
  
-   Default.aspx  
  
     Начальная страница по умолчанию для нового веб\-сайта.  Атрибут языка определяет язык и атрибут codebehind CodeFile определяет зависимый файл, содержащий код codebehind, связанный с данной страницей.  
  
-   Default.aspx.*расширение*  
  
     Зависимый файл, содержащий код codebehind для начальной страницы по умолчанию.  Codebehind определяет язык *расширение* этого файла.  
  
-   web.config  
  
     Корневой файл конфигурации web.site.  
  
-   WebApplication.vstemplate  
  
     Файл шаблона, определяющий содержимое решения веб\-сайта и обеспечивает создание папки App\_Data.  
  
-   WebApplication.webproj  
  
     Этот файл артефакт системы шаблона проекта.  Он удовлетворяет ссылка на файл проекта в файле WebApplication.vstemplate.  
  
### Проекты веб\-службы  
 Эти файлы создаются нового веб\-сайта в ответ на **Веб\-служба ASP.NET** команда доступны после которой он указывал на  **Новый веб\-узел** на  **Файл** меню:  
  
-   Service.asmx  
  
     Страница HTML для новой веб\-службы.  Атрибут language язык codebehind, а атрибут CodeBehind определяет зависимый файл, содержащий код codebehind, связанный с этой службой.  
  
-   Служба. *расширение*  
  
     Зависимый файл, который реализует класс службы.  Codebehind определяет язык *расширение* этого файла.  
  
-   web.config  
  
-   Корневой файл конфигурации web.site.  
  
-   WebService.vstemplate  
  
     Файл шаблона, определяющий содержимое сайта принудительное создание решения и папки App\_Data и App\_Code.  Служба.*расширение* файл копируется в папку App\_Code.  
  
-   WebService.webproj  
  
     Этот файл артефакт системы шаблона проекта.  Он удовлетворяет ссылка на файл проекта в файле WebService.vstemplate.  
  
## Папка шаблонов элементов проекта  
 Шаблоны шаблона элемента проекта обычно устанавливаются в интернете \[on*путь установки Visual Studio*\] \\ Common7 \\ ide \\ ItemTemplates \\ Интернет \\, каждое в подпапке с именем после его языка программирования Интернета.  
  
## Шаблоны элементов проектов  
 Шаблоны элементов проектов веб\-сайтов используются для добавления новых страниц на веб\-сайт в ответ на **Добавление существующего элемента** команды.  Эти типы страниц в настоящее время поддерживаются:  
  
-   Новый класс  
  
-   Новая страница HTML  
  
-   Новая форма Интернета  
  
-   Новая главная страница  
  
### Новый класс  
 Этот шаблон создает новый исходный файл, определяющий пустой класс в ответ на **добавьте новый класс** команды.  
  
-   Класс. *расширение*  
  
     Исходный файл, который реализует пустой класс.  Codebehind определяет язык *расширение* этого файла.  
  
-   Class.vstemplate  
  
     Файл шаблона, который создает исходный файл и указывает его содержимое.  
  
### Новая страница HTML  
 Этот шаблон создает новую страницу в ответ на **Добавьте новую html\-страницу** команды.  
  
-   HTMLPage.htm  
  
     Начальное содержимое страницы.  Эта страница обычно не имеет связанный файл зависимых codebehind.  Для создания страница интеллектуального со связанным файлом codebehind, используйте шаблон веб\-формы.  
  
-   HTMLPage.vstemplate  
  
     Файл шаблона, который создает страницы и указывает его содержимое.  
  
### Новое WebForm  
 Этот шаблон создает новую страница интеллектуального в ответ на **Добавьте новую форму Интернета** команды.  
  
 Создание зависимый исходный файл выберите codebehind **Поместите код в отдельном файле**.  В противном случае одна страница создана, которая не имеет пустой блок скриптов и директивы страниц %\> \<% подключить зависимый файл.  
  
 Создание страницы содержимого для выбранной главной страницы, выберите " **Выбрать главную страницу**.  
  
-   WebForm.aspx  
  
     Начальное содержимое страницы.  Эта страница не имеет связанный файл зависимых codebehind.  
  
-   WebForm\_cb.aspx  
  
     Начальное содержимое страницы.  Эта страница имеет связанный файл зависимых codebehind.  
  
-   Codebehind. *расширение*  
  
     Зависимый файл, который реализует класс для webform.  Codebehind определяет язык *расширение* этого файла.  
  
-   ContentPage.aspx  
  
     Начальное содержимое страницы страницы содержимого.  Эта страница не имеет связанный файл зависимых codebehind.  
  
-   ContentPage\_cb.aspx  
  
     Начальное содержимое страницы страницы содержимого.  Эта страница имеет связанный файл зависимых codebehind.  
  
-   WebForm.vstemplate  
  
     Файл шаблона, который определяет содержимое новой веб\-страницы и его зависимого файла, если таковые имеются.  
  
### Новая главная страница  
 Этот шаблон создает новую главную страницу в ответ на **Добавление новой главной страницы** команды.  
  
 Создание зависимый исходный файл выберите codebehind **Поместите код в отдельном файле**.  В противном случае одна страница создана, которая не имеет пустой блок скриптов и директивы страниц %\> \<% подключить зависимый файл.  
  
-   MasterPage.master  
  
     Начальное содержимое главной страницы.  Эта главная страница не имеет связанный файл зависимых codebehind.  
  
-   MasterPage\_cb.master  
  
     Начальное содержимое главной страницы.  Эта главная страница имеет связанный файл зависимых codebehind.  
  
-   Codebehind.*расширение*  
  
     Зависимый файл, который реализует класс главной страницы.  Codebehind определяет язык *расширение* этого файла.  
  
-   MasterPage.vstemplate  
  
     Файл шаблона, который определяет содержимое новой главной страницы и его зависимого файла, если таковые имеются.
