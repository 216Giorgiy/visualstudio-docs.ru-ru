# Применение параметров в нескольких соединениях проекта
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

Подключаемый модуль системы управления версиями, построенный с помощью api\-интерфейса 1,2 подключаемый модуль системы управления версиями, может использовать пакетную операцию для выполнения одной и той же операция системы управления версиями с помощью нескольких проектов или нескольких контекстах соединения.  Пакеты могут использоваться для исключения резервное копирование, диалоговые окна в\-проекта из этих от использования.  
  
 Если пользователь выделяет несколько элементов, которые относятся к более одного соединения в подключаемом модуле системы управления версиями, построенных с помощью API 1,1 подключаемый модуль системы управления версиями \(например, 2\) файл\-акции веб\-проекта на разных компьютерах и проверьте их, пользователь видит те же диалоговое окно повторно.  Это происходит даже если пользователь щелкает **Применить ко всем** флажок в диалоговом окне, поскольку интегрированная среда разработки сбросит свое состояние для каждого контекста соединения.  
  
## Новый пометить возможности  
 `SccBeginBatch` функция устанавливает  `SCC_CAP_BATCH` пометить для указания того, что выполняется операция пакета  
  
## Новые функции  
 Следующие новые функции поддерживают пакетную операцию.  
  
-   [SccBeginBatch](../../extensibility/sccbeginbatch-function.md)  
  
-   [SccEndBatch](../../extensibility/sccendbatch-function.md)  
  
 `SCCBeginBatch` функция начнет группа в составе операции системы управления версиями.  `SccEndBatch` закрывает группа.  Группы могут быть вложенными.
