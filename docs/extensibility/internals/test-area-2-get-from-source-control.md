# Область тестирования 2: Получение из системы управления версиями
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

Эта область тестирования охватывает тестовых случаев для извлечения элементов из хранилища версий с помощью команды Get. Эти тестовые случаи могут применяться для локальных и веб\-проектов.  
  
## Доступ к меню команд  
 Следующие [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] в тестовых случаях используются пути меню среды разработки.  
  
##### Получение последней версии:  
  
-   **Файл**, **систему управления версиями**, **получить последнюю версию**.  
  
-   **Файл**, **получить последнюю версию**.  
  
-   Контекстное меню **получить последнюю версию**.  
  
-   Get: **файл**, **систему управления версиями**, **получить**.  
  
## Ожидаемое поведение  
  
##### Получение последней версии:  
 Выполняется извлечение silent \(без пользовательского интерфейса\) последней версии элемента из хранилища версий.  
  
##### Получение:  
 Отображает **получить** диалоговое окно и пользователь может внести изменения в наборе файлов, будет получен, а также изменить параметры, влияющие на способ загрузки файлов.  
  
## Тестовые случаи

Удаление локальной копии элемента.<br />5.  Получить последнюю версию элемента \(контекстное меню **получить последнюю версию**\).|Файл элемента извлекается локально.|  
|Получить файл, не существует локально|1.  Создайте проект.<br />2.  Добавьте элемент в проект.<br />3.  Поместите проект в системе управления версиями.<br />4.  Удаление локальной копии элемента.<br />5.  Получение элемента \(**файл**, **системы управления версиями**, **получить** \< item \>\).|Файл элемента извлекается локально.|  
|Получить файл, монопольно извлечен и изменен локально|1.  Создайте проект.<br />2.  Добавьте элемент в проект.<br />3.  Поместите проект в системе управления версиями.<br />4.
