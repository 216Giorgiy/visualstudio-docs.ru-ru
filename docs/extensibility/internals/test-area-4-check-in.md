# Тестов областям 4: Возврат
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

Эта область тестирования подключаемый модуль системы управления версиями охватывает отправки обновленных элементов в хранилище версий через **вернуть** команды.  
  
## Доступ к меню команд  
 Следующие [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] в тестовых случаях используются пути меню среды разработки.  
  
##### Возврат.  
 **Файл**, **систему управления версиями**, **возврата**.  
  
 **Файл**, **возврата**.  
  
 Контекстное меню **вернуть**.  
  
## Общее ожидаемое поведение  
  
-   Проекты и файлы, добавленные в решение или проект в системе управления версиями отображаются в **вернуть** диалоговое окно и **Отложенные возвраты** окна.  
  
-   После возврата добавленные элементы отображаются в системе управления версиями.  
  
-   После возврата обновленные элементы номером версии в хранилище.  
  
## Тестовые случаи  
 Ниже приведены конкретных тестовых случаев для области проверки возврата.  
  
### Case 4а: элементы изменены  
 Описывается использование проверки действия для обновления файла в системе управления версиями, который был изменен.  
  
|Действие|Шаги теста|Ожидаемые результаты для проверки|  
|--------------|----------------|---------------------------------------|  
|Измените текстовый файл, который был извлечен, вернуть только файл \(**вернуть** диалоговое окно\)|1.  Создайте новый проект с помощью текстового файла.<br />2.  Добавьте решение в систему управления версиями.<br />3.  Извлечение и изменение текстового файла.<br />4.  Возврат через диалоговое окно «Возврат» \(**файл**, **системы управления версиями**, **вернуть**\).|Общее ожидаемое поведение.|  
|Измените текстовый файл, который был извлечен, вернуть только файл \(**Отложенные возвраты** окна\)|1.  Создайте новый проект с помощью текстового файла.<br />2.  Добавьте решение в систему управления версиями.<br />3.  Извлечение и изменение текстового файла.<br />4.  Возврат через **Отложенные возвраты** окна.|Общее ожидаемое поведение.|  
  
### Case 4b: Добавление файлов  
 При добавлении файла в проект или элемент в решение, проект или решение, необходимо изменить также. Таким образом родительский файл извлечен также и должны быть возвращены для завершения добавления.  
  
|Действие|Шаги теста|Ожидаемые результаты для проверки|  
|--------------|----------------|---------------------------------------|  
|Добавьте текстовый файл и возвращать все элементы \(**вернуть** диалоговое окно\)|1.  Создайте новый проект.<br />2.  Добавьте решение в систему управления версиями.<br />3.  Добавьте текстовый файл в проект.<br />4.  Примите Извлечение проекта при появлении соответствующего запроса.<br />5.  Выберите решение в **обозревателе решений**.<br />6.  Возврат из **вернуть** диалоговое окно.|Общее ожидаемое поведение.|  
|Добавьте текстовый файл и возвращать все элементы \(**Отложенные возвраты** окна\)|1.  Создайте новый проект.<br />2.  Добавьте решение в систему управления версиями.<br />3.  Добавьте текстовый файл в проект.<br />4.  Примите Извлечение проекта при появлении соответствующего запроса.<br />5.  Возврат решения из **Отложенные возвраты** окна.|Общее ожидаемое поведение|  
  
### Case 4c: добавление проектов  
 При добавлении проекта в решение, необходимо также изменить решение. Таким образом также извлечен файл решения и должны быть возвращены для завершения добавления.  
  
|Действие|Шаги теста|Ожидаемые результаты для проверки|  
|--------------|----------------|---------------------------------------|  
|Добавить проект в новое решение в системе управления версиями \(**вернуть** диалоговое окно\)|1.  Создайте пустое решение.<br />2.  Добавьте решение в систему управления версиями.<br />3.  Добавьте новый проект.<br />4.  Примите извлечение решения, при появлении соответствующего запроса.<br />5.  Возврат из **вернуть** диалоговое окно.|Общее ожидаемое поведение.|  
|Добавить проект в новое решение в системе управления версиями \(**Отложенные возвраты** окна\)|1.  Создайте пустое решение.<br />2.  Добавьте решение в систему управления версиями.<br />3.  Добавьте новый проект.<br />4.  Примите извлечение решения, при появлении соответствующего запроса.<br />5.  Возврат решения из **Отложенные возвраты** окна.|Общее ожидаемое поведение.|
