# <a name="test-area-4-check-in"></a>Тестов областям 4: Возврат
Эта область тестирования подключаемый модуль системы управления версиями охватывает отправка обновленных элементов в хранилище версий через **вернуть** команды.  
  
## <a name="command-menu-access"></a>Доступ к меню команд  
 Следующие [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] путей меню среды разработки, которые используются в тестовых случаях.  
  
##### <a name="check-in"></a>Зарегистрируйся:  
 **Файл**, **система управления версиями**, **возврата**.  
  
 **Файл**, **возврата**.  
  
 Контекстное меню **вернуть**.  
  
## <a name="common-expected-behavior"></a>Общее ожидаемое поведение.  
  
-   Проекты и файлы, добавленные в решение или проект в системе управления версиями, отображаются в **вернуть** диалоговое окно и **Отложенные возвраты** окна.  
  
-   После возврата добавленные элементы отображаются в системе управления версиями.  
  
-   После возврата измененных элементов номером версии в хранилище.  
  
## <a name="test-cases"></a>Тестовые случаи  
 Ниже приведены определенные тестовые случаи для области проверки возврата.  
  
### <a name="case-4a-modified-items"></a>Регистр 4а: элементы изменены  
 Описывает использование проверки в действие для обновления файла в системе управления версиями, который был изменен.  
  
|Действие|Шаги теста|Ожидаемые результаты для проверки|  
|------------|----------------|--------------------------------|  
|Измените текстовый файл, который был извлечен, проверьте в файле только (**вернуть** диалоговое окно «»)|1.  Создайте новый проект с помощью текстового файла.<br />2.  Добавьте решение в систему управления версиями.<br />3.  Извлечение и изменение текстового файла.<br />4.  Возврат через диалоговое окно «Возврат» (**файл**, **системы управления версиями**, **вернуть**).|Общее ожидаемое поведение.|  
|Измените текстовый файл, который был извлечен, проверьте в файле только (**Отложенные возвраты** окна)|1.  Создайте новый проект с помощью текстового файла.<br />2.  Добавьте решение в систему управления версиями.<br />3.  Извлечение и изменение текстового файла.<br />4.  Возврат через **Отложенные возвраты** окна.|Общее ожидаемое поведение.|  
  
### <a name="case-4b-adding-files"></a>Регистр 4b: Добавление файлов  
 При добавлении файла проекта или элемента решения, проекта или решения необходимо также поменять. Таким образом родительский файл извлечен также и должны быть возвращены для завершения добавления.  
  
|Действие|Шаги теста|Ожидаемые результаты для проверки|  
|------------|----------------|--------------------------------|  
|Добавьте в текстовый файл и возвращать все элементы (**вернуть** диалоговое окно «»)|1.  Создайте новый проект.<br />2.  Добавьте решение в систему управления версиями.<br />3.  Добавьте в текстовый файл в проект.<br />4.  Извлечение проекта следует примите при появлении соответствующего запроса.<br />5.  Выберите решение в **обозревателе решений**.<br />6.  Регистрация из **вернуть** диалоговое окно.|Общее ожидаемое поведение.|  
|Добавьте в текстовый файл и возвращать все элементы (**Отложенные возвраты** окна)|1.  Создайте новый проект.<br />2.  Добавьте решение в систему управления версиями.<br />3.  Добавьте в текстовый файл в проект.<br />4.  Извлечение проекта следует примите при появлении соответствующего запроса.<br />5.  Возврат решения от **Отложенные возвраты** окна.|Общее ожидаемое поведение.|
