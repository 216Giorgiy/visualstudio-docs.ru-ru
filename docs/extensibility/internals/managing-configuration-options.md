# Управление параметрами конфигурации
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

При создании нового типа проекта, необходимо управлять параметрами конфигурации проекта и решения, которые определяют, как будет построен проект, упакован, развертывания и выполнить.  В следующих подразделах обсуждаются конфигурацию проекта и решения.  
  
## В этом подразделе  
 [Обзор](../../extensibility/internals/configuration-options-overview.md)  
 Описание способов применения для проектов в пределах [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] может поддерживать несколько конфигураций.  
  
 [Страницы свойств](../../extensibility/internals/property-pages.md)  
 Объясняет, что пользователи могут просматривать и изменять свойства конфигурации проекта и независимые зависимые свойства с помощью страниц свойств.  
  
 [Конфигурация решения](../../extensibility/internals/solution-configuration.md)  
 Предоставляет сведения о том, что хранятся в конфигурациях решения и конфигурации решения направляет расширения функциональности **Запуск** и  **Построение** команды.  
  
 [Объект конфигурации проекта](../../extensibility/internals/project-configuration-object.md)  
 Объясняет, как объект конфигурации проекта управляет отображением сведений о конфигурации на пользовательский интерфейс.  
  
 [Настройка проекта для построения](../../extensibility/internals/project-configuration-for-building.md)  
 Объясняет, как списка конфигурации решений для конкретного решения управляться **Конфигурации решения** диалоговое окно.  
  
 [Настройка проекта для управления развертыванием](../../extensibility/internals/project-configuration-for-managing-deployment.md)  
 Определяет процесс развертывания и 2 способов [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] поддерживает проекты, которые поддерживают развертывание.  
  
 [Настройка проекта для вывода](../../extensibility/internals/project-configuration-for-output.md)  
 Описывает процессы построения, каждая конфигурация может поддерживать и интерфейсы и методы, с помощью которых элементы вывода могут быть доступны.  
  
## Связанные подразделы  
 [Типы проектов](../../extensibility/internals/project-types.md)  
 Общие сведения о проектах как базовые шаблоны [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] интегрированная среда разработки \(ide\).  Ссылки на дополнительные разделы, в которых объясняется, как построение проектов, элементов управления и кода следует компилировать.
