# Компилятор цвет VSIX
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

Средство компилятор цвет расширения Visual Studio является консольным приложением, которое принимает XML-файл, представляющий цвета для существующих тем Visual Studio и преобразует его в .pkgdef файлов, чтобы эти цвета можно использовать в Visual Studio. Поскольку это легко сравнить различия между XML-файлов, это средство используется для управления пользовательских цветов в системе управления версиями. Его также можно связать сред построения, чтобы выходные данные построения является допустимым pkgdef-файл.  
  
 **Темы XML-схемы**  
  
 В XML-файл темы выглядит следующим образом:  
  
```xml  
<Themes>  
      <!—one or Theme elements -->  
      <Theme>  
        <!-- one or more Category elements -->  
        <Category>  
          <!-- one or more Color elements -->  
          <Color>  
            <!-- zero or one Background element -->  
            <Background />  
            <!-- zero or one Foreground element -->  
            <Foreground />  
          </Color>  
        </Category>  
      </Theme>  
</Themes>  
```  
  
 **Темы**  
  
 \< Темы> элемент определяет целой темы. Тема должен содержать по крайней мере один \< категории> элемента. Элементы темы определяются следующим образом:  
  
```xml  
<Theme Name="name" GUID="guid">  
      <!-- one or more Category elements -->  
</Theme>  
```  
  
|||  
|-|-|  
|**Атрибут**|**Определение**|  
|Имя|[Required] Имя темы|  
|Идентификатор GUID|[Required] (Должно совпадать форматирование GUID) GUID темы|  
  
 При создании пользовательских цветов для Visual Studio, эти цвета должны быть определены следующие темы. Наличие цветов для определенной теме Visual Studio пытается загрузить отсутствующие цвета из Светлая тема.  
  
|||  
|-|-|  
|**Имя темы**|**Темы GUID**|  
|Светлый|{de3dbbcd-f642-433c-8353-8f1df4370aba}|  
|Темный|{1ded0138-47ce-435e-84ef-9ec1f439b749}|  
|Синий|{a4d6a176-b948-4b29-8c66-53c97a1ed7d0}|  
|Высокая контрастность|{a4d6a176-b948-4b29-8c66-53c97a1ed7d0}|  
  
 **Категории**  
  
 \< Категории> элемент определяет коллекцию цветов в теме. Имена категорий обеспечивают логические группы и должен быть определен как узко максимально. Категория должна содержать по крайней мере один \< цвет> элемента. Категории элементов определяются следующим образом:  
  
```xml  
<Category Name="name" GUID="guid">  
      <!-- one or more Color elements -->  
 </Category>  
```  
  
|||  
|-|-|  
|**Атрибут**|**Определение**|  
|Имя|[Required] Имя категории|  
|Идентификатор GUID|[Required] GUID категории (должна соответствовать форматирование GUID)|  
  
 **Цвет**  
  
 \< Цвет> элемент определяет цвет для компонента или состояние пользовательского Интерфейса. Предпочтительный схему именования для цвета — [тип пользовательского Интерфейса] [Состояние]. Не используйте слово «цвет», как он является избыточным. Цвет должен явно указывает тип элемента и ситуаций или «состояние», для которых будет применяться цвет. Цвет не может быть пустым и должно содержать один или оба \< фона> и \< переднего плана> элемента. Цвета элементов определяются следующим образом:  
  
```xml  
<Color Name="name">  
        <Background /> <!-- zero or one Background element -->  
        <Foreground /> <!-- zero or one Foreground element -->  
 </Color>  
```  
  
|||  
|-|-|  
|**Атрибут**|**Определение**|  
|Имя|[Required] Имя цвета|  
  
 **Фона и переднего плана**  
  
 \< Фона> и \< переднего плана> элементы определяют значение цвет и тип для фона и переднего плана элемента пользовательского Интерфейса. Эти элементы могут иметь дочерних элементов.  
  
```xml  
<Background Type="type" Source="int" />  
<Foreground Type="type" Source="int" />  
```  
  
|||  
|-|-|  
|**Атрибут**|**Определение**|  
|Тип|[Required] Тип цвета. Это может быть одно из следующих:<br /><br /> *CT_INVALID:* цвет недопустима или не задана.<br /><br /> *CT_RAW:* необработанное значение ARGB.<br /><br /> *CT_COLORINDEX:* СЛЕДУЕТ ИСПОЛЬЗОВАТЬ.<br /><br /> *CT_SYSCOLOR:* Windows системный цвет из SysColor.<br /><br /> *CT_VSCOLOR:* цвета Visual Studio из __VSSYSCOLOREX.<br /><br /> *CT_AUTOMATIC:* Автоматическое цвет.<br /><br /> *CT_TRACK_FOREGROUND:* СЛЕДУЕТ ИСПОЛЬЗОВАТЬ.<br /><br /> *CT_TRACK_BACKGROUND:* СЛЕДУЕТ ИСПОЛЬЗОВАТЬ.|  
|Источник|[Required] Значение цвета, представлены в шестнадцатеричном формате|  
  
 Все значения, поддерживаемые в перечислении __VSCOLORTYPE поддерживаются схемой в атрибуте типа. Тем не менее рекомендуется использовать только CT_RAW и CT_SYSCOLOR.  
  
 **Все вместе**  
  
 Это простой пример XML-файлу, действительная Тема:  
  
```xml  
<Themes>  
  <Theme Name="Light" GUID="{de3dbbcd-f642-433c-8353-8f1df4370aba}">  
    <Category Name="MyCategory" GUID="{0A96238B-70CE-4479-9170-EECEAA3FCD58}">  
      <Color Name="MyActiveBorder">  
        <Background Type="CT_RAW" Source="FFCCCEDB" />  
      </Color>  
    </Category>  
  </Theme>  
</Themes>  
```  
  
## <a name="how-to-use-the-tool"></a>Как использовать средство  
 **Синтаксис**  
  
 VsixColorCompiler \< XML-файл> \< файл PkgDef> \< необязательные аргументы>  
  
 **Аргументы**  
  
||||  
|-|-|-|  
|**Имя коммутатора**|**Примечания**|**Обязательный или необязательный**|  
|Неименованные (XML-файл)|Это первый неименованных параметров и путь к XML-файл для преобразования.|Обязательное|  
|Неименованные (pkgdef-файл)|Это является вторым параметром неименованные и выходной путь для созданного pkgdef-файл.<br /><br /> По умолчанию: \< имя XML-файла>.pkgdef|Optional|  
|/nologo|Этот флаг установлен останавливает продукта и авторских правах сведения из печати.|Optional|  
|/?|Распечатайте справочной информации.|Optional|  
|/help|Распечатайте справочной информации.|Optional|  
  
 **Примеры**  
  
-   D:\pkgdef\colors.pkgdef VsixColorCompiler D:\xml\colors.xml  
  
-   / Nologo VsixColorCompiler D:\xml\colors.xml  
  
## <a name="notes"></a>Примечания  
  
-   Это средство требует наличия последнюю версию среды выполнения VC ++.  
  
-   Поддерживаются только отдельные файлы. Пакетное преобразование через пути к папкам не поддерживается.
