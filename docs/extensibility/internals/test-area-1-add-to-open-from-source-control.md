# Область тестирования 1: Добавьте / открыть из системы управления версиями
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

Этот источник подключаемого модуля управления тестирования описаны области, размещение решений или проектов в системе управления версиями и получения их из системы управления версиями.  
  
## Доступ к меню команд  
 Следующие [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] в тестовых случаях используются пути меню среды разработки:  
  
-   Для [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)], открытых из системы управления версиями: **файл**, **Откройте**, **проекта**или**решение**; поиск в [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)] расположение.  
  
-   Для других источника подключаемые модули управления, открыть из системы управления версиями: **файл**, **системы управления версиями**, **Открыть из системы управления версиями**.  
  
-   Добавить в систему управления версиями: **файл**, **системы управления версиями**, **Добавить решение в исходный файл элемента управления**, **системы управления версиями**, **Добавить выбранные проекты в систему управления версиями**.  
  
-   Контекстное меню \(проект или решение\), **Добавить решение в систему управления версиями**.  
  
-   Добавить из системы управления версиями: **файл**, **системы управления версиями**, **Добавление проекта из системы управления версиями**.  
  
-   Для [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)], добавление источника элемента управления также доступна из **файл**, **Добавить**, **существующий проект**; поиск в [!INCLUDE[vsvss](../../extensibility/includes/vsvss_md.md)] расположение.  
  
    > [!NOTE]
    >  В этом тесте может использоваться путь локального файла или локальный сервер IIS \(веб\-сервер\).  
  
## Ожидаемое поведение  
  
-   Для каждого типа поддерживаемых проекта пользователь должен иметь возможность «Добавить» и «Открыть из «системы управления версиями.  
  
-   При добавлении проекта в систему управления версиями, соответствующий \<*ProjectName*\> создается файл .vspscc \(указание файла проекта\). Он содержит исключения файла списка и сведений о соединении. Не удаляйте этот файл, поскольку он содержит сведения, относящиеся к проекту.  
  
-   При добавлении решения в систему управления версиями, соответствующий \<*имя\_решения*\> создается файл .vssscc \(triple S\). В текстовом файле содержатся сведения о соединении и список исключений файла, аналогично указание файла проекта. Этот файл является временным и существует только в базе данных управления версиями.  
  
-   При открытии решения из системы управления версиями \<*имя\_решения*\> локально создается файл .vsscc \(double S\), который существует только в базе данных управления версиями во временном файле. Этот файл содержит путь из папки решения подключение к файлу решения. Этот файл является временным и локальная копия удаляется после завершения операции «Открыть из системы управления версиями».  
  
-   После добавления проекта в систему управления версиями можно выполнять действия управления любого источника на нем \(извлечение, Get и так далее\).  
  
## Тестовые случаи  
 Ниже перечислены конкретных тестовых случаев для добавления к \/ Open из области тестирования системы управления версиями.  
  
### Case 1a: Добавление решения в систему управления версиями  
 Этот тестовый случай рассматривается добавление решения в систему управления версиями.  
  
|Действие|Шаги теста|Ожидаемые результаты для проверки|  
|--------------|----------------|---------------------------------------|  
|Добавьте решение, содержащее проект клиента в систему управления версиями|1.  Создайте проект клиента.<br />2.  Добавьте решение в систему управления версиями \(**файл**, **системы управления версиями**, **Добавить решение в систему управления версиями**\).|Решение или проект был добавлен в систему управления версиями.|  
|Добавьте решение, содержащее файловой системы или локальные IIS веб\-проекта в систему управления версиями|1.  Создание файловой системы или локальные веб\-проекта IIS \(используйте кнопку обзора для указания расположения проекта; путь определяет, какой тип веб\-проекта создается\).<br />2.  Добавьте решение в систему управления версиями \(**файл**, **системы управления версиями**, **Добавить решение в систему управления версиями**\).|Решение или проект был добавлен в систему управления версиями.|  
|Добавьте решение, содержащее удаленного веб\-сайта проекта в систему управления версиями|1.  Создайте проект удаленного веб\-сайта.<br />2.  Добавьте решение в систему управления версиями \(**файл**, **системы управления версиями**, **Добавить решение в систему управления версиями**\).<br />3.  Щелкните **ОК** в диалоговом окне Предупреждение доступа FrontPage.|Решение был добавлен в систему управления версиями.<br /><br /> Проект удаленного сайта не существует в системе управления версиями. \(Удаленный узел проектов должна контролироваться с сервера IIS.\)|  
|Добавление одного проекта решения с помощью элемента управления источника **Добавить выбранные проекты в систему управления версиями**.|1.  Создание отдельного проекта решения.<br />2.  Добавить только решение в систему управления версиями как выделенной области \(**файл**, **системы управления версиями**, **Добавить выбранные проекты в систему управления версиями**\). Если этот шаг завершается успешно, перейдите к шагу Далее.<br />3.  Добавить проект в систему управления версиями, как выбор \(**файл**, **системы управления версиями**, **Добавить выбранные проекты в систему управления версиями**\).<br />4.  Щелкните **Да** для добавления проекта в том же расположении.<br />5.  Щелкните **извлечь** в **Извлечь для изменения** диалоговое окно.|`Result from Step 2:`<br /><br /> Проект и все файлы в проекте, на которых извлеченный индикатор управления источника и появляется подсказка «не системы управления версиями».<br /><br /> `Result from Step 5:`<br /><br /> Файлы проекта и решения, в той же папке в системе управления версиями.|  
|Отменить добавление решения в систему управления версиями|1.  Создание отдельного проекта решения.<br />2.  Попытка добавить проект и решение в систему управления версиями. Если этот шаг завершается успешно, перейдите к шагу Далее.<br />3.  Отмените после находятся в системе управления версиями.|`Result from Step 2:`<br /><br /> Набор расположение источника управления диалоговым окном проекта появляется только один раз.<br /><br /> `Result from Step 3:`<br /><br /> Проект Добавление отменено, не является решение или проект в системе управления версиями и добавить источник элемента управления меню по\-прежнему доступны.|

|Действие|Шаги теста|Ожидаемые результаты для проверки|  
|--------------|----------------|---------------------------------------|  
|Откройте решение, содержащее проект клиента из системы управления версиями|1.  Создайте проект клиента.<br />2.  Добавьте решение в систему управления версиями.<br />3.  Закройте решение.<br />4.  Откройте решение из системы управления версиями в новое расположение.|Открыть решение или проект из системы управления версиями.|  
|Откройте решение, содержащее локальный или веб\-проекта IIS из системы управления версиями|1.  Создайте локальный или веб\-проекта IIS.<br />2.  Добавьте решение в систему управления версиями.<br />3.  Закройте решение.<br />4.  Откройте решение из системы управления версиями в новое расположение.|Открыть решение или проект из системы управления версиями.|  
|Откройте решение, содержащее удаленного веб\-сайта проекта из системы управления версиями|1.  Создайте проект удаленного веб\-сайта.<br />2.  Добавьте решение в систему управления версиями. Если этот шаг завершается успешно, перейдите к шагу Далее.<br />3.  Закройте решение.<br />4.  Откройте решение из системы управления версиями в новое расположение.|`Result from Step 2:`<br /><br /> Удаленный веб\-сайта — не в системе управления версиями.<br /><br /> `Result from Step 4:`<br /><br /> Решение открывается из системы управления версиями.<br /><br /> Загрузить проект удаленного сайта, но не в системе управления версиями.|
