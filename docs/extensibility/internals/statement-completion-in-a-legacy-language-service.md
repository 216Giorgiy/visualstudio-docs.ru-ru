# Завершение операторов в языковую службу для прежних версий
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

Завершение операторов — это процесс, с помощью которого языковой службы помогает Готово ключевое слово языка или элемент, они начали ввод в редакторе основных пользователей. В этом разделе Описание работы завершения операторов и реализовать его в службе языка.  
  
 Устаревший языковые службы реализуются как частью VSPackage, но это использование расширений MEF новый способ реализации возможностей службы языка. Чтобы узнать больше о новый способ реализовать завершение операторов, в разделе [Пошаговое руководство: Отображение завершения операторов](../../extensibility/walkthrough-displaying-statement-completion.md).  
  
> [!NOTE]
>  Мы рекомендуем начать использовать новый редактор API как можно быстрее. Это улучшает производительность службы языка и позволяют воспользоваться преимуществами новых функций редактора.  
  
## Реализация завершение операторов  
 В редакторе core завершение операторов активирует специальный пользовательский Интерфейс, который интерактивно помогает легко и быстро писать код. Завершение операторов помогает, отображая нужные объекты или классов их при необходимости, это позволяет избежать, запоминая конкретные элементы или необходимости поиска их справочного раздела справки.  
  
 Для реализации завершение операторов, язык необходимо иметь триггер завершения инструкции, который может быть проанализирован. Например [!INCLUDE[vbprvb](../../code-quality/includes/vbprvb_md.md)] используется точка \(.\) оператор, хотя [!INCLUDE[vcprvc](../../debugger/includes/vcprvc_md.md)] использует стрелки \(\-\>\) оператор. Служба языка можно использовать более одного триггера для инициирования завершения операторов. Эти триггеры программируются в фильтре команды.  
  
## Фильтры команд и триггеров  
 Команды фильтров перехватывать вхождения триггера или триггеры. Добавление команды фильтр к представлению, реализовать <xref:Microsoft.VisualStudio.OLE.Interop.IOleCommandTarget> интерфейс и присоединить его к представлению, вызвав <xref:Microsoft.VisualStudio.TextManager.Interop.IVsTextView.AddCommandFilter%2A> метод. Можно использовать один и тот же фильтр команды \(<xref:Microsoft.VisualStudio.OLE.Interop.IOleCommandTarget>\) для всех аспектов службы языка, например завершение операторов, маркеры ошибок и метод советы. Для получения дополнительной информации см. [Перехват команд службы языка прежних версий](../../extensibility/internals/intercepting-legacy-language-service-commands.md).  
  
 При вводе в редакторе триггера — в частности, текстовый буфер — вызывает службу языка <xref:Microsoft.VisualStudio.TextManager.Interop.IVsTextView.UpdateCompletionStatus%2A> метод. В результате редактор для вызова пользовательского интерфейса, чтобы пользователь может выбирать из кандидатов завершения инструкции. Этот метод необходимо реализовать <xref:Microsoft.VisualStudio.TextManager.Interop.IVsCompletionSet> и <xref:Microsoft.VisualStudio.TextManager.Interop.UpdateCompletionFlags> флагов в качестве параметров. Список элементов завершения отображается в списке с прокруткой. По мере ввода пользователем выделенного фрагмента в окне списка обновляется для отражения типизированных близкое последних символов. Базовый редактор реализует пользовательский Интерфейс для выполнения инструкции, но языковой службы необходимо реализовать <xref:Microsoft.VisualStudio.TextManager.Interop.IVsCompletionSet> интерфейс, чтобы определить набор кандидатов элементы завершения инструкции.
