# Область тестирования 5: Изменение системы управления версиями
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

Эта область тестирования подключаемый модуль системы управления версиями рассматривается изменение системы управления версиями через **Изменение системы управления версиями** команды.  
  
 **Изменение системы управления версиями** команда предоставляет четыре основные функции для пользователя:  
  
-   **Привязки:**  
  
     Позволяет пользователю установить или восстановить связь с источником управления между решение или проект и хранилище версий.  
  
-   **Отменить привязку:**  
  
     Удаляет проект или решение из системы управления версиями на уровне подключения.  
  
-   **Подключение или отключение:**  
  
 Переключает подключенном или автономном состоянии управляемой решения, который рассматривается в области 3. Для получения дополнительной информации см. [Область тестирования 3: Извлечение и отменить извлечение](../../extensibility/internals/test-area-3-check-out-undo-checkout.md).  
  
## Доступ к меню команд  
 Следующие [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] путь меню среды разработки используется в тестовых случаях.  
  
 Изменение системы управления версиями:**файл**, **систему управления версиями**, **изменить систему управления версиями**.  
  
## Тестовые случаи  
 Ниже перечислены конкретных тестовых случаев для **Изменение системы управления версиями** команды области тестирования.  
  
### Case 5а: привязки  
 Привязка позволяет пользователю добавить дополнительную информацию о контроле исходного кода для выбранных проектов и решений. Пользователь обычно предлагается указать проект в системе управления версиями, к которому они должны быть добавлены. Пользователь не может создать новый проект в системе управления версиями в рамках этой операции \(Сравните с добавить в систему управления версиями\).

### Case 5b: привязку  
 Отменить привязку удаляет сведения об управления исходного кода из проектов и их решения. Измененные проекты и решения основаны на выбор пользователя и как они были добавлены в систему управления версиями.  
  
|Действие|Шаги теста|Ожидаемые результаты для проверки|  
|--------------|----------------|---------------------------------------|  
|Отменить привязку решения, содержащего один файловой системы или локальные веб\-проекта IIS и один клиентский проект|1.  Создание файловой системы или локальные веб\-проекта IIS.<br />2.  Добавьте решение в систему управления версиями.<br />3.  Добавление нового клиентского проекта в решение.<br />4.  Примите проверьте из решения, при появлении соответствующего запроса.<br />5.  Откройте **Изменение системы управления версиями** диалоговое окно.<br />6.  Щелкните **привязку**.<br />7.  Щелкните **ОК** чтобы закрыть диалоговое окно.<br />8.  Попытка извлечь решение, проект, элементы решения, элементы проекта.|Решения и проекты не в системе управления версиями.<br /><br /> Команды меню системы управления версиями не отображаются.|  
|Отменить привязку|1.  Создайте проект.<br />2.  Добавьте решение в систему управления версиями.<br />3.  Откройте **Изменение системы управления версиями** диалоговое окно.<br />4.  Щелкните **привязку всех**.<br />5.  Нажмите кнопку **Отмена**.|Решение находится под управлением системы управления версиями.|

 Повторная привязка состоит просто из unbind и привязка — процесс перепривязки проект или решение, ранее в системе управления версиями и было свободной.
