# <a name="source-control-integration-overview"></a>Общие сведения об интеграции управления источника
В этом разделе сравниваются два способа интеграции в систему управления версиями Visual Studio; системы управления версиями подключаемого модуля и VSPackage, который предоставляет решение для управления версиями и новыми возможностями управления источника. Visual Studio позволяет вручную переключение между исходным пакеты VSPackage и подключаемые модули управления версиями, а также автоматического решение основано переключения.  
  
## <a name="source-control-integration"></a>Интеграция системы управления версиями  
 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]поддерживает два типа параметров интеграции системы управления версиями. Во всех версиях [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)], по-прежнему можно интегрировать в зависимости от источника управления API подключаемых модулей (ранее также называют MSSCCI API), который предоставляет основные функциональные возможности элемента управления во время использования Visual Studio источника управления пользовательского интерфейса (подключаемый модуль ПОЛЬЗОВАТЕЛЬСКИЙ ИНТЕРФЕЙС). Системы управления версиями VSPackage, с другой стороны, предоставляет новые возможности, прямой интеграции [!INCLUDE[vsipsdk](../../extensibility/includes/vsipsdk_md.md)] подходит для интеграции системы управления версиями, которая требует высокого уровня квалификации и автономность в его модель управления версиями.  
  
 ![Обзор элемента управления источника](../../extensibility/internals/media/sourcectnrloverview.gif "SourceCtnrlOverview")  
  
## <a name="source-control-plug-in"></a>Подключаемый модуль системы управления версиями  
 Все версии Visual Studio поддерживает API подключаемых модулей управления источника спецификация версии 1.2 как путь к интеграции. Исполнитель подключаемого модуля управления источника записывает библиотеку DLL, которая реализует функции API подключаемых модулей для управления источника для интеграции системы управления версиями и регистрации в [Создание подключаемый модуль системы управления источника](../../extensibility/internals/creating-a-source-control-plug-in.md). При таком подходе интегрированной среды разработки (IDE) использует [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] пользовательского интерфейса для диалоговых окон, например возврат, извлечение, Сервис/Параметры страницы свойств, панели инструментов и глифы исходного элемента управления. Строгое соответствие API подключаемых модулей управления источника гарантирует интегрировать Visual Studio и надежной работы для пользователя. Это означает, что подключаемый модуль системы управления версиями необходимо реализовать большинство функций и обратные вызовы, описанные в API-интерфейса.  
  
 Реализация системы управления версиями, подключаемый модуль, с помощью API управления подключаемого модуля источника, выполните следующие действия.  
  
1.  Создать DLL, которая реализует функции, определенные в [подключаемых модулей системы управления версиями](../../extensibility/source-control-plug-ins.md).  
  
2.  Зарегистрировать библиотеку DLL, делая соответствующие записи в реестр (описано в [как: установить подключаемый модуль системы управления источника](../../extensibility/internals/how-to-install-a-source-control-plug-in.md)).  
  
3.  Создание вспомогательного класса пользовательского интерфейса и отображения при появлении запроса пакет адаптера системы управления версиями (Visual Studio компонент, который управляет функций системы управления версиями через подключаемые модули управления версиями)  
  
 В ответ на команда системы управления версиями Интегрированной среде разработки Visual Studio предоставляет стандартный пользовательский Интерфейс для основных операций и передает их в систему управления версиями подключаемого модуля через функции, определенные в API подключаемых модулей управления источника. Для настройки дополнительных параметров подключаемый модуль системы управления версиями может вызываться на для представления собственного пользовательского интерфейса, например, Обзор для проекта с управлением версиями. Это означает, что пользователь может появиться окно с двумя возможно различные стили пользовательского интерфейса, при работе с системой управления версиями: пользовательский Интерфейс, представленных в Visual Studio и пользовательского интерфейса, которая представляет подключаемый модуль системы управления версиями. Это наиболее заметны при использовании сложных операций управления версиями.  
  
### <a name="drawbacks-to-implementing-a-source-control-plug-in"></a>Недостатки реализации подключаемого модуля системы управления версиями  
  
-   Расширенные возможности пользователь может видеть два различных стилей интерфейсов, приведет к возможную неоднозначность.  
  
-   Подключаемый модуль системы управления версиями сводится к модель управления версиями, содержится в разрешении API подключаемых модулей управления источника.  
  
-   API подключаемых модулей управления источника может быть слишком строгие для некоторых сценариев управления версиями.  
  
### <a name="advantages-to-implementing-a-source-control-plug-in"></a>Преимущества реализации подключаемого модуля системы управления версиями  
  
-   Visual Studio предоставляет пользовательский Интерфейс для всех операций управления основных источника, чтобы подключаемый модуль системы управления версиями не должен реализовывать потенциально сложные пользовательского интерфейса.  
  
-   Из-за strict API подключаемый модуль системы управления версиями можно легко взаимодействовать с программами управления внешний источник для предоставления более сложных функций; Visual Studio не имеет значения, слишком гораздо как функций системы управления версиями выполняется, только то, что он выполняется в соответствии с API подключаемых модулей управления источника.  
  
-   Реализация системы управления версиями, подключаемый модуль, чем VSPackage системы управления версиями проще.  
  
## <a name="source-control-vspackage"></a>Исходный элемент управления VSPackage  
 [!INCLUDE[vsipsdk](../../extensibility/includes/vsipsdk_md.md)]обеспечивает глубокую интеграцию в Visual Studio с полный контроль над функций системы управления версиями и полная замена исходного предоставляемых системой Visual Studio элемента управления пользовательского интерфейса. Элемент управления источником VSPackage зарегистрированы в Visual Studio и предоставляет функциональные возможности элемента управления источника. Хотя с помощью Visual Studio можно зарегистрировать несколько системы управления версиями пакеты VSPackage, только один из них может быть активен в каждый момент времени. VSPackage системы управления версиями имеет полный контроль над функций системы управления версиями и внешний вид в Visual Studio, когда оно активно. Все другие системы управления версиями пакеты VSPackage, которые могут быть зарегистрированы в системе, находятся в неактивном состоянии и не будет отображаться пользовательского интерфейса, вообще.  
  
 Реализация системы управления версиями VSPackage необходима стратегия «все или ничего». Создатель VSPackage системы управления версиями необходимо приобрести значительный объем усилий в реализации интерфейсов управления источника и новые элементы пользовательского интерфейса (диалоговые окна, меню и панелей инструментов) для охвата всего исходного функциональные возможности элемента управления. В разделе [создания пакетов VSPackage управления источника](../../extensibility/internals/creating-a-source-control-vspackage.md) для получения дополнительных сведений.  
  
### <a name="drawbacks-to-implementing-a-source-control-vspackage"></a>Недостатки реализации элемента управления источника пакетов VSPackage  
  
-   VSPackage должен реализовать ряд сложных интерфейсов успешно интеграция с Visual Studio.  
  
-   Пакет VSPackage необходимо указать все пользовательского интерфейса, необходимые для системы управления версиями; Visual Studio предоставляет без помощи в этой области.  
  
-   Элемент управления источником VSPackage тесно связано с Visual Studio и не может работать с отдельные программы, поэтому функции не может совместно использоваться так же просто, внешние версии программы системы управления версиями.  
  
### <a name="advantages-to-implementing-a-source-control-vspackage"></a>Преимущества реализации элемента управления источника пакетов VSPackage  
  
-   Так как пакет VSPackage имеет полный контроль над пользовательского интерфейса системы управления версиями и функциональные возможности, пользователь видит удобный интерфейс для системы управления версиями.  
  
-   Пакет VSPackage не связана ни с моделью управления определенного источника.
