# <a name="source-control-integration-overview"></a>Общие сведения об интеграции управления версиями
В этом разделе сравниваются два способа интеграции в систему управления версиями Visual Studio; Система управления версиями подключаемого модуля и VSPackage, который предоставляет решение для управления версиями и новыми возможностями управления источника. Visual Studio позволяет вручную переключаться между исходным VSPackages и подключаемых модулей системы управления версиями, а также автоматическое решение основано переключение.  
  
## <a name="source-control-integration"></a>Интеграция системы управления версиями  
 [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)]поддерживает два типа параметров интеграции системы управления версиями. Во всех версиях [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)], по-прежнему можно интегрировать подключаемого модуля, в зависимости от источника управления API подключаемых модулей (ранее также называют MSSCCI API), который предоставляет основные функциональные возможности элемента управления при помощи Visual Studio исходного элемента управления пользовательского интерфейса (UI). Системы управления версиями VSPackage, с другой стороны, предоставляет новые возможности, интеграции сложного [!INCLUDE[vsipsdk](../../extensibility/includes/vsipsdk_md.md)] подходит для интеграцию системы управления версиями, которая требует высокого уровня квалификации и автономность в его модель управления исходным кодом.  
  
 ![Обзор контроля источника](../../extensibility/internals/media/sourcectnrloverview.gif "SourceCtnrlOverview")  
  
## <a name="source-control-plug-in"></a>Подключаемый модуль системы управления версиями  
 Все версии Visual Studio поддерживают API подключаемого модуля управления источника спецификация версии 1.2 как путь к интеграции. Записывает отдельный исполнитель подключаемый модуль управления источником DLL, который реализует функции API подключаемого модуля управления источника интеграцию системы управления версиями и регистрации, как описано в [Создание подключаемый модуль системы управления источника](../../extensibility/internals/creating-a-source-control-plug-in.md). При таком подходе интегрированной среды разработки (IDE) использует [!INCLUDE[vsprvs](../../code-quality/includes/vsprvs_md.md)] пользовательского интерфейса для диалоговых окон, например возврат, извлечение, Сервис/Параметры страницы свойств, панели инструментов и глифы исходного элемента управления. Строгое соответствие API подключаемого модуля управления источника обеспечивает простую интеграцию в Visual Studio и надежной работы пользователя. Это означает, что модуль управления версиями необходимо реализовать большинство функций и обратные вызовы, указанные в API.  
  
 Чтобы реализовать подключаемый модуль с помощью API подключаемого модуля управления источника, выполните следующие действия.  
  
1.  Создать библиотеку DLL, который реализует функции, указанной в [подключаемых модулей системы управления версиями](../../extensibility/source-control-plug-ins.md).  
  
2.  Зарегистрируйте библиотеки DLL, сделав соответствующие записи в реестр (описанные в [как: установить подключаемый модуль системы управления источника](../../extensibility/internals/how-to-install-a-source-control-plug-in.md)).  
  
3.  Создать вспомогательный класс пользовательского интерфейса и отображения при появлении запроса адаптера пакет системы управления версиями (Visual Studio компонент, который управляет функции системы управления версиями через подключаемые модули управления версиями)  
  
 В ответ на команду управления источника Интегрированной среде разработки Visual Studio предоставляет стандартный пользовательский Интерфейс для основных операций и передает их в систему управления версиями подключаемого модуля через функции, определенные в интерфейсе API подключаемого модуля управления источника. Для установки дополнительных параметров подключаемого модуля системы управления версиями может вызываться на представить свой UI, например, Обзор для проекта с управлением версиями. Это означает, что пользователь может появиться окно с двумя возможно различные стили пользовательского интерфейса при работе с системой управления версиями: Visual Studio отображает пользовательский Интерфейс и пользовательского интерфейса, которая представляет подключаемый модуль системы управления версиями. Это наиболее заметны при использовании сложных операций управления версиями.  
  
### <a name="drawbacks-to-implementing-a-source-control-plug-in"></a>Недостатки реализации подключаемого модуля системы управления версиями  
  
-   Расширенные возможности пользователь может видеть два различных стиля интерфейсов, вызывающие недоразумений.  
  
-   Подключаемый модуль системы управления версиями сводится к модель управления исходным кодом, который содержится в разрешении API подключаемого модуля управления источника.  
  
-   Возможно, слишком строгие для некоторых сценариев управления версиями API подключаемого модуля управления источника.  
  
### <a name="advantages-to-implementing-a-source-control-plug-in"></a>Преимущества реализации подключаемого модуля системы управления версиями  
  
-   Visual Studio предоставляет пользовательский Интерфейс для всех операций управления базовый источник, подключаемый модуль системы управления версиями не придется реализовать потенциально сложного интерфейса пользователя.  
  
-   Из-за strict API подключаемого модуля системы управления версиями можно легко взаимодействовать с программами управления внешнего источника для более глубокого функциональности. Visual Studio не имеет слишком гораздо как функции системы управления версиями выполняется, только то, что она выполняется в соответствии с API подключаемого модуля управления источника.  
  
-   Проще реализовать подключаемый модуль, чем VSPackage системы управления версиями.  
  
## <a name="source-control-vspackage"></a>Исходный элемент управления VSPackage  
 [!INCLUDE[vsipsdk](../../extensibility/includes/vsipsdk_md.md)]обеспечивает глубокую интеграцию в среду Visual Studio с полный доступ к функции системы управления версиями и полная замена элемента управления пользовательского интерфейса Visual Studio предоставляемый источника. Система управления версиями VSPackage зарегистрированы в Visual Studio и предоставляет функции системы управления версиями. Хотя некоторые системы управления версиями VSPackages могут быть зарегистрированы в Visual Studio, только один из них может быть активен в каждый момент времени. Система управления версиями VSPackage имеет полный контроль над внешний вид и функции системы управления версиями в Visual Studio при активном. Все другие системы управления версиями VSPackages, которые могут быть зарегистрированы в системе неактивных и не отобразит никаких элементов пользовательского интерфейса вообще.  
  
 Реализация системы управления версиями VSPackage необходима стратегия «все или ничего». Создатель VSPackage системы управления версиями необходимо вложить значительный объем усилий при реализации интерфейсов управления источника и новые элементы пользовательского интерфейса (диалоговые окна, меню и панелей инструментов) позволяет реализовать функциональные возможности управления всей исходной. В разделе [Создание VSPackage управления источника](../../extensibility/internals/creating-a-source-control-vspackage.md) подробнее.  
  
### <a name="drawbacks-to-implementing-a-source-control-vspackage"></a>Недостатки реализации VSPackage управления источника  
  
-   VSPackage необходимо реализовать целый ряд сложных интерфейсов для успешной интеграции с Visual Studio.  
  
-   VSPackage необходимо указать все пользовательский Интерфейс, необходимый для системы управления версиями; Visual Studio предоставляет без помощи в этой области.  
  
-   Система управления версиями VSPackage тесно связаны с Visual Studio и не работают отдельные программы, поэтому функции не может использоваться так же легко, внешняя версия программы системы управления версиями.  
  
### <a name="advantages-to-implementing-a-source-control-vspackage"></a>Преимущества реализации VSPackage управления источника  
  
-   Поскольку VSPackage имеет полный контроль над пользовательского интерфейса системы управления версиями и функциональные возможности, пользователю предоставляется простой интерфейс для системы управления версиями.  
  
-   Модель управления определенного источника не ограничен VSPackage.
