## Пакеты VSPackage  
 Пакеты VSPackage представляют собой программные модули, которые дополняют и расширяют с элементами пользовательского интерфейса, службы, проекты, редакторы и конструкторы Visual Studio. Пакеты VSPackage представляют собой единицы центра архитектуры Visual Studio. Для получения дополнительной информации см. [Пакеты VSPackage](../../extensibility/internals/vspackages.md).  
  
## Visual Studio \(оболочка\)  
 Оболочка Visual Studio обеспечивает базовые функции и поддержки взаимодействия между VSPackages и MEF расширения ее компонентов. Для получения дополнительной информации см. [Visual Studio \(оболочка\)](../../extensibility/internals/visual-studio-shell.md).

## Команды  
 Команды, функции, которые выполняют задачи, например печати документа, при обновлении представления или создании нового файла.  
  
 При расширении Visual Studio, можно создавать команды и зарегистрируйте их с помощью оболочки Visual Studio. Можно указать, как эти команды отображаются в Интегрированной среде разработки, например, меню или панели инструментов. Обычно отображается пользовательской команды на **средства** на появятся меню и команды для отображения окна средства **Другие окна** подменю **представление** меню.  
  
 При создании команды, необходимо также создать обработчик событий для него. Обработчик событий определяет, когда команда видим или он включен, позволяет изменять текст и гарантирует, что команда реагирует соответствующим образом при активации. В большинстве случаев IDE обрабатывает команды с помощью <xref:Microsoft.VisualStudio.OLE.Interop.IOleCommandTarget> интерфейса. Команды в Visual Studio, обрабатываются начиная с внутренней контекст командной строки, на основе локального выбора и продолжив внешний контекст, на основе глобального выделения. Команды, добавленные в главное меню сразу же доступны для использования в сценариях.  
  
 Для получения дополнительной информации см. [Команды, меню и панелей инструментов](../../extensibility/internals/commands-menus-and-toolbars.md).  
  
## Меню и панели инструментов  
 Меню и панели инструментов позволяют пользователям для вызова команд. Меню, строки или столбцы из команд, которые обычно отображаются в виде отдельных текстовых элементов сверху окна средства. Подменю\-это дополнительный меню, которые отображаются при щелчке команд, содержащих маленькую стрелку. Контекстные меню появляются при щелчке некоторых элементов пользовательского интерфейса. Некоторые общие имена меню являются **файл**, **Изменить**, **представление**, и **окна**. Для получения дополнительной информации см. [Расширение меню и команд](../../extensibility/extending-menus-and-commands.md).  
  
 Панели инструментов, строк или столбцов кнопок и других элементов управления, такие как поля со списком, списки и текстовые поля. Кнопки панели инструментов, обычно имеют значок изображения, такие как значок папки для **Открыть файл** команды или принтера для **печати** команды. Все элементы панели инструментов связаны с командами. При нажатии кнопки выполняется его связанной команды. В случае раскрывающийся элемент управления каждый элемент в раскрывающемся списке связан с другой команде. Некоторые элементы управления панели инструментов, например, элемент управления splitter являются гибридами. Одной стороны элемента управления кнопки панели инструментов, а другая сторона – стрелку вниз, которая отображает несколько команд, при щелчке.  
  
## Окна инструментов  
 Средство windows используются в Интегрированной среде разработки для отображения информации.**Элементов**, **обозревателе решений**, **Свойства** окна, и **веб\-обозреватель** приведены примеры окон инструментов.  
  
 Средство windows обычно предлагают различные элементы управления, с которыми может взаимодействовать пользователь. Например **Свойства** окна позволяет пользователю задавать свойства объектов, которые служат для определенной цели.**Свойства** окно специализированных в этом смысле, а также общие, так как он может использоваться во многих различных ситуациях. Аналогичным образом **вывода** окно специализированным, так как она обеспечивает вывод текста, но Общие, так как многие подсистем в Visual Studio можно использовать для представления пользователю Visual Studio.  
  
 Рассмотрим следующий рисунок Visual Studio, которая содержит несколько окон инструментов.  
  
 ![Снимок экрана](../../extensibility/internals/media/t1gui.png "T1gui")  
  
 Некоторые окна инструментов закреплены вместе на одну область отображает окно инструментов обозревателя решений и скрывает окна инструментов, но делает их нужно щелкнуть вкладку. На рисунке показаны два окна инструментов **Список ошибок** и **вывода** окно, находящееся вместе на одной панели.  
  
 Также показано, является основной области документа, показывающий несколько окон редактора. Несмотря на то, что средство windows обычно имеют только один экземпляр \(например, можно открыть только один **обозревателе решений**\), редактор windows может иметь несколько экземпляров, каждый из которых используется для редактирования отдельный документ, но все они закрепляются в той же области. На рисунке показана область документа, которая содержит два окна редактора, одно окно конструктора форм и окно браузера, который показывает страницу запуска. Все окна в окне документа, щелкнув вкладки доступны, но в окне редактора, содержащее файл EditorPane.cs является видимым и active.  
  
 При расширении Visual Studio, можно создать средство windows, которые позволяют пользователям Visual Studio взаимодействовать с расширением. Можно также создать свои собственные редакторов, которые позволяют пользователям Visual Studio редактирования документов. Поскольку окон инструментов и редакторов будут интегрированы в Visual Studio, не нужно программировать их, чтобы закрепить или неправильно отображаться на вкладке. Правильно зарегистрированы в Visual Studio, они будут автоматически после обычной функции окна инструментов и окна документов в Visual Studio. Для получения дополнительной информации см. [Расширение и настройка окна инструментов](../../extensibility/extending-and-customizing-tool-windows.md).  
  
## Окна документов  
 Окно документа — рамке дочернего окна многодокументного интерфейса \(MDI\) окна. Окна документов обычно используются для размещения текстовые редакторы, редакторы формы \(также известный как конструкторы\) или элементы управления для редактирования, но также позволяет размещать другие функциональные типы.**Новый файл** диалоговое окно содержит примеры окон документов, предоставляемых Visual Studio.  
  
 Большинства редакторов специфичны для языка программирования или тип файла, такие как HTML\-страницы, наборы фреймов, файлы C\+\+ или файлы заголовка. Выбрав шаблон в **новый файл** диалоговом пользователь динамически создает окно документа для редактора для типа файла, который связан с шаблоном. Окно документа также создается, когда пользователь открывает существующий файл.  
  
 Окна документов, ограничены клиентской области MDI. Каждый документ имеет вкладку в верхней части и вкладка заказ связан с другим windows, может быть открыт в области MDI. Щелкните правой кнопкой мыши вкладку окна документа отображается контекстное меню, содержит параметры, можно разбить на несколько групп вкладок горизонтальной или вертикальной области MDI. Разделение области MDI позволяет одновременно просматривать несколько файлов. Для получения дополнительной информации см. [Окна документов](../../extensibility/internals/document-windows.md).  
  
## Редакторы  
 В редакторе Visual Studio позволяет настраивать и использовать его для типа содержимого с помощью Managed Extensibility Framework \(MEF\). В большинстве случаев не потребуется создать VSPackage расширить редактор, несмотря на то, что если вы хотите включить функции из оболочки \(например, команду меню или сочетание клавиш\), расширение MEF можно объединить с VSPackage.  
  
 Также можно создать пользовательский редактор, например, для чтения и записи в базе данных, или если вы хотите использовать конструктор. Также можно использовать внешний редактор, например Блокнот или Microsoft WordPad. Дополнительные сведения см. в разделе [Редактор и расширения службы языка](../../extensibility/editor-and-language-service-extensions.md).  
  
## Языковые службы  
 Редактор Visual Studio для поддержки новых ключевых слов программирования или даже новый язык программирования, можно создать языковую службу. Каждая языковая служба может реализовать некоторых функций редактора, полностью, частично или вообще не. В зависимости от настроек служба языка может обеспечивать подсветку синтаксиса, парные фигурные скобки, поддержку технологии IntelliSense и другие функции в редакторе.  
  
 В сердце языковую службу — средство синтаксического анализа и сканер. Сканер \(или лексический анализатор\) разделяет исходный файл на элементы, которые известны как маркеры и средство синтаксического анализа устанавливает отношения между эти маркеры. При создании языковую службу, необходимо реализовать синтаксический анализатор и средство проверки, чтобы Visual Studio мог распознать маркеры и грамматику языка. Можно создавать службы управляемого или неуправляемого языка. Для получения дополнительной информации см. [Расширяемость устаревших языковой службы](../../extensibility/internals/legacy-language-service-extensibility.md).  
  
## Проекты  
 В Visual Studio проекты — это контейнеры, которые разработчики используют для организации и создания исходного кода и других ресурсов. Проекты позволяют организовывать, создавать, отлаживать и развертывать исходный код, ссылки на веб\-служб и баз данных и другие ресурсы. Пакеты VSPackage можно расширить система проектов Visual Studio, предоставляя типы проектов, подтипы проектов и пользовательские средства.  
  
 Проекты могут также можно собирать в решение, которое представляет собой способ группирования одного или нескольких проектов, работающих совместно для создания приложения. Проекта и состояние, относящееся к решению хранится в двух файлов решений, файл текстовых решения \(SLN\) и файл пользовательских параметров \(.suo\) двоичные решения. Эти файлы похожи на файлы группы \(VBG\), которые использовались в предыдущих версиях [!INCLUDE[vbprvb](../../code-quality/includes/vbprvb_md.md)], а рабочей области \(DSW\) и пользователь параметры файлов \(расширение — OPT\), которые использовались в предыдущих версиях [!INCLUDE[vcprvc](../../debugger/includes/vcprvc_md.md)].  
  
 Дополнительные сведения см. в разделах [Проекты](../../extensibility/internals/projects.md) и [Решения](../../extensibility/internals/solutions.md).  
  
## Шаблоны проектов и элементов  
 Visual Studio включает заранее определенных шаблонов проекта и шаблонов элементов проекта. Можно также создать собственные шаблоны или получите шаблонов от сообщества, а затем интегрировать их в Visual Studio.[Коллекции кода MSDN](http://code.msdn.microsoft.com/Project/ProjectDirectory.aspx?ProjectSearchText=visual%20studio) имеет место для поиска шаблонов и расширения.  
  
 Шаблоны содержат структуры проекта и основные файлы, необходимые для создания определенного типа приложений, управления, библиотеки или класса. При разработке программного обеспечения, подобное одному из шаблонов, создать проект, основанный на шаблоне и затем изменять файлы в этом проекте.  
  
> [!NOTE]
>  Эта архитектура шаблона не поддерживается для [!INCLUDE[vcprvc](../../debugger/includes/vcprvc_md.md)] проектов. Дополнительные сведения о создании [!INCLUDE[vcprvc](../../debugger/includes/vcprvc_md.md)] шаблонов проектов см. в разделе [Разработка мастера](/visual-cpp/ide/designing-a-wizard).  
  
 Дополнительные сведения см. в разделе [Добавление проекта и шаблоны элементов проекта](../../extensibility/internals/adding-project-and-project-item-templates.md).

## Службы Visual Studio  
 Служба предоставляет ряд интерфейсов компонентов для использования. Visual Studio предоставляет набор служб, которые могут использоваться любые компоненты, включая расширения. Например службы Visual Studio позволяют окна инструментов, показаны или скрыты динамически, включите доступ к справке, строка состояния или события пользовательского интерфейса. В редакторе Visual Studio также предоставляет службы, которые можно импортировать, расширения редактора. Для получения дополнительной информации см. [Использование и предоставления услуг](../../extensibility/using-and-providing-services.md).

Для получения дополнительной информации см. [Расширение возможностей отладчика Visual Studio](../../extensibility/debugger/visual-studio-debugger-extensibility.md).
