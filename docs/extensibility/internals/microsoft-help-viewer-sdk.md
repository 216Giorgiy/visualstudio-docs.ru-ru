# Средство просмотра справки Microsoft SDK
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

Данная статья включает следующие задачи для интеграторов просмотра справки Visual Studio.  
  
-   Создание раздела \(поддержка F1\)  
  
-   Создание фирменной символики содержимое пакета средства просмотра справки  
  
-   Развертывание набора статей  
  
-   Добавление справочных для оболочки Visual Studio \(интегрированная или изолированное\)  
  
-   Дополнительные ресурсы  
  
### Создание раздела \(поддержка F1\)  
 Этот раздел предоставляет обзор компонентов, представленных раздела, раздел требования, краткое описание способов создания раздела \(в том числе требования к поддержке F1\) и, наконец, пример раздела с его результат.  
  
 **Обзор раздела средства просмотра справки**  
  
 При вызове раздела для подготовки к просмотру средство просмотра справки возвращает элементы фирменной настройки пакета, связанных с разделом во время установки или последнего обновления вместе с раздела XHTML и объединяет два приведет представленных представления содержимого \(фирменной символики данных \+ раздела данных\).  Пакет с фирменной символикой содержит эмблемы, поддержку для поведения содержимого и текст фирменной символики \(об авторских правах, и т. д.\).  «Создание фирменной символики пакет» ниже приведены дополнительные сведения о фирменной символике элементов пакета.  В событии отсутствует пакет фирменной символики, связанных с разделом, средство просмотра справки будет использовать резервный фирменной символикой, расположенный в корневой папке приложения средства просмотра справки \(Branding\_en US.mshc\).  
  
 **Требования к статье средства просмотра справки**  
  
 Чтобы правильно отображаться в средстве просмотра справки, необработанные разделе содержимое должно быть W3C основные XHTML 1.1.  
  
 Раздел обычно содержит два раздела:  
  
-   Метаданные \(см. ссылку на метаданные содержимого\): данные по теме, например, уникальный идентификатор раздела, значение ключевого слова, раздел идентификатор Оглавление, родительский идентификатор узла и т. д.  
  
-   Текст содержимого: стандарту W3C основные XHTML 1.1, включающее поддерживается содержимого поведения \(сворачиваемой области, фрагмент кода, и т. д. Полный список приведен ниже\).  
  
 Пакет Visual Studio фирменной символики поддерживаемых элементов управления:  
  
-   Ссылки  
  
-   CodeSnippet  
  
-   CollapsibleArea  
  
-   Если сделать унаследованный член  
  
-   LanguageSpecificText  
  
 Поддерживаемый язык строки \(без учета регистра\):  
  
-   JavaScript  
  
-   c\# и c\#  
  
-   cplusplus или visualc \+\+ или c \+\+  
  
-   JScript  
  
-   VisualBasic или vb  
  
-   f \# или fsharp или fs  
  
-   другой – строка, представляющая имя языка  
  
 **Создание раздела справки**  
  
 Создание нового документа XHTML с именем ContosoTopic4.htm и включить тег заголовка \(см. ниже\).  
  
```html  
<html> <head> <title>Contoso Topic 4</title> </head> <body> </body> </html>  
  
```  
  
 Добавьте данные, чтобы определить, каким образом раздела должна присутствовать \(собственная типизированной или нет\), как для ссылки в этом разделе для F1, где существует этот раздел в Оглавление, его идентификатор \(ссылка по теме\), и т. д.  См. в таблице «Метаданные содержимого» ниже полный список поддерживаемых метаданных.  
  
-   В этом случае мы используем наш собственный пакет с фирменной символикой, вариант пакета с фирменной символикой просмотра справки Visual Studio.  
  
-   Добавление F1 meta имя и значение \(содержимое «Microsoft.Help.F1» \= «ContosoTopic4»\), будет соответствовать заданному F1 в контейнере свойств интегрированной среды разработки.  \(См. в разделе поддержка F1 для получения дополнительной информации\).   Это значение, которое соответствует F1 вызов из интегрированной среды разработки для отображения в этом разделе, при выборе F1 в Интегрированной среде разработки.  
  
-   Добавьте идентификатор раздела. Это строка, используемый для ссылки на этот раздел справки по другим темам.  Это идентификатор средство просмотра справки по этой теме.  
  
-   Для Оглавления добавьте родительский узел в этом разделе, чтобы определить, где будет отображаться узел содержание этого раздела.  
  
-   Для Оглавления добавьте порядок узлов в этом разделе. Если n количество дочерних узлов родительского узла, определение в порядке дочерних узлов места в этом разделе. Например, этот раздел является номер 4 из 4 дочерних разделов.\)  
  
 Пример метаданных:  
  
```html  
<html> <head> <title>Contoso Topic 4</title> <meta name="SelfBranded" content="false" /> <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" /> <meta name="Microsoft.Help.Id" content="ContosoTopic4" /> <meta name="Microsoft.Help.F1" content=" ContosoTopic4" /> <meta name="Language" content="en-us" /> <meta name="Microsoft.Help.TocParent" content="ContosoTopic0" /> <meta name="Microsoft.Help.TocOrder" content="4" /> </head> <body> </body> </html>  
  
```  
  
 **Текст раздела**  
  
 Текст \(не включая заголовок и нижний колонтитул\) раздела будет содержать ссылки на страницы, раздел примечание, сворачиваемой области, фрагмент кода и фрагмент текста определенного языка.  Сведения об этих областях представленных раздела в разделе фирменной символики.  
  
1.  Добавление тега заголовка раздела:  `<div class="title">Contoso Topic 4</div>`  
  
2.  Добавьте раздел Примечание: `<div class="alert"> add your table tag and text </div>`  
  
3.  Добавьте сворачиваемой области:  `<CollapsibleArea Expanded="1" Title="Collapsible Area Test Heading"> add text  </CollapsibleArea>`  
  
4.  Добавьте фрагмент кода:  `<CodeSnippet EnableCopyCode="true" Language="CSharp" ContainsMarkup="false" DisplayLanguage="C#" > a block of code </CodeSnippet>`  
  
5.  Добавьте код языка, определенный текст:  `<LanguageSpecificText devLangcs="CS" devLangvb="VB" devLangcpp="C++" devLangnu="F#" />` Обратите внимание, что devLangnu \= позволяет вводить других языков. Например, devLangnu \= «Fortran» будет отображаться Fortran при фрагмента кода DisplayLanguage \= Fortran  
  
6.  Добавьте ссылки на страницы: `<a href="ms-xhelp://?Id=ContosoTopic1">Main Topic</a>`  
  
> [!NOTE]
>  Примечание: для неподдерживаемых новый «язык» \(например, F \#, Cobol, Fortran\) выделение кода цветом в фрагменте кода будет монохромный.

 **Поддержка F1**  
  
 В Visual Studio нажатия клавиши F1 создает значения, полученные из положение курсора в Интегрированной среде разработки и заполняет «контейнер свойств» с предоставленными значениями \(в зависимости от текущего положения курсора. При прохождении курсора над компонентом x, функция x — активный или в фокус и заполняет контейнер свойств со значениями.  При выборе F1 заполняется контейнер свойств, и Visual Studio F1 код выглядит ли источник справки по умолчанию клиенты локальной или Интернет \(online используется по умолчанию\), затем создает соответствующую строку, в зависимости от пользователей, выполняющих установку \(online используется по умолчанию\) — оболочки \(в разделе Справка администратора руководство для EXE\-файла параметров запуска\) с параметрами для локальной справки \+ ключевые слова из контейнера свойств, если по умолчанию используется локальная справка , или URL\-адрес MSDN с помощью ключевого слова в списке параметров.  
  
 Если для F1 возвращаются три строки, называют как строка Многозначный воспользоваться первым условием, смотрите на попадание и если найден, завершена; в противном случае переход к следующей строке.  Порядок имеет значение. Представления ключевых слов Многозначный должно быть длинной строки короткой строкой.  Чтобы проверить это, в случае для нескольких ключевых слов, посмотрите на строке online F1 URL\-адреса, включающие выбранного ключевого слова.  
  
 В Visual Studio 2012 мы специально сделано сильнее деления между оперативным и автономным, так что если параметр пользователя для сети, затем мы просто передается запрос F1 нашей службе online запроса на MSDN, вместо маршрутизации через агент библиотек справки, который мы использовали в Visual Studio 2010. Затем мы полагаемся на состояние «установки поставщика содержимого \= true» для определения необходимости выполнение определенных действий в этом контексте. Если значение равно true, выполните эту логику синтаксического анализа и маршрутизации в зависимости от того, требуется поддержка для заказчиков. Если значение равно false, затем мы перейдем к MSDN. Если локальный параметр пользователя, все вызовы просто перейдите к механизм локальной справки.  
  
 F1 схема потока:  
  
 ![F1 flow](../../extensibility/internals/media/f1flow.png "F1flow")  
  
 Если источник содержимого справки средства просмотра справки по умолчанию задано значение online \(запуск в браузере\):  
  
-   Возможности Visual Studio партнера \(VSP\) выдавать значение в наборе свойств F1 \(prefix.keyword свойство контейнера и online URL\-адрес в качестве префикса в реестре\): F1 отправляет URL\-адрес VSP \+ параметров браузера.  
  
-   Возможности Visual Studio \(редактор языка, элементы меню Visual Studio, и т.д.\): F1 браузер отправляет URL\-адрес Visual Studio.  
  
 Если источник содержимого справки средства просмотра справки по умолчанию задано значение локальной справки \(запуск в средстве просмотра справки\):  
  
-   Функции VSP, где ключевое слово F1 контейнер свойств и совпадают индекс локального хранилища \(то есть контейнер prefix.keyword свойство \= значение, найденное в локальном хранилище индекса\): F1 отображает раздел в средстве просмотра справки.  
  
-   Возможности Visual Studio \(не параметр VSP переопределение контейнер свойств, извлекаемыми из возможностей Visual Studio\): F1 отображает раздел Visual Studio, в средстве просмотра справки.  
  
 Установите следующие значения реестра для включения для поставщиков содержимого справки F1 резервной. F1 резервной означает, что средстве просмотра справки для поиска содержимого справки F1 через Интернет, и поставщика содержимого устанавливается локально на жестком диске пользователя. Средство просмотра справки следует обратить внимание на локальную справку для содержимого, несмотря на то, что значение по умолчанию — для справки в Интернете.  
  
1.  Задайте **VendorContent** значение раздела реестра 2.1 справки:  
  
    -   Для 32\-разрядных операционных системах:  
  
         HKEY\_LOCAL\_MACHINE\\SOFTWARE\\Microsoft\\Help\\v2.1\\Catalogs\\VisualStudio12  
  
         «VendorContent» \= DWORD: 00000001  
  
    -   Для 64\-разрядных операционных системах:  
  
         HKEY\_LOCAL\_MACHINE\\SOFTWARE\\Wow6432Node\\Microsoft\\Help\\v2.1\\Catalogs\\VisualStudio12  
  
         «VendorContent» \= DWORD: 00000001  
  
2.  Регистрация имен партнеров помогут 2.1 с разделом реестра:  
  
    -   Для 32\-разрядных операционных системах:  
  
         HKEY\_LOCAL\_MACHINE\\SOFTWARE\\Microsoft\\Help\\v2.1\\Partner*\\ \< имен \>*  
  
         «расположение» \= «вне сети»  
  
    -   Для 64\-разрядных операционных системах:  
  
         HKEY\_LOCAL\_MACHINE\\SOFTWARE\\Wow6432Node\\Microsoft\\Help\\v2.1\\Partner*\\ \< имен \>*  
  
         «расположение» \= «вне сети»  
  
 **Синтаксический анализ базового собственного пространства имен**  
  
 Чтобы включить анализ базового собственного пространства имен, в реестре добавьте новый параметр DWORD с именем: BaseNativeNamespaces и присвойте ему значение 1 \(в разделе каталога, которые требуется поддерживать\).  Например если вы хотите использовать каталог Visual Studio, можно добавить ключ к пути:  
  
 HKEY\_LOCAL\_MACHINE\\SOFTWARE\\Wow6432Node\\Microsoft\\Help\\v2.1\\Catalogs\\VisualStudio12  
  
 Если ключевое слово F1 в формате, который обнаруживается заголовок или МЕТОДА, символ "\/" будет анализироваться, приведет к следующей конструкции:  
  
-   Заголовок: будет содержать пространство имен, который может использоваться для регистрации в реестре  
  
-   МЕТОД: это станет ключевое слово, которое передается через.  
  
 Например, если пользовательская библиотека называется CustomLibrary и метод с именем MyTestMethod, когда запрос поступает в ней F1 будут отформатированы как `CustomLibrary или MyTestMethod`.  
  
 Пользователь затем можно зарегистрировать CustomLibrary как пространство имен куста партнеров и предоставить ключ независимо от расположения, они хотят иметь и ключевое слово, передаваемый запрос будет MyTestMethod.  
  
 **Включение инструмента в Интегрированной среде отладки справки**  
  
 Добавьте следующий раздел реестра и значение:  
  
 Клавиша справки HKEY\_CURRENT\_USER\\Software\\Microsoft\\VisualStudio\\12.0\\Dynamic: отображение вывода данных отладки в розничной продаже: Да  
  
 В Интегрированной среде разработки, в пункте меню Справка выберите «Отладка контекст справки»  
  
 **Метаданные содержимого**  
  
 В следующей таблице любая строка, которая отображается в квадратных скобках — заполнитель, должен быть заменен распознаваемое значение. Например, в \< meta name\="Microsoft.Help.Locale» содержимое \= «\[код языка\]» или \>, «\[код языка\]» должен быть заменен значение например» en\-нам».  
  
|Свойства \(представление HTML\)|Описание|  
|-------------------------------------|--------------|  
|\< meta name\="Microsoft.Help.Locale «содержимое «\[язык code\] \= и \>|Задает языковой стандарт для этого раздела. Если этот тег используется в разделе, она должна использоваться только один раз и должен быть вставлен над теги справки \(Microsoft\). Если этот тег не используется, основной текст раздела индексируется с помощью средство разбиения по словам, связанный с языковой стандарт продукта, если он указан; в противном случае — en\-us используется средство разбиения по словам. Этот тег соответствует ISOC RFC 4646. Чтобы обеспечить корректную работу справки \(Microsoft\), это свойство можно используйте вместо общие атрибут языка.|  
|\< meta name\="Microsoft.Help.TopicLocale «содержимое «\[язык code\] \= и \>|Задает языковой стандарт для этого раздела, когда также используются в других языковых стандартов. Если этот тег используется в разделе, она должна использоваться только один раз. Используйте этот тег, если каталог содержит содержимое в более чем один язык. Несколько разделов в каталоге могут иметь тот же идентификатор, но каждый необходимо указать уникальный TopicLocale. ### Создание фирменной настройки пакета  
 В выпуске Visual Studio включает в себя ряд различных продуктов Visual Studio, включая изолированный и интегрированный оболочек для партнеров Visual Studio.  Каждый из этих продуктов требует некоторой степени раздела справки содержимого на основе фирменной символики поддержки, уникальные для продукта.  Например разделы Visual Studio требуется презентации согласованного торговой марки, тогда как SQL Studio, который является оболочкой оболочки ISO, требуется свой собственный уникальный справки содержимого фирменной символики для каждого раздела.  Партнера интегрированной оболочки можно их разделы справки находится в пределах родительского содержимого справки Visual Studio продукта сохраняя собственные статьи фирменной символики.  
  
 Пакеты устанавливаются на продукт, содержащий средства просмотра справки.  Для продуктов Visual Studio:  
  
-   Резервный пакет с фирменной символикой \(Branding\_ \< язык \> выполнения\) устанавливается в корне приложения 2.1 средства просмотра справки \(пример: C:\\Program Files \(x 86\) \\Microsoft Help Viewer\\v2.1\), языковой пакет средства просмотра справки.  Это используется в случаях, где не установлен продукт, пакет фирменной символики \(содержимое не установлен\) или где фирменной символики установленный пакет поврежден.  Обратите внимание, что элементы Visual Studio \(эмблема и отзыв\) игнорируются при использовании резервной корневого приложения, пакет фирменной символики.  
  
-   При установке Visual Studio содержимое из службы содержимого пакета \(для первого сценария установки содержимого времени\) также устанавливается пакет с фирменной символикой.  В случае обновления для фирменной настройки пакета в случае следующего обновления содержимого или дополнительный пакет установки действия установлено обновление.  
  
 Средство просмотра справки Microsoft поддерживает фирменной символики разделы, основываясь на раздел метаданных.  
  
-   Где раздел метаданных определяет self типизированной \= true, отображения раздела как есть, не выполняют никаких действий \(вплоть до фирменной символики\).  
  
-   Где раздел метаданных определяет self типизированной \= false, используйте пакет фирменной символики, связанный с TopicVendor значение метаданных.  
  
-   Где раздел метаданных определяет name\="Microsoft.Help.TopicVendor» содержимое \= \< фирменной символики имя пакета в поставщика MSHA \>, использовать пакет фирменной настройки, определенные в значение содержимого.  
  
-   Обратите внимание, что в каталоге Visual Studio приложении приоритета пакетов фирменной символики.  Первый Visual Studio фирменная символика по умолчанию применяется, а затем, если определено в разделе метаданных, а также поддерживает связанные фирменной символики пакета \(как определено в msha установки\), определяется поставщиком фирменной символики применяется переопределяет метод.  
  
 Фирменной символики элементы обычно делятся на три основные категории:  
  
-   Элементы заголовка \(например, ссылка обратной связи, текст заявления об отказе условного, эмблема\)  
  
-   Содержимое поведения \(Примерами могут служить текстовых элементов управления и свертывания и элементами фрагмент кода\)  
  
-   Нижний колонтитул элементы \(например, об авторских правах\)  
  
 Считать типизированные элементы включают в себя элементы \(описанные в этой спецификации\):  
  
-   Эмблема продукта или каталога \(например, Visual Studio\)  
  
-   Отзыв элементов ссылки и адрес электронной почты  
  
-   Текст заявления об отказе  
  
-   Текст об авторских правах  
  
 Вспомогательные файлы в пакет фирменной символики просмотра справки Visual Studio включают:  
  
-   Графики \(логотипы, значков, и т. д.\)  
  
-   Branding.js — такие вспомогательные файлы сценария  
  
-   Branding.XML — строки, которые всегда используется по содержимому каталога.  Примечание: для Visual Studio локализации текста в branding.xml, к ним относятся элементы \_locID \= «\< уникальное значение \>»  
  
-   Branding.CSS — определения стиля для согласованности презентации  
  
-   Printing.CSS — определения стиля для согласованного представления печати  
  
 Как отмечалось выше, пакеты фирменной символики связаны с раздела:  
  
-   Если SelfBranded \= false, определено в метаданных, раздел наследует каталога фирменной настройки пакета  
  
-   Или если SelfBranded \= false и существует уникальный пакет фирменной символики определяется в MSHA и доступны при установке содержимого  
  
 Для реализации пользовательских пакетов фирменной символики VSP \(VSP содержимое, SelfBranded \= True\), один способ — начать с фирменной символикой резервной \(устанавливается с помощью средства просмотра справки\) и измените имя файла соответствующим образом.  Branding\_ \< язык \> выполнения он ZIP\-файл с расширением изменен для выполнения, поэтому просто измените расширение с выполнения на ZIP и извлеките его содержимое.  Ниже приведены пакета элементов фирменной настройки и соответствующим образом изменить \(например, изменение логотипа эмблема VSP и ссылку на логотип в файле Branding.xml, обновляются Branding.xml в особенности VSP и т. д.\).  
  
 После этого все изменения, создание ZIP\-файл, содержащий нужные элементы фирменной символики и измените расширение на выполнения.  
  
 Чтобы связать пользовательский пакет фирменной символики, создайте MSHA, который содержит ссылку на файл фирменной символики mshc вместе с содержимым mshc \(содержащий разделы\).  См. ниже «MSHA» для создания основных MSHA.  
  
 Файл Branding.xml содержит список элементов, используемых для постоянно визуализации конкретных элементов в разделе, когда в этом разделе содержатся \< meta name\="Microsoft.Help.SelfBranded» содержимое \="false"\/ \>.  Visual Studio список элементов в файле Branding.xml перечислены ниже.  Обратите внимание, что этот список предназначен для использования в качестве шаблона для внедрений оболочки ISO, где они изменять эти элементы \(например эмблемы, отзывы и авторских правах\) в соответствии с их собственных продуктов потребностям фирменной символики.

### Развертывание набора разделов  
 Это очень простой и быстрый учебник по созданию развертывания содержимого справки задать содержать MSHA\-файл и набор CAB\-файлов или MSHC, содержащий темы. MSHA является XML\-файл, описывающий набор CAB\-файлы или файлы MSHC. Средство просмотра справки может считывать MSHA для получения списка содержимого \(. CAB\-ФАЙЛ или. Файлы MSHC\) для локальной установки.  
  
 Это только учебник описанием очень простой XML\-схемы для MSHA средство просмотра справки.  Обратите внимание, что пример реализации ниже Этот краткий обзор и примеры HelpContentSetup.msha.  
  
 Имя MSHA, этот учебник, в целях — HelpContentSetup.msha \(имя файла может быть что\-либо, с расширением. MSHA\). HelpContentSetup.msha \(пример ниже\) должен содержать список CAB\-файлов или MSHCs доступны.  Обратите внимание, что тип файла должно согласовываться в MSHA \(не поддерживает сочетание типов MSHA\- и CAB\-файлов\). Для каждого CAB или MSHC должно быть \< div класса \= «пакет» \>... \< \/ div \> \(см. пример ниже\).  
  
 Примечание: в следующем примере реализация приведены фирменной символикой. Это крайне важно для включения, чтобы получить необходимые элементы визуализация содержимого Visual Studio и содержимого поведения.  
  
 Образец файла HelpContentSetup.msha: \(Замените» содержимого имя набора 1» и «содержимого набор 2» и т.д. имя имен файлов.\)  
  
```  
<html> <head /> <body class="vendor-book"> <div class="details"> <span class="vendor">Your Company</span> <span class="locale">en-us</span> <span class="product">Your Company Help Content</span> <span class="name">Your Company Help Content</span> </div> <div class="package-list"> <div class="package"> <span class="name">Your_Company _Content_Set_1</span> <span class="deployed">True</span> <a class="current-link" href="Your_Company _Content_Set_1.mshc">Your_Company _Content_Set_1.mshc </a> </div> <div class="package"> <span class="name">Your_Company _Content_Set_2</span> <span class="deployed">True</span> <a class="current-link"href=" Your_Company _Content_Set_2.mshc "> Your_Company _Content_Set_2.mshc </a> </div>.  
  
```  
  
1.  Создайте локальную папку, например «C:\\SampleContent»  
  
2.  В этом примере мы используем MSHC файлы содержать разделы.  MSHC является zip с расширением изменилось с расширением ZIP для. MSHC.  
  
3.  Создайте ниже HelpContentSetup.msha как текстовый файл \(«Блокнот» использовалась для создания файла\) и сохраните его в папку отмечено выше \(см. шаг 1\).  
  
 Обратите внимание, что класс «Фирменной символики» существует и является уникальным. Mshc фирменной символики включается в этот учебник, чтобы установленное содержимое будет иметь фирменную символику и содержимого поведения, которые содержатся в MSHCs будет иметь поддержки элементов, содержащихся в пакете фирменной символики. Без этого возникнут ошибки, когда система выполняет поиск поддержки элементов, которые не являются частью дорожек \(установлен\) содержимого.  
  
 Чтобы получить Visual Studio, пакет фирменной символики, скопируйте файл Branding\_en US.mshc в \\Microsoft Help Viewer\\v2.1\\ C:\\Program Files \(x 86\) в рабочую папку.  
  
```html  
<html> <head /> <body class="vendor-book"> <div class="details"> <span class="vendor">Your Company</span> <span class="locale">en-us</span> <span class="product">Your Company Help Content</span> <span class="name">Your Company Help Content</span> </div> <div class="package-list"> <div class="package"> <span class="name">Your_Company _Content_Set_1</span> <span class="deployed">True</span> <a class="current-link" href="Your_Company _Content_Set_1.mshc">Your_Company _Content_Set_1.mshc </a> </div> <div class="package"> <span class="name">Your_Company _Content_Set_2</span> <span class="deployed">True</span> <a class="current-link"href=" Your_Company _Content_Set_2.mshc "> Your_Company _Content_Set_2.mshc </a> </div> <div class="package"> <span class="packageType">branding</span> <span class="name">Branding_en-US</span> <span class="deployed">True</span> <a class="current-link" href="Branding_en-US.mshc">Branding_en-US.mshc</a> </div> </div> </body> </html>
```  

### Добавление справки для оболочки Visual Studio \(интегрированный и изолированный режим\)  
 **Вступление**  
  
 В этом пошаговом руководстве показано, как включить содержимое справки в приложение оболочки Visual Studio, а затем развернуть его.  
  
 **Требования**  
  
1.  [!INCLUDE[vs_dev12](../../data-tools/includes/vs_dev12_md.md)]  
  
2.  [Visual Studio 2013 изолированной оболочки Redist](http://www.microsoft.com/visualstudio/11/downloads#vs-shell)  
  
 **Обзор**  
  
 [!INCLUDE[vs_dev12](../../data-tools/includes/vs_dev12_md.md)] Оболочки — это версия [!INCLUDE[vs_dev12](../../data-tools/includes/vs_dev12_md.md)] интегрированной среды разработки, в которой можно построить приложение. Такие приложения содержат изолированной оболочки вместе с расширения, которые можно создать. Использовать шаблоны проектов изолированной оболочки, которые входят в состав [!INCLUDE[vs_dev12](../../data-tools/includes/vs_dev12_md.md)] пакета SDK для создания расширений.  
  
 Основные шаги для создания приложения на основе изолированной оболочки и ее справку:  
  
1.  Получить [!INCLUDE[vs_dev12](../../data-tools/includes/vs_dev12_md.md)] распространяемый ISO оболочки \(загрузки Майкрософт\).  
  
2.  В Visual Studio создайте расширение справки на основе изолированной оболочки, например, модуль Contoso справки, который описан далее в этом пошаговом руководстве.  
  
3.  Включить расширения и оболочки ISO, распространяемый в развертывание MSI \(Настройка приложения\). В этом пошаговом руководстве не включает этап настройки.  
  
 Создание хранилища содержимого Visual Studio. Сценарии интегрированной оболочки измените Visual Studio12 к имени каталога продуктов следующим образом:  
  
-   Создайте папку C:\\ProgramData\\Microsoft\\HelpLibrary2\\Catalogs\\VisualStudio12.  
  
-   Создайте файл с именем CatalogType.xml и добавьте его в папку. Файл должен содержать следующие строки кода:  
  
    ```  
    <?xml version="1.0" encoding="UTF-8"?> <catalogType>UserManaged</catalogType>  
    ```  
  
 Определения содержимого хранилища в реестре. Для интегрированной оболочки измените имя каталога продуктов VisualStudio12:  
  
-   HKLM\\SOFTWARE\\Wow6432Node\\Microsoft\\Help\\v2.1\\Catalogs\\VisualStudio12  
  
     Ключ: LocationPath строковое значение: C:\\ProgramData\\Microsoft\\HelpLibrary2\\Catalogs\\VisualStudio12\\  
  
-   HKLM\\SOFTWARE\\Wow6432Node\\Microsoft\\Help\\v2.1\\Catalogs\\VisualStudio12\\en\-US  
  
     Ключа: Значение строка CatalogName: [!INCLUDE[vs_dev12](../../data-tools/includes/vs_dev12_md.md)] документации  
  
 **Создание проекта**  
  
 Чтобы создать расширение изолированной оболочки.  
  
1.  В Visual Studio в разделе **файл**, выберите **Новый проект**, в разделе **другие типы проектов** выберите **расширения**, и нажмите кнопку  **изолированной оболочки Visual Studio**. Назовите проект `ContosoHelpShell`\) для создания на основе изолированной оболочки Visual Studio шаблона проекта расширяемости.  
  
2.  В обозревателе решений откройте ApplicationCommands.vsct в проекте ContosoHelpShellUI в папке файлов ресурсов. Убедитесь, что эта строка закомментирован \(поиск «No\_Help»\): `<!-- <define name=“No_HelpMenuCommands”/> -->`  
  
3.  Нажмите клавишу F5, чтобы скомпилировать и запустить **отладки**. В экспериментальном экземпляре Интегрированной среды разработки изолированной оболочки, выберите **помочь** меню. Убедитесь, что **Просмотр справки**, **Добавление и удаление содержимого справки**, и **задать параметры справки** появляются следующие команды.  
  
4.  В обозревателе решений откройте ContosoHelpShell.pkgdef в проекте ContosHelpShell в папке Настройка оболочки. Чтобы определить каталог Contoso справки, добавьте следующие строки:  
  
    ```  
    [$RootKey$\Help] "Product"="Contoso" "Catalog"="Contoso" “Version"="100" "BrandingPackage"="ContosoBrandingPackage.mshc"  
    ```  
  
5.  В обозревателе решений откройте ContosoHelpShell.Application.pkgdef в проекте ContosHelpShell в папке Настройка оболочки. Чтобы включить справки F1, добавьте следующие строки:  
  
    ```  
    // F1 Help Provider [$RootKey$\HelpProviders\{C99BDC23-FF29-46bf-9658-ADD634CCAED8}] "Name"="13407" "Package"="{DA9FB551-C724-11d0-AE1F-00A0C90FFFC3}" @="Help3 Provider" [$RootKey$\HelpProviders] @="{C99BDC23-FF29-46bf-9658-ADD634CCAED8}" [$RootKey$\Services\{C99BDC23-FF29-46bf-9658-ADD634CCAED8}] "Name"="Help3 Provider" @="{4A791146-19E4-11D3-B86B-00C04F79F802}"  
    ```  
  
6.  В обозревателе решений в контекстном меню решения ContosoHelpShell выберите **Свойства** элемент меню. В разделе **Свойства конфигурации**, выберите **Configuration Manager**. В **Конфигурация** столбец, измените значения каждого «Отладка» на «Освободить».  
  
7.  Постройте решение. Это создает набор файлов в папке выпуска, которая будет использоваться в следующем разделе.  
  
 Чтобы проверить это, как если бы развертывания:  
  
1.  На компьютере выполняется развертывание Contoso, чтобы установить загруженный ISO оболочки \(выше\).  
  
2.  Создание папки в \\\\Program Files \(x 86\) \\ и назовите его `Contoso`.  
  
3.  Скопируйте содержимое из папки выпуска ContosoHelpShell \\Contoso\\ папку \\\\Program Files \(x 86\).  
  
4.  Запустите редактор реестра, выбрав  **запуска** в **запустить** меню и введите `Regedit`. В редакторе реестра выберите **файл**, а затем **импорта**. Перейдите в папку проекта ContosoHelpShell. В подпапке ContosoHelpShell выберите ContosoHelpShell.reg.  
  
5.  Создайте хранилище содержимого:  
  
     Для оболочки ISO \- создайте хранилище содержимого Contoso C:\\ProgramData\\Microsoft\\HelpLibrary2\\Catalogs\\ContosoDev12  
  
     Для [!INCLUDE[vs_dev12](../../data-tools/includes/vs_dev12_md.md)] интегрированной оболочки, создайте папку C:\\ProgramData\\Microsoft\\HelpLibrary2\\Catalogs\\VisualStudio12  
  
6.  Создайте CatalogType.xml и добавьте хранилище содержимого \(в предыдущем шаге\), содержащую:  
  
    ```  
    <?xml version="1.0" encoding="UTF-8"?> <catalogType>UserManaged</catalogType>  
    ```  
  
7.  Добавьте следующие разделы реестра:  
  
     HKLM\\SOFTWARE\\Wow6432Node\\Microsoft\\Help\\v2.1\\Catalogs\\VisualStudio12Key: LocationPath строковое значение.  
  
     Для оболочки ISO:  
  
     C:\\ProgramData\\Microsoft\\HelpLibrary2\\Catalogs\\VisualStudio12\\  
  
     [!INCLUDE[vs_dev12](../../data-tools/includes/vs_dev12_md.md)] Интегрированная оболочка:  
  
     C:\\ProgramData\\Microsoft\\HelpLibrary2\\Catalogs\\VisualStudio12\\\\en\-US  
  
     Ключа: Значение строка CatalogName: [!INCLUDE[vs_dev12](../../data-tools/includes/vs_dev12_md.md)] документации. Для оболочки ISO это имя вашего каталога.  
  
8.  Скопируйте содержимое \(CAB\-файлов или MSHC и MSHA\) в локальную папку.  
  
9. Пример интегрированная оболочка командной строки для тестирования локального хранилища содержимого. Для оболочки ISO измените значения каталога и launchingApp в зависимости от соответствует продукта.  
  
     Метод \/helpQuery\/CatalogName VisualStudio12 «C:\\Program Files \(x 86\) \\Microsoft Help Viewer\\v2.1\\HlpViewer.exe» \= «страница & id \= ContosoTopic0» \/launchingApp Microsoft VisualStudio, 12.0  
  
10. Запустите приложение Contoso \(от корня приложения Contoso\). В оболочке ISO выберите **помочь** пункт меню, а также изменение **задать параметры справки** для **Использование локальной справки**.  
  
11. В оболочке, выберите **помочь** элемент меню, затем **Просмотр справки**. Локальное средство просмотра справки должен быть запущен. Откройте вкладку **Управление содержимым**. В разделе **источник установки**, выберите **диск** переключатель. Выберите **...** кнопку и перейдите к локальной папке с содержимым Contoso \(копируется в локальную папку на предыдущем шаге\). Выберите HelpContentSetup.msha. Contoso должны отображаться как книгу в книге выбранные элементы. Выберите **Добавить**, а затем выберите **обновления** кнопки \(нижнем правом углу\).  
  
12. В Интегрированной среде Contoso нажмите клавишу F1 для проверки функциональности F1.
