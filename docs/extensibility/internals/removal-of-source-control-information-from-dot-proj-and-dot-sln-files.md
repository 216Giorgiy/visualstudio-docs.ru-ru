# Удаление сведения системы управления версиями из. Proj и. SLN-файлы
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

В версии 1.2 API системы управления версиями подключаемого модуля SCC данные хранятся в файле MSSCCPRJ.SCC.  Преимущество файла MSSCCPRJ.SCC, что данные SCC не является источником \- контролируемого, например его в файлах .proj и sln.  
  
## Изменения версии 1.2  
 В подключаемых модулей системы управления версиями, которые основаны на версии 1.1 API системы управления версиями вставляемой, сведения о системе управления версиями хранится в файлах проекта \(.proj\) и решения \(sln\).  Расположение базы данных чонтрольной сведения о системе управления версиями, задано AuxPath и конкретное расположение в базе данных определяется ProjName.  Эта функциональности может вызвать проблемы после ветвления, разветвить или скопировать операции, поскольку ProjName обычно будет недопустимым после любой из этих операций.  
  
 В версии 1.1 API системы управления версиями, вставляемой в интегрированной среде разработки использовать файлы ~SAK для обнаружения если подключаемый модуль поддерживает метод MSSCCPRJ.SCC хранения чонтрольная сведения о системе управления версиями.  Версия 1,2 API системы управления версиями подключаемых модулей предоставляет новую возможность обнаружения поддержка файла MSSCCPRJ.SCC без использования файла ~SAK.  Дополнительные сведения см. в разделе [Устранение ~ SAK файлов](../../extensibility/internals/elimination-of-tilde-sak-files.md).
