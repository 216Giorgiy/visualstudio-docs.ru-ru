# <a name="strings-element"></a>Элемент строки
Элемент строки должен содержать по крайней мере **ButtonText** дочерний элемент. Все дочерние элементы являются необязательными. Недопустимый XML символы, такие как «&» и "<" должны быть реализованы как сущности ("&amp;«и»&lt;" и т. д).  
  
 Знак амперсанда в текстовой строке задает сочетание клавиш для команды.  
  
## <a name="syntax"></a>Синтаксис  
  
```  
<Strings>  
  <ButtonText>... </ButtonText>  
  <CommandName>... </CommandName>  
</Strings>  
```  
  
## <a name="attributes-and-elements"></a>Атрибуты и элементы  
 В следующих разделах описаны атрибуты, дочерние и родительские элементы.

### <a name="child-elements"></a>Дочерние элементы  
  
|Элемент|Описание|  
|-------------|-----------------|  
|ButtonText|Это поле и пять следующие текстовые поля в определение команды позволяют указать текст, отображаемый в разных меню. По умолчанию `ButtonText` поле отображается в контроллеры меню. `ButtonText` Поле становится значением по умолчанию, если другие текстовые поля остаются пустыми. `ButtonText` Поля не может быть пустым, даже если указаны другие текстовые поля.|  
|ToolTipText|`ToolTipText` Поле указывает текст, отображаемый в подсказке для элемента меню.<br /><br /> Если `ToolTipText` поле не заполнено, `ButtonText` используется поле.|  
|MenuText|`MenuText` Поле текст, который отображается для команды, если в главном меню, панели инструментов, в контекстном меню или подменю. Если `MenuText` поле не заполнено, интегрированной среды разработки (IDE) использует `ButtonText` поля. `MenuText` Поле может также использоваться для локализации.<br /><br /> Для контекстного меню `MenuText` поля является имя, которое отображается в панель инструментов контекстного меню, который позволяет настраивать контекстные меню в Интегрированной среде разработки. Таким образом быть конкретному на имя вашего контекстного меню; Например используйте «Мини-приложения пакета контекстное меню» вместо «Ярлык».<br /><br /> Если `MenuText` поле не указано, `ButtonText` используется поле.|  
|CommandName|`CommandName` Поле указывает текст, отображаемый в категории «клавиатура» **команды** вкладку в **Настройка** диалоговое окно (выбрав **Настройка**на **средства** меню).|  
|canonicalName|Английский `CanonicalName` поле указывает имя команды в текст на английском языке, который может быть введен в **команда** окна для выполнения элемента меню. Удаляет все символы, которые не являются буквы, цифры, подчеркивания или внедренные точки интегрированной среды разработки. Затем этот текст добавляется к `ButtonText` поля для определения команды. Например **новый проект** на **файл** меню становится команда File.NewProject.<br /><br /> Если Английский `CanonicalName` поле не указано, интегрированная среда разработки использует `ButtonText` поля и полос все, за исключением буквы, цифры, знаки подчеркивания и внедренных точек. Например, на кнопке отображается надпись «& Определение команды...» становится DefineCommands, где удалить амперсанд, пробел и кнопку с многоточием.<br /><br /> Если `TextChanges` установлен флаг и изменяется текст команды, соответствующей команды, распознаваемые **команда** окна не приводит к изменению, он остается в канонической форме исходного `ButtonText` или Английский`CanonicalName` поля.|  
|LocCanonicalName|`LocCanonicalName` Поле работает аналогично Английский `CanonicalName` поля, но позволяет текст локализованного команды должен быть задан. Можно указать оба канонические поля. Поскольку интегрированной среды разработки выполняет только синтаксический анализ текста, введенного в **команда** окна и связывает его с помощью команды, английском и отличными от английского текста могут быть связаны с той же команде.|
