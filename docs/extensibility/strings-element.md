# <a name="strings-element"></a>Элемент строки
Элемент строки должен содержать по крайней мере **ButtonText** дочерний элемент. Все дочерние элементы являются необязательными. Недопустимый формат XML символы, такие как «&» и "<’ must="" be="" coded="" as="" entities=""></’>&amp;«и»&lt;" и так далее).  
  
 Знак амперсанда в текстовой строке задает сочетание клавиш для команды.  
  
## <a name="syntax"></a>Синтаксис  
  
```  
<Strings>  
  <ButtonText>... </ButtonText>  
  <CommandName>... </CommandName>  
</Strings>  
```  
  
## <a name="attributes-and-elements"></a>Атрибуты и элементы  
 В следующих разделах описаны атрибуты, дочерние и родительские элементы.  
  
### <a name="attributes"></a>Атрибуты  
  
|Атрибут|Описание|  
|---------------|-----------------|  
|язык|Необязательный. Language = «.».|  
  
### <a name="child-elements"></a>Дочерние элементы  
  
|Элемент|Описание|  
|-------------|-----------------|  
|ButtonText|Это поле и пять следующие текстовые поля в определении команды позволяют указать текст, отображаемый в разных меню. По умолчанию `ButtonText` поле отображается в меню контроллеров. `ButtonText` Поле становится значением по умолчанию, если другие текстовые поля остаются пустыми. `ButtonText` Поля не может быть пустым, даже если указаны другие текстовые поля.|  
|ToolTipText|`ToolTipText` Поле указывает текст, отображаемый в подсказке для элемента меню.<br /><br /> Если `ToolTipText` поле не заполнено, `ButtonText` используется поле.|  
|MenuText|`MenuText` Поле текст, если в главном меню, панели инструментов, в контекстном меню или в подменю отображается для команды. Если `MenuText` поле не заполнено, интегрированную среду разработки (IDE) использует `ButtonText` поле. `MenuText` Также можно использовать поле для локализации.<br /><br /> Для контекстного меню `MenuText` указано имя, отображаемое в панель инструментов контекстного меню, который позволяет настраивать контекстные меню в Интегрированной среде разработки. Таким образом четко имя меню; Например можно используйте «Мини-приложения пакета контекстное меню» вместо «Ярлык».<br /><br /> Если `MenuText` поле не указано, `ButtonText` используется поле.|  
|CommandName|`CommandName` Поле указывает текст, который отображается в категории «клавиатура» **команды** вкладку в **Настройка** диалоговое окно (выбрав **Настройка** в **средства** меню).|  
|CanonicalName|Английский `CanonicalName` поле указывает имя команды в текст на английском языке, который может быть введен в **команда** окна для выполнения элемента меню. Удаляет все символы, которые не являются буквы, цифры, подчеркивания или внедренные периоды интегрированной среды разработки. Затем этот текст добавляется к `ButtonText` поле для определения команды. Например **новый проект** на **файл** меню становится команде File.NewProject.<br /><br /> Если Английский `CanonicalName` поле не указано, используется IDE `ButtonText` полей и ленты все кроме буквы, цифры, подчеркивания и embedded периодов. Например, на кнопке отображается надпись «& определение команды...» становится DefineCommands, где удаляются амперсанд, пробел и кнопку с многоточием.<br /><br /> Если `TextChanges` указан флаг и изменяется текст команды, соответствующей команды, распознаваемые **команда** окна не изменяется; он остается в канонической форме исходного `ButtonText` или английского `CanonicalName` полей.|  
|LocCanonicalName|`LocCanonicalName` Поле идентична Английский `CanonicalName` поля, но позволяет текст локализованного команды должен быть задан. Можно указать оба канонические поля. Поскольку интегрированной среды разработки анализирует только текст, введенный в **команда** окна и связывает его с помощью команды, отличном от английского текста и английский можно связать с той же команды.|
