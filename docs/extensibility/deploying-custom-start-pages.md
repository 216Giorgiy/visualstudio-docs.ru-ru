# <a name="deploying-custom-start-pages"></a>Развертывание пользовательской начальной страницы
Настраиваемые начальные страницы можно развернуть с помощью развертывания VSIX или путем копирования файлов в правильное расположение на конечном компьютере.  
  
## <a name="vsix-deployment-by-using-the-start-page-project-template"></a>Развертывание VSIX с помощью шаблона проекта страницы запуска  
 При создании начальной страницы с помощью шаблона проекта начальной страницы и выполните построение проекта, Visual Studio создает файл VSIX, который можно передавать. Упаковка начальной страницы в VSIX-файл предоставляет следующие параметры для развертывания, в зависимости от предполагаемой аудитории:  
  
-   VSIX-файл можно поместить в общей сетевой папке или на общедоступном веб-сайте. При открытии файла начальной страницы устанавливается автоматически.  
  
-   Можно загрузить VSIX-файл для [коллекции Visual Studio](http://go.microsoft.com/fwlink/?LinkID=123847) веб-узел, чтобы пользователи могут установить его с помощью **Диспетчер расширений**.  
  
 Начальная страница шаблон проекта создает копию начальной страницы Visual Studio по умолчанию, можно изменять, копировать и сохранять исходный.  
  
 Начальная страница шаблона проекта можно получить с помощью **Диспетчер расширений** или загрузив его с веб-сайта.  
  
## <a name="vsix-deployment-without-using-the-start-page-project-template"></a>Развертывание VSIX без использования шаблона проекта страницы запуска  
 Успешное развертывание VSIX требуется расширение устанавливаются в папки, которые распознаются в процессе регистрации VSIX и по **Диспетчер расширений**. Шаблон проекта начальной страницы уже заданы правильные папки, поэтому рекомендуется использовать каждый раз, когда вам требуется создать пакет расширения для развертывания VSIX. Однако при наличии случай, в котором нельзя использовать шаблон развертывания VSIX можно создать без использования его.  
  
 Создание развертывания VSIX без использования шаблона проекта начальной страницы, сначала создайте файл VSIX для начальной страницы в одном из двух следующих способов:  
  
-   Добавляя пользовательские файлы начальной страницы в пустой проект VSIX. Дополнительные сведения см. в разделе [шаблона проекта VSIX](../extensibility/vsix-project-template.md).  
  
-   Вручную создав файл VSIX. Дополнительные сведения см. в разделе [как: вручную пакет расширения (VSIX развертывание)](../misc/how-to-manually-package-an-extension-vsix-deployment.md).  
  
 Для Visual Studio мог распознать начальную страницу `Content Element` манифест VSIX должна содержать `CustomExtension Element` с `Type` атрибут `"StartPage"`. Расширение начальной страницы, который был установлен с помощью развертывания VSIX отображается в **настроить начальную страницу** списке **запуска** страница параметров как **[установленных расширений]** *расширение*.  
  
 Если пакет начальной страницы включает сборки, необходимо добавить регистрации путь привязки, чтобы они были доступны при запуске Visual Studio. Для этого убедитесь, что пакет включает pkgdef-файл, который содержит следующую информацию.  
  
```  
[$RootKey$\BindingPaths\{Insert a new GUID here}]  
"$PackageFolder$"=""  
```  
  
### <a name="vsix-deployment-for-all-users"></a>Развертывание VSIX для всех пользователей  
 По умолчанию модули, развернутые в пакетах VSIX устанавливается только для текущего пользователя. Установить начальную страницу для всех пользователей на целевом компьютере, можно сделать, создав развертывания всех пользователей.  
  
##### <a name="to-create-an-all-users-deployment"></a>Чтобы создать, все пользователи  
  
1.  Откройте файл extension.vsixmanifest в представлении кода.  
  
2.  В `Identifier` добавьте элемент манифест vsix `AllUsers` элемент, который имеет значение `true`.  
  
    ```  
    <AllUsers>true</AllUsers>  
    ```  
  
     В результате установщик vsix, чтобы запрашивать разрешения администратора, а затем установить файлы \Common7\IDE\Extensions.  
  
3.  Откройте в pkgdef-файл.  
  
4.  Изменение .pkgdef для задания начальной страницы по умолчанию в разделе HKLM, добавив следующий код, где *MyStartPage.xaml* имя XAML-файл, содержащий начальной страницей.  
  
     [$RootKey$ \StartPage\Default]  
  
     «Uri «=» $PackageFolder$\\*MyStartPage.xaml*»  
  
     В этом случае Visual стояли в новое расположение домашней страницы.  
  
## <a name="file-copy-deployment"></a>Развертывание копии файла  
 Необходимо создать файл VSIX для развертывания на пользовательской странице. Вместо этого можно скопировать разметки и вспомогательные файлы непосредственно в папку \StartPages\ пользователя. **Настроить начальную страницу** списке **запуска** параметры странице перечислены все XAML-файлы в этой папке, а также путь — например, %USERPROFILE%\My документы\Visual Studio *версии*\StartPages\\*имя файла*.xaml. Если текущая начальная страница содержит ссылки на закрытых сборок, необходимо скопировать их и вставьте их в папку \PrivateAssemblies\.  
  
 Для распространения начальной страницы, созданные без упаковки в VSIX-файл, рекомендуется использовать стратегию копирования базовый файл, например, пакетный скрипт или другие технологии развертывания, которая позволяет поместить файлы в каталогах, требуется.  
  
#### <a name="to-manually-install-a-custom-start-page"></a>Чтобы вручную установить на пользовательской странице  
  
1.  XAML-файл, содержащий разметку начальной страницы, а также все вспомогательные файлы, отличные от сборки, скопируйте и вставьте их в папку \StartPages\ пользователя.  
  
2.  Если начальная страница требует сборки, их скопировать и вставить их в... \\ *Папка установки visual Studio*\Common7\IDE\PrivateAssemblies\\.  
  
3.  В **настроить начальную страницу** списке **запуска** параметры выберите новый начальной страницы. Дополнительные сведения см. в разделе [Настройка начальной страницы](../ide/customizing-the-start-page-for-visual-studio.md).
