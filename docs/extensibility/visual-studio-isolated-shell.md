# <a name="visual-studio-isolated-shell"></a>Visual Studio изолированной оболочки
Оболочка Visual Studio изолированного позволяет создавать автономных приложений, которые могут выполняться side-by-side с другими версиями Visual Studio. Это используется в основном для размещения специализированные средства, которые могут использовать службы Visual Studio, но также имеют настраиваемый внешний вид и фирменной символики. Функции Visual Studio и группы команд меню можно легко включать и отключать. Приложения, приложения значки и экраны-заставки являются полностью настраиваемыми. Список настраиваемых функций см. в разделе [Настройка изолированной оболочки](../extensibility/customizing-the-isolated-shell.md).  
  
 Для работы с проектом изолированной оболочки, необходимо установить пакет SDK для Visual Studio. Начиная с Visual Studio 2015, не установить пакет SDK для Visual Studio из центра загрузки. Она будет включена в качестве дополнительного компонента в установку Visual Studio. VS SDK также можно установить позже. Дополнительные сведения см. в разделе [установка Visual Studio SDK](../extensibility/installing-the-visual-studio-sdk.md).  
  
 Чтобы создать приложение изолированной оболочки, начните с изолированной оболочки Visual Studio проекта. Этот проект содержит все сведения, необходимые для разработки и тестирования приложения изолированной оболочки. Когда будете готовы написать программу установки, которая развертывает приложения, необходимо получить из распространяемого пакета изолированной оболочки [распространяемый пакет Microsoft Visual Studio Shell (изолированный режим)](http://go.microsoft.com/fwlink/?LinkId=616022).  
  
> [!NOTE]
>  Перед обращением к изолированной оболочки распространяемого пакета, вам будет предложено заполнить клиента краткий опрос.  После заполнения опрос, вы будете перенаправлены на страницу Visual Studio подключиться с помощью ссылки для загрузки распространяемого пакета.  Ссылки для загрузки можно найти при последующих посещениях узла Visual Studio подключиться под **программы | 2015 ИНТЕГРИРОВАННАЯ ОБОЛОЧКА VISUAL STUDIO и ИЗОЛИРОВАННОЙ** вкладки.  
  
> [!NOTE]
>  Дополнительные сведения о развертывании изолированного приложения на основе оболочки см. в разделе [Пошаговое руководство: создание простого приложения изолированной оболочки](../extensibility/walkthrough-creating-a-basic-isolated-shell-application.md).  
  
## <a name="working-with-the-isolated-shell"></a>Работа с изолированной оболочки  
 Приложение оболочки Visual Studio изолированного имеет полный доступ к службам Visual Studio и поддерживает специальные настройки и фирменной символики. Приложение изолированной оболочки можно настроить несколькими способами:  
  
-   Составные части VSPackages и Managed Extensibility Framework (MEF) можно использовать для расширения приложения изолированной оболочки так же, как их следует использовать в любое другое расширение Visual Studio. Дополнительные сведения см. в разделе [расширение изолированной оболочки](../extensibility/extending-the-isolated-shell.md).  
  
-   Чтобы функции Visual Studio и группы команд меню доступна или недоступна, обновите файл .vsct в проект пользовательского интерфейса пользователя приложения.  
  
-   Чтобы удалить **параметры** страниц или других компонентов оболочки Visual Studio из приложения, обновите файл .pkgundef приложения.  
  
-   Чтобы изменить другие аспекты внешнего вида и поведения оболочки, обновите pkgdef-файл приложения.  
  
-   Некоторые аспекты оболочки также можно указать при запуске приложения. Чтобы сделать это, обновление параметров в вызове начальную точку входа appenvstub.dll.  
  
 Дополнительные сведения о различных элементах, которые можно настраивать в разделе [элементы изолированной оболочки](../extensibility/elements-of-the-isolated-shell.md).
