# Макет для Visual Studio
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

Большинство диалоговых окон Visual Studio [Макет диалогового окна программы](../../extensibility/ux-guidelines/layout-for-visual-studio.md#BKMK_UtilityDialogLayout), являющиеся unthemed диалоговые окна этого стандарта выполните [принципы макет диалогового окна Windows Desktop](https://msdn.microsoft.com/en-us/library/windows/desktop/dn742499\(v=vs.85\).aspx). Перемещении обновить пользовательский Интерфейс Visual Studio более наглядными диалоговые окна есть новый дизайн, который устанавливает их работы как определение продукта. Эти [Тематический макет диалогового окна](../../extensibility/ux-guidelines/layout-for-visual-studio.md#BKMK_ThemedDialogLayout) имеют вид темой.  
  
##  <a name="BKMK_UtilityDialogLayout"></a> Макет диалогового окна программы  
  
-   Все элементы управления в диалоговое окно программа должна начинаются с вверх или влево и вниз потока.  
  
-   Никогда не center элементов управления в диалоговом окне, чтобы заполнить большую область.  
  
-   Используйте шрифт среды для всего текста в диалоговом окне. При написании visual спецификации, укажите вместо выбора конкретного шрифта и размера шрифта среды. См. раздел [Шрифт среды разработки](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md#BKMK_TheEnvironmentFont).  
  
-   Используйте размещения и согласованное Управление пространством для поддержки цели качества в мастерства.  
  
-   Диалоговые окна может быть затруднено из большого числа элементов управления и уникальное juxtaposition элементов управления. Для таких сложных случаев позволяют достаточно места между группы управления для предоставления пользователю логический поток для синтаксического анализа.

### Подробные сведения о макетах  
  
#### Поля  
  
-   Все диалоги должны иметь 12 пикселов границы.  
  
-   Поля внутри рамки группы должно быть 9 пикселей от края рамки.  
  
-   Поля внутри элемента управления tab должен быть 6 пикселей от края элемента управления "Вкладка".  
  
#### Кнопки команд  
  
-   Кнопки могут воздействовать на диалоговое окно, не на содержимое. Они должны размещаться в нижней правой и должно быть достаточно переменной выше, чтобы установить кнопки обособленные места.  
  
-   В случае горизонтального кнопок, которые работают в диалоговом окне кнопку Настройка альтернативного команд является вертикальный стек в правом верхнем. В разделе [Внутренний кнопки](../../extensibility/ux-guidelines/layout-for-visual-studio.md#BKMK_InteriorCommandButtons) ниже.  
  
-   Пробел слева от кнопки \(левой или центральной нижней части диалогового окна\) считается частью «диапазон» операции элементов управления диалоговых окон. Единственное, что следует вторгаться в это место находится ссылка на справку, относящиеся ко всей задачи или диалоговое окно.  
  
-   Кнопки должны иметь 75 x 23.  
  
-   Кнопки должны иметь 6 друг от друга.  
  
 ![Basic button alignment](../../extensibility/ux-guidelines/media/0801-c_buttonalign.png "0801\-c\_ButtonAlign")  
  
 **Рис 08.01 — c: Выравнивание основных кнопок**  
  
#### Подписи  
  
-   Выравнивание по левому краю все метки.  
  
-   Для меток, расположенные выше элемента управления они должны по левому краю точно с элементом управления под ним и в нижней части метки должны быть 5 пикселов над верхней границей другого элемента управления \(например, поле со списком\).  
  
-   Для меток, расположенные слева от элементов управления Минимальная ширина между меткой и элемент управления — 10 пикселей. Второй предполагаемого столбца должно быть установлено для выравнивания текстовые поля, поля со списком или других элементов управления.  
  
-   Метки являются прописная и затем двоеточие. См. раздел [Стиль текста](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md#BKMK_TextStyle).  
  
#### Расстояние между элементами управления  
 Достаточно стека элементов управления. Нет не абсолютный Ориентировочная расстояние между элементами управления с накоплением. Tightness между элементами управления может несколько отличаться от диалоговых окон. Рекомендуемый интервал — 20 пикселей для пар вертикальной метка и 9 пикселей для горизонтальной метка пар. Интервалы минимума для горизонтальной пары — 6 пикселов.  
  
 ![Recommended distance between controls](../../extensibility/ux-guidelines/media/0801-d_controldistance.png "0801\-d\_ControlDistance")  
  
 **Рисунок 08.01\-d: Рекомендации для расстояния между элементами управления**  
  
#### Отступ элемента управления  
 Когда есть вложенные элементы управления, выравнивание внутренние элементы управления по горизонтали относительно левого края элемента управления выше, обычно метки.  
  
 ![Nested control alignment](../../extensibility/ux-guidelines/media/0801-e_controlalign.png "0801\-e\_ControlAlign")  
  
 **Рис 08.01\-e: Выравнивание элемента управления вложенные**  
  
#### Ширина элемента управления  
 Ширина текстового поля или других схожих элементов управления не должен превышать среднее входных данных для поля. Среднее английского слова составляет 5 символов. Например, текстовое поле, необходимо имя длинный путь должен быть при условии, что позволяет горизонтальном расположении, а раскрывающийся список имен платформ только должно быть длиной, позволяющей самая длинная запись.  
  
#### Текст справки  
  
-   Диалоговое окно можно отобразить текст справки, предоставляющий дополнительные сведения о назначении в диалоговом окне. Это обычно располагается наверху и может быть предложения 1\-2.  
  
-   Длина строки должен быть знаком ширины для пользователя для синтаксического анализа и чтения. Средний диалоговое окно должно быть не более чем 550 пикселей в ширину.  
  
####  <a name="BKMK_InteriorCommandButtons"></a> Внутренний кнопки  
 В более сложные диалоговые окна внутреннего контроля может иметь свой собственный связанных кнопок, что может повлиять на находятся кнопки фиксации диалогового окна.  
  
-   Используйте вертикальное выравнивание \(столбца\) внутри кнопки, когда **ОК**или**Отменить** горизонтально ориентированного в правом нижнем углу.  
  
-   Горизонтальное выравнивание \(строки\) внутри кнопки, когда использование **ОК**или**Отменить** ориентированы вертикально в правом верхнем углу. Эта ситуация менее типична.  
  
-   Размер внутреннего кнопки должны целевой размер стандартной кнопки 75 x 23 точек, соответствующих размер **ОК**или**Отменить** кнопки, когда это возможно. Если метка кнопки делает кнопку превышает размер стандартной кнопки, другие кнопки в этом наборе должны соответствовать широкого размера.

#### \[Обзор...\] button  
 **\[Обзор...\]** должна описывать кнопки, выполните текстовое поле «Обзор...» в полном объеме в том числе с многоточием. Если не хватает места, или имеется несколько **\[Обзор...\]** кнопок на экране кнопку можно сократить до только с многоточием.  
  
##  <a name="BKMK_ThemedDialogLayout"></a> Тематический макет диалогового окна  
 Тематические диалоговых окон в Visual Studio имеют вид светлее и предоставлять дополнительные пробелы. Оформление предоставляет больше внимания и интерес, предлагают более открытого межстрочный интервал и разновидность размеры шрифтов и веса. Там, где это возможно, chrome и строки состояния были ограниченной или удалены. Макет эти диалоговые окна должны следовать этому простому шаблону:  
  
1.  Имеет белый фон диалогового окна.  
  
2.  Нет границы правило 1 пиксель серые среднего значения.  
  
3.  Заголовок диалогового окна больше не находится в строке заголовка, но предоставляет визуальную привлекательность и выделения увеличить размер точки. \(См. в разделе размер шрифта в [Стиль текста](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md#BKMK_TextStyle).\)  
  
4.  Метки в сочетании с дополнительный текст, например, описание, должно быть **шрифта среды \+ жирный**.  
  
5.  Внутренний столбцы разделяются правило 1 пиксель светло\-серый цвет.  
  
6.  Ссылки по умолчанию имеют без символов подчеркивания. При наведении указателя мыши и нажатия состояния имеют изменение цвета и подчеркивания.  
  
7.  Фиксация кнопки \(как **ОК**или**Отменить**\) расположен в правом нижнем углу.
