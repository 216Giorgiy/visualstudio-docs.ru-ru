# <a name="menus-and-commands-for-visual-studio"></a>Меню и команд для Visual Studio
## <a name="command-usage"></a>Использование команды  
  
### <a name="overview"></a>Обзор  
 В отличие от Microsoft Office, который представляет собой набор, который состоит из многих отдельных продуктов, Visual Studio содержит многие продукты, что каждый привести свои наборы команд для глобального IDE Visual Studio. IDE управляет коэффициентами сложности, тысяч команд с помощью фильтрации функциональные возможности, доступные для пользователя, в зависимости от контекста.  
  
 При изменении контекста пользователя - например переключения в окне конструктора в файл кода в окне редактирования — функциональные возможности, не относящуюся к новый контекст исчезает. В то же время новые функциональные возможности поверхности, а также связанные динамические сведения, такие как параметры свойств и элементов. Пользователь не следует заметить, требуется поменять местами набор доступных команд. Если пользователь является отвлекают или путать команды появление или исчезновение, разработки пользовательского интерфейса должен коррекции. Текущий контекст пользователя всегда указывается в один или несколько способов, например, в заголовке окна интегрированной среды разработки, окна «Свойства» или в диалоговом окне страницы свойств.  
  
 Панели команд для обеспечения гибкости в пользовательском Интерфейсе. Только команда структуры специфические для Visual Studio, среды, главного меню и панели основную команду, которая могут быть настроены и даже скрыты. Другие панели команд появляются и исчезают на основе на состоянии приложения. Окна инструментов и редакторов документа также может содержать внедренные панели инструментов в пределах их края окна.  
  
#### <a name="basic-guidelines"></a>Основные рекомендации  
  
##### <a name="use-existing-shared-commands-command-groups-and-menus-whenever-possible"></a>Использование существующих общих команд, группы команд и меню, когда это возможно.  
 Поскольку обычно отображаются команды на основе контекста, использование существующего общего меню и группы команд обеспечивает сохранение относительно устойчивой между изменениями в контексте структура команды. Повторное использование общих команд и размещение новых команд близко к связанных общих команд снижает сложность интегрированной среды разработки и создает удобство работы. Если новая команда должна быть задана, попробуйте поместить его в существующую группу общих команд. Если создается новая группа должна быть задана, поместите его в существующее меню общих близко к группы связанных команд перед созданием нового меню верхнего уровня.  
  
##### <a name="do-not-create-icons-for-every-command"></a>Не создавайте значки для каждой команды.  
 Следует тщательно перед созданием значок команды. Значки должны создаваться только для команд:  
  
-   отображаются на панели инструментов по умолчанию.  
  
-   скорее всего, добавленные пользователями на панели инструментов с помощью **Настройка...**  диалогового окна.  
  
-   имеют значок, связанный с одним действием в другим продуктом корпорации Майкрософт.  
  
##### <a name="limit-the-addition-of-keyboard-shortcuts"></a>Ограничить Добавление сочетания клавиш  
 Большинство пользователей, применяющих лишь небольшую часть всех доступных сочетаний клавиш. Если вы сомневаетесь, не привязан компонента сочетания клавиш. Работа с пользователя возникновение team перед добавлением новых сочетаний клавиш.  
  
##### <a name="give-commands-a-default-menu-placement"></a>Присвойте команд меню размещения по умолчанию.  
 Имейте в виду, что команды будет настраиваться другими пользователями и разрабатывать их соответствующим образом. Нет такого понятия скрытых команд. Отображаются все команды Visual Studio в **средства > Настройка** диалоговом окне командной строки автозаполнения, **Сервис > Параметры > клавиатуры** диалоговых окон и средств среды разработки (DTE). Убедитесь, что предоставление вашей команды, имя и подсказка в вашей файла ctc, чтобы их будет легко найти.  
  
##### <a name="do-not-duplicate-shared-commands-on-an-embedded-toolbar"></a>Не дублирования общих команд на встроенной панели инструментов.  
 Рекомендуется поместить команды в непосредственной близости от области фокус ввода пользователя. Один из способов сделать это является создание встроенной панели инструментов в верхней части окна инструментов или документов редактора. Команды на панели инструментов следует конкретной области содержимого внутри окна. Не дублирования общих команд на этих панелях инструментов. Например никогда не размещайте в встроенной панели инструментов значок «Сохранить».  
  
### <a name="content-and-command-visibility"></a>Видимость содержимого и команды  
 Команды существуют в следующих областях: **среды**, **иерархии**, и **документа**. Знаете каждой из областей, чтобы иметь уверенность в размещения команд.  
  
 Команды в **среды** область установления основного контекста и являются общими для нескольких контекстах. Они изменяют видимость или упорядочения документов и окна инструментов. Команды в окружении области перечислены **новый проект**, **соединение с сервером**, **присоединение процессу**, **Вырезать**,  **Копировать**, **вставить**, **найти**, **параметры**, **настроить**, **новое окно**, и **просмотра справки**.  
  
 Команды в **иерархии** область управления иерархиями в Visual Studio, включая **проекта**, **команды**, и **данные**. Они связаны с подконтекст проекта — например, **отладки**, **построения**, **тест**, **архитектура**, или **анализ** . Команды в иерархии области перечислены **Добавление нового элемента**, **новый запрос**, **параметры проекта**, **добавить новый источник данных**, **Запустить мастер производительности**, и **новую схему**.  
  
 Команды в **документа** act область содержимого документа, например код, конструктор или запрос рабочих элементов (WIQ). Они также действуют на представлении окна инструментов или в противном случае относятся только к этому окну инструментов. Команды области документа также действуют на объекты файлов, которые сами конкретной иерархии, такие как **удалить из проекта**. Среди команд в документе, область **рефакторинг > Переименовать**, **создать копию рабочего элемента**, **развернуть все**, **Свернуть все**, и **Создать задачу пользователя**.

### <a name="command-placement-in-menus"></a>Размещения команд в меню  
  
#### <a name="main-menu-bar"></a>Главное меню  
 В главном меню должно быть стандартное расположение для команд любых пакетов контекстные меню, задействованных в пользовательском Интерфейсе. В главном меню отличается от других структур команды в том, что в среде используется его к элементу управления, какие команды отображаются. Другие панели команд, просто отключите команды, которые находятся вне контекста, помещаются в меню или на панели инструментов.  
  
 Среды определяет набор встроенных в главном меню команды, которые являются общими для всей интегрированной среды разработки и нескольких доменов задач. Эти команды отображаются всегда вне зависимости от которого пакеты VSPackage, загружаются в среду. Несмотря на то, что пакеты VSPackage может расширить этот набор команд, набора команд из каждого продукта и размещения их команды отвечает каждого из членов команды.  
  
 Структура в главном меню Visual Studio можно разделить на следующие категории меню:  
  
##### <a name="core-menus"></a>Меню Core  
  
-   Файл  
  
-   Правка  
  
-   Просмотр  
  
-   Инструменты  
  
-   Окно  
  
-   Справка  
  
##### <a name="project-specific-menus"></a>Зависящее от проекта меню  
  
-   Проект  
  
-   Сборка  
  
-   Отладка  
  
##### <a name="context-specific-menus"></a>Контекстные меню  
  
-   Команда  
  
-   Данные  
  
-   Проверка  
  
-   Архитектура  
  
-   Анализ  
  
##### <a name="document-specific-menus"></a>Меню для конкретного документа  
  
-   Формат  
  
-   Таблица  
  
##### <a name="when-designing-main-menus-adhere-to-these-rules"></a>При разработке главных меню, соблюдать эти правила:  
  
-   Не превышает 25 элементы верхнего уровня в данном контексте  
  
-   Меню никогда не должен превышать 600 пикселей в высоту.  
  
-   Вычислить главного меню в нескольких контекстах, например в Ultimate SKU и общие профиль.  
  
-   Во всплывающем меню являются допустимыми.  
  
-   Во всплывающем меню должен содержать по крайней мере три элемента, а не более семи.  
  
-   Во всплывающем меню должен использоваться только один уровень вложенности — каскадные подменю имеют некоторые пункты меню Visual Studio, но этот шаблон не рекомендуется.  
  
-   Использовать не более шести разделители. Группирований должны соблюдаться на следующей иллюстрации:  
  
     ![Рекомендации по группированию главного меню](../../extensibility/ux-guidelines/media/0501-b_mainmenus.png "0501 b_MainMenus")  
  
-   Необязательно иметь каждой группы на рисунке, добавление дополнительных группирований ограничен.  
  
-   Каждой группы должны иметь от двух до семи элементов меню.  
  
#### <a name="main-menu-ordering"></a>Упорядочение главного меню  
 Прежде чем добавлять новый элемент верхнего уровня, рекомендуется поместить команду в существующее меню верхнего уровня. При добавлении нового меню верхнего уровня, не забудьте поместить его в правильное расположение. Определите, будет ли меню для проекта, контекст или документ. Сохранить краткое имя меню верхнего уровня и использовать только одно слово.  
  
 Меню core следует форзац остальная часть команды. Файл, редактирования и просмотра должен всегда быть слева и инструментов, окна, и Справка всегда должен быть справа.  
  
#### <a name="context-menus"></a>Контекстные меню  
 Размещение слишком много функциональности в контекстных меню приводит в интерфейсе сложно для изучения. Все основные функциональные возможности должны быть доступны через в главном меню. Положение команд должны быть согласованы с существующие команды, чтобы избежать повторяющихся команд. Контекстные меню оболочки определяет стандартные меню группы, которые должны быть включены в зависимости от того, является ли контекстное меню для решения, узел проекта или элемента проекта.  
  
 При разработке контекстные меню, следовать тем же правилам, как и для главного меню и в дополнение к этому:  
  
-   Не превышает 25 пункты меню верхнего уровня.  
  
-   Во всплывающем меню являются допустимыми, но должно не превышать глубину в один уровень — никогда не используйте каскадные всплывающие окна.  
  
-   Использовать не более шести разделители.  
  
### <a name="command-placement-in-toolbars"></a>Размещения команд на панелях инструментов  
  
#### <a name="general-toolbars"></a>Общие панели инструментов  
 При разработке и упорядочение панели инструментов, выполните эти стандарты.  
  
-   Не используйте больше одной команды на кнопку. Одна кнопка = одно действие.  
  
-   Текст для элемента значок следует используйте только в том случае, если он должен быть принудительно обеспечивается с меткой.  
  
-   Используйте поле со списком исключительно для свойств, которые будут переключаться несколько раз в одном сеансе. В противном случае — предоставить свойство в другом месте.  
  
-   Ширина длинного элемента в поле + 30%, должно быть равно ширину поля со списком. Например если максимальному элементу — 200 пикселей, поле со списком необходимо 260 пикселей в ширину.  
  
-   Ограничьте использование разделителей. Разделитель рядом с раскрывающийся список является антишаблон, поскольку фигуру из раскрывающегося списка, сам выступает в качестве визуального разделителя.  
  
-   Значок группы должен содержать от 3 до шести значков.  
  
-   Если несколько полезные команды квалификаторы, используйте элемент разворачивающейся кнопки, которая сохраняет последний параметр:  
  
     ![Разбивка кнопок в Visual Studio](../../extensibility/ux-guidelines/media/0501-c_splitbuttons.png "0501 c_SplitButtons")  
  
     **Пример разворачивающейся кнопки. Шесть команд в левой части экрана можно вместо этого помещаются в одну кнопку.**  
  
#### <a name="product-specific-toolbars"></a>Панели инструментов конкретного продукта  
 Предоставляют каждого продукта по умолчанию панель инструментов, которая содержит часто используемые и важные команды и панели инструментов по умолчанию для каждого продукта появится при первом запуске Visual Studio после установки продукта.  
  
 Продукты также следует использовать группы общих команд и меню, предоставляемых интегрированной среды разработки. Каждая группа общих команд помещается в общих меню предназначен для упорядочения связанных команд в интуитивно понятной форме для пользователя. Очень важно использовать эту структуру общих команд для уменьшения сложности.  
  
#### <a name="global-toolbars"></a>Глобальные панелей инструментов  
 Глобальные панели инструментов не требуется поместиться на одной строке сразу же после установки. При создании новой главной панели инструментов, следуйте инструкциям для этого типа панели инструментов.  
  
 **Инструкции по главной панели инструментов:**  
  
-   Панели инструментов имеет 24 точки в стандартных элементов управления (захвата, переполнения).  
  
-   Все кнопки панели инструментов — 22 пикселей в ширину, включая заполнение. Делая значок разворачивающейся кнопки добавляет другой 11 пикселей от ширины.  
  
-   Допускается дублирование команд на панели инструментов.  
  
 **Панели инструментов для конкретного документа** отображаются при включенном файлов определенных типов и исчезает, когда становится активным в файл другого типа.  
  
-   Панели инструментов для конкретного документа не может иметь более 12 кнопок.  
  
-   Общая ширина панели инструментов не может превышать 300 пикселей.  
  
-   Каждый тип файла может иметь один встроенной панели инструментов или одного конкретного документа главной панели инструментов, но не оба.  
  
 **Панели инструментов в зависимости от контекста** отображаются при определенном контексте задано и, как правило, остаются активными в течение длительного времени.  
  
-   Кнопка панели инструментов все контекстные ограничен 18.  
  
-   Если большинство пользователей не будет постоянно использовать команды эта панель инструментов, при активном контексте, то не связать эту панель инструментов с контекстом.  
  
-   Убедитесь, что панель инструментов исчезнет при выходе из контекста. Ни один из этих панелей инструментов появится при запуске.  
  
 **Панели инструментов без контекста** никогда не отображаются автоматически. Они показывают, только когда пользователь активирует их. Сохраните максимальную ширину ниже 200 пикселей.  
  
### <a name="general-organization-and-shell-defined-groups"></a>Общая организация и определенные оболочки группы  
 Используйте существующие общие команды, группы команд и меню. Если новая команда должна быть задана, попробуйте поместить его в существующую группу общих команд. Если создается новая группа должна быть задана, попробуйте поместить его в существующее меню общих близко к группе соответствующей командой перед созданием нового меню верхнего уровня. Это снижает сложность команды, обеспечивая согласованность команд размещения в Интегрированной среде разработки.  
  
 Возможно, общий **формат** меню, обычно указывается в контексте окон документов стиль конструктора, показан на следующем рисунке:

### <a name="reducing-and-reusing-commands"></a>Сокращение и повторное использование команд  
 Обычно отображаются команды на основе контекста, чтобы сократить количество команд, которые пользователь видит в любой момент времени. Однако следует также повторно использовать существующие общие меню и группы команд, чтобы убедиться, что структура команды остается относительно стабильной между изменениями в контексте.  
  
 Повторное использование общих команд и размещение новых команд близко к связанных общих команд снижает сложность интегрированной среды разработки и создает удобство работы.  
  
## <a name="naming-commands"></a>Об именах команд  
  
### <a name="naming-conventions"></a>Соглашения об именах  
 Согласованное команда именования очень важно, чтобы пользователи могут найти и выполнять команды, либо с помощью командной строки или привязка сочетания клавиш. Имена команд также помочь пользователю понять, какие цели команды служит, когда она появится на панели инструментов или в каскадной или команду контекстного меню.  
  
#### <a name="when-naming-commands"></a>При именовании команды:  
  
-   Создание текста, чтобы ее было удобно локализовать. Дополнительные сведения о локализации см. в разделе [готовность к международному использованию для Visual Studio](http://msdn.microsoft.com/en-us/1cc35051-8126-441f-bea9-059245a47b1d).  
  
-   Быть кратким. Команды следует использовать не более трех слов.  
  
-   Используйте заглавные буквы регистр: должна быть прописной первую букву каждого слова. Дополнительные сведения о форматировании в Visual Studio см. в разделе [стиль текста](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md#BKMK_TextStyle).  
  
-   Имейте в виду, куда будет помещена команда. Это в меню верхнего уровня или всплывающее окно? Например, при команды выравнивания группировки в плавающее окно, а команды верхнего уровня должен быть «Выравнивание» и всплывающее окно команд должны быть «Слева,» «Справа», «Center», «выравнивание» и так далее. Излишне имя команды всплывающее окно «Выровнять по левому краю» или «Выровнять справа».  
  
     ![Меню формата Visual Studio](../../extensibility/ux-guidelines/media/0502-a_formatmenu.png "0502 a_FormatMenu")  
  
### <a name="using-icons-with-commands"></a>С помощью значка с командами  
 Быть резервирования в использовании значок связывание с командами. Несмотря на то, что связь является уникальным образом с помощью команды hastens возможность идентификации этой команды, загромождения и неэффективность возникать злоупотребление изображения. Следующие правила помогут при принятии решения о значок команды "Создать".  
  
#### <a name="use-an-icon-with-a-command-only-if"></a>Используйте значок с команды, только если:  
  
-   Та же команда имеет значок, связанный с ним в другой Показательным продуктах корпорации Майкрософт, такие как одно из приложений Microsoft Office.  
  
-   Команда будет размещено на панели инструментов по умолчанию.  
  
-   Команда — это специальная команда, которая пользователи, вероятно добавить панель инструментов с помощью **«Изменить...»** диалогового окна.  
  
## <a name="access-and-shortcut-keys"></a>Доступ и сочетания клавиш  
  
### <a name="overview"></a>Обзор  
 Существует два вида назначения клавиш клавиатуры.  
  
-   **Ключи доступа** (также известный как сочетания клавиш) доступа клавиатуры через меню для команды и каждая метка в диалоговом окне пользовательского интерфейса. Клавиши доступа, главным образом для специальных возможностей, назначаются всем меню и большинство элементов управления диалоговых, не обязательно быть запомнен — влияют только текущее окно и локализованы.  
  
-   **Сочетания клавиш** главным образом использовать элемент управления (Ctrl) и функции (Fn) сочетания клавиш. Они более предназначены для опытных пользователей и получения помощи в производительности. Они назначаются только наиболее часто используемых команд и позволяют быстрого доступа, если обход в главном меню. Сочетания клавиш предназначены для быть запомнен и, поэтому должны быть назначены соответствуют схеме профиля. Схемы ключа клавиш может меняться от профиль для создания профиля. Пользователь может настроить сочетания клавиш, через **Сервис > Параметры > клавиатуры**.  
  
### <a name="assigning-access-keys"></a>Назначение клавиш доступа  
 Ключи доступа состоят из клавиш Alt, а также буквенно-цифровых. Назначение клавиши доступа для каждого пункта меню без исключений. Выполните Windows и общие соглашения для назначение клавиш доступа. Например, ключ доступа для **файл > создать** всегда должен быть **Alt, F, N**.  
  
 Не используйте буквы одним ширина в пикселях, например, «i» (в верхнем или нижнем регистре) или строчная буква «l» и не используйте символы с подстрочных элементов (g, j, p, q и y), как их трудно отличить.  
  
 Избегайте использования повторяющихся ключей, когда это возможно. В случаях, где неизбежна дублирования системе меню обрабатывает конфликты по перебирать все команды, использующие ключ. Например, для гипотетического «Number» команды в меню «файл», которое дублирует ключ доступа «N» **Alt, F, N** создаст новый файл и **Alt "," F "," N "," N** выполнит команду «Число».  
  
### <a name="assigning-shortcut-keys"></a>Назначение сочетаний клавиш  
 Так как они не являются обязательными для каждой команды и налогах системы (и пользовательской памяти), если поскольку вызовет перегрузку, не следует назначать новые сочетания клавиш. Данные из программы улучшения качества клиента (CEIP) показывает, что пользователям Visual Studio использовать небольшое подмножество интеграции сочетания клавиш.  
  
 При определении сочетания клавиш, следуют следующим правилам.  
  
-   **Используйте элемент управления (Ctrl) и функции (Fn) сочетания клавиш.**  
  
-   **Сохранить часто используемые сочетания клавиш.** Ведение наиболее популярных сочетания клавиш.  
  
-   **Облегчить введите клавиш редактора.** Привязки клавиш на тип к командам, что разработчики должны большинство при написании кода. Например **Edit.InvokeSmartTag** должен иметь быстрый сочетания клавиш, например Ctrl +/ и не Alt + Shift + F10.  
  
-   **Стремиться к постоянно теме сочетания клавиш.**  
  
-   **Следуйте правилам Windows, чтобы определить, какие модификатор ключей используют.** Команды, имеющие крупномасштабных эффекты, такие как команды, которые применяются ко всему документу с помощью сочетания клавиш Ctrl. Использование сочетаний клавиш Shift для команд, которые расширяют или дополняют действия стандартные сочетания клавиш. Не используйте сочетания Ctrl + Alt.  
  
-   **Удалите лишние сочетания клавиш.** Если у вас есть устаревшие функции, рекомендуется удалить сочетания клавиш, которые используются с extreme infrequency (менее 10 раз из данных программы улучшения качества по) или средняя infrequency (менее 100 раз из данных программы улучшения качества по), если ключ доступа предоставляет быстрый доступ к той же команде. Например: C Alt, H, откроется справка/содержимое.  
  
 Не является простой способ проверить доступность ярлык. Если вы хотите добавить ярлык, выполните следующие действия.  
  
1.  Проверьте список [клавиш Visual Studio 2013](http://visualstudioshortcuts.com/2013/) для получения сведений об аналогичных команд для группировки вас с.  
  
2.  Последовательно выберите пункты **Сервис > Параметры > среда > клавиатуры** и тестирования ярлыка. Убедитесь, что каждая схема назначений клавиш клавиатуры в списке «Применять следующие дополнительную схему назначений клавиш». Проверьте профили Общие, C#, VB и C++, как те, совместно использовать уникальные сочетания клавиш. Ярлык доступен в том случае, если он не сопоставлен ни в одном из этих мест.
