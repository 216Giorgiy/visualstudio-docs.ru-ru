# <a name="images-and-icons-for-visual-studio"></a>Изображения и значки для Visual Studio
##  <a name="BKMK_ImageUseInVisualStudio"></a> Использование изображений в Visual Studio  
 Перед созданием иллюстрации, попробуйте сделать использование изображений в 1 000 + [библиотека изображений Visual Studio](http://www.microsoft.com/en-my/download/details.aspx?id=35825).  
  
### <a name="types-of-images"></a>Типы образы  
  
-   **Значки**. Маленькие изображения, отображаемые в командах, иерархий, шаблоны и т. д. Размер значка по умолчанию, используемых в Visual Studio — PNG 16 x 16. Значки, автоматически созданные службы образов в генерировать формат XAML для поддержки HDPI.  
  
     **Примечание:** хотя образы используются в системе меню, не следует создавать значок для каждой команды. Обратитесь к [меню и команд для Visual Studio](../../extensibility/ux-guidelines/menus-and-commands-for-visual-studio.md) следует ли команде получили значка.  
  
-   **Эскизы.** Изображения, используемые в области предварительного просмотра диалогового окна, такие как диалоговое окно нового проекта.  
  
-   **Изображения диалогового окна.** Изображения, которые отображаются в диалоговых окон и мастеров, как описательный графики или индикаторы сообщения. Используйте редко и только при необходимости для демонстрации основных принципов трудно или получения внимания пользователя (предупреждение, предупреждение).  
  
-   **Изображения анимированного.** Использовать индикаторы выполнения, строки состояния и диалоговые окна для операции.  
  
-   **Курсоры.** Используется для указания, разрешены ли операции с помощью мыши, где объект может быть прекращена и так далее.  
  
##  <a name="BKMK_IconDesign"></a> Значок конструктора  
  
### <a name="overview"></a>Обзор  
 Visual Studio использует современный стиль значков, которые чистой геометрии и средствам баланс положительное или отрицательное (светлой и темной) и используйте метафоры прямым, понятным. Разработке ключевой значков указывает сосредоточено вокруг ясности, упрощения и контекста.  
  
-   **Для ясности:** сосредоточиться на основных метафоры, причем значок его значение индивидуальность.  
  
-   **Упрощение:** уменьшить его значения core значок - получение темы только необходимые элементы и без дополнительных возможностей.  
  
-   **Контекст:** учитывать все аспекты значок роли во время разработки концепции, который крайне важен при определении того, какие элементы составляют метафоры core значок.  
  
 С помощью значков существует несколько точек конструктора, чтобы избежать:  
  
-   Не используйте значки, обозначающие элементы пользовательского интерфейса, за исключением случаев, когда это необходимо. Выберите более абстрактный или символьной подход, если элемент пользовательского интерфейса не общий, очевидно, уникальное.  
  
-   Не злоупотребление Общие элементы, такие как документы, папки, стрелки и значок лупы. Такие элементы только в том случае, если основные значок, что означает используйте. Например, значок лупы рядом вправо должно быть указано, поиск, просмотр и поиск.  
  
-   Несмотря на то, что некоторые элементы устаревший значок поддерживать использование перспективы, нельзя создавать новые значки с точки зрения, если элемент не имеет ясности без него.  
  
-   Не cram слишком много сведений в значок. Простой изображение, которое легко распознается или узнали, как распознать символ является намного более полезными, чем изображение слишком сложным. Значок не может определить общую картину.

### <a name="production-details"></a>Сведения о рабочей среде  
 Все новые элементы пользовательского интерфейса должны быть созданы с помощью Windows Presentation Foundation (WPF) и все новые значки для WPF должны иметь формат PNG 32-разрядной. 24-разрядный PNG-устаревший формат, не поддерживают прозрачность и поэтому не рекомендуется для значков.  
  
 Сохраните разрешение 96 точек на ДЮЙМ.  
  
#### <a name="file-types"></a>Типы файлов  
  
-   **32-разрядных PNG:** предпочтительным форматом значков. Формат файла сжатие без потерь данных, который может хранить растрового единого (в пикселях) изображения. 32-разрядных файлов PNG поддерживают прозрачность альфа канала, гамма-коррекция и чередование.  
  
-   **32-разрядные BMP:** для элементов управления, отличных от WPF. Также называется XP или высокое, 32-разрядных BMP имеет формат изображения RGB-значение, true цветного изображения с прозрачностью альфа канала. Альфа-канал представляет собой уровень прозрачности, указанных в Adobe Photoshop, которая затем сохраняется в растровое изображение как дополнительные (четвертый) цветового канала. Во время иллюстрации эксплуатацию, чтобы все 32-разрядные BMP-файлы для обеспечения быстрого визуальную подсказку о глубину цвета добавляется черном фоне. Это черном фоне представляет область, чтобы быть скрыт в пользовательском Интерфейсе.  
  
-   **32-разрядное ICO:** значков проекта и добавить элемент. Все файлы ICO, 32-разрядное значение true, цвет, прозрачность альфа канала (RGB-значение). Поскольку ICO-файлы можно хранить несколько размеров и цветов, значки, Vista, часто в формате ICO, содержащий 16 x 16, 32 x 32, 48 x 48 и размеры изображения 256 x 256. Для правильного отображения в обозревателе Windows ICO-файлы должны быть сохранены раскрывающегося списка для 24-разрядный и 8-разрядных цветов для каждого размера изображения.  
  
-   **XAML:** для областей конструктора и графических элементов Windows. Значки XAML, файлы изображений на основе вектора, которые поддерживают масштабирование, вращение, хранения и прозрачности. Они не являются общими в Visual Studio сегодня, но становятся все более широкое из-за своей гибкостью.  
  
-   **SVG**  
  
-   **24-разрядный BMP:** для панели команд Visual Studio. Формат изображения RGB true color, в 24-разрядный BMP — Значок соглашение, которое создает прозрачности слоя с помощью Пурпурный (R = 255, G = 0, B = 255) как ключ цвета прозрачности открывающейся слоя. В 24-разрядный BMP отображаются все пурпурный поверхности цветом фона.  
  
-   **24-разрядный GIF:** для панели команд Visual Studio. Формат изображения true цветов RGB, поддерживающим прозрачность. GIF-файлы часто используются в мастер иллюстрации и анимации GIF.  
  
### <a name="icon-construction"></a>Значок создания  
 Минимальный размер значка в Visual Studio — 16 x 16. Самый большой общего использования — 32 x 32. Имейте в виду не для того, чтобы заполнить всю рамку 16 x 16, 24 x 24 или 32 x 32, при разработке значка. Построение читаемым, универсальный значок крайне важен для распознавания пользователя. При построении значки соблюдать следующие моменты.  
  
-   Значки должны иметь ясно, понятным и согласованность.  
  
-   Рекомендуется для использования в качестве единого значки элементов уведомления о состоянии и не к их наложению значок базового элемента. В некоторых контекстах пользовательского интерфейса может потребоваться элемент status для связывания с базового элемента.  
  
-   Значки проекта обычно являются ICO-файлы, содержащие несколько размеров. Обновляются только значков 16 x 16, 24 x 24 и 32 x 32. Большинство значков 16 x 16 и 24 x 24 будет содержать те же элементы. Значки 32 x 32 содержат дополнительные сведения, включая язык типа проекта, где это применимо.  
  
-   Значки 32 x 32 базовые элементы обычно имеют толщину линии 2 пикселя. 1 или 2 пикселя толщину линии можно использовать для детализации элементов. Позволяет определить, какой более подходящими небольшими.  
  
-   Иметь по крайней мере 1 пиксель интервалы между элементами для 16 x 16 и значки 24 x 24. Значки 32 x 32 используйте 2 пикселя интервалы между элементами и модификатор и базового элемента.  
  
 ![Элемент интервалы для размера значков 16 x 16, 24 x 24 и 32 x 32](../../extensibility/ux-guidelines/media/0404-47_elementspacing.png "0404 47_ElementSpacing")<br />Интервалы между элементами для значков размером 16 x 16, 24 x 24 и 32 x 32
