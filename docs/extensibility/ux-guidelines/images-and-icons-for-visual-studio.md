# Изображения и значки для Visual Studio
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

##  <a name="a-namebkmkimageuseinvisualstudioa-image-use-in-visual-studio"></a><a name="BKMK_ImageUseInVisualStudio"></a> Использование изображений в Visual Studio  
 Перед созданием иллюстрации, рассмотрите возможность создания использование изображений в 1 000 + [Библиотека изображений Visual Studio](http://www.microsoft.com/en-my/download/details.aspx?id=35825).  
  
### <a name="types-of-images"></a>Типы образов  
  
-   **Значки**. Маленькие изображения, отображаемые в команды, иерархий, шаблоны и т. д. Размер значка по умолчанию, используемых в Visual Studio — PNG 16 x 16. Значки, автоматически созданные службой образов в создания формата XAML для поддержки HDPI.  
  
     **ПРИМЕЧАНИЕ:** образы используются в системе меню, не следует создавать значок для каждой команды. Обратитесь к [меню и команд для Visual Studio](../../extensibility/ux-guidelines/menus-and-commands-for-visual-studio.md) ли команда должна получили значка.  
  
-   **Эскизы.** Рисунки, используемые в области предварительного просмотра диалогового окна, такие как диалоговое окно нового проекта.  
  
-   **Диалоговое окно изображения.** Изображения, отображаемые в мастеров для описания графики или индикаторы сообщения или диалоговые окна. Используйте редко и только при необходимости для демонстрации основных принципов трудно или получения внимания пользователя (, предупреждение).  
  
-   **Анимированные изображения.** Использовать индикаторы хода выполнения, строки состояния и диалоговые окна для операции.  
  
-   **Курсоры.** Используется для указания, разрешены ли операции с помощью мыши, когда объект может быть удален и т. д.  
  
##  <a name="a-namebkmkicondesigna-icon-design"></a><a name="BKMK_IconDesign"></a> Значок конструктора  
  
### <a name="overview"></a>Обзор  
 Visual Studio использует современный стиль значков, которые имеют чистую геометрию и средствам баланс положительные и отрицательные (света и тени) и использовать прямой, понятные метафоры. Важно значок центр разработки точки вокруг ясности, упрощения и контекста.  
  
-   **Для ясности:** сосредоточиться на основных метафоры, причем значок его значение индивидуальность.  
  
-   **Упрощение:** уменьшить его ядра, что означает значок – получите темы только необходимые элементы и без дополнительных возможностей.  
  
-   **Контекст:** учитывать все аспекты значок роли во время разработки концепция, которая имеет решающее значение при принятии решения, какие элементы составляют метафоры основной значок.  
  
 С помощью значков существует несколько точек конструктора, чтобы избежать:  
  
-   Не следует использовать значки, обозначающие элементы пользовательского Интерфейса, за исключением, когда это необходимо. Выберите более абстрактный или символических подход, если элемент пользовательского Интерфейса не общий, очевидно, уникальное.  
  
-   Чрезмерное использование общих элементов не нравится документов, папок, стрелки и увеличительное стекло. Используйте такие элементы, только если essential, что означает значок. Например, увеличительное стекло вправо следует указывать поиска, просмотра и поиска.  
  
-   Несмотря на то, что некоторые элементы устаревший значок поддерживать использование перспективы, не создавайте новых значков с точки зрения, если элемент не имеет ясности без него.  
  
-   Не cram слишком много информации в значок. Простое изображение, которое можно легко распознать или научились запоминающееся символов гораздо полезнее, чем изображение слишком сложным. Значок не раскрыт полный механизм.

### <a name="production-details"></a>Сведения о рабочей среде  
 Все новые элементы пользовательского Интерфейса должны создаваться с помощью Windows Presentation Foundation (WPF) и всех новых значков для WPF должны быть в формате PNG 32-разрядной. 24-разрядный PNG является формат прежних версий, которые не поддерживают прозрачность и поэтому не рекомендуется для значков.  
  
 Сохраните разрешением 96 точек на дюйм.  
  
#### <a name="file-types"></a>Типы файлов  
  
-   **32-разрядных PNG:** предпочтительный формат значков. Формат файла сжатие без потерь данных, который может хранить один растрового (пикселей) изображения. 32-разрядных PNG-файлы поддерживают прозрачность альфа канала, гамма-коррекция и чередование.  
  
-   **32-разрядные BMP:** для элементов управления, не относящееся к WPF. Также называется XP или высокое, 32-разрядные BMP-это формат изображения RGB-значение, true цветного изображения с прозрачностью альфа канала. Альфа-канал — это уровень прозрачности, назначенные в Adobe Photoshop, который затем сохраняется в растровое изображение как дополнительные (четвертый) цветового канала. Во время производства логотип на все 32-разрядные BMP-файлов для обеспечения быстрого визуальную подсказку о глубина цвета добавляется черный фон. Это черный фон представляет области скрывается в пользовательском Интерфейсе.  
  
-   **32-разрядные ICO:** значки проекта и добавление элемента. Все файлы ICO, 32-разрядной палитрой с прозрачностью альфа канала (RGB-значение). Поскольку ICO-файлы можно сохранить несколько размеров и цветов, часто являются значков Vista формат ICO, содержащий 16 x 16, 32 x 32, 48 x 48 и размеры изображения 256 x 256. Для правильного отображения в обозревателе Windows, ICO-файлы должны быть сохранены список для 24- и 8-разрядных цветов для каждого размера образа.  
  
-   **XAML:** для конструирования и графические элементы Windows. Значки XAML, файлы векторное изображение, поддерживающие масштабирования, поворота, хранения и прозрачности. Они не распространены в Visual Studio уже сегодня, но становятся все более популярным благодаря своей гибкостью.  
  
-   **SVG**  
  
-   **24-разрядный BMP:** командной строки Visual Studio. Формат изображения RGB true color, 24-разрядный BMP — Значок соглашение, которое создает уровень прозрачности с помощью Пурпурный (R = 255, G = 0, B = 255) как ключ цвета для слоя открывающейся прозрачности. В 24-разрядный BMP все поверхности пурпурного отображаются цветом фона.  
  
-   **24-разрядный GIF:** командной строки Visual Studio. Формат изображения RGB true color, который поддерживает прозрачность. GIF-файлы часто используются в мастере логотип и анимации GIF.  
  
### <a name="icon-construction"></a>Значок создания  
 Наименьший размер значка в Visual Studio — 16 x 16. Наибольшего общего использования — 32 x 32. Имейте в виду не для заполнения всего 16 x 16, 24 x 24 или 32 x 32 кадра при разработке значка. Построение читаемым, универсальный значок является необходимым для распознавания пользователя. При построении значки соблюдать следующие моменты.  
  
-   Значки должны иметь четкой, понятно и согласованность.  
  
-   Рекомендуется для использования в качестве единого значки элементов состояния уведомления и не к их наложению значок базового элемента. В некоторых контекстах пользовательского Интерфейса может потребоваться сочетаться с базовым элементом состояние элемента.  
  
-   Значки проекта обычно являются .ico файлы, содержащие несколько размеров. Обновляются только значки 16 x 16, 24 x 24 и 32 x 32. Большинство значков 16 x 16 и 24 x 24 будет содержать те же элементы. Значки 32 x 32 содержат дополнительные сведения, включая языка тип проекта, если применимо.  
  
-   Значки 32 x 32 базовые элементы обычно имеют толщину линии 2 пикселя. Толщина линии 1 или 2 пикселя можно использовать для детализации элементов. Небольшими используется для определения, который является более подходящим.  
  
-   Иметь по крайней мере 1 пиксель интервалы между элементами для 16 x 16 и значки 24 x 24. Значки 32 x 32 используйте 2 пикселя интервалы между элементами и модификатор и базового элемента.

 Значки в Visual Studio, в основном одноцветным. Цвет зарезервирован для передачи информации о конкретных и никогда не для оформления. Цвет используется:

-   Чтобы предупредить пользователя о уведомление о состоянии  
  
-   Чтобы назначить принадлежность языка  
  
-   Чтобы отличить элементы в IntelliSense
