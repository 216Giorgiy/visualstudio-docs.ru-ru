# Анимации для Visual Studio
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

## Основы анимации  
  
### Рекомендации по анимации в Visual Studio  
 Выполните эти правила, чтобы обеспечить Стили анимации согласованный и удобным в интегрированной среде разработки Visual Studio.  
  
-   **Будьте осторожны.** Ограничить анимации, те, которые предназначены для определенных целей.  
  
-   **Скорости и времени являются важными** чтобы убедиться, что переходы доверяете быстрого и естественным:  
  
    -   Завершите анимировать переходы в пределах одного полсекунды \(500 миллисекунд\).  
  
    -   Анимации, которые будут происходить часто должны быть достаточно быстро, что они не будут прерывать работу пользователя рабочего процесса.  
  
    -   Не должно быть анимации, поэтому быстрым или резать глаз, что это сложно понять, но не настолько велико, что он делает один нетерпеливый перехода для завершения.  
  
    -   Чтобы подчеркнуть важность использования переменных времени. Например при перемещении по последовательности элементов на схеме классов, ускоренно переходы между элементами, затем замедлить сосредоточиться на важных элементов.  
  
-   **Использовать постепенное замедление нелинейного** из одного состояния в другое, предоставляя смысле тихо и естественным перемещения  
  
-   По возможности **использовать слабая анимации при наведении курсора мыши** для указания интерактивных элементов под указателем мыши.  
  
-   Если серьезно зависит от анимации в компоненты затем **предоставляют средства, чтобы отключить их** локально \(для всех функций\) как параметр в **Сервис \> Параметры** диалогового окна.  
  
-   **Только одна анимация должна выполняться одновременно** и передает только часть информации.  
  
-   **Важно тонкостей.** В большинстве случаев анимации не обязательно должны внимания пользователя запросу обслуживать его назначение. Незначительные изменения в синхронизации, виртуализации и поведение может существенно повлиять на восприятие и может отличаться в зависимости от эффективного и неэффективного анимации.  
  
-   При использовании анимации для привлечения внимания к одному, **Убедитесь, что имеет смысл прерывать пользователя**в ход мыслей.  
  
-   **При отображении состояния или хода выполнения** посредством анимации:  
  
    -   Прекратить отображение хода выполнения перемещения при базовый процесс не перемещения.  
  
    -   Различают неопределенное процессов определенный процессов.  
  
    -   Убедитесь, что анимация имеет определяемый состояния завершения и ошибки.  
  
    -   Сведите к минимуму использование эффект анимации, показывающих состояние и иметь действительное значение, предоставляя дополнительную информацию фактическое использование. Примерами могут служить временными состояния изменения и аварийных ситуациях  
  
#### Не следует:  
  
-   Использовать небольшой движения \(перемещение небольших размеров\), предпочитая затенения и изменяется перемещения объектов.  
  
-   Используйте анимацию, выполняемые через большую область реальный объем экрана. Независимо от размера этот стиль анимации отвлекает для пользователя.  
  
-   Использование анимации, которые не связаны с объекта, к которому пользователь в настоящее время занимается или взаимодействия с.  
  
-   Используйте анимацию, требующие взаимодействия с пользователем для сброса состояния, например, чтобы пользователь реагировать на мигающий уведомлений, чтобы сделать его остановить мерцание. Взаимодействия с ними каким\-либо образом должно быть достаточно закрыть их.  
  
 Дополнительные сведения о приложениях, эти рекомендации см. в разделе [Шаблоны анимации](../../extensibility/ux-guidelines/animations-for-visual-studio.md#BKMK_AnimationPatterns).  
  
### Метрики анимации  
  
-   Система запросит должен реагировать на жесты пользователя в менее чем 10 миллисекунд.  
  
-   Анимации не может занять больше времени, чем 500 миллисекунд для выполнения.  
  
-   Один способ компенсировать переходы, которые требуют более длительное время — разделите ее на две части; Например первая часть анимации может быть пустой содержимого контейнера \(до 500 миллисекунд\), следуют содержимого затухание в контейнер \(до 500 миллисекунд\).  
  
-   Для времени загрузки, которые могут быть рассчитаны определяющих индикатора \(индикатор выполнения сделать процентов\) является предпочтительным.  
  
-   Для времени загрузки, не может быть вычислен подходит занят индикатор, например курсора или внедренные вращающийся анимации \(загрузки или индикатор работы\).  
  
### Анимация в communicator  
 В пользовательском Интерфейсе Visual Studio анимация функционирует только как о средстве коммуникаций.  Он используется для обмена данными различные сведения, например структурных изменений в пользовательском Интерфейсе; Например, когда меню откроется, или закрывает. Анимация может помочь визуализировать поведение зависит от времени сложных систем, таких как представление хода выполнения установки или использоваться для привлечения внимания оповещения и уведомления.  
  
 Обычно функции пользовательского интерфейса анимации четырьмя способами: визуализировать, привлечения внимания, имитации и указать ответ раз и ходе выполнения.  
  
#### Визуализация  
 Анимации можно выделить характер трехмерных объектов и упростить визуализировать их пространственных структуры. Для этого анимация может требуется на вращение объекта в полный круг, медленно включить вперед и назад или перевести объект ближе и немного увеличить его размер, чтобы подчеркнуть смены или фокус.  
  
 Несмотря на то, что для трехмерных объектов может быть перемещен с пользовательским элементом управления, конструктор должен определить заранее \(программно или вручную\) как наилучшим образом анимация движения, который обеспечивает оптимальное представление объекта. При этом запрограммировать анимация может затем активировать пользователем, поместите указатель на объект, тогда как управляемая пользователем движения, необходимо понять, как управлять объектом. Ограничить перемещение одной оси или ориентации одновременно; либо масштабирования, поворота, или преобразования, но не более одного одновременно.  
  
 Категория визуализировать включает в себя аспекты данных, связи и состояние, структуры, последовательности и времени.  
  
##### Данные  
 Показаны сведения о сложных и переменных.  
  
-   Перемещение по сведения визуализации, такие как диаграммы и графики  
  
-   Пошаговое выполнение последовательности, Обзор и разбиение по страницам  
  
-   Вызов сведений, выбрав и выделения определенных сведений  
  
-   Наложение сведения и Дополнительные сведения на основе фокус элемента  
  
-   Трансформация из одного представления структуры и организации в другую  
  
-   Представляющий изменения во времени, используя ползунки времени, регистрацию и выбора колес и элементы управления движением \(воспроизведения, остановки и приостановки\).  
  
##### Отношения  
  
-   Показано, как элементы относятся друг к другу или элементы, которые относятся к данному элементу.  
  
-   Показать связи иерархий "и" родители потомки "или" того же уровня  
  
-   Один элемент порождает другой  
  
-   Сводит к минимуму одного элемента к другому элементу  
  
-   Один элемент, привязанным к другой  
  
##### Состояние  
  
-   Обновления содержимого.  
  
-   Фокус и выбор  
  
-   Ход выполнения  
  
-   Ошибки  
  
##### Структура  
  
-   Сведение структуры на одном узле  
  
-   Переориентация  
  
-   Свернуть и развернуть, или развернуть и свернуть  
  
##### Sequence  
  
-   Последовательность слайд\-шоу  
  
-   Пролистывание изображения  
  
##### Время  
  
-   Изменение отображения за промежуток времени, времени и презентация  
  
-   Переместить в мусор, отмены и повтора  
  
-   Восстановить состояние журнала  
  
#### Привлечение внимания  
 Если целью является привлечь внимание пользователя к одному элементу из нескольких или предупредить пользователя о обновленные сведения, анимация может быть соответствующий. Например на начальную страницу приложения могут использовать кнопку Приступая к работе, слайды в месте, после загрузки страницы.  
  
 Как правило последний перемещения элемента на экране для привлечения внимания пользователя.  В ряде анимированных элементов внимание пользователя последует последнего перемещения объекта.  
  
##### Предупреждения  
  
-   Предупредить пользователя, получение внимания, Показать ход выполнения  
  
-   Показать, что что\-то проводится верно или неверно или отображение хода выполнения или выполняющиеся изменения  
  
-   Запрашивать у пользователей во время выполнения задачи, такие как найти дополнительные сведения в Интернете или изучения текущей задачи  
  
##### Уведомления  
  
-   Оповещать пользователя об условиях возникновения ошибки  
  
-   Прерывать пользователя для просмотра, если они хотели заняться другой работой  
  
-   Аккуратно проинформировать пользователя, что процесс завершена успешно или изменить, например при завершении загрузки.  
  
#### Имитации  
 Эта категория охватывает физической природы и размерность.  
  
-   Иллюстрации, откуда берутся объекты или когда они поступают  
  
-   Развернуть и свернуть или открыть и закрыть  
  
-   Включает панорамирования, прокрутка и страницы  
  
-   Наложение и z упорядочивание  
  
-   Карусель и accordion  
  
-   Отражение и поворот пользовательского интерфейса  
  
#### Индикаторы ответа и хода выполнения  
 Индикаторы хода выполнения имеют несколько существенных преимуществ:  
  
-   Определенный и неопределенные индикаторы заверить пользователя, система не произошел и работает над проблемы.  
  
-   Определенный индикаторы предоставить пользователю, которое идет чувство насколько вдоль действие, а также чувство получения ближе к окончанию.  
  
##  <a name="BKMK_AnimationPatterns"></a> Шаблоны анимации  
  
### Обзор  
 Анимации в Visual Studio предназначены для обслуживания определенной функции и не снизить производительность работы пользователей. Анимация общие характеристики придерживаться включают:  
  
-   Небольшой и выделялся  
  
-   Естественными и реалистичными  
  
-   Слабая и subdued  
  
-   Быстрое и эффективное  
  
-   Ослаблены, не hurried  
  
 На следующем рисунке Стили анимации, рекомендуется использовать в Visual Studio. Без анимации и элегантной анимации такие как появление \/ исчезания используются наиболее часто. Ограниченные приложения движения анимации, такие как разворачивать и сворачивать, изменение и поворота координат X и Y.  
  
 ![Recommended animation styles for Visual Studio](../../extensibility/ux-guidelines/media/1202-a_vsanimstyles.png "1202\-a\_VSAnimStyles")  
  
 **Рекомендуемые стили анимации для Visual Studio**  
  
#### Отображение и скрытие  
 С этим шаблоном элемента переключается с видимым для ожидания вида и обратно без анимации перехода:  
  
 ![Appear&#47;Disappear animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-b_appearanddisappear.png "1202\-b\_AppearAndDisappear")  
  
##### Правильное использование  
 Новые элементы пользовательского интерфейса, которые необходимо сразу же отображаются и скрываются, чтобы пользователь не имеет избежать или действия. Кроме того медленное перемещение анимации может быть воспринято как перетаскивание производительности не будет происходить с использованием стиля, отображаются и исчезают.  
  
##### Неправильное использование  
 Случаи, в которых пользовательский Интерфейс отображается неожиданно пользователь не имеет ни малейшего представления произошедшего и добавления анимации может помочь контекстные сведения.  
  
##### Свойства анимации  
 Промежуток времени, обычно — 0 секунд.  
  
##### Примеры  
  
-   Автоматическое скрытие окон инструментов  
  
-   Редактор активации клавиатуры пользовательского интерфейса, такие как IntelliSense и Справка по параметрам  
  
-   Участки кода развернуть свернуть  
  
#### Постепенное и затухания  
 С этим шаблоном элемента пользовательского интерфейса переходит из не отображается \(0% непрозрачность\) отображается \(непрозрачность 100%\), или наоборот:  
  
 ![Fade&#45;in&#47;Fade&#45;out animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-c_fadeinfadeout.png "1202\-c\_FadeInFadeOut")  
  
##### Правильное использование  
 Это наиболее часто рекомендуется анимации пользовательского интерфейса. Это Слабый эффект, добавлении процента без прерывания потока. В некоторых случаях пользователь может и не подозревать, что анимацию и просто воспринимает и передаче пользовательского интерфейса системы.  
  
##### Свойства анимации  
  
-   Запуск непрозрачности: 0% для постепенного появления, 100% для затухания  
  
-   Завершение непрозрачности: 100% для постепенного появления, затухания % 0  
  
-   Длительность: автономный 200 миллисекунд, 100 миллисекунд, при использовании в качестве части последовательности анимации комбинации  
  
-   Замедление стиль: InOut синус  
  
##### Примеры  
  
-   Автоматическое скрытие окон инструментов  
  
-   Меню открытия и закрытия  
  
-   Вкладка Переходы фона и переднего плана  
  
#### Переход цвета от A до B  
 С этим шаблоном элемента пользовательского интерфейса изменяется от A к цвету б.  
  
 ![Color blend animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-d_colorblend.png "1202\-d\_ColorBlend")  
  
##### Правильное использование  
 Как анимированный перехода, когда элемент пользовательского интерфейса изменяет цвет из одного контекста или состояния в другое.  
  
##### Свойства анимации  
  
-   Начальный цвет: конкретного пользовательского интерфейса  
  
-   Конечный цвет: конкретного пользовательского интерфейса  
  
-   Длительность: автономный 200 миллисекунд, 100 миллисекунд, при использовании в качестве части последовательности анимации комбинации  
  
-   Замедление стиль: InOut синус  
  
##### Примеры  
  
-   Переходы состояния окна документа \(активных, последние активные и неактивные\)  
  
-   Переходы состояния окна инструментов \(фокус и не в фокусе\)  
  
#### Разворачивать и сворачивать  
 С этим шаблоном элемента пользовательского интерфейса при развертывании в X, Y или в обоих направлениях:  
  
 ![Expand&#47;Contract animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-e_expandcontract.png "1202\-e\_ExpandContract")  
  
##### Правильное использование  
 Как анимации перехода, когда элемента пользовательского интерфейса изменяется размер из одного контекста на другой.  
  
##### Свойства анимации  
  
-   Шкала x: % или конкретного измерения \(в пикселях\)  
  
-   Масштаб Y: % или конкретного измерения \(в пикселях\)  
  
-   Закрепить положение: обычно верхний левый \(для языков слева направо\) или правую \(для языков справа налево\)  
  
-   Длительность: автономный 200 миллисекунд, 100 миллисекунд, при использовании в качестве части последовательности анимации комбинации  
  
##### Примеры  
  
-   Панель обозревателя архитектуры развертывания и свертывания  
  
-   Начальная страница элемент развертывания и свертывания  
  
#### Изменение позиции X\-Y  
 С этим шаблоном элемента пользовательского интерфейса изменяется его положение X или Y или оба:  
  
 ![X&#47;Y position change animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-f_xypositionchange.png "1202\-f\_XYPositionChange")  
  
##### Правильное использование  
 Как анимированный перехода, когда элемент пользовательского интерфейса изменяет положение из одного контекста на другой.  
  
##### Свойства анимации  
  
-   Запуск X и Y позиции: конкретного пользовательского интерфейса  
  
-   Завершение X и Y позиции: конкретного пользовательского интерфейса  
  
-   Путь перемещения: нет  
  
-   Длительность: автономный 200 миллисекунд, 100 миллисекунд, при использовании в качестве части последовательности анимации комбинации  
  
-   Замедление стиль: InOut синус  
  
##### Пример  
 Изменение порядка вкладки  
  
#### Поворот  
 С этим шаблоном поворачивает элемент пользовательского интерфейса:  
  
 ![Rotate animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-g_rotate.png "1202\-g\_Rotate")  
  
##### Правильное использование  
 Только для неопределенного вращающийся индикатор выполнения.  
  
##### Свойства анимации  
  
-   Угол поворота: 360  
  
-   Центр вращения: средний объекта  
  
-   Длительность: непрерывные  
  
##### Пример  
 Неопределенный индикатор \(вращающийся\)  
  
### Общие действия пользовательского интерфейса оболочки и рекомендуемые анимации  
  
#### Откройте вкладку
