# <a name="notifications-and-progress-for-visual-studio"></a>Уведомления и хода выполнения для Visual Studio
##  <a name="BKMK_NotificationSystems"></a> Системы уведомления  
  
### <a name="overview"></a>Обзор  
 Существует несколько способов, чтобы информировать пользователей, выполняемых в Visual Studio о своих задач разработки программного обеспечения.  
  
 При реализации уведомления любого вида:  
  
-   **Количество уведомлений к минимуму** действующие номер. Сообщения уведомления, должны применить большинство пользователей Visual Studio или пользователям из определенной функции или функциональной области. Чрезмерное использование уведомлений может sidetrack пользователя или снижает предполагаемых простоту работы системы.  
  
-   **Убедитесь, вы представляете четкие, позволяющих выполнить действие сообщения** , пользователь может использовать для вызова соответствующего контекста для принятия более сложные решения и предпринимает дополнительные меры.  
  
-   **Представления синхронных и асинхронных сообщений соответствующим образом.** Синхронные уведомления указывают, что что-то требует немедленного внимания, например при сбое веб-службы или код исключения. Пользователь должен быть извещен этих ситуациях сразу таким способом, который требует входные данные, такие как модальное диалоговое окно. Асинхронные уведомления, которые следует знать о пользователя, но не нужно будет действовать немедленно, такие как завершается при завершении операции построения и развертывания веб-сайта. Эти сообщения необходимо более внешней и не прерывать поток задач пользователя.  
  
-   **Используйте модальные диалоговые окна, только когда требуется запретить пользователям предпринимает дополнительные меры** перед подтверждение сообщения или принятия решения, представленные в диалоговом окне.  
  
-   **Удалите окружения уведомления, когда они больше не являются допустимыми.** Пользователь закрыть уведомление, если они уже выполнены действия для устранения этой проблемы, которую они были уведомлены о не требуется.  
  
-   **Имейте в виду, что уведомления могут привести к могут возникнуть ложные корреляции.** Пользователи могут думать, один или несколько своих действий запуск уведомление при на самом деле не было причин. Быть сообщение уведомления о контексте, триггер и источник уведомления.

####  <a name="BKMK_ModalErrorMessageDialogs"></a> Модальные диалоговые окна сообщений  
 Ошибка модальное диалоговое окно сообщения используется для отображения сообщения об ошибке, требуется подтверждение или действия пользователя.  
  
 ![Модальное сообщение об ошибке](../../extensibility/ux-guidelines/media/0901-01_modalerrormessage.png "0901 01_ModalErrorMessage")

####  <a name="BKMK_IDEStatusBar"></a> Строки состояния интегрированной среды разработки  
 Вероятность того, что пользователи заметить текст строки состояния сопоставляет их работу всемирной компьютера и конкретных опыт работы с платформой Windows. Базовый клиента Visual Studio, как правило, иметь опыт в обеих областях, хотя даже опытных пользователей Windows может пропустить изменения в строке состояния. Таким образом в строке состояния наилучшим образом используется исключительно в ознакомительных целях, так и избыточных подсказки для сведения представляются в других местах. В диалоговом окне или в окне средства уведомления, должны предоставляться любого вида важной информации, пользователь должен разрешить немедленно.  
  
 В строке состояния Visual Studio предназначена для нескольких типов данных для отображения. Он состоит из регионов для обратной связи, конструктор, индикатор выполнения, анимации и клиента.  
  
 Область обратной связи и область конструктора отображаются всегда. Индикатор хода выполнения и анимации области всегда являются динамическими и зависимости от контекста пользователя. Область конструктора имеет статический ширину определяется длина строки, извлекается из ресурса сопутствующие текстовых сообщений. Это позволяет локализации изменить ширину без необходимости изменения кода. Для английского языка ширина этой строки равна примерно 220 пикселей. В области конструктора будет работать в обычном режиме и область обратной связи величину оставшегося свободного места.  
  
 В строке состояния также выделяется цветом для добавления визуальные эффекты и функциональной значения путем взаимодействия различных изменения состояния интегрированной среды разработки, например, если интегрированной среды разработки в режиме отладки.  
  
 ![Изменения цвета строки состояния интегрированной среды разработки](../../extensibility/ux-guidelines/media/0901-02_idestatusbar.png "0901 02_IDEStatusBar")  
  
 **Цвета строки состояния интегрированной среды разработки**  
  
####  <a name="BKMK_EmbeddedInfobar"></a> Встроенная информационная панель  
 В верхней части окна документа или окна инструментов можно использовать информационную панель для оповещения пользователя о состоянии или условие. Он также может предложить команд, чтобы пользователь мог иметь возможность легко действий. Информационная панель — это стандартная оболочка элемент управления. Не следует создавать собственные, которой будет работать и отображаться Несогласованная с другими пользователями в Интегрированной среде разработки. В разделе [информационных панелей](../../extensibility/ux-guidelines/notifications-and-progress-for-visual-studio.md#BKMK_Infobars) сведения о реализации и инструкции по использованию.  
  
 ![Внедренные информационная панель](../../extensibility/ux-guidelines/media/0901-03_embeddedinfobar.png "0901 03_EmbeddedInfobar")  
  
 **Информационная внедряются в окне документа, предупреждающее, что, в режиме журнала отладки интегрированной среды разработки и редактор не будет отвечать так же, как и в стандартном режиме отладки.**  
  
####  <a name="BKMK_MouseCursorChanges"></a> Изменения курсора мыши  
 При изменении указателя, используйте цвета, которые связаны с VSColor службы и уже связаны с курсором. Изменения курсора может использоваться для указания, за текущей операции, а также попаданий зонах, где пользователь наведении указателя мыши на целевой объект, который можно перетащить, помещенной на или используется для выбора объекта.  
  
 Используйте курсор мыши занятости ожидания только в том случае, если все доступного времени ЦП должно быть зарезервировано для операции, предотвращая выражения любой дальнейшего ввода данных пользователем. В большинстве случаев с грамотно сконструированные приложения при использовании многопоточности раз, когда пользователи не могут выполнять другие операции должно быть редко.  
  
 Имейте в виду, курсор изменяется полезны, представленный избыточные ключевые сведения в другом месте. Не следует полагаться на изменение курсора как единственный способ взаимодействия с пользователем, особенно при попытке передачи, что-то, является критическим, что пользователю необходимо устранить.  
  
####  <a name="BKMK_NotSysProgressIndicators"></a> Индикаторы выполнения  
 Индикаторы выполнения важны для предоставления отзывов пользователей во время процессов, которые выполняются дольше нескольких секунд для завершения. Можно отображать индикаторы хода выполнения на месте (рядом с точкой запуска действия выполняется), встроенная строка состояния, в модальное диалоговое окно или в строке состояния Visual Studio. Следуйте указаниям в [индикаторы хода выполнения](../../extensibility/ux-guidelines/notifications-and-progress-for-visual-studio.md#BKMK_ProgressIndicators) в отношении использования и реализации.

 Окно уведомления не является в настоящее время расширяемым.

Например в строке состояния интегрированной среды разработки не подходит для уведомления о несколько запусков или выбранные элементы немедленных результаты тестового запуска. Важно сохранять такие сведения о состоянии в контексте окна документа или инструмента, где пользователь выбирает элемент или запускает процесс.

####  <a name="BKMK_WindowsTray"></a> Уведомления панели задач Windows

 Уведомления уровня среды следует отображена в концентраторе уведомления Visual Studio, не области уведомлений Windows.

Пузырьки противоречат важной информации, пользователь должен решить прямо сейчас. Если вы используете пузырьки уведомления в Visual Studio, выполните [руководство Windows Desktop пузырьков уведомления](https://msdn.microsoft.com/en-us/library/windows/desktop/dn742472\(v=vs.85\).aspx).

##  <a name="BKMK_ProgressIndicators"></a> Индикаторы выполнения  
  
### <a name="overview"></a>Обзор  
 Индикаторы выполнения являются важной частью системы оповещения для предоставления отзывов пользователей. Они сообщают пользователя, когда будет выполнена, процессы и операции. Знакомые индикатор типы включают индикаторы выполнения, вращающийся курсоров и анимированных значков. Тип и расположение индикатор хода выполнения зависит от контекста, в том числе формируется отчет и как долго процессу или операции потребуется для завершения.  
  
#### <a name="factors"></a>Факторы  
 Чтобы определить, какие из этих типов индикаторов, необходимо определить следующие факторы.  
  
1.  **Время:** продолжительность операции займет  
  
2.  **Модальность:** ли операция является модальным среду (блокировки пользовательского интерфейса, пока не завершится процесс)  
  
3.  **Постоянный/динамической:** ли результат хода выполнения должен быть отчета и (или) для просмотра в дальнейшем  
  
4.  **Определенный или не определено:** ли время окончания операции и ход выполнения можно вычислить  
  
5.  **График/текстовая расположение:** ли о ходе выполнения или процесс является записанного встроенным образом, в тексте сообщения или определенный элемент управления, такие как дерево  
  
6.  **Выражение с учетом:** ли ход выполнения в непосредственной близости от пользовательского интерфейса, связанного с. (Например, она может быть в строке состояния, который может быть далеко, или рядом с кнопки, которая запускается процесс имеет?)

### <a name="progress-indicator-types"></a>Типами индикаторов выполнения  
  
#### <a name="progress-bars"></a>Индикаторы выполнения  
  
##### <a name="indeterminate"></a>Неопределенный  
 ![Неопределенный индикатор](../../extensibility/ux-guidelines/media/0901-04_indeterminate.png "0901 04_Indeterminate")  
  
 **Неопределенный индикатор выполнения**  
  
 «Неопределенное» означает общий ход выполнения операции или процесс не может быть определено. Используйте индикаторы выполнения определен в операциях, требующих неограниченное количество времени, а также, доступа неизвестного числа объектов. Используйте текстовое описание сопровождающее происходящем. Использование времени ожидания для предоставления границы для операций на основе времени. Индикаторы выполнения неопределенное используйте анимацию, чтобы показать, что становится хода выполнения, но не предоставляют других сведений о. Не следует выбирать Неопределенный индикатор, на основе только возможно отсутствие только точность.  
  
##### <a name="determinate"></a>Определенный  
 ![Определенный индикатор выполнения](../../extensibility/ux-guidelines/media/0901-05_determinate.png "0901 05_Determinate")  
  
 **Определенный индикатор выполнения**  
  
 «Определенный» означает, что операция или процесс требует ограниченное количество времени, даже если этот период времени не было точно спрогнозировать. Четко отобразить выполнение. Не позволяйте индикатор перейдите на 100 процентов, если операция завершена. Панель анимации прогресса перемещает слева направо от 0 до 100%.  
  
 Никогда не переместить обратно во время операции индикатор хода выполнения. В строке должен перемещаться вперед стабильно когда операция начинается и доступа 100%, при ее завершении. Индикатор важно для предоставления пользователю о том, то вся операция продолжительности, независимо от того, сколько шагах.  
  
##### <a name="concurrent-reporting-stacked-progress-bars"></a>Количество одновременных reporting (индикаторы выполнения с накоплением)  
 Если операция будет выполняться долго - возможно может использоваться несколько минут - затем индикаторы выполнения две, одна для отображения общий ход выполнения операции, а другая — для продвижения текущего шага. Например если программа установки копирует много файлов, один индикатор можно использовать для указания того, как долго выполняет процесс, во время секунды можно указать, какой процент текущего файла или каталога при копировании. Не сообщать более пяти одновременных операций или процессов, использующих индикаторы выполнения с накоплением. При наличии более пяти одновременных операций или процессов для отчетов, следует используйте модальное диалоговое окно, с помощью кнопки "Отмена" и отчетов сведения о ходе выполнения в окно вывода.  
  
##### <a name="textual-descriptions"></a>Текстовые описания  
 Используйте текстовое описание, сопровождающее выполняемых и предполагаемое время выполнения. Если невозможно определить, сколько времени занимает операции, лучшим выбором для Предоставление отзывов может быть анимированный значок, а не индикатор хода выполнения.  
  
 Visual Studio предоставляет стандартный индикатор в строке состояния, который может использоваться с любой продукт, интегрированные в Visual Studio. Текстовое описание выполняемых во время анимации индикатор выполнения можно обновить текст строки состояния.

##  <a name="BKMK_Infobars"></a> Информационные панели  
  
### <a name="overview"></a>Обзор  
 Информационные панели предоставляют пользователю индикатора близко к своей точке внимания и с помощью общей информационной панели управления обеспечивает согласованность в внешний вид и взаимодействия.  
  
 ![Infobar](../../extensibility/ux-guidelines/media/0904-01_infobar.png "0904-01_Infobar")  
  
 **Информационные панели в Visual Studio**  
  
#### <a name="appropriate-uses-for-an-infobar"></a>Использование для информационной панели  
  
-   Чтобы разрешить пользователям без блокировки, но важные сообщения, относящиеся к текущему контексту  
  
-   Указывает, что пользовательского интерфейса в состоянии или условие, которое выполняет некоторые особенности взаимодействия, например, отладка с ведением журнала  
  
-   Уведомлять пользователя о том, что система обнаружила проблемы, например, если расширение является причиной проблем с производительностью  
  
-   Чтобы предоставить пользователю возможность легко выполнять действия, например, когда редактор обнаруживает, что файл смешанный знаки табуляции и пробелы  
  
##### <a name="do"></a>Выполните действия.  
  
-   Сохраните текст сообщения информационную панель и помещение их в точку.  
  
-   Сохраните краткий текст ссылки и кнопки.  
  
-   Убедитесь, что параметры «действие» предоставляют пользователям минимальны, отображаются только необходимые действия.  
  
##### <a name="dont"></a>Не надо:  
  
-   С помощью информационной панели обеспечивать стандартные команды, которые должны размещаться на панели инструментов.  
  
-   Используйте информационная вместо модальное диалоговое окно.  
  
-   Создайте сообщение с плавающей запятой за пределами окна.  
  
-   Используйте несколько информационных панелей в нескольких местах, в том же окне.  
  
#### <a name="can-multiple-infobars-show-at-the-same-time"></a>Можно ли отображать несколько информационных панелей в то же время?  
 Да, в то же время покажет несколько информационных панелей. Они отображаются в порядке первым получен — первым обслужен с первой информационную панель, на отображение верхнего и дополнительных информационных панелей ниже.  
  
 Пользователь будет видеть более трех информационные панели за раз, после которого, если доступно несколько информационных панелей, области информационной панели станут прокручиваемый.  
  
### <a name="creating-an-infobar"></a>Создание информационной панели  
 Информационную панель состоит из четырех разделов, слева направо.  
  
-   **Значок:** это, где необходимо добавить любой значок требуется отобразить для информационной панели, такие как значок предупреждения.  
  
-   **Текст:** можно добавить текст, описывающий пользователя сценарии и ситуации возможности, а также ссылок в тексте, если это необходимо. Не забудьте сохранить краткий текст.  
  
-   **Действия:** в этом разделе, содержали ссылки и кнопки для действий, которые может выполнять пользователь в вашей информационной панели.  
  
-   **Кнопка закрытия:** последний раздел справа может иметь кнопку Закрыть.  
  
#### <a name="creating-a-standard-infobar-in-managed-code"></a>Создание стандартных сведений в управляемом коде  
 Класс InfoBarModel можно использовать для создания источника данных для информационной панели. Используйте один из этих четырех конструкторов.  
  
```  
public InfoBarModel(IEnumerable<IVsInfoBarTextSpan> textSpans, ImageMoniker image = default(ImageMoniker), bool isCloseButtonVisible = true);
```  

#### <a name="getting-an-infobar-uielement-from-an-infobar"></a>Получение из информационной панели на информационную панель элементов пользовательского интерфейса  
 Реализация InfoBarModel или IVsInfoBar находится модели данных, которые необходимо включить в элементов пользовательского интерфейса для пользовательского интерфейса. В службе SVsInfoBarUIFactory/IVsInfoBarUIFactory могут извлекаться UIElement.  
  
```  
private bool TryCreateInfoBarUI(IVsInfoBar infoBar, out IVsInfoBarUIElement uiElement)  
{  
    IVsInfoBarUIFactory infoBarUIFactory = serviceProvider.GetService(typeof(SVsInfoBarUIFactory)) as IVsInfoBarUIFactory;  
    if (infoBarUIFactory == null)  
    {  
        uiElement = null;  
        return false;  
    }  
  
    uiElement = infoBarUIFactory.CreateInfoBar(infoBar);  
    return uiElement != null;  
}  
```  
  
### <a name="placement"></a>Размещение  
 Информационные панели могут отображаться в одном или нескольких из следующих расположений:  
  
-   Окна инструментов  
  
-   На вкладке документа  
  
> [!IMPORTANT]
>  Можно разместить на информационную панель, чтобы предоставить сообщение о глобальном контексте. Это будет выглядеть между панелями элементов и в контейнере документа. Это не рекомендуется, так как это вызывает проблемы с «перехода и jerk» интегрированной среды разработки, поэтому следует избегать, если только абсолютно необходимые и соответствующие.  
  
#### <a name="placing-an-infobar-in-a-toolwindowpane"></a>Размещение на информационную панель в ToolWindowPane  
 Метод ToolWindowPane.AddInfoBar(IVsInfoBar) может использоваться для добавления на информационную панель в окно инструментов. Этот API можно либо добавить IVsInfoBar (который InfoBarModel является реализация по умолчанию), или IVsUIElement.  
  
#### <a name="placing-an-infobar-in-a-document-or-non-toolwindowpane"></a>Размещение на информационную панель в документ или не ToolWindowPane  
 Чтобы поместить информационную панель в любой IVsWindowFrame, свойство VSFPROPID_InfoBarHost позволяет получить IVsInfoBarHost для кадра и добавляйте информационную панель элементов пользовательского интерфейса.  
  
```  
private void AddInfoBar(IVsWindowFrame frame, IVsUIElement uiElement)  
{  
    IVsInfoBarHost infoBarHost;  
    if (TryGetInfoBarHost(frame, out infoBarHost))  
    {  
        infoBarHost.AddInfoBar(uiElement);  
    }  
}  
private bool TryGetInfoBarHost(IVsWindowFrame frame, out IVsInfoBarHost infoBarHost)  
{  
    object infoBarHostObj;  
    if (ErrorHandler.Failed(frame.GetProperty((int)__VSFPROPID7.VSFPROPID_InfoBarHost, out infoBarHostObj)))  
    {  
        infoBarHost = null;  
        return false;  
    }  
  
    infoBarHost = infoBarHostObj as IVsInfoBarHost;  
    return infoBarHost != null;  
}  
  
```  
  
#### <a name="placing-an-infobar-in-the-main-window"></a>Размещение на информационную панель в главном окне  
 Разместить информационную панель в главном окне, используйте VSSPROPID_MainWindowInfoBarHost IVsShell службы, чтобы получить IVsInfoBarHost главное окно и добавить информационную панель элементов пользовательского интерфейса.  
  
### <a name="will-i-know-when-the-user-takes-action-in-my-infobar"></a>Будет определить, когда пользователь выполняет действие в моей информационную панель?  
 Да, мы возвращаем каждое действие события автору информационной панели. Именно затем автор информационная панель действий в Интегрированной среде разработки на основе выбора пользователя на информационной панели. Информационные панели будут автоматически удалены из узла, в которых закрытие кнопка была нажата, но дополнительные действия является обязательным, если другие информационные панели необходимо удалить после закрытия. Данные телеметрии также потребуется войти независимо друг от друга, каждая информационной панели.  
  
#### <a name="receiving-infobar-events-in-a-toolwindowpane"></a>Получение событий информационную панель в ToolWindowPane  
 ToolWindowPane имеет два события для информационных панелей. Событие InfoBarClosed возникает при закрытии информационную панель в ToolWindowPane. Событие InfoBarActionItemClicked возникает при щелчке гиперссылки или кнопка внутри информационную панель.  
  
#### <a name="receiving-infobar-events-directly-from-the-uielement"></a>Получение событий информационная панель непосредственно из элементов пользовательского интерфейса  
 IVsInfoBarUIElement.Advise можно использовать для подписки на события непосредственно из информационной панели элементов пользовательского интерфейса. Реализация IVsInfoBarUIEvents позволит автору получать close и выберите события.  
  
```  
public interface IVsInfoBarUIEvents  
{  
    void OnActionItemClicked(IVsInfoBarUIElement infoBarUIElement, IVsInfoBarActionItem actionItem);  
    void OnClosed(IVsInfoBarUIElement infoBarUIElement);  
}  
  
```  
  
##  <a name="BKMK_ErrorValidation"></a> Ошибка проверки  
 Когда пользователь вводит данные, не является допустимым, например, если обязательное поле будет пропущено, или при вводе данных в неправильном формате, лучше использовать проверки элемента управления или обратной связи, рядом с элементом управления, вместо того чтобы использовать диалоговое окно ошибки блокировки всплывающее окно.  
  
### <a name="field-validation"></a>Проверка полей  
 Проверка формы и поле состоит из трех компонентов: элемент управления, значок и подсказку. Хотя это использовать несколько типов элементов управления, текстовое поле будет использоваться в качестве примера.  
  
 ![Поле проверки &#40;пустой&#41;](../../extensibility/ux-guidelines/media/0905-01_fieldvalidation.png "0905 01_FieldValidation")  
  
 Если поле является обязательным, то необходимо водяного знака текст, указывающий  **\<необходимые >** и фон поля должен быть светлым желтый (VSColor: `Environment.ControlEditRequiredBackground`) и должен быть серым, переднего плана (VSColor: `Environment.ControlEditRequiredHintText`):  
  
 ![Поле проверки с меткой «Required»](../../extensibility/ux-guidelines/media/0905-02_fieldvalidationrequired.png "0905 02_FieldValidationRequired")  
  
 Программа может определить, что элемент управления находится в состоянии *недопустимое содержимое, введенное* при перемещении фокуса на другой элемент управления или когда пользователь нажимает кнопку «ОК» фиксации или когда пользователь сохраняет документ или форму.  
  
 После определения недопустимое состояние содержимого внутри элемента управления или просто рядом с ним появится значок. При наведении курсора мыши, значок или элемент управления появится всплывающая подсказка с описанием ошибки. Кроме того границы 1 пиксель должны отображаться вокруг элемента управления, который создает недопустимое состояние.
