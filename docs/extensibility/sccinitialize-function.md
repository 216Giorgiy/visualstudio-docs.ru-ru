# <a name="sccinitialize-function"></a>Функция SccInitialize
Эта функция инициализирует подключаемый модуль системы управления версиями и предоставляет возможности и ограничения для интегрированной среды разработки (IDE).  
  
## <a name="syntax"></a>Синтаксис  
  
```cpp  
SCCRTN SccInitialize (  
   LPVOID* ppvContext,  
   HWND    hWnd,  
   LPCSTR  lpCallerName,  
   LPSTR   lpSccName,  
   LPLONG  lpSccCaps,  
   LPSTR   lpAuxPathLabel,  
   LPLONG  pnCheckoutCommentLen,  
   LPLONG  pnCommentLen  
);  
```  
  
#### <a name="parameters"></a>Параметры  
 `ppvContext`  
 [in] Подключаемый модуль системы управления версиями может поместить указатель на структуру контекста здесь.  
  
 `hWnd`  
 [in] Дескриптор окна интегрированной среды разработки, подключаемый модуль системы управления версиями можно использовать в качестве родительского для все диалоговые окна, которые он предоставляет.  
  
 `lpCallerName`  
 [in] Имя вызывающей подключаемый модуль системы управления версиями программы.  
  
 `lpSccName`  
 [in, out] Буфер, который помещает собственное имя в подключаемый модуль системы управления версиями (не должно превышать `SCC_NAME_LEN`).  
  
 `lpSccCaps`  
 [out] Возвращает флаги подключаемого модуля возможностей системы управления версиями.  
  
 `lpAuxPathLabel`  
 [in, out] Буфер, который подключаемый модуль системы управления версиями помещает строку, описывающую `lpAuxProjPath` параметром, возвращенным [SccOpenProject](../extensibility/sccopenproject-function.md) и [SccGetProjPath](../extensibility/sccgetprojpath-function.md) (не должно превышать `SCC_AUXLABEL_LEN`).  
  
 `pnCheckoutCommentLen`  
 [out] Возвращает максимально допустимую длину для извлечения комментария.  
  
 `pnCommentLen`  
 [out] Возвращает максимально допустимую длину для других комментариев.  
  
## <a name="return-value"></a>Возвращаемое значение  
 Реализация подключаемого модуля управления источника этой функции должен возвращать одно из следующих значений:  
  
|Значение|Описание:|  
|-----------|-----------------|  
|SCC_OK|Успешно инициализации элемента управления источника.|  
|SCC_E_INITIALIZEFAILED|Не удалось инициализировать систему.|  
|SCC_E_NOTAUTHORIZED|Пользователь не может выполнить указанную операцию.|  
|SCC_E_NONSPECFICERROR|Неспецифичную сбоя; Система управления версиями не был инициализирован.|  
  
## <a name="remarks"></a>Примечания  
 IDE вызывает эту функцию при ее первой загрузке системы управления версиями. Она позволяет IDE, чтобы передать определенную информацию, например имя вызывающего объекта в подключаемый модуль. IDE также возвращает определенные сведения, такие как комментарии, а также возможности подключаемые в максимально допустимую длину.  
  
 `ppvContext` Указывает `NULL` указателя. Подключаемый модуль системы управления версиями можно выделить структуру для внутреннего использования и хранения указателя на этой структуре в `ppvContext`. IDE передает этот указатель остальными функциями VSSCI API, позволяя подключаемый модуль имеет информации о контексте без обращения к глобального хранилища и поддерживать несколько экземпляров подключаемого модуля. Эта структура должна быть освобождена при [SccUninitialize](../extensibility/sccuninitialize-function.md) вызывается.  
  
 `lpCallerName` И `lpSccName` параметры позволяют интегрированной среды разработки и подключаемый модуль системы управления версиями для обмена имена. Эти имена можно использовать только для различения нескольких экземпляров, или они действительно могут появляться в диалоговых окнах и меню.  
  
 `lpAuxPathLabel` Параметр является строкой, используемой как комментарий определение вспомогательного проекта пути, который хранится в файле решения и передан в систему управления версиями подключаемого модуля в вызове [SccOpenProject](../extensibility/sccopenproject-function.md). [!INCLUDE[vsvss](../extensibility/includes/vsvss_md.md)]использует строку «проект SourceSafe:»; другие источника подключаемые модули управления рекомендуется воздержаться от использования данной конкретной строки.  
  
 `lpSccCaps` Дает системы управления версиями подключаемого модуля место для хранения, указывающее, подключаемые в возможности битовые флаги. (Полный список возможностей битовые флаги см. в разделе [флаги возможностей](../extensibility/capability-flags.md)). Например если подключаемый модуль планы для записи результатов в функцию обратного вызова, предоставленный вызывающим объектом, подключаемый модуль нужно задать возможность бит SCC_CAP_TEXTOUT. Это будет отправлять сигнал интегрированной среды разработки для создания окна для управления результаты версии.
