# Конструктор манифеста VSIX
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Изменяет файл манифеста пакета VSIX, который задает режим установки для расширения Visual Studio.  
  
 **Конструктор манифеста VSIX** сопоставляется схемой VSIX. Каждый элемент в схеме можно задать с помощью соответствующего элемента управления в конструкторе. Дополнительные сведения о схеме см. в разделе [Справочник по схеме 2.0 расширения VSIX](../extensibility/vsix-extension-schema-2-0-reference.md).  
  
 Чтобы открыть **Конструктор манифеста VSIX**, найдите файл source.extension.vsixmanifest в **обозревателе решений**, и откройте файл. Если файл не содержит допустимый XML\-код, не будут открывать конструктора манифестов.  
  
> [!NOTE]
>  Source.Extension.vsixmanifest выводится extension.vsixmanifest при построении пакета.  
  
## Список элементов пользовательского интерфейса  
 **Конструктор манифеста VSIX** содержит четыре раздела, которые соответствуют этих элементов верхнего уровня схемы:  
  
-   Метаданные  
  
-   Установка целевых объектов  
  
-   Активы  
  
-   Зависимости  
  
 Область заголовка содержит следующие элементы управления.  
  
 **Название продукта**  
 Описывает имя расширения.  
  
 **Идентификатор продукта**  
 Указывает уникальные идентификационные данные для этого пакета.  
  
 **Автор**  
 Задает имя автора модуля.  
  
 **Версия**  
 Указывает номер версии расширения.  
  
 **Метаданные** вкладка содержит следующие элементы управления.  
  
 **Описание**  
 Предоставляет текстовое описание расширения, отображаемый в **Диспетчер расширений**.  
  
 **Язык**  
 Задает язык по умолчанию для пакета, который соответствует текстовые данные в манифесте.`Language` Атрибут соответствует соглашению языка среды выполнения \(CLR\) языкового стандарта кода общих сборок ресурсов, например, en\-us, en, fr\-fr. По умолчанию значение нейтрального; Это означает, что пакет будет запущен в любой языковой версии Visual Studio.  
  
 **Лицензии**  
 Задает текстовый файл, содержащий лицензию пользователя при его наличии.  
  
 **Значок**  
 Указывает графический файл \(формат PNG, BMP, .jpeg, .ico\), содержащего значок, отображаемый в **Диспетчер расширений**, отображается значок. Изображение значка должно быть 32 x 32 пиксела или размеры этих измерений. Если значок не задан, **Диспетчер расширений** используется значок по умолчанию.  
  
 **Изображение для предварительного просмотра**  
 Указывает графический файл \(формат PNG, BMP, .jpeg, .ico\), содержащего изображение предварительного просмотра для отображения в **Диспетчер расширений**, при наличии изображения предварительного просмотра. Предварительный просмотр изображения должен быть 200 x 200 пикселов. Если нет изображения предварительного просмотра, **Диспетчер расширений** использует изображение по умолчанию.  
  
 **Теги**  
 Добавляет текст для поиска ссылок.  
  
 **Заметки о выпуске**  
 Задает файл \(.txt, .rtf\), содержащий заметки о выпуске. Также принимает URL\-адрес веб\-сайта отображается в заметках о выпуске.  
  
 **Руководство по началу работы**  
 Задает файл \(.txt, .rtf\), который содержит сведения об использовании расширения или содержимого в пакете VSIX. В этом руководстве появляется после завершения установки расширения. Также принимает URL\-адрес веб\-сайта, отображающий руководства.  
  
 **Дополнительные сведения о URL\-адрес**  
 Указывает URL\-адрес веб\-сайта, который содержит дополнительные сведения о продукте.  
  
 **Цели установки** вкладка содержит следующие элементы управления.  
  
 **Тип установки**  
 Список **расширение Visual Studio** и **SDK расширения** объект типы установки. Параметры различаются в зависимости от выбранного типа.  
  
 **Расширение Visual Studio**  
 Список **InstallationTarget** элементы, которые описывают, как пакет может быть установлен и в какие продукты Visual Studio можно установить это расширение. Каждого продукта отдельно идентифицируется по имени и версии или версии диапазона.  Продукты можно добавляется в список, изменены и удалены. Имя и версию продукта соответствуют **идентификатор** и **версии** атрибуты связанного **InstallationTarget** элемента.  
  
 **Диапазон версий** — \[12.0, 14.0\] и используется следующая запись:  
  
-   \[— Минимальная версия включительно  
  
-   \] — максимальный номер версии включительно  
  
-   \(\-монопольного Минимальная версия  
  
-   \) — максимальный номер версии монопольной  
  
-   Отдельная версия \# — только указанную версию  
  
 **Пакет SDK расширения**  
 Задает глобальные установки, не ограничиваются конкретного продукта и версии.**Идентификатор целевой платформы** является платформы, например «Windows», которое вы ориентированы.**Версия целевой платформы** имеет версию, например 8.0 целевой платформы.**Имя SDK** и **версия пакета SDK** — имя и номер версии пакета SDK, соответственно.  
  
 **Этот VSIX установлен для всех пользователей \(требуется повышение прав при установке\)** флажок  
 Если этот флажок установлен, этот модуль устанавливается для всех пользователей; в противном случае устанавливается только для текущего пользователя.  
  
 **Этот VSIX установлен с помощью установщика Windows** флажок  
 Если этот флажок установлен, этот модуль устанавливается с помощью установщика Windows \(MSI\-файл\); в противном случае он устанавливается как типичный пакет VSIX \(VSIX\-файл\).  
  
 **Активы** вкладка содержит следующие элементы управления.  
  
 **Список активов**  
 Список активов элементы, описывающие элементы расширения или содержимое, пакет поверхности. Каждый модуль или содержимого элемента отображаются отдельно, источник, тип и путь. Расширения и содержимое элементов можно добавляется в список, изменены и удалены. Тип, а также путь расширения или содержимое элемента соответствует `Type` и `Path` атрибуты связанного `Asset` элемента. Известны следующие типы:  
  
-   Microsoft.VisualStudio.Package  
  
-   Microsoft.VisualStudio.MefComponent  
  
-   Microsoft.VisualStudio.ToolboxControl  
  
-   Microsoft.VisualStudio.Samples  
  
-   Microsoft.VisualStudio.ProjectTemplate  
  
-   Microsoft.VisualStudio.ItemTemplate  
  
-   Microsoft.VisualStudio.Assembly  
  
-   Microsoft.ExtensionSDK  
  
 Чтобы добавить или изменить ресурс, необходимо указать тип актива ли ресурс находится проект в текущее решение или файл в файловой системе и имя проекта. Можно также указать имя папки, в которой необходимо внедрить.  
  
 Можно также создать собственные типы и присвойте им уникальные имена.  
  
 **Зависимости** вкладка содержит следующие элементы управления.  
  
 **Имя источника и диапазон версий**  
 Список элементов зависимостей пакета, которые являются другие пакеты, от которых зависит данный пакет. Если пакета зависимостей, его необходимо установить перед установкой этого пакета; в противном случае необходимо установить этот пакет.  
  
 Идентификатор, имя, диапазон версий, источник и как должна быть устранена зависимость задаются зависимостей пакетов. По имени, версии и источник каждого пакета зависимостей отображаются отдельно. Пакеты зависимостей можно добавляется в список, изменены и удалены.  
  
 Идентификатор должен соответствовать `ID` атрибут метаданных зависимостей пакетов. Источником может быть проект в текущее решение, установленные расширения или файл.**Как — разрешения зависимости** вложенный пакет относительный путь или URL\-адрес расположение загрузки для зависимостей можно установить. Идентификатор версии и разрешение зависимостей пакета соответствуют `Id`, `Version`, и `Location` атрибуты связанного `Dependency` элемента.
