# <a name="vsix-manifest-designer"></a>Конструктор манифеста VSIX
Изменяет файл манифеста пакета VSIX, который задает режим установки для расширения Visual Studio.  
  
 **Конструктор манифеста VSIX** сопоставляется с базовой схемой VSIX. Каждый элемент в схеме можно задать с помощью соответствующего элемента управления в конструкторе. Дополнительные сведения о схеме см. в разделе [Справочник по схеме 2.0 расширения VSIX](../extensibility/vsix-extension-schema-2-0-reference.md).  
  
 Чтобы открыть **конструктор манифеста VSIX**, найдите файл source.extension.vsixmanifest в **обозревателе решений**и откройте его. Если файл не содержит допустимый XML-код, не будут открывать конструктора манифестов.  
  
> [!NOTE]
>  Source.Extension.vsixmanifest выводится extension.vsixmanifest при построении пакета.  
  
## <a name="uielement-list"></a>Список элементов пользовательского интерфейса  
 **Конструктор манифеста VSIX** содержит четыре раздела, которые соответствуют этих элементов верхнего уровня схемы:  
  
-   Метаданные  
  
-   Целевые объекты установки  
  
-   Ресурсы  
  
-   Зависимости  
  
 В области заголовка содержатся следующие элементы управления.  
  
 **Название продукта**  
 Описывает имя расширения.  
  
 **Идентификатор продукта**  
 Указывает уникальные идентификационные данные для этого пакета.  
  
 **Автор**  
 Задает имя автора модуля.  
  
 **Version**  
 Указывает номер версии расширения.  
  
 **Метаданные** вкладка содержит следующие элементы управления.  
  
 **Описание**  
 Предоставляет текстовое описание расширения для отображения в **Диспетчер расширений**.  
  
 **Язык**  
 Задает язык по умолчанию для пакета, который соответствует текстовые данные в манифесте. `Language` Атрибут соответствует соглашению кода общие языковой стандарт языка среды выполнения (CLR) для сборки ресурсов, например, en-us, en, fr-fr. По умолчанию значение является нейтральным; Это означает, что пакет будет запускаться на любой языковой версии Visual Studio.  
  
 **Лицензии**  
 Указывает текстовый файл, содержащий пользовательской лицензии, если такая имеется.  
  
 **Значок**  
 Указывает графический файл (.png, .bmp, .jpeg, .ico), содержащего значок, отображаемый в **Диспетчер расширений**, если отображается значок. Изображение значка должен быть 32 x 32 пикселей или размеры для этих измерений. Если значок не указан, **Диспетчер расширений** используется значок по умолчанию.  
  
 **Изображение предварительного просмотра**  
 Указывает графический файл (.png, .bmp, .jpeg, .ico), содержащего изображение предварительного просмотра для отображения в **Диспетчер расширений**, если присутствует изображения предварительного просмотра. В изображении предварительного просмотра должен быть 200 x 200 пикселей. Если указано без предварительного просмотра, **Диспетчер расширений** использует изображение по умолчанию.  
  
 **Теги**  
 Добавляет текст для поиска ссылок.  
  
 **Заметки о выпуске**  
 Задает файл (.txt, .rtf), содержащий заметки о выпуске. Кроме того, принимает URL-адрес веб-сайта, отображенные в заметках о выпуске.  
  
 **Руководство по началу работы**  
 Задает файл (.txt, .rtf), содержащий сведения об использовании расширения или содержимое в пакет VSIX. В этом руководстве появляется после завершения установки расширения. Также принимает URL-адрес веб-сайта, отображающий руководству.  
  
 **Дополнительные сведения о URL-адрес**  
 Указывает URL-адрес веб-сайт, содержащий дополнительные сведения о продукте.  
  
 **Цели установки** вкладка содержит следующие элементы управления.  
  
 **Тип установки**  
 Перечислены **расширение Visual Studio** и **пакет SDK расширений** объект типы установки. Параметры отличаются в зависимости от выбранного типа.  
  
 **Расширение Visual Studio**  
 Перечислены **InstallationTarget** элементы, которые описывают, как пакет может быть установлен и в какие продукты Visual Studio можно установить это расширение. Имя и версия или версия диапазона отдельно идентифицируется каждого продукта.  Продукты можно добавляется к списку, изменить и удалить. Имя и версия продукта соответствуют **идентификатор** и **версии** атрибуты связанного **InstallationTarget** элемента.  
  
 **Диапазон версий** является [12.0, 14.0] и используется следующая запись:  
  
-   [-Минимальная версия включительно  
  
-   ]-максимальный номер версии включительно  
  
-   (-эксклюзивное Минимальная версия  
  
-   ) — максимальная версия монопольного  
  
-   Одна версия # — только указанная версия  
  
 **Пакет SDK расширения**  
 Задает глобальные установку, не распространяется действие к определенному продукту и версии. **Целевой идентификатор платформы** является платформы, например «Windows», который целевым процессором. **Целевая версия платформы** версией, например 8.0 целевой платформы. **Имя пакета SDK** и **версию пакета SDK** представляют собой имя и номер версии пакета SDK, соответственно.  
  
 **Этот VSIX-ФАЙЛ устанавливается для всех пользователей (требуются повышенные права на установку)** флажок  
 Если этот флажок установлен, это расширение установлено для всех пользователей; в противном случае он устанавливается только для текущего пользователя.  
  
 **Этот VSIX-ФАЙЛ устанавливается с помощью установщика Windows** флажок  
 Если этот флажок установлен, это расширение установлено с помощью установщика Windows (MSI-файл); в противном случае он устанавливается как типичный пакет VSIX (VSIX-файл).  
  
 **Активы** вкладка содержит следующие элементы управления.  
  
 **Список активов**  
 Список активов элементы, описывающие элементы расширения или содержимое этим пакет поверхности. Каждый модуль или содержимого элемента отображаются отдельно, источник, тип и путь. Расширения и содержимое элементов можно добавляется в список, изменены и удалены. Тип и путь модуля или содержимое элемента соответствует `Type` и `Path` атрибуты связанного `Asset` элемента. Известны следующие типы:  
  
-   Microsoft.VisualStudio.Package  
  
-   Microsoft.VisualStudio.MefComponent  
  
-   Microsoft.VisualStudio.ToolboxControl  
  
-   Microsoft.VisualStudio.Samples  
  
-   Microsoft.VisualStudio.ProjectTemplate  
  
-   Microsoft.VisualStudio.ItemTemplate  
  
-   Microsoft.VisualStudio.Assembly  
  
-   Microsoft.ExtensionSDK  
  
 Чтобы добавить или изменить ресурс, необходимо указать тип актива актива находится ли проект в текущем решении или файл в файловой системе и имя проекта. Можно также указать имя папки, в которой для внедрения.  
  
 Можно также создать собственные типы и дать им уникальные имена.  
  
 **Зависимости** вкладка содержит следующие элементы управления.  
  
 **Имя источника и диапазон версий**  
 Список элементов зависимостей этого пакета, которые являются другие пакеты, от которых зависит данный пакет. Если зависимость пакета указан, его необходимо установить перед установкой этого пакета; в противном случае необходимо установить этот пакет.  
  
 Идентификатор, имя, диапазон версий, источник и как зависимость будет разрешаться задаются зависимостей пакетов. Каждый пакет зависимостей отображаются отдельно по имени, версии и источник. Пакеты зависимостей можно добавляется в список, изменены и удалены.  
  
 Идентификатор должен соответствовать `ID` атрибут метаданных зависимостей пакета. Источник может быть проект в текущем решении, установленные расширения или файл. **Как можно разрешить зависимость** параметр может иметь вложенных пакетов относительный путь или URL-адрес расположения загрузки для зависимости. Идентификатор, версию и разрешение зависимостей пакета соответствуют `Id`, `Version`, и `Location` атрибуты связанного `Dependency` элемента.
