# <a name="source-control-plug-in-glossary"></a>Глоссарий подключаемого модуля управления источника
В документации SDK подключаемого модуля управления источника относятся следующие полезные термины и определения.  
  
## <a name="definitions"></a>Определения  
 Возврат  
 Когда пользователь вносит изменения в рабочей копии, пользователь должен отправить изменения из рабочей копии в хранилище контроля центрального источника. Это создает новую редакцию файла, которая доступна для других пользователей. Этот процесс называется возврат.  
  
 Извлечь  
 Процесс запроса рабочей копии из репозитория, сообщающий репозитории намерение изменить его. Рабочая копия отражает состояние проекта, начиная с момента, когда он извлечен.  
  
 "Клиент";  
 Программа, использующая системы управления исходным кодом. В этой документации очень Интегрированной среде разработки Visual Studio.  
  
 Комментарий  
 Сообщение, описывающее внесенные изменения пользователя можно присоединить к исправлению при выполнении операции управления версиями.  
  
 Конфликт  
 Ситуация, когда два пользователя пытаются возврата изменений в одном регионе и тот же файл. Как правило необходимо выполнить слияние.  
  
 Каталог  
 Локальная папка клиентского называют каталог. Это копия, в котором фактически будут внесены изменения. Может быть много рабочие копии данного проекта. Обычно каждый разработчик имеет собственную копию.  
  
 Получение  
 Операция получения переводит пользователя рабочей копии в актуальном состоянии с копией репозитория. В отличие от извлечение get выполняется, когда пользователь просто должен самую последнюю версию, и планирует не вносить изменения.  
  
 Журнал  
 Обычно это сводка всех извлечения, возвраты, обновлений, теги и выпусках, в репозитории системы управления версиями.  
  
 IDE  
 Обычно означает интегрированной среды разработки Visual Studio. Однако она также может указывать на другие клиентские среды, которые распознают API подключаемого модуля управления источника.  
  
 Слияние  
 Процесс, во время исходного два или более файлов кода были объединены новый файл, который включает в себя все возможности предыдущих файлов. Эта концепция важно в системе управления версиями, когда два или несколько разработчиков работают с файлами, одновременно.  
  
 Проект  
 Папка системы управления версиями часто называется проекта. Это не имеет никакой связью с проектов или решений в Visual Studio.  
  
 Подключаемый модуль  
 Библиотеку DLL, которая предоставляет функции системы управления версиями при реализации API подключаемого модуля управления источника.  
  
 Репозиторий  
 Основной копией, где система управления версиями хранит журнал полной версии проекта. Каждый проект имеет ровно один репозиторий.  
  
 Номер редакции  
 Зафиксированные изменения в журнал файла или набора файлов. Версии — один моментальный снимок в постоянно меняющаяся проекта.
