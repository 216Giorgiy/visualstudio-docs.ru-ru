# Шаблон проекта VSIX
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Можно использовать шаблон проекта VSIX для упаковки одно или несколько расширений Visual Studio в проекте VSIX и затем опубликовать пакет на [коллекции Visual Studio](http://go.microsoft.com/fwlink/?LinkID=123847) веб\-сайта.  
  
 Развертывание VSIX поддерживает пакеты VSPackage, сборки, MEF компонентов, шаблонов проектов, шаблонов элементов, элементов управления и настраиваемые типы расширений.  
  
> [!NOTE]
>  Чтобы использовать проекты VSIX, необходимо установить пакет SDK для Visual Studio. Дополнительные сведения о Visual Studio SDK см. в разделе [SDK для Visual Studio](../extensibility/visual-studio-sdk.md).  
  
## Где найти шаблон проекта VSIX  
 Шаблон проекта VSIX доступен в **Новый проект** диалоговое окно. Разверните **Visual Basic** узел или **Visual C\#** узел и нажмите кнопку **расширения**.  
  
> [!TIP]
>  Следует убедиться, что .NET Framework 4.5 или более поздней версии, указан в раскрывающемся списке в верхней части **Новый проект** диалоговое окно.  
  
## Использование шаблона проекта VSIX  
 Шаблон проекта VSIX имеет два основных назначения:  
  
-   Развертывание шаблонов проектов, шаблонов элементов и другие расширения, которые уже не поддерживает VSIX.  
  
-   Чтобы поместить выходные данные нескольких расширений в одно развертывание пакета.  
  
 Необходимо использовать шаблон проекта VSIX для развертывания пакетов VSPackages или другие виды расширений, которые уже имеют поддержки VSIX.  
  
## Упаковка расширения в пустом проекте VSIX  
 Можно упаковать существующее расширение или расширение, не имеет поддержки VSIX, путем помещения его в пустой проект VSIX. Расширение инкапсулируемый должен иметь тип, поддерживаемый [Схема VSIX](../extensibility/vsix-extension-schema-2-0-reference.md).  
  
#### Пакет расширения с помощью проекта VSIX  
  
1.  Построение проектов, составляющих расширения.  
  
2.  Создайте проект VSIX с помощью **проект VSIX** шаблона.  
  
     Открывает Source.Extension.vsixmanifest в **Конструктор манифеста**.  
  
3.  На **активы** выберите **New** кнопки.  
  
     **Добавить новый актив** откроется диалоговое окно.  
  
4.  В **тип** выберите тип добавляемого расширения.  
  
5.  Чтобы добавить расширение или содержимое элемента, включенного в текущем решении \(например, шаблон элемента или скомпилированную сборку\), выполните следующие действия:  
  
    1.  В **источника** выберите **проект в текущем решении**.  
  
    2.  В **проекта** выберите имя расширения.  
  
    3.  В **внедрить в этой папке** введите имя папки, в которой для внедрения ресурса, а затем выберите **ОК** кнопки.  
  
6.  Чтобы добавить расширение или элемент содержимого, не содержится в текущем решении, выполните следующие действия.  
  
    1.  В **источника** поле списка, выберите **файл в файловой системе**.  
  
    2.  В **путь** поля, введите полный путь к файлу расширение скомпилированных или сжатый или использовать **Обзор** кнопку, чтобы найти файл.  
  
    3.  В **внедрить в этой папке** введите имя папки, в которой для внедрения ресурса, а затем выберите **ОК** кнопки.  
  
7.  Если необходимо включить дополнительные расширения к пакету, добавьте их таким же образом.  
  
8.  Постройте решение.  
  
     [!INCLUDE[vsprvs](../code-quality/includes/vsprvs_md.md)] Строит VSIX\-файл, содержащий файл манифеста VSIX, файл \[Content\_Types\] .xml и всех средств расширения, добавленные в проект.
