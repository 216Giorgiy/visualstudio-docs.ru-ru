# <a name="binding-keyboard-shortcuts-to-menu-items"></a>Привязка сочетания клавиш к пунктам меню
Чтобы привязать сочетания клавиш для команды пользовательского меню, просто добавьте запись vsct-файле для пакета. В этом разделе объясняется, как для сопоставления с настраиваемой кнопки, пункт меню или панели инструментов команды сочетания клавиш и по применению назначения клавиш в редакторе по умолчанию или ограничить их специализированный редактор.  
  
 Чтобы назначить сочетания клавиш для существующих элементов меню Visual Studio, в разделе [определение и Настройка сочетаний клавиш](../ide/identifying-and-customizing-keyboard-shortcuts-in-visual-studio.md).  
  
## <a name="choosing-a-key-combination"></a>Выбор сочетания клавиш  
 Многие сочетания клавиш уже используются в Visual Studio. Не следует присваивать того же сочетания для нескольких команд, так как повторяющиеся привязки, которые трудно обнаружить и также может привести к непредсказуемым результатам. Таким образом рекомендуется проверить доступность ярлык до назначения его.  
  
#### <a name="to-verify-the-availability-of-a-keyboard-shortcut"></a>Чтобы проверить доступность сочетания клавиш  
  
1.  В **Сервис / Параметры / Среда** выберите **клавиатуры**.  
  
2.  Убедитесь, что **клавиш в** равно **Global**.  
  
3.  В **клавиши** введите сочетание клавиш, которое вы хотите использовать.  
  
     Если сочетание клавиш уже используется в Visual Studio **сочетанием, используемой в настоящий момент** окно отобразит команду, которая в настоящее время вызывает ярлыка.  
  
4.  Попробуйте различные сочетания клавиш, пока не найдете один, который не сопоставлен.  
  
    > [!NOTE]
    >  Сочетания клавиш, нажмите клавиши ALT может открыть меню и команды выполняются не напрямую. Таким образом **сочетанием, используемой в настоящий момент** при вводе ярлык, который включает в себя ALT поле может быть пустым. Убедитесь, что ярлык не открывается меню, закрывая **параметры** диалоговое окно и нажав клавиши.  
  
 В следующей процедуре предполагается, что имеется существующий пакет VSPackage с помощью команды меню. Если требуется помощь, взгляните на [создания расширения с помощью команды меню](../extensibility/creating-an-extension-with-a-menu-command.md).  
  
#### <a name="to-assign-a-keyboard-shortcut-to-a-command"></a>Чтобы назначить сочетание клавиш для команды  
  
1.  Откройте vsct-файл для пакета.  
  
2.  Создать пустой `<KeyBindings>` статьи после `<Commands>` , если он не существует.  
  
    > [!WARNING]
    >  Дополнительные сведения о сочетания клавиш см. в разделе [соответствующие клавиши](../extensibility/keybinding-element.md).  
  
     В `<KeyBindings>` создайте `<KeyBinding>` входа.  
  
     Задать `guid` и `id` атрибуты те команды, которую необходимо вызвать.  
  
     Задать `mod1` атрибут **управления**, **Alt**, или **Shift**.  
  
     В разделе привязки клавиш должен выглядеть примерно следующим образом:  
  
    ```xml  
    <KeyBindings>  
        <KeyBinding guid="<name of command set>" id="<name of command id>"  
            editor="guidVSStd97" key1="1" mod1="CONTROL"/>  
    </KeyBindings>  
  
    ```  
  
 Если сочетание клавиш требуется более двух ключей, задайте `mod2` и `key2` атрибуты.  
  
 В большинстве случаев **Shift** не должны использоваться без модификатора второй, так как уже ее нажатие вызывает большинство буквенно-цифровых клавиш к типу буквой верхнего регистра или символ.  
  
 Ключ виртуальной коды позволяют открывать специальные клавиши, которые не имеют знака, связанных с ними, например, сочетания клавиш и **BACKSPACE** ключа. Дополнительные сведения см. в разделе [ключ виртуальной коды](http://go.microsoft.com/fwlink/?LinkID=105932).  
  
 Чтобы сделать команды доступны в Visual Studio редактора, задать `editor` атрибут `guidVSStd97`.  
  
 Чтобы сделать команды доступны только в специализированный редактор, установите `editor` атрибут имени настраиваемого редактора, созданное [!INCLUDE[vsprvs](../code-quality/includes/vsprvs_md.md)] шаблона пакета при создании пакета VSPackage, который включает в себя пользовательский редактор. Чтобы найти значение имени, найдите `<Symbols>` , в разделе `<GuidSymbol>` узел которого `name` атрибут заканчивается на «`editorfactory`.» Это имя настраиваемого редактора.  
  
## <a name="example"></a>Пример  
 В этом примере команда привязывает сочетание клавиш CTRL + ALT + C `cmdidMyCommand` в пакет с именем `MyPackage`.  
  
```  
<CommandTable>  
. . .  
<Commands>  
. . .  
</Commands>  
<KeyBindings>  
  <KeyBinding guid="guidMyPackageCmdSet" id="cmdidMyCommand"   
      key1="C" mod1="CONTROL" mod2="ALT" editor="guidVSStd97" />  
</KeyBindings>  
. . .  
</CommandTable>  
```  
  
## <a name="example"></a>Пример  
 В этом примере команда привязывает сочетание клавиш CTL + B `cmdidBold` в проект с именем `TestEditor`. Команда доступна только в пользовательском редакторе, а не в других редакторах.  
  
```xml  
<KeyBinding guid="guidVSStd97" id="cmdidBold" editor="guidTestEditorEditorFactory" key1="B" mod1="Control" />  
```
