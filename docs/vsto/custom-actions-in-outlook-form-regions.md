# <a name="custom-actions-in-outlook-form-regions"></a>Пользовательские действия в областях форм Outlook
  Действия отображения кнопок, которые позволяют пользователям ответить на сообщение Microsoft Office Outlook. Например, для ответа на сообщение, пользователи щелкают **ответ**, **ответить всем**, или **вперед** кнопки действий. Каждое из этих действий создается новый элемент почты и заполняет поля элемента с помощью данных из исходного элемента.  
  
 Можно создать настраиваемое действие, открывающее любой элемент Outlook. Например можно добавить настраиваемое действие для открытия нового элемента встречи или задачи. Настройка параметров настраиваемого действия или использовать пользовательский код для заполнения полей нового элемента. Настраиваемые действия отображаются в **настраиваемые действия** раскрывающегося элемента, который открыт в окне инспектора Outlook.  
  
 [!INCLUDE[appliesto_olkallapp](../vsto/includes/appliesto-olkallapp-md.md)]  
  
## <a name="adding-custom-actions-to-a-form-region"></a>Добавление настраиваемых действий в область формы  
 Чтобы добавить настраиваемое действие в область формы, используйте **настраиваемые действия** диалоговое окно. Можно открыть **настраиваемые действия** диалоговое окно в **обозревателе решений** , развернув **манифеста** узла, при выборе **CustomActions**свойства, а затем нажмите кнопку с многоточием (![эллипс конструктора ASP.NET Mobile](../sharepoint/media/mwellipsis.gif "эллипс конструктора ASP.NET Mobile")).  
  
 Можно использовать **настраиваемые действия** для указания *целевым формы*. Целевая форма — это форма, отображаемая при выполнении пользователем настраиваемого действия.  
  
 Можно также использовать **настраиваемые действия** диалоговое окно «» можно указать, каким образом сведения из исходного элемента в целевой формы.  
  
 В следующей таблице описаны свойства, доступные в **настраиваемые действия** диалоговое окно.  
  
|Свойство.|Описание:|  
|--------------|-----------------|  
|**AddressLike**|Указывает, каким образом будет направлено конечной форме.|  
|**Текст**|Указывает, как текст исходного элемента добавляется к конечной форме.|  
|**Включено**|Указывает, включено ли настраиваемое действие. Если это свойство имеет значение **false**, настраиваемое действие отключено.|  
|**Метод**|Указывает тип ответа, которые доступны при выполнении настраиваемого действия. Настраиваемое действие отправки формы, откройте форму или запрашивает у пользователя разрешение для отправки или открыть форму.|  
|**Name**|Указывает внутреннее имя, которое можно использовать для ссылки на это настраиваемое действие в коде.|  
|**ShowOnRibbon**|Указывает, следует ли отображать настраиваемого действия на ленте исходного элемента.|  
|**SubjectPrefix**|Задает текст, который вставляется в начале строки темы целевой формы.|  
|**TargetForm**|Указывает имя класса сообщений целевой формы. Например, введите **IPM. Задача** открыть форму задач.|  
|**Заголовок**|Указывает метку кнопки настраиваемого действия.|  
  
## <a name="customizing-a-custom-action-at-run-time"></a>Настройка настраиваемого действия во время выполнения  
 Также можно добавить поведение к пользовательскому действию с помощью кода. Например можно добавить код, который принимает имена получателей электронной почты и добавляет эти имена в качестве участников в нового элемента встречи. Чтобы сделать это, необходимо обработать событие [настраиваемое действие](http://msdn.microsoft.com/library/office/ff862186.aspx) событие [объект MailItem](http://msdn.microsoft.com/library/office/ff861332.aspx).
