# <a name="use-command-line-parameters-to-install-visual-studio-2017"></a>Использование параметров командной строки для установки Visual Studio 2017
При установке Visual Studio 2017 из командной строки вы можете использовать параметры командной строки, которые перечислены ниже.  

## <a name="list-of-command-line-parameters"></a>Список параметров командной строки  
 В параметрах командной строки для Visual Studio регистр не учитывается.  

| **Команда командной строки** | **Описание** |
| ----------------------- | --------------- |  
| ```modify``` | Изменяет установленный продукт. |
| ```update``` | Обновляет установленный продукт. |
| ```repair``` | Восстанавливает установленный продукт. |
| ```uninstall``` | Удаляет установленный продукт. |

Если команда не указана, продукт устанавливается.

| **Параметр командной строки** | **Описание** |
| ----------------------- | --------------- |  
| ```--installPath <dir>``` | Каталог установки экземпляра, с которым выполняется операция. Для команды установки это место установки экземпляра. Для других команд это место, где экземпляр был установлен ранее. |
| ```--productId <id>``` | Идентификатор продукта для устанавливаемого экземпляра. Требуется только для команды установки; для других команд игнорируется, если указан параметр --installPath. |
| ```--layout <dir>``` | **Необязательно**. Указывает каталог для создания автономного кэша установки. При выборе этого параметра неявно добавляется параметр "--wait". |
| ```--lang <language-locale>``` *[&#60;язык-языковой стандарт&#62; ...]* | **Необязательно**. Устанавливает или удаляет пакеты ресурсов для указанных языков. Дополнительные сведения см. в разделе [Список языковых стандартов](#list-of-language-locales) на этой странице.|
| ```--add <workload or component ID>``` *[&#60;идентификатор рабочей нагрузки или компонента&#62; ...]* | **Необязательно**. Один или несколько идентификаторов рабочих нагрузок или компонентов, которые нужно добавить. Необходимые компоненты артефакта установлены, однако не установлены рекомендуемые или необязательные компоненты. Дополнительными компонентами можно управлять глобально с помощью параметра "--includeRecommended" и (или) "--includeOptional". Для более детального контроля можно добавить ";includeRecommended" и (или) ";includeOptional" к artifactId (например, "--add Workload1;includeRecommended" или "--add Workload2;includeOptional;includeRecommended"). Дополнительные сведения см. в статье о [рабочей нагрузке и идентификаторах компонента](workload-and-component-ids.md).|
| ```--remove <workload or component ID>``` *[&#60;идентификатор рабочей нагрузки или компонента&#62; ...]* | **Необязательно**. Один или несколько идентификаторов рабочих нагрузок или компонентов, которые нужно удалить. Дополнительные сведения см. в статье о [рабочей нагрузке и идентификаторах компонента](workload-and-component-ids.md).|
| ```--all``` | **Необязательно**. Полная установка всех рабочих нагрузок и компонентов продукта. |
| ```--allWorkloads``` | **Необязательно**. Установка всех рабочих нагрузок и необходимых компонентов для них, но без рекомендуемых и дополнительных компонентов. |
| ```--includeRecommended``` | **Необязательно**. Включает в установку рекомендуемые компоненты для всех устанавливаемых рабочих нагрузок, но не дополнительные компоненты. Рабочие нагрузки указываются с помощью параметра --allWorkloads или --add. |
| ```--includeOptional``` | **Необязательно**. Включает в установку дополнительные компоненты для всех устанавливаемых рабочих нагрузок, но не рекомендуемые компоненты. Рабочие нагрузки указываются с помощью параметра --allWorkloads или --add.  |
| ```--quiet, -q``` | **Необязательно**. Не отображать пользовательский интерфейс во время установки. |
| ```--passive, -p``` | **Необязательно**. Отображать пользовательский интерфейс, но не запрашивать действия со стороны пользователя. |
| ```--norestart``` | **Необязательно**. Если этот параметр указан, команды с параметром --passive или --quiet не будут перезагружать компьютер автоматически. Если параметр --passive или --quiet не указан, этот параметр игнорируется.  |
| ```--locale <language-locale>``` | **Необязательно**. Изменяет язык пользовательского интерфейса установщика. Настройка сохраняется. Дополнительные сведения см. в разделе [Список языковых стандартов](#list-of-language-locales) на этой странице.|
| ```--nickname <name>``` | **Необязательно**. Определяет псевдоним, назначаемый установленному продукту. Длина псевдонима не может превышать 10 символов.  |
| ```--help, --?, -h, -?``` | Вывод сведений об использовании параметра. |

>Примечание. При указании нескольких рабочих нагрузок и компонентов требуется повторять параметр командной строки `--add` или `--remove` для каждого элемента.

| **Дополнительный параметр командной строки** | **Описание** |
| ----------------------- | --------------- |  
| ```--channelId <id>``` | **Необязательно**. Идентификатор канала для устанавливаемого экземпляра. Требуется только для команды установки; для других команд игнорируется, если указан параметр --installPath. |
| ```--channelUri <uri>``` | **Необязательно**. Универсальный код ресурса (URI) манифеста канала. Может использоваться для команды установки; для других команд игнорируется. |
| ```--installChannelUri <uri>``` | **Необязательно**. Универсальный код ресурса (URI) манифеста канала, используемого для установки. Универсальный код ресурса (URI), указанный с помощью параметра --channelUri (который должен быть задан при наличии параметра --installChannelUri), будет использоваться для обнаружения обновлений. Если обновления нежелательны, параметр --channelUri следует указать без аргумента. Может использоваться для команды установки; для других команд игнорируется. |
| ```--installCatalogUri <uri>``` | **Необязательно**. Универсальный код ресурса (URI) манифеста каталога, используемого для установки. Если этот параметр задан, диспетчер канала попытается скачать манифест каталога по этому коду URI перед использованием кода URI, указанного в манифесте канала установки. Этот параметр служит для поддержки автономной установки, при которой кэш макета будет создан с уже скачанным каталогом продуктов. Может использоваться для команды установки; для других команд игнорируется. |
| ```--in <path>``` | **Необязательно**. Универсальный код ресурса (URI) или путь к файлу ответов.  |
| ```--addProductLang <language-locale>``` | **Необязательно**. Определяет язык устанавливаемого артефакта (группы, рабочей нагрузки или компонента). Может использоваться в командной строке несколько раз. Является необязательным для команд установки и изменения; игнорируется для команд обновления, восстановления и удаления. Если этот параметр не задан, при установке используется языковой стандарт компьютера. Дополнительные сведения см. в разделе [Список языковых стандартов](#list-of-language-locales) на этой странице.|
| ```--removeProductLang <language-locale>``` | **Необязательно**. Определяет язык удаляемого артефакта (группы, рабочей нагрузки или компонента). Может использоваться в командной строке несколько раз. Является необязательным для команд установки и изменения; игнорируется для команд обновления, восстановления и удаления. Дополнительные сведения см. в разделе [Список языковых стандартов](#list-of-language-locales) на этой странице.|
| ```--wait``` | **Необязательно**. Процесс дождется завершения установки, прежде чем возвращать код выхода. Это удобно при автоматизации установки, где требуется ожидать завершения установки для обработки ее кода возврата. |
| ```--productKey``` | **Необязательно**. Определяет ключ продукта, используемый для установленного продукта. Он состоит из 25 буквенно-цифровых символов в формате "xxxxx-xxxxx-xxxxx-xxxxx-xxxxx" или "xxxxxxxxxxxxxxxxxxxxxxxxx". |
