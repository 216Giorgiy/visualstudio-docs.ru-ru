# <a name="use-command-line-parameters-to-install-visual-studio-2017-rc"></a>Использование параметров командной строки для установки версии-кандидата Visual Studio 2017
При установке версии-кандидата Visual Studio 2017 из командной строки можно использовать перечисленные ниже параметры командной строки.  

## <a name="list-of-command-line-parameters"></a>Список параметров командной строки  
 В параметрах командной строки для Visual Studio регистр не учитывается.  

| **Команда командной строки** | **Описание** |
| ----------------------- | --------------- |  
| ```modify``` | Изменяет установленный продукт. |
| ```update``` | Обновляет установленный продукт. |
| ```repair``` | Восстанавливает установленный продукт. |
| ```uninstall``` | Удаляет установленный продукт. |

Если команда не указана, продукт устанавливается.

| **Параметр командной строки** | **Описание** |
| ----------------------- | --------------- |  
| ```--installPath <dir>``` | Каталог установки экземпляра, с которым выполняется операция. Для команды установки это место установки экземпляра. Для других команд это место, где экземпляр был установлен ранее. |
| ```--productId <id>``` | Идентификатор продукта для устанавливаемого экземпляра. Требуется только для команды установки; для других команд игнорируется, если указан параметр --installPath. |
| ```--layout <dir>``` | (Необязательно) Указывает каталог для создания автономного кэша установки. При выборе этого параметра неявно добавляется параметр "--wait". |
| ```--lang <language-locale>``` *[&#60;язык-языковой стандарт&#62; ...]* | (Необязательно) Устанавливает или удаляет пакеты ресурсов для указанных языков. |
| ```--add <workload or component ID>``` *[&#60;идентификатор рабочей нагрузки или компонента&#62; ...]* | (Необязательно) Один или несколько идентификаторов рабочих нагрузок или компонентов, которые нужно добавить. Необходимые компоненты артефакта установлены, однако не установлены рекомендуемые или необязательные компоненты. Дополнительными компонентами можно управлять глобально с помощью параметра "--includeRecommended" и (или) "--includeOptional". Для более детального контроля можно добавить ";includeRecommended" и (или) ";includeOptional" к artifactId (например, "--add Workload1;includeRecommended" или "--add Workload2;includeOptional;includeRecommended"). |
| ```--remove <workload or component ID>``` *[&#60;идентификатор рабочей нагрузки или компонента&#62; ...]* | (Необязательно) Один или несколько идентификаторов рабочих нагрузок или компонентов, которые нужно удалить. |
| ```--all``` | (Необязательно) Следует ли установить все рабочие нагрузки и компоненты продукта. |
| ```--allWorkloads``` | Устанавливает все рабочие нагрузки и необходимые компоненты, не устанавливает рекомендуемые или дополнительные компоненты (необязательно). |
| ```--includeRecommended``` | (Необязательно) Включает рекомендуемые компоненты для всех устанавливаемых рабочих нагрузок, но не дополнительные компоненты. Рабочие нагрузки указываются с помощью параметра --allWorkloads или --add. |
| ```--includeOptional``` | (Необязательно) Включает дополнительные компоненты для всех устанавливаемых рабочих нагрузок, но не рекомендуемые компоненты. Рабочие нагрузки указываются с помощью параметра --allWorkloads или --add.  |
| ```--quiet, -q``` | (Необязательно) Не отображать пользовательский интерфейс во время установки. |
| ```--passive, -p``` | (Необязательно) Отображать пользовательский интерфейс, но не запрашивать действия со стороны пользователя. |
| ```--norestart``` | (Необязательно) Если этот параметр указан, команды с параметром --passive или --quiet не будут перезагружать компьютер автоматически (если это требуется). Если параметр --passive или --quiet не указан, этот параметр игнорируется.  |
| ```--locale <language-locale>``` | (Необязательно) Изменяет язык пользовательского интерфейса установщика. Настройка сохраняется. |
| ```--nickname <name>``` | (Необязательно) Определяет псевдоним, назначаемый установленному продукту. Длина псевдонима не может превышать 10 символов.  |
| ```--help, --?, -h, -?``` | Вывод сведений об использовании параметра. |

>Примечание. При указании нескольких рабочих нагрузок и компонентов требуется повторять параметр командной строки `--add` или `--remove` для каждого элемента.

| **Дополнительный параметр командной строки** | **Описание** |
| ----------------------- | --------------- |  
| ```--channelId <id>``` | (Необязательно) Идентификатор канала для устанавливаемого экземпляра. Требуется только для команды установки; для других команд игнорируется, если указан параметр --installPath. |
| ```--channelUri <uri>``` | (Необязательно) Универсальный код ресурса (URI) манифеста канала. Может использоваться для команды установки; для других команд игнорируется. |
| ```--installChannelUri <uri>``` | (Необязательно) Универсальный код ресурса (URI) манифеста канала, используемого для установки. Универсальный код ресурса (URI), указанный с помощью параметра --channelUri (который должен быть задан при наличии параметра --installChannelUri), будет использоваться для обнаружения обновлений. Если обновления нежелательны, параметр --channelUri следует указать без аргумента. Может использоваться для команды установки; для других команд игнорируется. |
| ```--installCatalogUri <uri>``` | (Необязательно) Универсальный код ресурса (URI) манифеста каталога, используемого для установки. Если этот параметр задан, диспетчер канала попытается скачать манифест каталога по этому коду URI перед использованием кода URI, указанного в манифесте канала установки. Этот параметр служит для поддержки автономной установки, при которой кэш макета будет создан с уже скачанным каталогом продуктов. Может использоваться для команды установки; для других команд игнорируется. |
| ```--in <path>``` | (Необязательно) Универсальный код ресурса (URI) файла ответов или путь к нему.  |
| ```--addProductLang <language-locale>``` | (Необязательно) Определяет язык артефакта (группы, рабочей нагрузки или компонента), который необходимо установить. Может использоваться в командной строке несколько раз. Является необязательным для команд установки и изменения; игнорируется для команд обновления, восстановления и удаления. Если этот параметр не задан, при установке используется языковой стандарт компьютера. |
| ```--removeProductLang <language-locale>``` | (Необязательно) Определяет язык артефакта (группы, рабочей нагрузки или компонента), который необходимо удалить. Может использоваться в командной строке несколько раз. Является необязательным для команд установки и изменения; игнорируется для команд обновления, восстановления и удаления. |
| ```--wait``` | Перед возвратом кода выхода процесс будет ожидать завершения установки (необязательно). Это удобно при автоматизации установки, где требуется ожидать завершения установки для обработки ее кода возврата. |
| ```--productKey``` | Определяет ключ продукта, используемый для установленного продукта (необязательно). Он состоит из 25 буквенно-цифровых символов в формате "xxxxx-xxxxx-xxxxx-xxxxx-xxxxx" или "xxxxxxxxxxxxxxxxxxxxxxxxx". |
## <a name="list-of-workload-ids-and-component-ids"></a>Список идентификаторов рабочих нагрузок и компонентов
Список идентификаторов рабочих нагрузок и компонентов, отсортированный по продуктам Visual Studio, см. на странице [Идентификаторы рабочих нагрузок и компонентов Visual Studio 2017](https://aka.ms/vs2017componentids).

> [!WARNING]
> Параметр --layout приведет к ошибке, если имя EXE-файла программы установки содержит цифры. Чтобы обойти эту проблему, следует удалить цифры из имени файла &mdash; например переименовав *vs_community__198521760.1486960229.exe* в ***vs_community.exe***.
