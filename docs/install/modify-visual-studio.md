# <a name="modify-visual-studio-2017-rc"></a>Изменение версии-кандидата Visual Studio 2017
Мы упростили не только персонализацию Visual Studio в соответствии с выполняемыми задачами, но и изменение среды Visual Studio. Для этого больше не нужно обращаться к панели управления: просто запустите установщик Visual Studio и внесите нужные изменения.  

 Ниже описывается порядок действий.  

## <a name="modify-workloads"></a>Изменение рабочих нагрузок  
 Рабочие нагрузки содержат функции, которые требуются для используемого языка программирования или платформы. С помощью рабочих нагрузок можно изменить среду Visual Studio так, чтобы она поддерживала выполнение нужных задач в любое время.  

1.  Найдите установщик Visual Studio на своем компьютере.  

     Например, на компьютере с юбилейным обновлением Windows 10 нажмите кнопку **Пуск** и прокрутите список до буквы **У**. Вы увидите пункт **Установщик Microsoft Visual Studio**.  

     ![00-ModifyingDev15Prev5-FindTheVisualStudioInstaller](../install/media/00-modifyingdev15prev5-findthevisualstudioinstaller.png "Нахождение установщика Microsoft Visual Studio")

2.  Щелкните установщик, чтобы запустить его, и нажмите кнопку **Изменить**.  

     ![Изменение версии-кандидата Visual Studio 2017: запуск или изменение](media/vs2017modify-ModifyOrLaunch.png "Запуск или изменение Visual Studio 2017")  

3.  На экране **Рабочие нагрузки** выберите рабочие нагрузки, которые нужно установить, или отмените выбор тех, которые нужно удалить.  

    ![Диалоговое окно программы установки версии-кандидата Visual Studio 2017](media/vs2017modify-ChooseWorkload.png "Выбор рабочей нагрузки в Visual Studio 2017")

4. Нажмите кнопку **Изменить**.  

5. После установки новых рабочих нагрузок и компонентов нажмите кнопку **Запуск**.

## <a name="modify-individual-components"></a>Изменение отдельных компонентов

Если вы не хотите пользоваться удобной функцией рабочих нагрузок для настройки установленного экземпляра Visual Studio, выберите в установщике Visual Studio вариант **Отдельные компоненты**, выберите нужные компоненты и следуйте указаниям.  

> [!IMPORTANT]
> Хотя версию-кандидат Visual Studio 2017 в общем можно использовать в рабочей среде, рабочие нагрузки и компоненты, которые в пользовательском интерфейсе установки доступны в режиме предварительной версии, в рабочей среде не поддерживаются.
