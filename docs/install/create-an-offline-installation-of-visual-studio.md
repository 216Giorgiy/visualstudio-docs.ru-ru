# <a name="create-an-offline-installation-of-visual-studio-2017-rc"></a>Создание автономной установки версии-кандидата Visual Studio 2017

## <a name="create-a-layout"></a>Создание макета
Если нужно установить [версию-кандидат Visual Studio 2017](https://www.visualstudio.com/vs/visual-studio-2017-rc/) на другом компьютере, на котором нет доступа к Интернету, можно сначала создать макет автономной установки, содержащий все необходимые файлы и компоненты Visual Studio.

Затем можно установить Visual Studio на целевом компьютере с помощью созданного макета автономной установки.     

> [!WARNING]
> В настоящее время пакет SDK для Android не поддерживает автономную установку. Если попытаться установить компоненты пакета SDK для Android на компьютере, который не подключен к Интернету, может произойти сбой установки. Дополнительные сведения об этой проблеме см. в подразделе [Устранение неполадок автономной установки](#tshootofflineinstall) этого раздела.


#### <a name="to-create-an-offline-installation-layout-of-visual-studio"></a>Создание макета автономной установки Visual Studio
1. Скачайте исполняемый файл программы установки Visual Studio на диск локального компьютера.
  Например, [скачайте файл vs_enterprise.exe](https://www.visualstudio.com/vs/visual-studio-2017-rc/).
2. В командной строке выполните команду `vs_enterprise.exe` со следующими аргументами (параметрами):

   1. Добавьте параметр `--layout <path>`, где `<path>` — это место, куда необходимо скачать макет. Обратите внимание на то, что в настоящее время относительные пути (например, `..\vs2017`) не поддерживаются. По умолчанию скачиваются все языки. (См. пример А.)

   2. Чтобы ограничить скачиваемые языки некоторым подмножеством, укажите аргумент `--lang <language>`, где `<language>` — это одна или несколько строк типа "язык-языковой стандарт".  (См. примеры Б и В.)

   В. Чтобы ограничить скачиваемые рабочие нагрузки и компоненты некоторым подмножеством, укажите аргумент `--add <package ID>`. В результате будут скачаны только указанные рабочие нагрузки и компоненты (а также их зависимости). (См. примеры Г и Д.)

   Полный список идентификаторов рабочих нагрузок и компонентов, отсортированный по продуктам Visual Studio, см. на странице [Идентификаторы рабочих нагрузок и компонентов Visual Studio 2017](https://aka.ms/vs2017componentids).

### <a name="examples"></a>Примеры
**Пример А**. Скачивание всех рабочих нагрузок и компонентов для всех языков
  > ```vs_enterprise.exe --layout C:\vs2017```

**Пример Б**. Скачивание всех рабочих нагрузок и компонентов для одного языка  
  > ```vs_enterprise.exe --layout C:\vs2017 --lang en-US```

**Пример В**. Скачивание всех рабочих нагрузок и компонентов для нескольких языков
  > ```vs_enterprise.exe --layout C:\vs2017 --lang en-US de-DE ja-JP```

**Пример Г**. Скачивание одной рабочей нагрузки для всех языков
  > ```vs_enterprise.exe --layout C:\vs2017 --add Microsoft.VisualStudio.Workload.Azure ```

**Пример Д**. Скачивание двух рабочих нагрузок и одного дополнительного компонента для трех языков
  > ```vs_enterprise.exe --layout C:\vs2017 --add Microsoft.VisualStudio.Workload.Azure Microsoft.VisualStudio.Workload.ManagedDesktop Component.GitHub.VisualStudio --lang en-US de-DE ja-JP ```

  > [!WARNING]
  > Параметр --layout приведет к ошибке, если имя EXE-файла программы установки содержит цифры. Чтобы обойти эту проблему, следует удалить цифры из имени файла &mdash; например переименовав *vs_community__198521760.1486960229.exe* в ***vs_community.exe***.

## <a name="install-from-a-layout"></a>Установка на основе макета
#### <a name="to-install-visual-studio-from-an-offline-installation-layout"></a>Установка Visual Studio на основе макета автономной установки
1. На целевом компьютере перейдите в папку **Сертификаты**, которая находится в папке макета.
2. Щелкните правой кнопкой мыши и установите каждый сертификат в папке **Сертификаты**.

  (Если после установки сертификата появляется запрос на ввод пароля, нажмите кнопку **Продолжить**.)  
3. Запустите файл `vs_enterprise.exe` из папки **макета**.

Примечание. Если вы производите установку из частичного макета и выбираете рабочие нагрузки, компоненты или языки, которых нет в макете, программа установки попытается скачать их.  Если у вас нет доступа к Интернету, установить эти элементы не удастся.

> [!CAUTION]
> Макет автономной установки в настоящее время создает ряд файлов (списков управления доступом), которые запрещают доступ всем пользователям.  *Прежде* чем предоставлять доступ к автономной установке, необходимо настроить разрешения (списки управления доступом), предоставив доступ для чтения другим пользователям.

## <a name="update-an-installation-layout"></a>Обновление макета установки
При появлении обновлений для версии-кандидата Visual Studio 2017 вы можете выполнить команду `--layout` еще раз, указав ту же папку макета, чтобы обеспечить актуальность версий компонентов. Будут скачаны только те компоненты, которые обновлялись с момента последнего выполнения команды `--layout`.

## <a id="tshootofflineinstall"> </a>Устранение неполадок макета установки
При автономной установке из кэша установки могут выводиться предупреждения о невозможности установить некоторые компоненты и пакеты. В таблице ниже приведены возможные решения таких проблем.

| Компонент или пакет | Решение |
| -------------------- | -------- |
|Программа установки Android SDK (уровень API)| Для установки пакетов Android SDK (уровень API) требуется подключение к Интернету. Если вы находитесь в сети с ограниченным доступом, при установке Visual Studio необходимо разрешить доступ к следующим URL-адресам: <br><br> — http://dl.google.com:443 <br>— http://dl-ssl.google.com:443 <br>  — https://dl-ssl.google.com/android/repository/*<br><br>Дополнительные сведения об устранении возможных проблем с параметрами прокси-сервера см. в записи блога [Сбои установки Visual Studio (программа установки Android SDK) при использовании прокси-сервера](https://blogs.msdn.microsoft.com/peterhauge/2016/09/22/visual-studio-2015-install-failures-android-sdk-setup-behind-a-proxy/).  |  

 > [!IMPORTANT]
 > Хотя версию-кандидат Visual Studio 2017 в общем можно использовать в рабочей среде, рабочие нагрузки и компоненты, которые в пользовательском интерфейсе установки доступны в режиме предварительной версии, в рабочей среде не поддерживаются.
