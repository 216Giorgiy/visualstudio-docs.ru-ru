# <a name="view-the-call-stack-and-use-the-call-stack-window-in-the-visual-studio-debugger"></a>Просмотр стека вызовов и использование окна стека вызова в отладчике Visual Studio

С помощью **стек вызовов** окне можно просматривать вызовы функций и процедур, которые в настоящее время находятся в стеке. **Стек вызовов** окна показан порядок, в котором находятся получение называется методов и функций. Стек вызовов — хороший способ проверять и анализировать ход выполнения приложения.
  
Когда [отладочные символы](#bkmk_symbols) недоступны для части стека вызовов, **стек вызовов** окно возможно не удастся отобразить правильные сведения об этой части стека вызовов. Если это произошло, появится следующая запись:  
  
`[Frames below may be incorrect and/or missing, no symbols loaded for name.dll]`

> [!NOTE]
> **Стек вызовов** окно аналогична точки зрения отладки в некоторых IDE, например Eclipse. 
> 
> [!NOTE]
>  Отображаемые диалоговые окна и команды меню могут отличаться от описанных здесь в зависимости от текущих параметров или выпуска. Чтобы изменить параметры, выберите **Импорт и экспорт параметров** на **средства** меню.  См. в разделе [Персонализация среды IDE](../ide/personalizing-the-visual-studio-ide.md)
  
## <a name="view-the-call-stack-while-in-the-debugger"></a>Просмотр стека вызовов в отладчике 
  
- Во время отладки, в **Отладка** меню, выберите **Windows > стек вызовов**.

  ![Окно стека вызовов](../debugger/media/dbg_basics_callstack_window.png "CallStackWindow")

Желтая стрелка указывает на кадр стека, в котором находится указатель выполнения. По умолчанию, это кадр стека, сведения которого отображаются в источнике, **"Локальные"**, **"Видимые"**, **Watch**, и **Дизассемблированный код** windows . Если требуется изменение контекста отладчика на другой кадр стека, это можно сделать [переключение на другой кадр стека](#bkmk_switch).   
  
## <a name="display-non-user-code-in-the-call-stack-window"></a>Отображение другого кода в окне стека вызовов  
  
-   Щелкните правой кнопкой мыши **стек вызовов** и выберите **Показать внешний код**.

Непользовательский код — это любой код, который не отображается, когда [Just My Code](../debugger/just-my-code.md) включен. В управляемом коде не написанный пользователем код кадров скрыты по умолчанию. Следующая запись отображается вместо кадры не написанный пользователем код:  
  
**[\<Внешний код >]**  
  
## <a name="bkmk_switch"></a> Переключиться на другой кадр стека (изменение контекста отладчика)
  
1.  В **стек вызовов** окно, щелкните правой кнопкой мыши кадр стека, код и данные, которые вы хотите просмотреть.

    Или можно дважды щелкнуть кадр в **стек вызовов** окно, чтобы переключиться в режим выбранного кадра. 
  
2.  Выберите **перейти к кадру**.  
  
     Зеленая стрелка с фигурным концом появится рядом с выбранным фреймом стека. Указатель выполнения остается в исходном кадре, который по-прежнему отмечен желтой стрелкой. При выборе **шаг** или **Продолжить** из **Отладка** меню выполнение продолжится в исходном кадре, не с выбранного фрейма.  
  
## <a name="view-the-source-code-for-a-function-on-the-call-stack"></a>Просмотреть исходный код для функции в стеке вызовов  
  
-   В **стек вызовов** окно, щелкните правой кнопкой мыши функцию, исходный код которой хотите просматривать и выбирать **к исходному коду**.

## <a name="run-to-a-specific-function-from-the-call-stack-window"></a>Выполнение до определенной функции из окна «Стек вызовов»  
  
-  В **стек вызовов** окно, выберите функцию, щелкните правой кнопкой мыши и выберите **выполнить до текущей позиции**.  
  
## <a name="set-a-breakpoint-on-the-exit-point-of-a-function-call"></a>Установите точку останова в точке выхода вызова функции  
  
-   См. в разделе [установите точку останова в функции в стеке вызова](../debugger/using-breakpoints.md#BKMK_Set_a_breakpoint_in_the_call_stack_window).

## <a name="display-calls-to-or-from-another-thread"></a>Отображаются вызовы, или из другого потока  
  
-   Щелкните правой кнопкой мыши **стек вызовов** и выберите **включить вызовы между потоками**.   
  
## <a name="visually-trace-the-call-stack"></a>Визуальном отслеживании стека вызовов  

Если вы используете Visual Studio Enterprise (только), можно просматривать карты кода для стека вызовов во время отладки.

- В **стек вызовов** окно, откройте контекстное меню. Выберите **Показать стек вызовов на карте кода**. (Клавиатура: **CTRL** + **SHIFT** + **`**)  
  
    Подробные сведения см. в разделе [сопоставление методов в стеке вызовов при отладке](../debugger/map-methods-on-the-call-stack-while-debugging-in-visual-studio.md).

## <a name="view-the-disassembly-code-for-a-function-on-the-call-stack"></a>Просмотреть дизассемблированный код для функции в стеке вызовов  
  
-   В **стек вызовов** окно, щелкните правой кнопкой мыши функцию, Дизассемблированный код которой вы хотите просматривать и выбирать **к дизассемблированному**.    

## <a name="change-the-optional-information-displayed"></a>Изменить отображение дополнительных сведений  
  
-   Щелкните правой кнопкой мыши **стек вызовов** окна и установите или снимите флажок **Показать \<**  _информацией, которая должна_ **>**.  
  
## <a name="bkmk_symbols"></a> Загрузить символы для модуля
В **стек вызовов** окно, вы можете загрузить отладочные символы для кода, который в данный момент нет символы загружены. Это могут быть символы платформы .NET Framework или системные символы, загруженные с общедоступных серверов символов корпорации Microsoft, или символы в каталоге символов на компьютере, на котором производится отладка.  
  
См. в разделе [Указание файлов символов (.pdb) и исходных файлов](../debugger/specify-symbol-dot-pdb-and-source-files-in-the-visual-studio-debugger.md)  
  
### <a name="to-load-symbols"></a>Чтобы загрузить символы  
  
1.  В **стек вызовов** окно, щелкните правой кнопкой мыши кадр стека для символы не загружены. Кадр затеняется.  
  
2.  Пункты **загрузить символы** и нажмите кнопку **серверы символов Microsoft** (если доступно) или укажите путь к символам.  
  
### <a name="to-set-the-symbol-path"></a>Установка пути к символам  
  
1.  В **стек вызовов** окно, выберите **параметры символов** в контекстном меню.  
  
     **Параметры** откроется диалоговое окно и **символы** откроется страница.  
  
2.  Нажмите кнопку **параметры символов**.  
  
3.  В **параметры** диалоговом окне щелкните значок папки.  
  
     В **символов (PDB)** поле появится курсор.  
  
4.  Введите путь к каталогу с символами на компьютере, на котором производится отладка. Для локальной и удаленной отладки, это путь на локальном компьютере.
  
5.  Нажмите кнопку **ОК**, чтобы закрыть диалоговое окно **Параметры**.
