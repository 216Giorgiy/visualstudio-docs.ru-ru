# Messages View
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

У каждого окна есть связанный с ним поток сообщений.  Этот поток сообщений отображается в окне представления сообщений.  Отображаются дескриптор окна, код сообщения и само сообщение.  Для потока или процесса также можно создать представление сообщений.  Это позволяет просматривать сообщения, отправляемые во все окна, принадлежащие конкретному процессу или потоку, что особенно полезно при отслеживании сообщений об инициализации окон.  
  
 Ниже показано типичное окно представления сообщений.  Обратите внимание, что в первом столбце показан дескриптор окна, а во втором — код сообщения \(пояснения см. в разделе [Коды сообщений](../debugger/message-codes.md)\).  Декодированные параметры сообщений и возвращаемые значения отображаются справа.

Представление сообщений Spy\+\+  
  
## Процедуры  
  
#### Открытие представления сообщений для окна, процесса или потока  
  
1.  Переместите фокус на окно [Представление окон](../debugger/windows-view.md), [Представление процессов](../debugger/processes-view.md) или [Представление потоков](../debugger/threads-view.md).  
  
2.  Найдите узел элемента, сообщения которого необходимо изучить, и выберите его.  
  
3.  В меню **Spy** выберите **Записывать сообщения**.  
  
     Откроется диалоговое окно [Параметры сообщения](../debugger/message-options-dialog-box.md).  
  
4.  Выберите параметры сообщения, которые необходимо отобразить.  
  
5.  Нажмите кнопку **ОК**, чтобы начать запись сообщений.  
  
     Откроется окно представления сообщений, и на панель инструментов Spy\+\+ добавится меню **Сообщения**.  В зависимости от выбранных параметров сообщения начнут передаваться потоком в активное окно представления сообщений.  
  
6.  После получения достаточного количества сообщений выберите в меню **Сообщения** команду **Остановить запись**.
