# <a name="debug-at-design-time-in-visual-studio"></a>Отладка во время разработки в Visual Studio

В некоторых сценариях может потребоваться отладить код на этапе разработки времени, а не во время работы приложения. Это можно сделать с помощью **Интерпретация** окна. Если вы хотите выполнить отладку XAML-код, который взаимодействует с другим кодом, такие как код привязки данных, можно использовать **Отладка** > **присоединение к процессу** это сделать.
  
### <a name="debug-at-design-time-using-the-immediate-window"></a>Отладка во время разработки, используя окно "Интерпретация"  

Можно использовать Visual Studio **Интерпретация** окне, чтобы выполнить функцию или подпрограмму, когда приложение не запущено. Если в функции или подпрограмме есть точка останова, Visual Studio прервет выполнение на соответствующей точке. При этом можно использовать окна отладчика для просмотра состояния программы. Эта возможность называется отладкой времени разработки.  

Ниже приведен в Visual Basic, но **Интерпретация** окно также поддерживается в приложениях C# и C++.
  
1.  Вставьте следующий код в консольное приложение Visual Basic:  
  
    ```vb  
    Module Module1  
  
        Sub Main()  
            MySub()  
        End Sub  
  
        Function MyFunction() As Decimal  
            Static i As Integer  
            i = i + 1  
            Dim s As String  
  
            s = "Add Breakpoint here"  
            Return 4  
        End Function  
  
        Sub MySub()  
            MyFunction()  
        End Sub  
    End Module  
    ```  
  
2.  Установите точку останова на строке `s="Add BreakPoint Here"`.  
  
3.  Откройте **Интерпретация** окна (**Отладка** > **Windows** > **Интерпретация**) и введите следующую команду в окно: `?MyFunction<enter>`  
  
4.  Убедитесь, что случился останов и что стек вызовов правилен.  
  
5.  На **Отладка** меню, щелкните **Продолжить**и убедитесь, что вы по-прежнему в режиме конструктора.  
  
6.  Введите следующую команду в **Интерпретация** окна: `?MyFunction<enter>`  
  
7.  Введите следующую команду в **Интерпретация** окна: `?MySub<enter>`  
  
8.  Убедитесь, что достигнута точка останова и посмотрите значение статической переменной `i` в **"Локальные"** окна. Оно должно иметь значение 3.  
  
9. Убедитесь, что стек вызовов верен.  
  
10. На **Отладка** меню, щелкните **Продолжить**и убедитесь, что вы по-прежнему в режиме конструктора.  

## <a name="debug-at-design-time-from-the-xaml-designer"></a>Отладка во время разработки в конструкторе XAML

Может быть полезным для отладки кода программной части из конструктора XAML в некоторых сценариях привязки декларативных данных.

1. В проекте, добавьте новую страницу XAML, таких как *temp.xaml*. Оставьте пустым на новой странице XAML. 

1. Скомпилируйте решение.

1. Откройте *temp.xaml*, который загружает конструктор (*UwpSurface.exe* в приложении UWP, или *XDesProc.exe*) позволяет подключить к нему позже. 

1. Откройте новый экземпляр Visual Studio. В новом экземпляре откройте **присоединение к процессу** диалоговое окно (**Отладка** > **присоединение к процессу**), задайте **присоединить к** поле в тип правильный код, таких как **управляемого кода (CoreCLR)** или тип правильный код зависимости от используемой версии .NET. Выберите правильный конструктора из списка и выберите **Attach**.

    Для универсальной платформы Windows в проектах, предназначенных для сборки 16299 или более поздней, процесс конструктора *UwpSurface.exe*. WPF или версии универсальной платформы Windows до 16299, конструктора происходит *XDesProc.exe*.

1. Во время присоединения к процессу, в проект, откройте кода программной части, где требуется отладить и установите точку останова.

1. Наконец откройте страницу, содержащую код XAML, который содержит привязку данных.

    Например вы установите точку останова в коде преобразователь типа для следующих XAML, который привязывает TextBlock во время разработки.

    ```xaml
    <TextBlock Text="{Binding title, ConverterParameter=lower, Converter={StaticResource StringFormatConverter}, Mode=TwoWay}"  />
    ```
   При загрузке страницы, достижении точки останова.
