# <a name="feature-tour-of-the-visual-studio-debugger"></a>Обзор возможностей отладчика Visual Studio

Этот раздел знакомит с возможностями отладчика Visual Studio. Если вы хотите попробовать, открыв свое приложение в Visual Studio, это можно сделать, или можно изучить пример приложения с использованием [руководство для начинающих](../debugger/getting-started-with-the-debugger.md).

Описываемые здесь функции относятся к C#, C++, Visual Basic, JavaScript и другие языки, поддерживаемые Visual Studio (за исключением иное).

## <a name="set-a-breakpoint-and-start-the-debugger"></a>Установите точку останова и запустите отладчик

Для отладки, необходимо запустить приложение с прикрепленным процесс приложения отладчиком. F5 (**Отладка > Начать отладку**) является наиболее распространенным способом сделать это. Однако код сейчас возможно не задано ни одной точки останова, чтобы проверить приложение, мы, установите и запустите отладку.

Если файл открыт в редакторе кода, можно установить точку останова, щелкнув в поле слева от строки кода.

![Установите точку останова](../debugger/media/dbg-tour-set-a-breakpoint.gif "установить точку останова")

Нажмите клавишу F5 (**Отладка > Начать отладку**) и отладчиком обнаружении первой точки останова. Если приложение не запущено, F5 для запуска отладчика и останавливается на первой точки останова.

Точки останова — полезная возможность, если известно, что строка кода или раздел кода, который вы хотите подробно рассмотреть.

## <a name="navigate-code-in-the-debugger-using-step-commands"></a>Навигация по коду в отладчике, с помощью команд

Мы предоставляем сочетания клавиш для большинства команд, так как они делают навигации из кода вашего приложения быстрее. (Эквивалент команды, такие как команды меню отображаются в круглых скобках).

Чтобы запустить приложение с подключенным отладчиком, нажмите клавишу F11 (**Отладка > шаг с заходом**). — F11 **шаг с заходом** команды и перемещает приложения выполнения одной инструкции за раз. При запуске приложения с F11 отладчик останавливается на первой инструкции, который исполняется.

![F11 Шаг с заходом](../debugger/media/dbg-tour-f11.png "F11 шаг с заходом")

Желтая стрелка представляет инструкцию, на котором отладчик приостановил, который также приостанавливает выполнение приложения в той же точке (она еще не выполнена).

F11 является удобным способом для анализа потока выполнения большинства подробно. (Чтобы переместить быстрее через код, мы продемонстрируем некоторые другие параметры также.) По умолчанию, отладчик пропускает не написан пользователем (если для получения дополнительных сведений см. раздел [только мой код](../debugger/just-my-code.md)).

>[!NOTE]
> В управляемом коде вы увидите диалоговое окно с вопросом, следует получать уведомление об автоматически обход свойств и операторов (по умолчанию). Если вы хотите изменить параметр более поздней версии, отключите **обход свойств и операторов** в **Сервис > Параметры** меню в разделе **Отладка**.

## <a name="step-over-code-to-skip-functions"></a>Шаг с обходом кода для пропуска функции

Находясь в строку кода, которая является вызовом функции или метода, можно нажать клавишу F10 (**Отладка > шаг с обходом**) вместо F11.

Отладчик F10 без шаг с заходом в функции или методы в коде приложения (по-прежнему выполняется код). С помощью клавиши F10, вы можете пропустить код, который вас не интересует. Таким образом, можно быстро получить код, который пользователи будут заинтересованы в.

## <a name="step-into-a-property"></a>Шаг с заходом в свойства

Как упоминалось ранее, по умолчанию, отладчик пропускает управляемого свойства и поля, но **шаг с заходом в конкретное** команда позволяет переопределить это поведение.

Щелкните правой кнопкой мыши на свойство или поле и выберите **шаг с заходом в конкретное**, а затем выберите одну из доступных вариантов.

![Шаг с заходом в конкретное](../debugger/media/dbg-tour-step-into-specific.png "шаг с заходом")

В этом примере **шаг с заходом в конкретное** получает нам кода для `Path.set`.

![Шаг с заходом в конкретное](../debugger/media/dbg-tour-step-into-specific-2.png "шаг с заходом")

## <a name="run-to-a-point-in-your-code-quickly-using-the-mouse"></a>Выполнение до точки в коде быстро с помощью мыши

В отладчике, наведите указатель мыши на строке кода до **запуска, щелкните** кнопки (запустить выполнение сюда) ![запуска щелкните](../debugger/media/dbg-tour-run-to-click.png "RunToClick") отображается слева.

![Запустите щелкните](../debugger/media/dbg-tour-run-to-click-2.png "запуска, щелкните")

>  [!NOTE] 
> **Запуска щелкните** кнопки (запустить выполнение сюда) новые возможности [!include[vs_dev15](../misc/includes/vs_dev15_md.md)].

Нажмите кнопку **запуска щелкните** кнопку (запустить выполнение здесь). Отладчик перемещается на строку кода щелчка.

С помощью этой кнопки аналогично установке временную точку останова. Эта команда также удобен для быстрого решения проблем в видимой области кода приложения. Можно использовать **запуска щелкните** в любом открытом файле.

## <a name="advance-the-debugger-out-of-the-current-function"></a>Переместить отладчик выходит из текущей функции

В некоторых случаях может потребоваться продолжить сеанс отладки, но перемещено отладчика и вплоть до текущей функции.

Нажмите клавиши Shift + F11 (или **Отладка > шаг с выходом**).

Эта команда возобновляет выполнение приложения (и отладчик) до возврата текущей функции.

## <a name="run-to-cursor"></a>Выполнить до текущей позиции

Остановите отладчик, нажав клавишу **остановить отладку** красная кнопка ![остановить отладку](../debugger/media/dbg-tour-stop-debugging.png "остановить отладку") или Shift + F5.

Щелкните правой кнопкой мыши строку кода в приложении и выберите **выполнить до текущей позиции**. Эта команда запускает отладку и задает временную точку останова в текущей строке кода.

![Выполнить до текущей позиции](../debugger/media/dbg-tour-run-to-cursor.png "выполнить до курсора")

Если установлены точки останова, отладчик останавливается на первой точки останова, который он попадает.

Нажмите клавишу F5, пока не достигнете строки кода, если вы выбрали **выполнить до текущей позиции**.

Эта команда полезна при редактировании кода и требуется быстро настроить временную точку останова и запустить отладчик.


> [!NOTE]
> Можно использовать **выполнить до текущей позиции** в **стек вызовов** окно появляется при отладке.

## <a name="restart-your-app-quickly"></a>Перезапустите приложение быстро

Нажмите кнопку **перезапустите** ![перезапустить приложение](../debugger/media/dbg-tour-restart.png "перезапустить приложение") на панели инструментов отладки (**Ctrl + Shift + F5**).

При нажатии клавиши **перезапустите**, он позволяет сэкономить время и остановки приложения и перезапуска отладчика. Отладчик останавливается на первой точки останова будет достигнута при выполнении кода.

Если вы хотите остановить отладчик и вернуться в редактор кода, можно нажать клавишу красный Стоп ![остановить отладку](../debugger/media/dbg-tour-stop-debugging.png "остановить отладку") вместо кнопки **перезапустите**.

## <a name="inspect-variables-with-data-tips"></a>Отслеживание значений переменных с помощью подсказки данных

Теперь, когда известно как немного имеется удобная возможность запуска проверки с помощью отладчика (переменные) состояние приложения. Компоненты, позволяют просматривать значения переменных, некоторые из наиболее полезных функций отладчика, и можно различными способами, чтобы сделать это. Часто при попытке выполнить отладку проблемы вы пытаетесь выяснить ли переменные хранятся значения, которые они должны быть в состоянии конкретного приложения.

Во время приостановки в отладчике, наведите указатель на объект с помощью мыши и посмотрите значение свойства по умолчанию (в данном примере имя файла `market 031.jpg` является значением свойства по умолчанию).

![Просмотр подсказке](../debugger/media/dbg-tour-data-tips.gif "просмотра подсказки")

Разверните объект для просмотра его свойств (например, `FullPath` свойству в этом примере).

Часто при отладке, требуется быстрый способ проверить значения свойств для объектов, подсказки данных — это удобный способ сделать это.

> [!TIP]
> В наиболее поддерживаемых языков можно изменить код во время сеанса отладки. Дополнительные сведения см. в разделе [изменить и продолжить](../debugger/edit-and-continue.md).

## <a name="inspect-variables-with-the-autos-and-locals-windows"></a>Отслеживание значений переменных с помощью окна «Видимые» и «локальные переменные

Во время отладки, посмотрите на **видимые** окно в нижней части редактора кода.

![Видимые окна](../debugger/media/dbg-tour-autos-window.png "видимые-окно")

В **видимые** отображаются переменные вместе с их текущие значения и их типов. **Видимые** в окне отображаются все переменные, используемые в текущей строке или предыдущую строку (в C++ в окне отображается переменные в предыдущем три строки кода. См. в документации для поведения, зависящие от языка).

> [!NOTE]
> В языке JavaScript **локальные** окна, но не поддерживается **видимые** окна.

Теперь рассмотрим **локальные** окна. **Локальные** окна показаны переменные, которые в настоящее время находятся в области действия.

![Окно локальных переменных](../debugger/media/dbg-tour-locals-window.png "локальные-окно")

В этом примере `this` и объектов `f` находятся в области действия. Дополнительные сведения см. в разделе [проверки переменных в окнах локальных переменных и видимые](../debugger/autos-and-locals-windows.md).

## <a name="examine-the-call-stack"></a>Проверить стек вызовов

Нажмите кнопку **стек вызовов** окно появляется при отладке, по умолчанию, откройте в нижней правой области.

![Проверить стек вызовов](../debugger/media/dbg-tour-call-stack.png "проверить стек вызовов")

**Стек вызовов** окна показан порядок, в котором находятся вызван методы и функции. В верхней строке отображена текущая функция ( `Update` метод в этом примере). Вторая строка показывает, что `Update` был вызван с `Path.set` свойства и т. д. Стек вызовов является удобным способом проверки и понимать ход выполнения приложения.

> [!NOTE]
> **Стек вызовов** окна аналогична отладочной перспективы в некоторых интегрированных средах разработки, таких как Eclipse.

Можно дважды щелкнуть строку кода для перехода просмотрите исходного кода, а также изменяет текущую область, проверяемое отладчиком. Это не перемещает отладчик.

Можно также использовать контекстные меню из **стек вызовов** окна для выполнения других действий. Например, вставлять точки останова в определенных функций, перезапустите приложение, нажав **выполнить до текущей позиции**и изучите исходный код. В разделе [как: проверить стек вызовов](../debugger/how-to-use-the-call-stack-window.md).

## <a name="examine-an-exception"></a>Изучите исключение

При создании исключения в приложении, отладчик переходит к строке кода, вызвавшего исключение.
     
![Помощник по исправлению ошибок](../debugger/media/dbg-tour-exception-helper.png "помощника по исправлению ошибок")

В этом примере **помощника по исправлению ошибок** показывает `System.Argument` исключение и сообщение об ошибке, информирующее о том, что путь не является недопустимую форму. Таким образом мы знаем, что ошибка произошла в аргумент метода или функции.

В этом примере `DirectoryInfo` вызов присваивает ошибку в пустую строку, хранящиеся в `value` переменной.

Помощник исключений — прекрасная возможность, которая может помочь в отладке ошибок. Также можно выполнять следующие действия представления сведения об ошибке и Добавить контрольное значение из помощника по исправлению ошибок. Или, при необходимости можно изменить условия для конкретного исключения.

>  [!NOTE] 
> Помощник по исправлению ошибок заменяет помощник по исключениям в [!include[vs_dev15](../misc/includes/vs_dev15_md.md)].

Разверните **параметры исключений** узел, чтобы просмотреть дополнительные параметры для обработки исключения этого типа, но нет необходимости изменять что-либо в этом обзоре!
