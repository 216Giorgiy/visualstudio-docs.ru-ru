# <a name="view-strings-in-a-string-visualizer-in-visual-studio"></a>Просмотр строк в визуализатор строки в Visual Studio
При отладке, вы можете открыть визуализатор строки, чтобы просмотреть строки, которые слишком длинны для просмотра в окне подсказки или отладчик данных. Во многих сценариях визуализатор может помочь для определения строк имеет неправильный формат.

Стандартная строка встроенные визуализаторы включать обычный текст, XML, HTML и JSON. Несколько других типов, таких как WPF-объектов, которые отображаются в отладчике, например windows **"Видимые"** окне также можно открыть визуализаторы.

## <a name="open-a-string-visualizer"></a>Открыть визуализатор строки

Чтобы просмотреть обычного текста, JSON, HTML или XML-строку, щелкните значок лупы ![VisualizerIcon](../debugger/media/dbg-tips-visualizer-icon.png "значок визуализатор") при наведении указателя переменная, содержащая строковое значение. Вы должны быть отладчик приостановит выполнение, чтобы появился значок лупы.

![Открыть визуализатор строки](../debugger/media/dbg-tips-string-visualizers.png "OpenStringVisualizer")

## <a name="view-string-data"></a>Просмотр строковых данных

**Выражение** поля в визуализатор строки показывает текущую переменную или выражение, при прохождении курсора над в отладчике.

**Значение** поле отображается строковое значение. Средство визуализации текста показывает в виде обычного текста.

Пустое **значение** указывает, что определенные визуализатор не может распознать тип string. Например, визуализатора XML показано пустое **значение** для строку в формате простую текстовую строку (с не XML-теги) или JSON. Если вам нужно просмотреть содержит нераспознаваемый строку в средстве визуализации, используйте средство визуализации текста.

### <a name="view-json-string-data"></a>Представление JSON строковых данных

Правильный формат строки JSON будет выглядеть на следующем рисунке в визуализатор JSON. Неправильно сформированный JSON может отображать значок ошибки (или остается пустым, если нераспознанный). Если вы видите значок ошибки, скопируйте и вставьте строку JSON в средство анализа кода JSON, такие как [JSLint](https://www.jslint.com/) для определения ошибки JSON.

![Визуализатор строки JSON](../debugger/media/dbg-tips-string-visualizer-json.png "визуализатор строки JSON")

### <a name="view-xml-string-data"></a>Строка XML-представление данных

На следующем рисунке в визуализаторе XML будет выглядеть корректную XML-строку. Без тегов XML (или остается пустым, если нераспознанный) могут отображаться неправильно сформированный XML.

![Визуализатор строки XML](../debugger/media/dbg-string-visualizers-xml.png "визуализатор строки XML")

### <a name="view-html-string-data"></a>Просмотр HTML строковых данных

Правильный формат строки HTML отображается аналогично представлению, которую мы привыкли Если строка подготавливается к просмотру в браузере, как показано на следующем рисунке. Неправильный формат HTML могут отображаться в виде обычного текста.

![Визуализатор HTML-строки](../debugger/media/dbg-string-visualizers-html.png "визуализатор HTML-строки")
