# Вкладка &quot;Пространство&quot; диалогового окна &quot;Свойства процесса&quot;
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Вкладка **Пространство** используется для просмотра сведений об адресном пространстве процесса.  Для отображения диалогового окна [Свойства процесса](../debugger/process-properties-dialog-box.md) переместите фокус в окно [Представление процессов](../debugger/processes-view.md).  Выберите любой узел процесса в дереве, а затем выберите в меню **Вид** пункт **Свойства**.  
  
 На вкладке **Пространство** доступны следующие параметры:  
  
|Entry|Описание|  
|-----------|--------------|  
|**Показать место, помеченное как**|Этот список используется для выбора категории пространства \(образ, сопоставлено, зарезервировано или не сопоставлено\).|  
|**Исполняемый файл \(байт\)**|Для выбранной категории — суммарный объем всего адресного пространства, используемого процессом.  Исполняемая память — это память, содержимое которой может выполняться программами, но недоступная для чтения или записи данных.|  
|**Байты Exec\-Read\-Only**|Для выбранной категории — суммарный объем всех адресных пространств со свойствами "только для чтения", используемых процессом.  Память exec\-read\-only — это память, которая позволяет получать исполняемый код программы и считывать данные.|  
|**Байты Exec\-Read\-Write**|Для выбранной категории — суммарный объем всех адресных пространств со свойствами "для чтения и записи", используемых процессом.  Память exec\-read\-write позволяет получать исполняемый код программы, считывать и изменять данные.|  
|**Байты Exec\-Write\-Copy**|Для выбранной категории — суммарный объем всего адресного пространства, которое может использоваться для выполнения программ, считывания и записи кода.  Этот тип защиты используется в том случае, когда память необходимо делить между процессами.  Если процессы, совместно использующие память, только выполняют чтение, они все могут использовать одну и ту же память.  Если при совместном использовании памяти требуется доступ для записи, для процесса создается копия памяти.|  
|**Нет доступа \(байт\)**|Для выбранной категории — суммарный объем всего адресного пространства, использование которого процессом запрещено.  При попытке чтения или записи создается ошибка нарушения прав доступа.|  
|**Только чтение \(байт\)**|Для выбранной категории — суммарный объем всего адресного пространства, которое может использоваться для выполнения программ и считывания кода.|  
|**Чтение и запись \(байт\)**|Для выбранной категории — суммарный объем всего адресного пространства, используемого для чтения и записи кода.|  
|**Запись копии \(байт\)**|Для выбранной категории — суммарный объем всего адресного пространства, используемого совместно для чтения, но не для записи кода.  Когда процессы выполняют чтение кода из этой памяти, они могут использовать одну и ту же память совместно.  Тем не менее, когда при совместном доступе одному из процессов необходимо выполнить запись в общую память, создается копия.|
