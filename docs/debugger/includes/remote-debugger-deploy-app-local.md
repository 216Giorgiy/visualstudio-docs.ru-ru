1. В **обозревателе решений**, щелкните правой кнопкой мыши узел проекта и выберите **публикации** (для веб-формы, **публикации веб-приложение**).

2. В **публикации** выберите **папки**, нажмите кнопку **Обзор**и создайте новую папку **C:\Publish**.

    ![RemoteDBG_Publish_Local](../media/remotedbg_publish_local.png "RemoteDBG_Publish_Local")

    Для приложения веб-форм выберите **настраиваемый** в диалоговом окне «Публикация» введите имя профиля и выберите **ОК**.

3. Нажмите кнопку **Опубликовать**.

    Visual Studio публикует проект в папку. Ход выполнения указывает в окне вывода.

4. В **публикации** диалоговое окно, нажмите кнопку **параметры** связь, а затем выберите **параметры** вкладки.

5. Задайте для конфигурации **отладки**выберите **удалить все существующие файлы перед публикацией**, а затем нажмите кнопку **Сохранить**.

    > [!NOTE]
    > При использовании сборки выпуска отключается отладку в файле web.config, при публикации.

6. Нажмите кнопку **Опубликовать**.

    ![RemoteDBG_Publish_Debug_Config](../media/remotedbg_publish_debug_config.png "RemoteDBG_Publish_Debug_Config")
    
    Публикует приложения **отладки** конфигурации проекта в локальной папке.

5. Скопируйте каталог проекта ASP.NET с компьютера Visual Studio в локальный каталог, настроено для приложения ASP.NET (в этом примере **C:\Publish**) на компьютере с Windows Server. В этом учебнике предполагается копировании вручную, но можно использовать другие средства, такие как PowerShell, Xcopy или Robocopy.

    > [!CAUTION]
    >  Если необходимо внести изменения в код или перестроения, необходимо повторно опубликовать и повторите этот шаг. Исполняемый файл, скопированный на удаленный компьютер, должен в точности совпадать с локальным исходным кодом и символами.

6. На сервере Windows убедитесь, что вы правильно запустите приложение, откройте приложение в браузере.

    Если приложение работает неправильно, возможно несоответствие между версией ASP.NET, установленной на сервере и на компьютере Visual Studio или может возникнуть проблема с конфигурацией IIS или веб-сайта. Повторная проверка предыдущих шагах.
