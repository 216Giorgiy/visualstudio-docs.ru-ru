# <a name="how-to-search-for-a-message-in-messages-view"></a>Практическое руководство. Поиск сообщения в представлении сообщений
Можно поиск конкретного сообщения в представлении сообщений с помощью его дескриптор, тип или идентификатор сообщения в качестве критерия поиска. Одной из них — или сочетание — будет допустимое условие. Также можно указать начальное направление поиска. Предопределенные поля в диалоговом окне с атрибутами, выбранного в данный момент сообщения.  
  
### <a name="to-search-for-a-message-in-messages-view"></a>Для поиска сообщения в представлении сообщений  
  
1.  Расположение окон, поэтому Spy ++ и активной [представления сообщений](../debugger/messages-view.md) окна являются видимыми.  
  
2.  Из **поиска** меню, выберите **найти сообщение**.  
  
     [Диалоговое окно "Поиск сообщений"](../debugger/message-search-dialog-box.md) открывается.  
  
3.  Перетащите **инструмент поиска** на нужное окно. При перетаскивании, **поиск сообщений** диалоговом отображаются сведения о выбранном окне.  
  
     - или  
  
     Если дескриптор окна сообщения, для которого необходимо просмотреть, введите его в **обработки** текстовое поле.  
  
     - или  
  
     Если известно, что тип сообщения и/или требуется идентификатор сообщения, выберите их из **тип** и **сообщение** раскрывающиеся меню и снимите **обработки** текстовое поле.  
  
4.  Очистите все поля, для которых не требуется задать значения.  
  
    > [!TIP]
    >  Чтобы не загромождать экран, выберите **Скрыть Spy** параметр. Этот параметр позволяет скрывать главное окно Spy ++, оставляя только **Поиск окна** диалоговое окно поверх других приложений. Главное окно Spy ++ восстанавливается при нажатии кнопки **ОК** или **отменить**, или при удалении **Скрыть Spy ++** параметр.  
  
5.  Выберите **копирование** или **работу** для исходное направление поиска.  
  
6.  Нажмите кнопку **ОК**.  
  
 При обнаружении соответствующего сообщения, он выделяется в окне представления сообщений. В разделе [представлением "сообщения"](../debugger/messages-view.md).
