# Практическое руководство. Добавление и удаление компонентов и элементов в пакете с помощью обозревателя пакетов
  Настроить пакет для развертывания компонентов и элементов SharePoint можно с помощью обозревателя решений.  Компоненты и элементы проекта SharePoint, находящиеся в WSP\-файле, можно изменять.  
  
 Или же можно использовать конструктор пакетов для просмотра изменения порядка компонентов, а также для изменения порядка их активации.  Для получения дополнительной информации см. [Практическое руководство. Добавление и удаление компонентов и элементов в пакете с помощью конструктора пакетов](../sharepoint/how-to-add-and-remove-features-and-items-to-a-package-by-using-the-package-designer.md).  
  
## Запуск обозревателя пакетов  
 Обозреватель пакетов можно открыть согласно приведенным ниже инструкциям, если в решении Visual Studio имеется хотя бы один проект SharePoint.  Кроме того, обозреватель пакетов автоматически открывается при просмотре конструктора компонентов или пакетов.  Если закрыть все окна конструкторов компонентов и пакетов, обозреватель пакетов также закроется.  
  
#### Запуск обозревателя пакетов  
  
1.  В строке меню выберите **Вид**, **Другие окна**, **Обозреватель пакетов**.  
  
     **Обозреватель пакетов** появится на **панели инструментов**.  
  
## Добавление компонента в пакет  
 Добавлять новые и существующие компоненты в пакет можно с помощью обозревателя пакетов.  
  
#### Добавление компонента SharePoint  
  
1.  Откройте **Обозреватель пакетов**, откройте контекстное меню для проекта, а затем выберите **Добавить компонент**.  
  
#### Перемещение существующего компонента SharePoint  
  
1.  Откройте **Обозреватель пакетов**, а затем выполните одно из следующих действий.  
  
    -   Перетащите **компонент** из одного проекта в другой.  
  
    -   Откройте контекстное меню для компонента, выберите **Вырезать**, откройте контекстное меню для проекта, в который необходимо вставить компонент, а затем выберите **Вставить**.  
  
    > [!NOTE]  
    >  Эта процедура используется в случае, если в решении имеется несколько проектов SharePoint.  
  
## Проверка компонента или пакета  
 Проверка файлов позволяет выявлять потенциальные проблемы в компонентах и пакетах SharePoint.  Предупреждения и ошибки отображаются в окне вывода и в окне списка ошибок.  
  
#### Проверка компонента или пакета SharePoint  
  
1.  Откройте **Обозреватель пакетов**.  
  
2.  Откройте контекстное меню для компонента или пакета, а затем выберите **Проверить**.
