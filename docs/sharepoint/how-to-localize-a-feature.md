# Практическое руководство. Локализация компонента
  По умолчанию заголовки компонентов и описаний используют жестко запрограммированные строковые значения.  Для локализации заголовков компонента и описания замените строки выражениями, ссылающимися на локализованные ресурсы.  
  
## Локализация компонента  
  
#### Локализация компонента  
  
1.  В области **Обозреватель решений** откройте контекстное меню для узла **Feature1** и выберите **Добавить ресурс компонента**.  
  
2.  В диалоговом окне **Добавить ресурс** выберите из списка **Универсальный язык** в качестве языка и региональных параметров для языка файла ресурса компонента по умолчанию.  
  
3.  Повторите предыдущий шаг для каждого языка локализации, выбирая языки для локализованных файлов ресурсов компонента.  
  
     Разделите создаваемые файлы ресурсов компонента: один для языка по умолчанию и по одному для каждого языка локализации, который предполагается поддерживать.  
  
4.  Откройте каждый файл ресурсов в "Редакторе ресурсов", а затем введите все строковые идентификаторы и их значения.  
  
     Например, в файле ресурсов компонента, установленном по умолчанию, введите строковый идентификатор заголовка со значением "Мой заголовок компонента" и второй строковый идентификатор описания со значением "Мое описание компонента".  Для каждого локализованного файла ресурсов используйте строковые идентификаторы, аналогичные использованным в ресурсе компонента по умолчанию, но введите локализованные строки для значений.  
  
5.  После введения всех значений ресурсов, необходимо открыть контекстное меню для компонента \(например, Feature1.feature\), а затем выбрать команду **Открыть в конструкторе**, чтобы открыть компонент в конструкторе компонентов.  
  
6.  Чтобы локализовать поля компонента **Заголовок** и **Описание**, введите в них значения в следующем формате:  
  
     `$Resources:` *String ID*  
  
     Например, введите $Resources:Title в поле **Заголовок компонента** и $Resources:Description в поле **Описание компонента**.  
  
     Строковые идентификаторы должны совпадать с теми, которые используются в файлах ресурсов.  
  
7.  Нажмите клавишу F5, чтобы собрать и запустить приложение.  
  
8.  В SharePoint откройте меню **Действия сайта**, выберите **Настройки сайта**, а затем, в разделе **Действия сайта** щелкните ссылку **Управление компонентами сайта**.  
  
9. Измените в SharePoint заданный по умолчанию язык отображения.  
  
     Локализованные заголовок и описание компонента появятся в приложении.  Для отображения локализованных ресурсов на сервере SharePoint должен быть установлен языковой пакет, соответствующий языку и региональным параметрам файла ресурсов.
