# Поддержка форм в рабочих процессах
  В рабочих процессах можно использовать четыре типа форм: ассоциация, запуск, задача и изменение.  Данные типы форм могут основываться на форме ASPX или InfoPath.  Уровень поддержки, который предоставляется [!INCLUDE[vsprvs](../sharepoint/includes/vsprvs-md.md)] для конкретной формы, зависит от нескольких факторов, описанных в следующих таблицах.  Дополнительные сведения о типах форм рабочего процесса см. в разделе [Workflow Forms Overview](http://go.microsoft.com/fwlink/?LinkId=185228) на веб\-сайте MSDN.  
  
## Рефакторинг XML  
 При добавлении ASPX\-формы ассоциации или запуска в элемент проекта рабочего процесса [!INCLUDE[vsprvs](../sharepoint/includes/vsprvs-md.md)] рефакторинг XML в файле рабочего процесса Elements.xml будет автоматически выполняться [!INCLUDE[vsprvs](../sharepoint/includes/vsprvs-md.md)] для сохранения атрибута, который ссылается на форму ассоциации или запуска, в процессе синхронизации при обновлении названия формы, пути развертывания или удалении формы.  Однако при использовании в рабочем процессе других типов форм, таких как форма задач или изменений, файл Elements.xml не подвергается рефакторингу.

## Поддержка форм в импортированных рабочих процессах SharePoint с возможностью повторного использования  
 В следующей таблице описывается поддержка, предоставляемая [!INCLUDE[vsprvs](../sharepoint/includes/vsprvs-md.md)] для различных типов форм \(формы ASPX или InfoPath\) в рабочих процессах SharePoint с возможностью повторного использования, которые импортированы в [!INCLUDE[vsprvs](../sharepoint/includes/vsprvs-md.md)].  
  
|Тип формы|Рабочий процесс с возможностью повторного использования, который содержит импортированную из SharePoint Designer форму ASPX|Рабочий процесс с возможностью повторного использования, который содержит импортированную из SharePoint Designer форму InfoPath|  
|---------------|---------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------|  
|Ассоциация|-   Форма указывается в файле рабочего процесса Elements.xml.<br />-   Рефакторинг файла Elements.xml происходит при переименовании или удалении формы, а также при изменении пути развертывания.|-   Эта форма импортируется, но не указывается в файле рабочего процесса Elements.xml.<br />-   Файл рабочего процесса Elements.xml не подвергается рефакторингу.|  
|форма запуска;|-   Форма указывается рабочим процессом в файле рабочего процесса Elements.xml.<br />-   Рефакторинг файла Elements.xml происходит при переименовании или удалении формы, а также при изменении пути развертывания.|-   Эта форма импортируется, но не указывается в файле рабочего процесса Elements.xml.<br />-   Файл рабочего процесса Elements.xml не подвергается рефакторингу. **Note:**  Для правильной работы данного сценария необходимо добавить и изменить правила и свойства.|  
|Задача|-   Форма указывается в файле рабочего процесса Elements.xml.<br />-   Файл рабочего процесса Elements.xml не подвергается рефакторингу.|-   Эта форма импортируется, но не указывается в файле рабочего процесса Elements.xml.<br />-   Файл рабочего процесса Elements.xml не подвергается рефакторингу. **Note:**  Для правильной работы данного сценария необходимо добавить и изменить правила и свойства.|  
|Изменение|Неприменимо.  В SharePoint Designer нельзя создавать формы изменений ASPX.|Неприменимо.  В SharePoint Designer нельзя создавать формы изменений InfoPath, за исключением встроенного рабочего процесса SharePoint Server, который не включается в WSP\-файл при экспорте рабочего процесса.|
