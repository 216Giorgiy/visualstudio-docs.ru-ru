# <a name="packaging-and-deploying-sharepoint-solutions"></a>Упаковка и развертывание решений SharePoint
  Как правило решения SharePoint развертывается на сервере SharePoint с помощью файла пакета (.wsp) решения. Visual Studio можно использовать для организации элементов проекта SharePoint в компоненты и создать пакет для развертывания компонентов на SharePoint.  
  
 В данном разделе содержатся следующие сведения:  
  
-   [Создание компонентов и пакетов](#Creating)  
  
-   [Компонентов и пакетов средство поддержки](#Tools)  
  
-   [Развертывание решений SharePoint](#Deploying)  
  
-   [Развертывание файлов решения SharePoint](#DeployingFiles)  
  
##  <a name="Creating"></a>Создание компонентов и пакетов  
 Visual Studio можно использовать для группировки связанных элементов SharePoint в *функция*. Например компонент для определения списка контактов может включать экземпляр списка и определение списка. Эти два элемента можно объединить в один компонент в целях развертывания. Дополнительные сведения о функциях см. в разделе [стандартный блок: функции](http://go.microsoft.com/fwlink/?LinkID=169183).  
  
 Затем можно создать пакет решения SharePoint (.wsp) чтобы объединять несколько компонентов, определений сайтов, сборок и других файлов в один пакет, который хранит файлы в формате, необходимые для развертывания файлов на сервере SharePoint. Дополнительные сведения см. в разделе [стандартный блок: решений](http://go.microsoft.com/fwlink/?LinkID=169186).  
  
##  <a name="Tools"></a>Компонентов и пакетов средство поддержки  
 Средства разработки SharePoint в Visual Studio позволяют быстро организовать файлы SharePoint компоненты и пакеты решений для упрощения развертывания. Чтобы настроить пакет компонентов и решений можно использовать следующие средства.  
  
-   Конструктор компонентов и пакетов.  
  
-   Обозреватель пакетов, окно инструментов.  
  
-   Обозреватель решений.  
  
### <a name="feature-designer-and-package-designer"></a>Конструктор компонентов и пакетов  
 Создание функции, задавать их области и отмечать другие компоненты как зависимости с помощью конструктора функций. Конструктор также отображает конечный XML-файл с описанием каждого компонента. Дополнительные сведения см. в разделе [создание компонентов SharePoint](../sharepoint/creating-sharepoint-features.md).  
  
 Компонент можно применять для определенного веб-сайта или группы веб-сайтов, установив его *область* в конструкторе компонентов. Если функция активирована для отдельных веб-сайта, действует только для этого сайта. Если компонент активирован для семейства веб-сайтов, элементы в компоненте применяются для всего семейства сайтов. Дополнительные сведения см. в разделе [область элемента](http://go.microsoft.com/fwlink/?LinkID=169189).  
  
 Если функция зависит от других компонентов, можно установить *зависимость активации компонентов* чтобы перед активацией компонента отметить зависимые компоненты. Зависимость активации компонентов проверяет, если зависимые компоненты активированы в этой области. Дополнительные сведения см. в разделе [зависимости активации и область](http://go.microsoft.com/fwlink/?LinkID=169190).  
  
 В конструкторе пакетов можно сгруппировать элементы SharePoint в единый пакет решения и указать, следует ли выполнить сброс веб-сервера во время развертывания. Чтобы задать тип сервера развертывания, используйте **свойства** окна. Конструктор также создает XML-файл, описывающий содержимое пакета. Дополнительные сведения см. в разделе [создание пакетов решений SharePoint](../sharepoint/creating-sharepoint-solution-packages.md).  
  
 Во время развертывания службы Internet Information Services (IIS) остановлена для копирования файлов решения на сервер SharePoint. С помощью конструктора пакетов в Visual Studio, можно выбрать веб-сервер должен быть перезапущен. Чтобы настроить это решение развертывается на веб-сервере или на сервере приложений, используйте **свойства** окна. Дополнительные сведения см. в разделе [элемент решения (решение)](http://go.microsoft.com/fwlink/?LinkID=169191).  
  
### <a name="packaging-explorer"></a>Обозреватель пакетов  
 В дополнение к конструкторов компонентов и пакетов, можно использовать обозреватель пакетов для группирования элементов SharePoint в компоненты и пакеты. Кроме того, можно просмотреть иерархическое представление проекта SharePoint пакетов, компонентов, элементов и файлов. Обозреватель пакетов представляет собой окно инструментов, которое можно использовать для выполнения следующих задач:  
  
-   Открыть файлы и элементы проектов SharePoint.  
  
-   Перетаскивание элементов проекта SharePoint из одного компонента в другой.  
  
-   Перетаскивание элементов проектов SharePoint и компоненты из одного пакета в другой.  
  
-   Добавьте новый компонент в пакет.  
  
-   Откройте конструктор компонента или пакета.  
  
-   Проверка компонентов и пакетов.  
  
 Средства разработки SharePoint в Visual Studio содержат правила проверки, чтобы обеспечить правильность формирования пакета решения. Кроме того эти правила позволяют проверить том WSP-файл решения может быть успешно развернута и активирована на сервере SharePoint. Дополнительные сведения о схеме XML для компонентов см. в разделе [схемы компонентов](http://go.microsoft.com/fwlink/?LinkID=169192).  
  
 Для системы проектов SharePoint можно добавить пользовательские функции и правила проверки пакета. Дополнительные сведения см. в разделе [как: Создание пользовательских компонентов и правила проверки пакетов для решений SharePoint](../sharepoint/how-to-create-custom-feature-and-package-validation-rules-for-sharepoint-solutions.md).  
  
 Дополнительные сведения об обозревателе пакетов см. в разделе [как: Добавление и удаление компонентов и элементов в пакете с помощью обозревателя пакетов](../sharepoint/how-to-add-and-remove-features-and-items-to-a-package-by-using-the-packaging-explorer.md).  
  
### <a name="solution-explorer"></a>обозреватель решений  
 Обозреватель решений можно использовать для перемещения и открытия файлов проекта SharePoint. Используйте контекстное меню в обозревателе решений для добавления функций приемников событий компонентов и ресурсы компонентов. Кроме того можно открыть конструкторы компонентов и пакетов и настройка компонентов и пакетов для развертывания.  
  
##  <a name="Deploying"></a>Развертывание решений SharePoint  
 После настройки компонентов и пакетов в Visual Studio, можно создать WSP-файл для развертывания на серверах SharePoint. Visual Studio можно использовать для отладки и тестирования WSP-файл только на сервере SharePoint на компьютере разработчика. Дополнительные сведения о развертывании решений SharePoint на удаленном сервере SharePoint см. в разделе [развертывание решения](http://go.microsoft.com/fwlink/?LinkID=169194).  
  
 Можно также настроить действия по развертыванию на компьютере разработчика. Дополнительные сведения см. в разделе [развертывание, публикация и обновление пакетов решений SharePoint](../sharepoint/deploying-publishing-and-upgrading-sharepoint-solution-packages.md).  
  
##  <a name="DeployingFiles"></a>Развертывание файлов решения SharePoint  
 Как правило при добавлении элемента проекта SharePoint в решение SharePoint, все необходимые файлы включены. (Файлы кода) компиляции файлов, которые могут быть встроены в выходную сборку решения. Тем не менее также может потребоваться добавить в проект SharePoint не компилируемых файлов, например, XML-файла, txt или файлов ресурсов. Эти файлы не упаковываются автоматически в решении. Чтобы убедиться, что они собраны, либо добавьте файлы сопоставленную папку или в элемент проекта SharePoint.  
  
 Файлы, добавленные в сопоставленную папку, автоматически копируются во куста SharePoint при развертывании решения. Файлы, добавленные в элемент проекта SharePoint, развернутых в расположение, указанное в **расположение развертывания** свойство для каждого файла, которое частично задается на основе **тип развертывания** свойство. По умолчанию **тип развертывания** значение свойства **NoDeployment**, что означает, что файл не развернуты вместе с решением. Необходимо задать другое значение для свойства к файлу в пакете.  
  
 Например чтобы добавить в XML-файл в проект SharePoint, выполните одно из следующих действий.  
  
-   Добавление сопоставленной папки «Layouts» SharePoint в проект. Будет создан **обозревателе решений** папку с именем **макеты** , содержит вложенную папку для проекта. Добавьте XML-файл в новую вложенную папку. По умолчанию этот файл разворачивается в файловой системе SharePoint в разделе... \TEMPLATE\LAYOUTS\\*имя папки*\\. Сведения о том, как добавить сопоставленные папки в разделе [как: Добавление и удаление папок Mapped](../sharepoint/how-to-add-and-remove-mapped-folders.md).  
  
-   Добавьте XML-файл в папку элемента проекта SharePoint, а затем измените **тип развертывания** свойство XML-файла из **NoDeployment** с другой настройкой например **RootFile** или **ElementFile**. Соответствующий **тип развертывания** параметр зависит от файла и проекта. Дополнительные сведения о **тип развертывания** параметры свойств в разделе [разработке решений SharePoint](../sharepoint/developing-sharepoint-solutions.md).  
  
 Если добавленный файл не применяется для любого конкретного проекта в решении, можно добавить в решение новый проект SharePoint и добавить дополнительные файлы. Другим способом развертывания файлов в SharePoint, особенно в базе данных содержимого, является добавление модуля в проект и затем добавьте файлы в модуль. Дополнительные сведения см. в разделе [с использованием модулей, чтобы включить файлы в решении](../sharepoint/using-modules-to-include-files-in-the-solution.md).
