# Практическое руководство. Развертывание, публикация и обновление решений SharePoint на удаленном сервере
  Помимо развертывания решений SharePoint в локальной системе, можно публиковать изолированные решения SharePoint на удаленные сайты или локальные сайты SharePoint.  Удаленный процесс публикации копирует .wsp\-файл на сервер SharePoint, устанавливает решение, а затем позволяет активировать решение.  Можно также обновить удаленную установку решения SharePoint после внесения в нее изменений.  
  
## Публикация изолированного решения SharePoint на удаленный сервер SharePoint  
  
1.  В **Обозревателе решений** откройте контекстное меню для изолированного проекта SharePoint, который требуется опубликовать, и выберите **Опубликовать**.  
  
2.  В диалоговом окне **Опубликовать** выберите переключатель **Опубликовать на сайт SharePoint**, а затем введите URL\-адрес сайта публикации в сети, как в следующем примере: https:\/\/mytestsite.sharepoint.microsoftonline.com.  
  
3.  Выберите переключатель **Открыть страницу коллекции решений в браузере после публикации**, чтобы просмотреть список решений на странице **Коллекция решений** после публикации.  
  
4.  Нажмите кнопку **Опубликовать**.  
  
5.  Выполните вход на удаленном сервере, если требуется проверка подлинности пользователя.  
  
     Ход выполнения публикации отображается в окне **Вывода** Visual Studio.  После завершения процесса, файл решения \(.wsp\-файл\) устанавливается на удаленном сервере SharePoint.  Однако его по\-прежнему необходимо активировать до использования в SharePoint.  
  
6.  На странице **Коллекция решений** выберите приложение SharePoint, а затем нажмите кнопку **Активировать** на ленте.  
  
7.  В диалоговом окне **Активировать решение** выберите на ленте кнопку **Активировать** еще раз.  
  
     Столбец **Состояние** на странице **Коллекции решений** указывает, что приложение активно.  
  
## Обновление изолированных решений SharePoint на удаленном сервере SharePoint  
 Если изолированное решение SharePoint уже опубликовано на удаленном сервере, то следующий процесс позволяет обновить его после внесения изменений в приложение в [!INCLUDE[vsprvs](../sharepoint/includes/vsprvs-md.md)].  
  
1.  Переименуйте пакет SharePoint в [!INCLUDE[vsprvs](../sharepoint/includes/vsprvs-md.md)].  В области **Обозревателя решений** откройте пакет.  Он отображается в **Обозревателе пакетов**.  
  
2.  В **Обозревателе пакетов** в поле **Имя**, измените имя пакета на уникальное имя.  
  
3.  Сохраните проект.  
  
4.  В области **Обозреватель решений** откройте контекстное меню для проекта и выберите пункт **Опубликовать**.  
  
5.  В диалоговом окне **Опубликовать** выберите переключатель **Опубликовать на сайте SharePoint**, а затем, если отсутствует URL\-адрес удаленного сервера, где сохраняется решение, введите его.  
  
6.  Выберите переключатель **Открыть страницу коллекции решений в браузере после публикации**, чтобы просмотреть список решений на странице **Коллекция решений** после публикации.  
  
7.  Нажмите кнопку **Опубликовать**.  
  
8.  Выполните вход на удаленном сервере, если требуется проверка подлинности пользователя.  
  
     Если вход на удаленный сервер был выполнен недавно, проверка подлинности может не потребоваться.  
  
     Если на сервере SharePoint все еще существует старая версия приложения с тем же именем, возникнет ошибка, что пакет с таким именем уже существует на сервере SharePoint.  Необходимо изменить имя пакета на уникальное имя до публикации.  
  
9. Выберите новое приложение в SharePoint, а затем на ленте нажмите кнопку **Обновить**.  
  
10. В диалоговом окне **Обновить решение** выберите на ленте кнопку **Обновить** еще раз.  Столбец **Состояние** на странице **Коллекции решений** теперь должен указывать, что приложение активно.  
  
     Старая версия решения отключена, новая версия решения обновляется с поддерживаемыми данными из старого решения и новое решение активируется в SharePoint.
