# Шаблоны проектов и элементов проектов SharePoint
  В следующих разделах описываются доступные проекты SharePoint и шаблоны элементов проекта, а также способы их использования.  
  
 [!INCLUDE[autoOutline](../Token/autoOutline_md.md)]  
  
##  <a name="project_items_templates_overview"></a> Обзор шаблонов проектов и элементов проектов SharePoint  
 При создании нового проекта SharePoint в Visual Studio, проект SharePoint добавляется к решению вместе со всеми элементами проекта, необходимыми этому типу проекта.  Например, когда создается проект веб\-части Silverlight, Visual Studio создает решение, которое содержит проект визуальной веб\-части и проект приложения Silverlight, вместе со всеми файлами, необходимыми этим проектам.  Шаблоны элементов проектов используются для добавления элементов в существующие проекты SharePoint, например для добавления приемника событий, столбца сайта или списка.  
  
 Дополнительные сведения об основах SharePoint см. в разделе [SharePoint Foundation Building Blocks](http://go.microsoft.com/fwlink/?LinkId=179404).  Опытные пользователи могут создавать пользовательские проекты и шаблоны элементов проекта.  Для получения дополнительной информации см. [Extending the SharePoint Project System](../sharepoint/extending-the-sharepoint-project-system.md).  
  
##  <a name="project_templates"></a> Шаблоны проектов  
 Ниже приводится список шаблонов проектов SharePoint.  Для просмотра шаблонов проектов SharePoint в Visual Studio в диалоговом окне **Создать проект** разверните узел **SharePoint** в **Visual C\#** или **Visual Basic**, а затем выберите пункт **2010**.  
  
### Проект SharePoint 2010  
 Содержимое *Проект SharePoint 2010* включается в каждый шаблон проекта SharePoint.  Проект SharePoint 2010 содержит:  
  
-   Файл проекта.  
  
-   страницу свойств проекта.  
  
-   папку **References**, содержащую ссылки на все сборки в проекте.  
  
-   папку **Features**, содержащий файл конфигурации с расширением .feature, используемый для развертывания компонентов на сервере SharePoint;  
  
-   папку **Пакет**, которая содержит файл Package.package, используемый для развертывания решения в SharePoint.  
  
-   файл key.snk \(ключ строгого имени\) для подписания сборки строгим именем, используется для расширенной безопасности.  
  
### Веб\-часть Silverlight SharePoint 2010  
 Проекты *Веб\-часть Silverlight SharePoint 2010* позволяют создать веб\-части для SharePoint, отображающие приложения Silverlight.  При создании проекта можно указать, следует ли добавить к нему новое приложение Silverlight или ссылку на существующее.  Дополнительные сведения см. в разделах [Создание веб-частей для SharePoint](../sharepoint/creating-web-parts-for-sharepoint.md) и [Пошаговое руководство. Создание веб-части Silverlight, отображающей данные OData для SharePoint](../sharepoint/walkthrough-creating-a-silverlight-web-part-that-displays-odata-for-sharepoint.md).  
  
### Визуальная веб\-часть SharePoint 2010  
 Проект *Визуальная веб\-часть SharePoint 2010* включает файл определения Elements.xml, элемент **Веб\-часть** и элемент **Пользовательский элемент управления**.  Чтобы настроить внешний вид веб\-части, нужно перетащить или скопировать элементы управления с панели элементов Visual Studio на поверхность пользовательского элемента управления.  Дополнительные сведения см. в разделах [Практическое руководство. Создание веб-части SharePoint с помощью конструктора](../sharepoint/how-to-create-a-sharepoint-web-part-by-using-a-designer.md) and [Building Block: Web Parts](http://go.microsoft.com/fwlink/?LinkId=179438).  
  
### Импорт пакета решения SharePoint 2010  
 Проекты *Импорт пакета решения SharePoint 2010* позволяют импортировать все или часть существующего сайта SharePoint 2010, экспортированного в файл решения SharePoint \(wsp\-файл\), в Visual Studio.  Импортировав элемент в Visual Studio, можно настраивать и повторно развертывать его элементы.  Для получения дополнительной информации см. [Импорт элементов из существующего сайта SharePoint](../sharepoint/importing-items-from-an-existing-sharepoint-site.md).  
  
### Импорт рабочего процесса SharePoint 2010 с возможностью повторного использования  
 Проекты *Импорт рабочего процесса SharePoint 2010 с возможностью повторного использования* позволяют импортировать в Visual Studio декларативный процесс с возможностью повторного использования, созданный в SharePoint Designer 2010.  Рабочий процесс экспортируется из сайта SharePoint в виде WSP\-файла.  Импортировав рабочий процесс в Visual Studio, можно его настраивать, добавлять в него код и повторно развертывать его на сайтах SharePoint.  Для получения дополнительной информации см. [Пошаговое руководство. Импорт рабочего процесса SharePoint Designer с возможностью повторного использования в Visual Studio](../sharepoint/walkthrough-import-a-sharepoint-designer-reusable-workflow-into-visual-studio.md).  
  
##  <a name="project_item_templates"></a> Шаблоны элементов проектов  
 Ниже приводится список шаблонов элементов проектов SharePoint.  Шаблоны элементов проектов добавляют файлы в решение SharePoint для поддержки такой функциональности SharePoint, как столбцы сайтов, списков и типов содержимого.  Например, добавление столбца сайта в решение добавляет проекта столбца сайта, содержащий файл определения Elements.xml.  Добавление визуальной веб\-части добавляет проект визуальной веб\-части, содержащий файл Elements.xml, пользовательский элемент управления и элемент визуальной веб\-части.  
  
 Для просмотра шаблонов элементов проектов SharePoint в **Обозревателе решений** необходимо открыть контекстное меню для проекта SharePoint, а затем выбрать **Добавить**, **Создать элемент**.  Разверните узел **SharePoint**, расположенный в области **Visual C\#** или **Visual Basic**, и выберите **2010**.  
  
### Страница приложения \(только для решения фермы\)  
 Элемент **Страница приложения \(только для решения фермы\)** позволяет разрабатывать веб\-страницы [!INCLUDE[vstecasp](../sharepoint/includes/vstecasp-md.md)] для сайтов SharePoint.  Страницы приложений можно использовать только в решениях фермы.  Можно добавить этот элемент проекта только в решения фермы.  Дополнительные сведения см. в разделе [Практическое руководство. Создание страницы приложения](../sharepoint/how-to-create-an-application-page.md) и [Тип страницы \_макеты приложения](http://go.microsoft.com/fwlink/?LinkId=179434).  
  
### Модель подключения к бизнес\-данным \(только для решения фермы\)  
 Элемент **модели подключения к бизнес\-данным \(только для решений фермы\)** позволяет интегрировать бизнес\-данные в SharePoint.  Бизнес\-данные могут поступать от серверных приложений, таких как [!INCLUDE[ssNoVersion](../sharepoint/includes/ssnoversion-md.md)], Siebel и протокол SAP.  Модели подключения к бизнес\-данным могут использоваться только в решениях фермы.  Можно добавить этот элемент проекта только в решения фермы.  Дополнительные сведения см. в [Практическое руководство. Создание модели подключения к бизнес-данным](../sharepoint/how-to-create-a-bdc-model.md), [Практическое руководство. Использование файла ресурсов для задания локализованных имен, свойств и разрешений](../sharepoint/how-to-use-a-resource-file-to-specify-localized-names-properties-and-permissions.md) и [What's New: Business Connectivity Services](http://go.microsoft.com/fwlink/?LinkId=179411).  
  
### Тип содержимого  
 Элементы *типа содержимого* позволяют создавать пользовательские типы содержимого на основе существующих \(базовых\) типов, таких как документ, объявление и задача.  Пользовательский тип содержимого имеет те же поля и атрибуты, что и базовый, а также столбцы сайтов \(поля\), заданные пользователем.  Например, можно создать пользовательский тип содержимого "Контакт", основанный на базовом типе содержимого "Контакт", входящем в SharePoint.  Можно настраивать тип содержимого, изменяя существующие столбцы сайтов или добавляя больше столбцов сайтов к тем, что уже включены в базовый тип содержимого.  
  
> [!NOTE]  
>  Вследствие ограничений SharePoint невозможно создать тип контента решения фермы на основе типа контента обезвреженного решения.  
  
 Дополнительные сведения см. в [Пошаговое руководство. Создание столбца сайта, типа содержимого и списка для SharePoint](../sharepoint/walkthrough-create-a-site-column-content-type-and-list-for-sharepoint.md) и [Building Block: Content Type](http://go.microsoft.com/fwlink/?LinkId=179413).  
  
### Пустой элемент  
 *Пустые элементы* обычно используются для определения элементов проектов SharePoint, для которых в Visual Studio отсутствует шаблон проекта или элемента проекта.  При добавлении пустого элемента в проект создается узел с именем EmptyElement\[x\], где {x} — уникальный номер.  EmptyElement\[x\] содержит только файл Elements.xml.  Используйте выписки [!INCLUDE[TLA2#tla_xml](../sharepoint/includes/tla2sharptla-xml-md.md)] для указания нужных элементов в Elements.xml.  
  
### Приемник событий  
 *Приемники событий* обрабатывают события для элементов на сайте SharePoint, например, когда элемент был добавлен в список, когда веб\-элемент был удален, или когда рабочий процесс запущен.  Шаблон элемента проекта приемника событий позволяет обрабатывать следующие типы событий:  
  
-   события списков;  
  
-   события элементов списков;  
  
-   события списков электронной почты;  
  
-   Веб\-события  
  
-   события рабочих процессов списков.  
  
 Элемент проекта приемника событий создает папку **Приемник событий**, в которой имеется только один файл класса, содержащий обработчики для всех событий, указанных при создании проекта в **Мастере настройки SharePoint**.  Класс event receiver может обрабатывать события, возникающие на сайте SharePoint при добавлении, обновлении и удалении таких элементов, как файлы, поля, элементы, списки, вложения, веб\-части и рабочие процессы.  Дополнительные сведения см. в. [Практическое руководство. Создание приемника событий](../sharepoint/how-to-create-an-event-receiver.md) и [Building Block: Event Handling](http://go.microsoft.com/fwlink/?LinkId=179416).  
  
### List  
 Список — это экземпляр базового определения списка SharePoint, такого как календарь или список задач.  После добавления списка в решение, конструктор списка позволяет добавлять столбцы сайтов в список и создавать пользовательские столбцы списка.  Это включает столбцы сайта из типов содержимого.  Можно определить *представление* для списка, которое определяет столбцы, которые отображаются в списке.  Дополнительные сведения см. в [Пошаговое руководство. Создание столбца сайта, типа содержимого и списка для SharePoint](../sharepoint/walkthrough-create-a-site-column-content-type-and-list-for-sharepoint.md) и [Building Block: Lists and Document Libraries](http://go.microsoft.com/fwlink/?LinkId=179421).  
  
### Module  
 *Модули* \(не путать с модулями [!INCLUDE[vbprvb](../sharepoint/includes/vbprvb-md.md)]\) могут содержать любые файлы, которые требуется развернуть на сервере SharePoint, например, изображения или примечания.  Элемент проекта модуля содержит узел **Модуль**.  Узел "Модуль" содержит шаблоны двух элементов проекта: XML\-файла определений, выполняющего роль манифеста модуля, и файла sample.txt, прилагаемого для примера.  Дополнительные сведения см. в [Использование модулей для включения файлов в решение](../sharepoint/using-modules-to-include-files-in-the-solution.md) и [Modules](http://go.microsoft.com/fwlink/?LinkId=179425).  
  
### Последовательный рабочий процесс \(только для решения фермы\)  
 *Последовательный рабочий процесс* — это набор действий бизнес\-логики, выполняемых последовательно до завершения последнего действия.  Последовательные рабочие процессы используются для управления процессами, связанными с элементами SharePoint, такими как списки и документы.  Можно создавать рабочие процессы как на уровне сайта \(глобальные\), так и на уровне списка \(локальные\), а также указывать способ запуска рабочего процесса: автоматически или вручную.  Данный элемент проекта можно использовать только в решениях фермы.  Можно добавить этот элемент проекта только в решения фермы.  Дополнительные сведения см. в [Создание решений рабочих процессов SharePoint](../sharepoint/creating-sharepoint-workflow-solutions.md), [Workflows in SharePoint Server 2010](http://go.microsoft.com/fwlink/?LinkId=260555), и [What's New: Workflow Improvements](http://go.microsoft.com/fwlink/?LinkId=179418).  
  
### Веб\-часть Silverlight  
 Элементы проектов *Веб\-часть Silverlight* позволяют создать веб\-части для SharePoint, отображающие приложения Silverlight.  При добавлении этого элемента проекта в решение, можно позже выбрать, следует ли добавить новое приложение Silverlight или ссылку на существующее.  Дополнительные сведения см. в разделах [Создание веб-частей для SharePoint](../sharepoint/creating-web-parts-for-sharepoint.md) и [Пошаговое руководство. Создание веб-части Silverlight, отображающей данные OData для SharePoint](../sharepoint/walkthrough-creating-a-silverlight-web-part-that-displays-odata-for-sharepoint.md).  
  
### Столбец сайта  
 *Cтолбец сайта*, также известный как *поле*, является одним из наиболее основных элементов, которые можно добавить в проект SharePoint.  Столбец сайта представляет собой тип данных, например, телефонный номер, текстовый комментарий или название города контакта в списке контактов.  Дополнительные сведения см. в [Создание столбцов сайта, типов содержимого и списков для SharePoint](../sharepoint/creating-site-columns-content-types-and-lists-for-sharepoint.md) и [Columns](http://go.microsoft.com/fwlink/?LinkId=226840).  
  
### Определение сайта \(только для решения фермы\)  
 Элементы проекта *Определения сайта* содержат папку определения сайта, которая содержит следующие файлы:  
  
-   страница default.aspx, используемая как веб\-страница по умолчанию для этого сайта;  
  
-   Файл onet.xml с определениями компонентов сайта;  
  
-   XML\-файл webtemp, описывающий настройки определения сайта, которые находятся в разделе **Выбор шаблона** страницы **Новый сайт SharePoint**.  
  
 После добавления определения сайта можно добавлять код и файлы для реализации функциональных возможностей.  Данный элемент проекта можно использовать только в решениях фермы.  Можно добавить этот элемент проекта только в решения фермы.  Дополнительные сведения см. в [Создание определений сайтов SharePoint](../sharepoint/creating-site-definitions-for-sharepoint.md) и [Site Definitions and Configurations](http://go.microsoft.com/fwlink/?LinkId=260554).  
  
### Рабочий процесс конечного автомата \(только для решения фермы\)  
 *Рабочий процесс конечного автомата* — это набор состояний, переходов и действий бизнес\-логики.  Действия рабочего процесса конечного автомата выполняются не последовательно: они инициируются действиями и состояниями.  Как и последовательные рабочие процессы, рабочие процессы конечного автомата связаны с элементами SharePoint, такими как списки и документы.  Опять же, можно создавать рабочие процессы как на уровне сайта \(глобальные\), так и на уровне списка \(локальные\).  Также можно указывать способ запуска рабочих процессов: автоматически или вручную.  Данный элемент проекта можно использовать только в решениях фермы.  Можно добавить этот элемент проекта только в решения фермы.  Дополнительные сведения см. в [Создание решений рабочих процессов SharePoint](../sharepoint/creating-sharepoint-workflow-solutions.md), [Workflows in SharePoint Server 2010](http://go.microsoft.com/fwlink/?LinkId=260555), и [What's New: Workflow Improvements](http://go.microsoft.com/fwlink/?LinkId=179418).  
  
### Пользовательский элемент управления \(только для решения фермы\)  
 *Пользовательский элемент управления* является настраиваемым, повторно используемым элементом управления, к которому можно добавлять другие элементы управления ASP.NET и SharePoint.  Пользовательский элемент управления можно добавлять к страницам приложений и веб\-частей, выполняемых в SharePoint.  Данный элемент проекта можно использовать только в решениях фермы.  Можно добавить этот элемент проекта только в решения фермы.  Дополнительные сведения см. в [Создание многократно используемых пользовательских элементов управления для веб\-частей или страниц приложений](http://go.microsoft.com/fwlink/?LinkId=226841).  
  
### Визуальная веб\-часть  
 Элемент проекта *визуальной веб\-части* включает файл определения Elements.xml, элемент **Веб\-часть** и элемент **Пользовательский элемент управления**.  Чтобы настроить внешний вид веб\-части, нужно перетащить или скопировать элементы управления с панели элементов Visual Studio на поверхность пользовательского элемента управления.  Дополнительные сведения см. в разделах [Практическое руководство. Создание веб-части SharePoint с помощью конструктора](../sharepoint/how-to-create-a-sharepoint-web-part-by-using-a-designer.md) and [Building Block: Web Parts](http://go.microsoft.com/fwlink/?LinkId=179438).  
  
### Веб\-часть  
 *Веб\-часть* — серверный элемент управления, который запускается внутри страницы специального типа, называемой страница веб\-частей.  Они являются стандартными блоками страниц, отображаемых на сайте SharePoint.  Элемент веб\-часть содержит файлы, используемые для разработки веб\-частей для сайтов SharePoint.  Дополнительные сведения см. в разделах [Практическое руководство. Создание веб-части SharePoint](../sharepoint/how-to-create-a-sharepoint-web-part.md) and [Building Block: Web Parts](http://go.microsoft.com/fwlink/?LinkId=179438).
