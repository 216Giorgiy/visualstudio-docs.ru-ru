# <a name="sandboxed-solution-considerations"></a>Sandboxed Solution Considerations
  *Изолированные решения* — это функция Microsoft SharePoint 2010, позволяющая пользователям семейства сайта загружать собственные решения с пользовательским кодом. Общие изолированное решение — пользователей, отправка собственные веб-части.  
  
 Изолированное приложение SharePoint работает в безопасном, отслеживаемом процессе, который имеет доступ к ограниченной части веб-фермы. Microsoft SharePoint 2010 использует сочетание функций, коллекций решений, контроля решений и структуры проверки обезвреженных решений.  
  
## <a name="specifying-project-trust-level"></a>Указание уровня доверия проекта  
 [!INCLUDE[vsprvs](../sharepoint/includes/vsprvs-md.md)]поддерживает изолированные решения через свойство логическое проекта, который называется *изолированное решение*. Это свойство можно задать в любое время в проекте, или он может быть указан при создании проекта в **мастер настройки SharePoint**.  
  
> [!NOTE]  
>  Изменение *изолированное решение* свойства проекта после его создания может вызвать ошибки проверки.  
  
 Решение считается решением из области фермы, если *изолированное решение* свойству **false** или выборе **развернуть как решение фермы** параметр. Тем не менее, решение обрабатывается иначе, чем решение фермы при *изолированное решение* свойству **true** или выборе **развернуть как изолированное решение** параметр в мастере.  
  
## <a name="sharepoint-site-hierarchy"></a>Иерархия сайта SharePoint  
 Чтобы понять, как изолированные решения работы, полезно знать сайтов SharePoint являются иерархическими в области видимости. Верхний элемент называется веб-фермы и другие элементы ему подчинены.  
  
 Веб-фермы  
  
 Веб-приложение A  
  
 Семейство сайтов A1  
  
 A1a сайта  
  
 Веб-приложение B  
  
 Семейство сайтов B1  
  
 B1a сайта  
  
 B1b сайта  
  
 Коллекция В2 сайта  
  
 B2a сайта  
  
 Как видите, веб-фермы, может содержать один или несколько веб-приложений, которые в свою очередь, могут содержать один или несколько семейств веб-сайтов, которые могут иметь дочерние узлы и так далее. Изменения, внесенные в один узел коллекции затрагивают коллекции и никакие другие. Однако изменения на уровне веб-фермы влияют на всех семейств веб-сайтов в ферме.  
  
 Службы Windows SharePoint Services (WSS) 3.0 позволяет развертывать решения только на уровне фермы, но [!INCLUDE[wss_14_long](../sharepoint/includes/wss-14-long-md.md)] позволяет развертывать их на уровне фермы (решения фермы), так и на уровне семейства сайтов (изолированного решения).  
  
## <a name="why-sandboxed-solutions"></a>Почему изолированные решения?  
 В WSS 3.0 решений может развертываться только на уровне фермы. Это означает, что потенциально вредоносных или нестабильных позволяют развертывать решения, затронутых всей веб-фермы и всех семейств веб-сайтов и приложений, работающих под ним. Тем не менее с помощью изолированные решения, можно развернуть решения для дочерней области фермы коллекции конкретного сайта. Для обеспечения дополнительной защиты, что сборка решения не загружается в главном [!INCLUDE[TLA2#tla_iis5](../sharepoint/includes/tla2sharptla-iis5-md.md)] процесса (w3wp.exe). Вместо этого он загружается в отдельном процессе (SPUCWorkerProcess.exe). Этот процесс контролируется и внедряет квоты и регулирования для защиты фермы от изолированных решений, которые выполняют вредоносных действий, таких как запуск непрерывные циклы, которые поглощают циклы ЦП.  
  
## <a name="site-collection-solution-gallery"></a>Коллекция решений семейств сайтов  
 В [!INCLUDE[sharepointShort](../sharepoint/includes/sharepointshort-md.md)] 2010 входит компонент под названием "коллекция решений семейств сайтов". Эту функцию можно открыть со страницы центра администрирования SharePoint 2010 или путем открытия **действия сайта** меню, выбрав **параметры сайта**и затем выбрав **решения** ссылку в списке **галерей** на сайте SharePoint. Коллекции решений являются репозиториев решений, позволяющие Администраторы семейства сайтов для управления решениями в этих семействах сайтов.  
  
 Коллекция решений — это библиотека документов, хранящихся в корневом веб-сайте SharePoint. Коллекция решений заменяет собой шаблоны сайтов и поддерживает пакеты решений. При отправке файла решения SharePoint (.wsp) пакета, оно обрабатывается как изолированное решение.  
  
## <a name="sandboxed-solution-limitations"></a>Ограничения изолированных решений  
 При развертывании изолированное решение с целью сокращения какие-либо уязвимости, которые он может иметь ограничено массив функций SharePoint, доступные для него. Ниже приведены некоторые из этих ограничений:  
  
-   Изолированные решения имеют ограниченное подмножество доступных элементов развертываемых решений. Потенциально уязвимой шаблонов проектов SharePoint, например, определений сайтов и рабочие процессы, недоступны.  
  
-   SharePoint запускает код изолированное решение в процессу (SPUCWorkerProcess.exe) отдельно от основной [!INCLUDE[TLA2#tla_iis5](../sharepoint/includes/tla2sharptla-iis5-md.md)] процесс (w3wp.exe) пула приложений.  
  
-   Невозможно добавить сопоставленные папки в проект.  
  
-   Типы в [!INCLUDE[moss_14_long](../sharepoint/includes/moss-14-long-md.md)] сборки Microsoft.Office.Server не может использоваться в изолированных решениях. Кроме того, только типы [!INCLUDE[wss_14_long](../sharepoint/includes/wss-14-long-md.md)] сборки Microsoft.SharePoint может использоваться в изолированных решениях.  
  
 Это важно отметить, что при указании решения SharePoint как изолированное решение не оказывает никакого влияния на сервере SharePoint. он только определяет, как проект SharePoint развертывается в SharePoint из [!INCLUDE[vsprvs](../sharepoint/includes/vsprvs-md.md)] и какими сборками он связывается. Не влияет на созданный WSP-файл и WSP-файл не содержит данных, непосредственно сопоставляемых со *изолированное решение* свойство.  
  
## <a name="capabilities-and-elements-in-sandboxed-solutions"></a>Возможности и элементы в изолированных решениях  
 Изолированные решения поддерживают следующие возможности и элементы:  
  
-   Поля и типы содержимого  
  
-   Настраиваемые действия  
  
-   Декларативных рабочих процессов  
  
-   приемники событий;  
  
-   Вызовы компонентов  
  
-   Определения списков  
  
-   Экземпляры списков  
  
-   И файлы модулей  
  
-   Навигация  
  
-   файл onet.XML  
  
-   SPItemEventReceiver  
  
-   SPListEventReceiver  
  
-   SPWebEventReceiver  
  
-   Поддержка для всех веб-частей, которые являются производными от`System.Web.UI.WebControls.WebParts.WebPart`  
  
-   Веб-части  
  
-   Элементы компонента WebTemplate (вместо Webtemp.xml)  
  
-   Визуальные веб-части  
  
 Изолированные решения не поддерживают следующие возможности и элементы:  
  
-   Страницы приложения  
  
-   Настраиваемые действия группы  
  
-   Компоненты уровня фермы  
  
-   `HideCustomAction` - элемент  
  
-   Веб-компоненты уровня приложения  
  
-   Рабочие процессы с кодом
